## 1. OpenAI 系列

### GPT-1 (2018.06)

**费曼解释**：OpenAI的第一个语言模型，证明"先预训练，再微调"的有效性。

**参数**：117M

**论文**：
- "Improving Language Understanding by Generative Pre-Training" (Radford et al., 2018)

**核心贡献**：
1. 无监督预训练 + 有监督微调
2. Transformer decoder架构
3. 证明预训练的价值

**局限**：
- 模型小，能力有限
- 只在特定任务有效

---

### GPT-2 (2019.02)

**费曼解释**：参数从1.17亿到15亿，OpenAI声称"太危险不敢发布"。

**参数**：1.5B

**论文**：
- "Language Models are Unsupervised Multitask Learners" (Radford et al., 2019)

**核心观点**：
- 足够大的语言模型可以zero-shot完成各种任务
- 不需要专门训练，只要"学会语言"

**历史趣闻**：
- 初始因"担心滥用"分阶段发布
- 最后全部开源

**局限**：
- Zero-shot能力仍然有限
- 生成质量不稳定

---

### GPT-3 (2020.06) ⭐

**费曼解释**：参数暴涨到1750亿，发现"大模型会涌现新能力"，few-shot学习惊艳世界。

**参数**：175B

**论文**：
- "Language Models are Few-Shot Learners" (Brown et al., 2020)

**核心发现**：
1. **Few-shot learning** - 给几个示例就能学会新任务
2. **In-context learning** - 不更新参数，从prompt中学习
3. **Scaling works** - 模型越大，能力越强

**技术细节**：
- 96层Transformer
- 12288隐藏维度
- 96个attention heads
- 2048 context window

**训练**：
- 数据：300B tokens（Common Crawl + WebText + Books + Wikipedia）
- 计算：3.14×10²³ FLOPS
- 成本：估计$4.6M

**影响**：
- 开启LLM时代
- 证明scale的力量
- 引发scaling laws研究

**局限**：
- 没有对齐，可能生成有害内容
- 推理能力仍然有限
- 成本极高

---

### Codex (2021.08)

**费曼解释**：GPT-3的代码版本，GitHub Copilot背后的模型。

**参数**：12B

**论文**：
- "Evaluating Large Language Models Trained on Code" (Chen et al., 2021)

**训练数据**：
- 54M GitHub仓库
- 代码 + 注释

**能力**：
- HumanEval pass@1: 28.8%
- 支持Python、JavaScript等12种语言

**应用**：
- GitHub Copilot
- OpenAI API code-davinci

**局限**：
- 有时会生成有bug的代码
- 不理解复杂架构

---

### InstructGPT / ChatGPT (2022.01/2022.11) ⭐⭐⭐

**费曼解释**：GPT-3加上RLHF，让它"听话"且"有用"。ChatGPT引爆AI应用热潮。

**参数**：175B（GPT-3.5基础）

**论文**：
- "Training language models to follow instructions with human feedback" (Ouyang et al., 2022)

**三步训练**：
```
1. SFT (Supervised Fine-Tuning)
   - 用人类标注的"指令-回答"对
   - 约13k训练样本
   
2. Reward Model
   - 让人类对模型输出排序
   - 训练奖励模型
   - 约33k比较数据
   
3. PPO (Reinforcement Learning)
   - 用奖励模型指导优化
   - 约31k prompts
```

**关键改进**：
- **遵循指令** - 真正"听话"了
- **减少有害输出** - 拒绝不当请求
- **承认不确定性** - 会说"我不知道"

**ChatGPT爆发**：
- 2022.11.30发布
- 5天用户破100万
- 2个月用户破1亿（史上最快）

**影响**：
- AI走进大众视野
- 引发全球AI竞赛
- 推动监管讨论

**局限**：
- 仍然会幻觉
- 知识截止（2021年）
- 推理能力有限
- RLHF可能导致过度安全

### GPT-4 (2023.03) ⭐⭐⭐⭐⭐

**费曼解释**：多模态、强推理、长上下文，当时的最强模型。在bar exam中进入前10%。

**参数**：未公开（传闻1.76T MoE）

**论文**：
- "GPT-4 Technical Report" (OpenAI, 2023)

**核心能力**：

| 能力 | 表现 |
|------|------|
| **多模态** | 图像理解 + 文本 |
| **推理** | MMLU 86.4% |
| **代码** | HumanEval 67% |
| **长上下文** | 8K/32K tokens |
| **对齐** | 大幅减少有害输出 |

**技术亮点**：
1. **多模态** - 原生支持图像输入
2. **推理增强** - CoT能力强
3. **可控性** - 系统提示更有效
4. **安全性** - 6个月red teaming

**训练推测**：
- 数据量：推测10T+ tokens
- 计算：推测10²⁵ FLOPS级别
- 成本：推测$100M+

**评估成绩**：
```
MMLU:           86.4% (人类专家 ~89%)
Bar Exam:       前10%
GRE Writing:    99%
Codeforces:     前20%
```

**局限**：
1. **幻觉仍然存在** - 但频率降低
2. **知识截止** - 2023.04
3. **不透明** - 技术细节几乎未公开
4. **成本高** - API价格昂贵
5. **推理慢** - 比GPT-3.5慢很多

**争议**：
- 参数量猜测（MoE? 稠密?）
- 训练数据来源
- "AGI"宣传

### GPT-4o (2024.05) ⭐⭐⭐⭐

**费曼解释**：原生多模态，音频/图像/文本统一处理，速度快如闪电。

**参数**：未公开

**核心创新**：
1. **端到端多模态** - 不是拼接，而是原生统一
2. **实时语音** - 232ms延迟，接近人类对话
3. **性价比** - 比GPT-4 Turbo便宜50%

**能力**：
```
文本：   与GPT-4 Turbo相当
音频：   新增实时对话
图像：   更强的视觉理解
视频：   实时视频理解
```

**技术特点**：
- 单一模型处理所有模态
- 不需要拼接多个模型
- 推理成本大幅降低

**局限**：
- 某些复杂推理不如GPT-4
- 音频质量仍有提升空间

### o1 (2024.09) ⭐⭐⭐⭐⭐

**费曼解释**：专门训练来"思考"的模型，会花时间推理再回答。像学生考试前深思熟虑。

**核心创新**：
1. **思维链推理** - 内部生成推理链
2. **推理时计算** - 可以"想"很久
3. **强化学习训练** - 学会推理过程

**能力对比**：
```
AIME数学竞赛:
  GPT-4o:   13.4%
  o1:       83.3%

GPQA (博士级科学问题):
  GPT-4o:   56.7%
  o1:       78.0%

Codeforces:
  GPT-4o:   前11%
  o1:       前89%
```

**技术**：
- Inference-time compute
- Chain-of-Thought training
- 可能是"搜索" + LLM

**局限**：
- 慢（推理时间长）
- 贵
- 简单任务可能过度思考

**意义**：
- 新的scaling维度：推理时计算
- 证明"思考"可以训练
- 开启"推理模型"时代

## 4. Meta (Llama) 系列

### Llama 1 (2023.02) ⭐⭐⭐⭐⭐

**费曼解释**：Meta开源的模型，虽然性能不如GPT-3.5，但开启了开源LLM时代。

**参数**：7B / 13B / 33B / 65B

**论文**：
- "LLaMA: Open and Efficient Foundation Language Models" (Touvron et al., 2023)

**训练数据**：
- 1T tokens（公开数据集）
- CommonCrawl, C4, GitHub, Wikipedia, Books

**关键决策**：
1. **开源权重** - 研究可用，但有许可证限制
2. **训练更久** - 超过Chinchilla最优
3. **多尺寸** - 覆盖不同资源场景

**影响**：
- 开启开源LLM时代
- 证明开源可以闭源竞争
- Alpaca, Vicuna等基于Llama的微调模型涌现

**局限**：
- 许可证限制商用
- 能力不如GPT-3.5
- 不对齐

---

### Llama 2 (2023.07) ⭐⭐⭐⭐⭐

**费曼解释**：Llama 1的升级版，商用友好许可证，真正开启开源商业化。

**参数**：7B / 13B / 70B

**论文**：
- "Llama 2: Open Foundation and Fine-Tuned Chat Models" (Touvron et al., 2023)

**关键改进**：
1. **商用许可证** - 可免费商用（部分限制）
2. **对话版本** - Llama 2 Chat，RLHF对齐
3. **更强性能** - 70B接近GPT-3.5

**Llama 2 Chat训练**：
```
1. SFT (Supervised Fine-Tuning)
2. RLHF (Reinforcement Learning from Human Feedback)
3. Safety fine-tuning
```

**评估**：
```
MMLU (70B):    69.8%
HumanEval:     29.9%
```

**技术特点**：
- Grouped Query Attention (GQA) - 70B模型使用
- 更长的训练
- 多轮对话优化

**影响**：
- 成为开源模型标准
- 大量企业采用
- 生态系统繁荣

**局限**：
- 70B参数仍然较大
- 推理能力有限
- 许可证限制（月活>700M需单独许可）

### Llama 3 (2024.04) ⭐⭐⭐⭐⭐

**费曼解释**：Llama系列的最新版，8B和70B两个尺寸，能力大幅提升。

**参数**：8B / 70B（405B即将发布）

**训练数据**：15T tokens（Llama 2的7倍）

**Llama 3 70B评估**：

MMLU:       82%

**技术改进**：
1. **更大训练数据** - 15T tokens
2. **更好的tokenizer** - 128K词汇表
3. **GQA** - 更高效的推理
4. **改进的RoPE** - 长上下文

**Llama 3 8B**：
- 性能接近Llama 2 70B
- 可在消费级GPU运行

**影响**：
- 再次定义开源标准
- 8B成为新的"小模型"标杆

### Llama 3.1 (2024.07) ⭐⭐⭐⭐⭐

## 5. Mistral 系列

### Mistral 7B (2023.09) ⭐⭐⭐⭐

**费曼解释**：法国初创公司的第一个模型，7B参数却打爆13B模型。

**参数**：7B

**论文**：
- "Mistral 7B" (Jiang et al., 2023)

**技术创新**：
1. **Sliding Window Attention** - 4K窗口，但有全局信息
2. **GQA** - 高效推理
3. **Rolling Buffer Cache** - 内存优化

**评估**：
```
在大多数基准上超越Llama 2 13B
接近Llama 1 34B
```

**影响**：
- 证明小模型可以很强
- Apache 2.0许可证
- 欧洲AI崛起

---

### Mixtral 8x7B (2023.12) ⭐⭐⭐⭐⭐

**费曼解释**：MoE架构的8个7B expert，每次只激活2个，总共47B参数但只用14B。

**架构**：
- 8个expert（每个7B）
- 每个token激活2个expert
- 总参数：47B
- 激活参数：14B

**评估**：
```
性能 >= Llama 2 70B
推理成本 ~= 12B模型
```

**意义**：
- MoE架构在开源中的成功
- 高效率+高性能
- 推动MoE普及
