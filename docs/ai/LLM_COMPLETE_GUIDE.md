## 核心概念

### LLM 本质

1. **知识来源**: 训练数据压缩 + 统计模式识别
2. **可靠性**: 常识 > 专业 > 时效
3. **能力边界**: 擅长语言/代码/创意，不擅长最新/推理/实操

### 历史发展

```
2017  Transformer - "Attention is All You Need"
2018  BERT/GPT-1 - 预训练时代
2019  GPT-2 - "too dangerous to release"
2020  GPT-3 - Few-shot learning (175B)
2021  Codex, CLIP - 多模态萌芽
2022  ChatGPT - 对话式AI爆发
2023  GPT-4, Claude 2, Llama - 多模态+开源
2024  Claude 3, GPT-4o, Llama 3 - 持续进步
```

### 核心架构

```
Transformer
├── Encoder (BERT系列)
│   └── 理解任务：分类、抽取、嵌入
│
└── Decoder (GPT系列)
    └── 生成任务：对话、写作、代码
```

## Prompt 设计

### 基本公式

```
角色 + 任务 + 背景 + 约束 + 格式 = 高质量Prompt
```

### 设计原则

1. **清晰性** - 一句话说明要什么
2. **具体性** - 明确输入输出
3. **结构化** - 分步骤、给格式
4. **渐进性** - 从简单到复杂
5. **可验证性** - 有判断标准

### 核心技巧

| 技巧 | 适用场景 | 示例 |
|------|----------|------|
| 角色设定 | 专业领域 | "作为资深架构师..." |
| 思维链 | 复杂推理 | "让我们一步步思考..." |
| Few-shot | 格式要求 | 给2-3个示例 |
| 结构化 | 多条件 | 用表格/列表组织 |
| 负面约束 | 避免错误 | "不要使用..." |

## 职业实践

### 程序员

```
高价值: 代码生成、Bug分析、重构建议、文档生成
需谨慎: 安全关键代码、性能关键路径
不适合: 生产环境直接部署
```

### 产品经理

```
高价值: 需求分析、竞品调研、文档撰写、创意发散
需谨慎: 用户数据、商业机密
不适合: 最终决策
```

### 数据分析师

```
高价值: SQL生成、图表建议、报告撰写
需谨慎: 原始数据、精确计算
不适合: 最终数字确认
```

## 模板库

### 代码生成

```
角色：资深[语言]开发者
任务：实现[功能描述]
要求：
- 使用[框架/库]
- 包含错误处理
- 添加注释
输出：完整可运行代码
```

### 文档写作

```
角色：技术文档专家
任务：为[项目/功能]编写文档
受众：[目标用户]
要求：结构清晰、示例丰富
格式：Markdown
```

### 问题诊断

```
当前情况：[描述问题]
已尝试：[列出尝试过的方法]
环境：[相关版本/配置]
期望：[期望的结果]
错误：[错误信息]
```
