# PRD 编写

## 概述

PRD（Product Requirements Document，产品需求文档）是产品经理输出给设计、开发、测试等团队的核心文档，详细描述产品功能的实现需求。

## PRD 的作用

### 主要作用

```
产品经理 → PRD → 设计师
              ↓
              → 开发工程师
              ↓
              → 测试工程师
              ↓
              → 运营人员
```

### 具体价值

1. **沟通工具**：统一团队理解
2. **开发依据**：指导开发实现
3. **测试基准**：验收测试的依据
4. **文档留存**：知识传承和记录

### 4. 交互逻辑

#### 交互说明模板

```markdown
## 交互说明

### 正常流程
1. 用户点击"提交"按钮
2. 前端验证表单
3. 显示 loading 状态
4. 调用后端接口
5. 接口返回成功
6. 显示成功提示
7. 跳转到结果页

### 异常流程

#### 网络异常
1. 接口调用超时（> 10秒）
2. 显示"网络异常，请检查网络后重试"
3. 提供"重试"按钮

#### 参数验证失败
1. 后端返回参数错误
2. 显示具体错误信息
3. 定位到错误字段

#### 业务异常
1. 后端返回业务错误码
2. 根据错误码显示对应提示
```

#### 错误码定义

| 错误码 | 错误信息 | 处理方式 |
|--------|---------|---------|
| 10001 | 参数错误 | 显示具体错误字段 |
| 10002 | 用户不存在 | 引导注册 |
| 10003 | 密码错误 | 提示重新输入 |
| 20001 | 系统繁忙 | 提示稍后重试 |

## PRD 编写技巧

### 1. 清晰简洁

**不好的写法**：
> 系统应该提供一个功能，让用户能够在某个地方看到一些相关的信息，以便于他们能够更好地了解情况。

**好的写法**：
> 在订单详情页顶部显示订单状态，包括：当前状态、预计送达时间、物流轨迹。

### 2. 结构化表达

**使用表格**：

| 场景 | 条件 | 结果 |
|------|------|------|
| 场景1 | 条件A | 结果X |
| 场景2 | 条件B | 结果Y |

**使用流程图**：

```
开始 → 步骤1 → 步骤2 → 结束
```

### 3. 完整性

**检查清单**：

- [ ] 正常流程描述完整
- [ ] 异常流程考虑全面
- [ ] 边界条件明确
- [ ] 接口需求清晰
- [ ] 数据埋点明确
- [ ] 非功能需求完整

### 4. 可追溯性

**版本管理**：

| 版本 | 日期 | 修改内容 | 修改人 |
|------|------|---------|--------|
| V1.0 | 01-01 | 初始版本 | 张三 |
| V1.1 | 01-05 | 增加异常处理 | 张三 |

## 常见问题

### 问题1：PRD 写得太细还是太粗？

**原则**：详细到开发可以实施，但不要过度设计

**判断标准**：
- 开发是否清楚要做什么？
- 是否还有模糊的地方？
- 是否包含了必要的边界条件？

### 问题2：PRD 和设计稿的关系？

**协作流程**：
```
PRD（初稿）→ 设计评审 → PRD（完善）→ 设计稿 → PRD（定稿）
```

**PRD 包含**：
- 功能描述
- 交互逻辑
- 业务规则

**设计稿包含**：
- 视觉设计
- 交互细节
- 动效说明

### 问题3：PRD 变更如何管理？

**变更流程**：
1. 提出变更需求
2. 评估影响范围
3. 更新 PRD 文档
4. 通知相关方
5. 记录变更历史

## PRD 工具

### 常用工具

| 工具 | 特点 | 适用场景 |
|------|------|---------|
| Confluence | 协作功能强 | 团队协作 |
| Notion | 灵活可定制 | 各种规模 |
| Word | 通用性强 | 小团队 |
| 飞书文档 | 国产、集成度高 | 中国团队 |

### 最佳实践

1. **使用模板**：建立标准化模板
2. **版本管理**：保留历史版本
3. **协作编辑**：支持多人编辑
4. **评论功能**：支持讨论和反馈
5. **权限管理**：控制访问和编辑权限

## 总结

写好 PRD 的关键：

1. **结构清晰**：遵循标准结构
2. **表达准确**：避免歧义
3. **内容完整**：考虑全面
4. **图文结合**：提高可读性
5. **及时更新**：保持文档最新
6. **团队协作**：与相关方共同完善
