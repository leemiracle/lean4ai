# OKR 管理

## 概述

OKR（Objectives and Key Results，目标与关键结果）是一种目标管理方法，用于定义和跟踪目标及其完成情况。

## 核心概念

### 基本结构

```
Objective（目标）
    │
    ├── Key Result 1（关键结果1）
    ├── Key Result 2（关键结果2）
    └── Key Result 3（关键结果3）
```

### 定义

#### Objective（目标）

**定义**：要达成的定性目标

**特点**：
- 激励人心
- 有挑战性
- 可理解
- 可行动

**示例**：
- 成为用户首选的协作工具
- 打造极致的用户体验
- 建立高效的获客体系

#### Key Result（关键结果）

**定义**：衡量目标达成程度的定量指标

**特点**：
- 可衡量
- 有时限
- 有挑战
- 可验证

**示例**：
- MAU 从 10 万增长到 50 万
- NPS 从 30 提升到 50
- 获客成本降低 30%

## OKR 公式

### 标准公式

```
我将通过 [Key Results] 来衡量 [Objective] 的完成
```

### 示例

```markdown
## Objective：提升用户活跃度

我将通过以下关键结果来衡量目标的完成：

- KR1：DAU 从 5 万提升到 15 万
- KR2：用户日均使用时长从 10 分钟提升到 20 分钟
- KR3：7 日留存率从 40% 提升到 60%
```

## OKR vs KPI

### 对比

| 维度 | OKR | KPI |
|------|-----|-----|
| 定义 | 目标管理系统 | 绩效指标 |
| 目的 | 挑战目标、创新 | 维持运营、考核 |
| 公开性 | 全员公开 | 通常保密 |
| 挑战度 | 有挑战（60-70% 完成率） | 需要达成（100%） |
| 与绩效挂钩 | 不直接挂钩 | 直接挂钩 |
| 周期 | 季度/月度 | 年度/季度 |

### 关系

```
战略目标
    │
    ├── OKR：实现突破性目标
    │       (提升 50% 活跃度)
    │
    └── KPI：维持日常运营
            (保持 99.9% 可用性)
```

## 制定好的 OKR

### 好的 Objective

#### 标准

1. **激励人心**：激发团队热情
2. **有挑战性**：超出舒适区
3. **可理解**：所有人都能理解
4. **可行动**：团队可以影响

#### 示例

```
❌ 不好的 Objective：
- 提高收入（太模糊）
- 做 100 个功能（不是目标）
- 大家一起努力（没有方向）

✓ 好的 Objective：
- 打造用户最爱的协作工具
- 建立行业领先的推荐系统
- 实现爆发式用户增长
```

### 好的 Key Result

#### 标准

1. **可衡量**：有具体数字
2. **有时限**：有完成时间
3. **有挑战**：需要努力才能达成
4. **可影响**：团队可以影响结果

#### 公式

```
[动作] + [指标] + [从 X 到 Y] + [时间]
```

#### 示例

```markdown
## 好的 KR 示例

**格式**：动作 + 指标 + 从X到Y + 时间

✓ 好的 KR：
- KR1：在 Q2 将 MAU 从 10 万提升到 30 万
- KR2：在 Q3 将用户满意度 NPS 从 30 提升到 50
- KR3：在 Q4 将获客成本从 50 元降低到 30 元

❌ 不好的 KR：
- 提高用户活跃度（不可衡量）
- 上线新功能（是任务不是结果）
- 做到最好（没有具体标准）
```

### KR 类型

| 类型 | 示例 |
|------|------|
| 增长型 | MAU 从 10 万到 30 万 |
| 提升型 | 留存率从 40% 到 60% |
| 降本型 | 成本降低 30% |
| 质量型 | NPS 从 30 到 50 |
| 进度型 | 完成功能上线 |
| 布尔型 | 完成/未完成 |

## OKR 周期

### 标准周期

```
年度 OKR（年初制定）
    │
    ├── Q1 OKR（季度初制定）
    │       │
    │       ├── 周跟进
    │       ├── 月度检查
    │       └── 季度复盘
    │
    ├── Q2 OKR
    │
    ├── Q3 OKR
    │
    └── Q4 OKR
```

### 时间分配

| 活动 | 时间占比 |
|------|---------|
| 制定 OKR | 5% |
| 周跟进 | 10% |
| 月度检查 | 5% |
| 季度复盘 | 5% |
| 日常执行 | 75% |

## OKR 流程

### 1. 制定阶段

```
公司战略 → 公司 OKR → 部门 OKR → 团队 OKR → 个人 OKR
```

#### 制定流程

```
1. 回顾战略
     ↓
2. 草拟 OKR
     ↓
3. 向上对齐
     ↓
4. 向下分解
     ↓
5. 横向协调
     ↓
6. 最终确定
```

### 2. 执行阶段

#### 周例会

```markdown
## OKR 周会模板

### 1. 本周进展
- KR1 进度：X/Y（Z%）
- KR2 进度：X/Y（Z%）
- KR3 进度：X/Y（Z%）

### 2. 关键成果
- 完成了什么？
- 有什么亮点？

### 3. 问题与风险
- 遇到什么问题？
- 需要什么支持？

### 4. 下周计划
- 重点做什么？
```

#### 月度检查

```markdown
## OKR 月度检查

### 1. 进度评估
- OKR 整体完成度
- 各 KR 完成情况

### 2. 趋势分析
- 进度是否正常？
- 是否需要调整？

### 3. 问题分析
- 遇到什么问题？
- 根本原因是什么？

### 4. 行动计划
- 如何追赶进度？
- 是否需要调整 OKR？
```

### 3. 复盘阶段

#### 季度复盘

```markdown
## OKR 季度复盘

### 1. 结果评估
| KR | 目标值 | 实际值 | 完成率 | 评分 |
|----|--------|--------|--------|------|
| KR1 | 300万 | 280万 | 93% | 0.9 |
| KR2 | 60% | 55% | 92% | 0.9 |
| KR3 | 30元 | 35元 | 117% | 0.7 |

### 2. 成功经验
- 做对了什么？
- 如何复制？

### 3. 失败教训
- 哪里出了问题？
- 如何改进？

### 4. 改进计划
- 下季度如何做得更好？
```

## OKR 评分

### 评分标准

```
1.0：超额完成（100%+）
0.7：完成目标（70%-100%）
0.5：部分完成（50%-70%）
0.3：进展有限（30%-50%）
0.1：几乎没有进展（<30%）
```

### 理想分数

```
目标完成率：60-70%（0.6-0.7 分）

太高（>0.8）：目标太简单
太低（<0.4）：目标太难或执行有问题
适中（0.6-0.7）：目标有挑战，激发了潜力
```

### 评分示例

```markdown
## OKR 评分示例

### O1：实现用户规模化增长

**KR1**：MAU 从 100 万增长到 300 万
- 实际：280 万
- 完成率：93%
- 评分：0.9

**KR2**：获客成本从 50 元降低到 30 元
- 实际：35 元
- 完成率：117%（未达成目标值）
- 评分：0.7

**KR3**：7 日留存率从 40% 提升到 60%
- 实际：55%
- 完成率：75%
- 评分：0.7

**Objective 平均分**：(0.9 + 0.7 + 0.7) / 3 = 0.77
```

## OKR 常见问题

### 问题1：OKR 与绩效挂钩

**问题**：将 OKR 直接用于绩效考核

**后果**：
- 员工会降低目标难度
- 失去挑战性
- 不敢设定有风险的目标

**解决**：OKR 与绩效解耦，OKR 用于目标管理，绩效考核综合评估

### 问题2：OKR 数量过多

**问题**：O 或 KR 数量太多

**后果**：
- 精力分散
- 重点不突出
- 难以跟踪

**解决**：
- Objective：2-5 个
- Key Result：每个 O 对应 2-5 个 KR

### 问题3：KR 不可衡量

**问题**：KR 没有具体数字

**示例**：
```
❌ 提高用户满意度
✓ NPS 从 30 提升到 50
```

**解决**：确保每个 KR 都有具体数字和时间

### 问题4：KR 是任务而非结果

**问题**：把要做的事当成了 KR

**示例**：
```
❌ 上线搜索功能
✓ 搜索使用率达到 50%
```

**解决**：KR 应该是结果（Outcome），不是产出（Output）

### 问题5：没有定期跟进

**问题**：制定了 OKR 就不再看

**后果**：
- 流于形式
- 没有实际作用

**解决**：
- 每周跟进
- 每月检查
- 季度复盘

## OKR 最佳实践

### 1. 公开透明

**原则**：全员可以看到所有人的 OKR

**好处**：
- 促进协作
- 发现依赖
- 增强责任感

### 2. 自下而上

**原则**：不只是自上而下分解，也要自下而上对齐

**比例**：
- 自上而下：60%
- 自下而上：40%

### 3. 挑战适度

**原则**：目标要有挑战但可达成

**标准**：60-70% 完成率

### 4. 持续跟踪

**原则**：不是定完就结束

**方法**：
- 每周同步进度
- 每月深度检查
- 季度全面复盘

### 5. 灵活调整

**原则**：根据实际情况调整

**时机**：
- 市场环境变化
- 公司战略调整
- 发现目标不合理

## 总结

OKR 的核心是：

1. **聚焦**：聚焦最重要的目标
2. **对齐**：上下左右对齐
3. **追踪**：持续跟踪进度
4. **透明**：全员公开透明
5. **挑战**：设定有挑战的目标
6. **学习**：通过复盘持续改进
