# 用户中心设计（UCD）

## 概述

用户中心设计（User-Centered Design, UCD）是一种将用户需求、期望和限制置于设计过程核心的设计方法论。

## 核心原则

### 1. 以用户为出发点

- 设计决策基于用户研究
- 用户参与设计过程
- 持续收集用户反馈

### 2. 迭代设计

```
研究 → 设计 → 原型 → 测试 → 评估
  ↑                                  ↓
  └──────────────────────────────────┘
```

### 3. 全流程参与

从需求分析到最终发布，用户参与每个阶段。

## UCD 流程

### 阶段一：理解用户

#### 用户研究方法

| 方法 | 适用场景 | 优势 | 劣势 |
|------|---------|------|------|
| 用户访谈 | 深度了解需求 | 信息丰富 | 样本有限 |
| 问卷调查 | 大规模数据收集 | 数据量大 | 深度不足 |
| 用户观察 | 真实行为分析 | 客观真实 | 耗时耗力 |
| 焦点小组 | 群体讨论 | 激发创意 | 群体思维 |

#### 用户画像（Persona）

```markdown
## 用户画像模板

**姓名**：张三
**年龄**：28岁
**职业**：产品经理
**技术熟练度**：高

**目标**：
- 提高工作效率
- 更好地管理团队

**痛点**：
- 信息过载
- 跨部门沟通困难

**行为特征**：
- 移动端使用频繁
- 重视数据可视化
```

### 阶段二：定义需求

#### 用户需求层次

```
            △ 忘我需求（Delighters）
           △△ 表达需求（Performance）
          △△△ 基本需求（Basic）
```

#### 需求优先级矩阵

| 紧急度 \ 重要度 | 重要 | 不重要 |
|---------------|------|--------|
| 紧急 | 立即做 | 计划做 |
| 不紧急 | 安排做 | 不做 |

### 阶段三：设计方案

#### 信息架构设计

```
首页
├── 产品
│   ├── 产品列表
│   └── 产品详情
├── 解决方案
│   ├── 行业方案
│   └── 场景方案
└── 关于我们
    ├── 公司介绍
    └── 联系方式
```

#### 交互设计原则

1. **一致性**：相似操作有相似反馈
2. **可见性**：操作结果清晰可见
3. **反馈性**：及时反馈用户操作
4. **容错性**：允许用户撤销操作
5. **效率性**：支持快捷操作

### 阶段四：原型验证

#### 原型类型

| 保真度 | 适用阶段 | 特点 |
|--------|---------|------|
| 低保真 | 早期验证 | 快速、低成本 |
| 中保真 | 交互验证 | 有基本交互 |
| 高保真 | 最终验证 | 接近真实产品 |

#### 可用性测试流程

```
1. 制定测试计划
   ↓
2. 招募测试用户
   ↓
3. 准备测试环境
   ↓
4. 执行测试任务
   ↓
5. 收集反馈数据
   ↓
6. 分析优化建议
```

### 阶段五：迭代优化

#### 优化循环

1. **数据收集**：用户行为数据、反馈数据
2. **问题识别**：发现用户体验问题
3. **方案设计**：提出优化方案
4. **A/B 测试**：验证方案效果
5. **全量发布**：推广有效方案

## 用户旅程地图

### 模板结构

```
┌─────────────────────────────────────────────────────┐
│ 阶段        │ 认知 → 考虑 → 购买 → 使用 → 忠诚    │
├─────────────────────────────────────────────────────┤
│ 用户行为    │                                      │
├─────────────────────────────────────────────────────┤
│ 触点        │                                      │
├─────────────────────────────────────────────────────┤
│ 情绪曲线    │                                      │
├─────────────────────────────────────────────────────┤
│ 痛点        │                                      │
├─────────────────────────────────────────────────────┤
│ 机会点      │                                      │
└─────────────────────────────────────────────────────┘
```

### 示例：电商购物旅程

| 阶段 | 用户行为 | 情绪 | 痛点 | 机会 |
|------|---------|------|------|------|
| 浏览 | 搜索商品 | 期待 | 找不到想要的 | 智能推荐 |
| 选择 | 对比商品 | 纠结 | 信息不全 | 对比工具 |
| 下单 | 填写信息 | 焦虑 | 流程繁琐 | 一键购买 |
| 收货 | 等待物流 | 着急 | 不知进度 | 实时追踪 |
| 使用 | 体验商品 | 满足 | 不会操作 | 使用指南 |

## 用户参与度模型

### 用户参与度金字塔

```
           △ 倡导者
          △△ 活跃用户
         △△△ 注册用户
        △△△△ 访问用户
       △△△△△ 潜在用户
```

### 提升参与度策略

1. **新手引导**：降低入门门槛
2. **成就系统**：激励持续使用
3. **社交功能**：增强用户连接
4. **个性化**：提供定制体验

## UCD 的组织实施

### 团队角色

- **UX 设计师**：负责用户体验设计
- **UI 设计师**：负责视觉界面设计
- **用户研究员**：负责用户研究和测试
- **产品经理**：负责产品规划和需求
- **开发工程师**：负责技术实现

### 流程整合

```
产品需求 → 用户研究 → 设计方案 → 技术评审 → 开发实现 → 用户测试 → 发布上线
    ↑                                                                   ↓
    └───────────────────────────────────────────────────────────────────┘
```

## 常见误区

### 1. 把自己当用户

**问题**：用自己的需求代替用户需求
**解决**：基于真实用户研究，避免主观臆断

### 2. 过度设计

**问题**：功能过度复杂，用户难以理解
**解决**：聚焦核心需求，保持简洁

### 3. 一次到位

**问题**：追求完美，忽视迭代
**解决**：快速验证，持续优化

## 实践案例

### 案例：Airbnb 的 UCD 实践

**用户研究**：
- 创始人亲自住进房东家里
- 深度了解用户痛点

**关键洞察**：
- 房东担心陌生人入住
- 房客担心安全和卫生

**设计决策**：
- 建立信任体系（评价、认证）
- 提供保险保障
- 优化照片展示

## 总结

用户中心设计不是一次性工作，而是持续的过程。成功的关键在于：

1. **真正理解用户**：深入用户场景
2. **持续验证假设**：用数据说话
3. **快速迭代优化**：小步快跑
4. **团队共同参与**：全员用户思维
