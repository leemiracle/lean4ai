# 决策框架

## 概述

决策框架是帮助产品经理系统化思考和做出决策的工具和方法集合，能够提高决策质量和效率。

## 决策类型

### 按重要性分类

| 类型 | 特点 | 示例 | 决策方式 |
|------|------|------|---------|
| 战略决策 | 影响深远 | 产品方向 | 高层决策 |
| 战术决策 | 中等影响 | 功能优先级 | 团队决策 |
| 执行决策 | 影响有限 | 界面细节 | 个人决策 |

### 按确定性分类

| 类型 | 确定性 | 决策方法 |
|------|--------|---------|
| 确定性决策 | 结果可预测 | 分析比较 |
| 风险决策 | 概率可知 | 期望值分析 |
| 不确定决策 | 概率不可知 | 情景分析 |

## 决策流程

### 标准流程

```
识别问题
    ↓
收集信息
    ↓
定义目标
    ↓
生成方案
    ↓
评估方案
    ↓
选择方案
    ↓
执行实施
    ↓
评估反馈
```

### 详细步骤

#### 1. 识别问题

```markdown
## 问题定义模板

**现象**：观察到什么问题？

**影响**：问题带来什么影响？

**原因**：问题产生的根本原因是什么？

**目标**：希望达成什么结果？
```

#### 2. 收集信息

```markdown
## 信息收集清单

- [ ] 用户数据
- [ ] 业务数据
- [ ] 竞品信息
- [ ] 技术约束
- [ ] 资源情况
- [ ] 风险因素
```

#### 3. 定义目标

**SMART 原则**：
- Specific：具体
- Measurable：可衡量
- Achievable：可实现
- Relevant：相关
- Time-bound：有时限

#### 4. 生成方案

**方法**：
- 头脑风暴
- 类比思维
- 逆向思维
- 组合创新

#### 5. 评估方案

**维度**：
- 可行性
- 价值
- 成本
- 风险

## 常用决策框架

```

│    内部因素     │    外部因素     │
├────────────────┼────────────────┤
│ Strengths      │ Opportunities  │
│ 优势            │ 机会            │
│                │                │
│ · ...          │ · ...          │
│ · ...          │ · ...          │
├────────────────┼────────────────┤
│ Weaknesses     │ Threats        │
│ 劣势            │ 威胁            │
│                │                │
│ · ...          │ · ...          │

```

```
                  是否做功能X？
                       │

            做功能X            不做
              │                 │
```

### 5. 成本效益分析

```markdown
## 成本效益分析

### 方案A
**成本**：
- 开发成本：2周 × 3人 = 6人周
- 机会成本：延后其他功能

**收益**：
- 预期提升转化率：5%
- 预期增加收入：¥100,000/月

**ROI**：(100,000 - 50,000) / 50,000 = 100%

### 方案B
**成本**：
- 开发成本：1周 × 2人 = 2人周
- 机会成本：较小

**收益**：
- 预期提升转化率：2%
- 预期增加收入：¥40,000/月

**ROI**：(40,000 - 20,000) / 20,000 = 100%

### 结论
两个方案 ROI 相同，方案A收益更高但风险更大
```

### 6. 期望值分析

```markdown
## 期望值计算

**公式**：期望值 = Σ (结果 × 概率)

### 方案A
- 成功（60%）：收益 100 万
- 失败（40%）：损失 20 万
- 期望值 = 100 × 0.6 + (-20) × 0.4 = 52 万

### 方案B
- 成功（80%）：收益 60 万
- 失败（20%）：损失 10 万
- 期望值 = 60 × 0.8 + (-10) × 0.2 = 46 万

### 结论
方案A 期望值更高，但风险更大
```

### 7. RACI 矩阵

```markdown
## RACI 矩阵

| 决策项 | PM | 技术 | 设计 | 业务 | 老板 |
|-------|----|----|------|------|------|
| 产品方向 | A | C | C | C | R |
| 功能优先级 | R/A | C | C | C | I |
| 技术方案 | C | R/A | I | I | I |
| 设计方案 | A | C | R | C | I |
| 发布时间 | R/A | C | C | C | I |

R = Responsible（负责执行）
A = Accountable（最终负责）
C = Consulted（被咨询）
I = Informed（被告知）
```

## 决策偏差

### 常见认知偏差

| 偏差 | 描述 | 示例 | 应对 |
|------|------|------|------|
| 确认偏差 | 倾向于寻找支持现有观点的信息 | 只看支持自己的数据 | 主动寻找反面证据 |
| 沉没成本谬误 | 已投入的成本影响决策 | 因为已经投入很多而继续 | 关注未来成本收益 |
| 锚定效应 | 过度依赖第一个信息 | 被第一个报价影响 | 多方比较 |
| 可得性偏差 | 容易想到的事情被认为更重要 | 最近的问题影响判断 | 系统化分析 |
| 幸存者偏差 | 只看成功案例 | 只分析成功产品 | 分析失败案例 |

### 减少偏差的方法

1. **数据驱动**：基于数据而非直觉
2. **多元视角**：听取不同意见
3. **结构化思维**：使用决策框架
4. **延迟判断**：不急于下结论
5. **持续学习**：了解认知偏差

## 团队决策

### 决策方式

| 方式 | 描述 | 适用场景 |
|------|------|---------|
| 独裁决策 | 一人决定 | 紧急情况 |
| 协商决策 | 征求意见后决定 | 重要决策 |
| 投票决策 | 多数人决定 | 方案选择 |
| 共识决策 | 所有人同意 | 重大决策 |

### 达成共识的方法

```markdown
## 共识决策流程

1. 充分讨论
   - 每人表达观点
   - 理解不同立场

2. 识别分歧
   - 分歧点是什么？
   - 根本原因是什么？

3. 寻找方案
   - 有没有折中方案？
   - 有没有新方案？

4. 达成共识
   - 所有人同意
   - 或至少不反对

5. 承诺执行
   - 共同承诺
   - 全力支持
```

## 总结

好的决策框架应该：

1. **结构化**：有清晰的流程和步骤
2. **数据驱动**：基于事实和数据
3. **多方参与**：听取不同观点
4. **减少偏差**：识别和避免认知偏差
5. **可复盘**：记录和回顾决策
6. **持续改进**：不断优化决策方法
