# 数据驱动决策

## 概述

数据驱动决策（Data-Driven Decision Making, DDDM）是一种基于数据分析和事实证据来做决策的方法论，而非仅凭直觉或经验。

## 核心理念

### 传统决策 vs 数据驱动决策

| 维度 | 传统决策 | 数据驱动决策 |
|------|---------|-------------|
| 依据 | 直觉、经验 | 数据、证据 |
| 过程 | 主观判断 | 客观分析 |
| 可重复性 | 低 | 高 |
| 可解释性 | 弱 | 强 |
| 风险 | 较高 | 较低 |

### 决策三角

```
        数据
         △
        ╱ ╲
       ╱   ╲
      ╱     ╲
     ╱       ╲
    ╱         ╲
 业务理解 ───── 分析能力
```

## 数据类型

### 1. 定量数据

**定义**：可以用数字表示的数据

**类型**：
- 连续型：时间、金额、温度
- 离散型：用户数、订单数

**示例**：
- DAU：10,000
- 转化率：5.2%
- 客单价：¥128

### 2. 定性数据

**定义**：非数字的描述性数据

**类型**：
- 用户反馈
- 访谈记录
- 开放式问卷

**示例**：
- "搜索功能很难用"
- "希望能增加夜间模式"

### 3. 数据来源

| 来源 | 类型 | 示例 |
|------|------|------|
| 行为数据 | 定量 | 页面浏览、点击、停留时长 |
| 交易数据 | 定量 | 订单、支付、退款 |
| 用户反馈 | 定性 | 客服工单、用户评论 |
| 第三方数据 | 混合 | 行业报告、竞品数据 |

## 数据指标体系

### 1. 北极星指标（North Star Metric）

**定义**：最能反映产品核心价值的单一指标

**特点**：
- 反映用户价值
- 与商业价值相关
- 可衡量、可行动

**示例**：

| 产品 | 北极星指标 |
|------|-----------|
| Spotify | 收听时长 |
| Airbnb | 预订天数 |
| Slack | 发送消息数 |
| Medium | 阅读时间 |

### 2. OMTM 模型

**One Metric That Matters**：每个阶段聚焦一个核心指标

| 阶段 | 关注指标 |
|------|---------|
| 验证期 | 留存率 |
| 增长期 | 增长率 |
| 变现期 | ARPU |
| 成熟期 | LTV |

### 3. 指标金字塔

```

      ╱    (过程指标)      ╲
     ╱                     ╲

```

**示例**：

## 数据分析方法

### 1. 描述性分析

**目的**：描述发生了什么

**方法**：
- 统计汇总
- 趋势分析
- 分布分析

**示例**：
```
过去 30 天：
- 新增用户：10,000
- 日均活跃：8,000
- 平均留存率：45%
```

### 2. 诊断性分析

**目的**：理解为什么发生

**方法**：
- 下钻分析
- 相关性分析
- 对比分析

**示例**：
```
留存率下降 5%，原因分析：
- 新用户渠道变化
- 产品体验问题
- 竞品影响
```

### 3. 预测性分析

**目的**：预测将来会发生什么

**方法**：
- 时间序列预测
- 回归分析
- 机器学习模型

**示例**：
```
基于历史数据预测：
- 下月 DAU：12,000
- 转化率：5.5%
- 流失率：8%
```

### 4. 指导性分析

**目的**：建议应该做什么

**方法**：
- 优化算法
- 推荐系统
- 决策支持系统

**示例**：
```
系统建议：
- 增加搜索功能投入
- 优化注册流程
- 调整定价策略
```

## 常用分析模型

### 1. 漏斗分析

```
访问（100%）
   │
   ↓ 转化 60%
注册（60%）
   │
   ↓ 转化 40%
激活（24%）
   │
   ↓ 转化 50%
付费（12%）
   │
   ↓ 转化 80%
复购（9.6%）
```

**应用**：
- 识别转化瓶颈
- 优化关键步骤
- 提升整体转化率

### 2. 同期群分析（Cohort Analysis）

```
         第1周  第2周  第3周  第4周
1月 cohort  100%   40%   30%   25%
2月 cohort  100%   45%   35%   28%
3月 cohort  100%   50%   40%   32%
```

**应用**：
- 分析用户留存趋势
- 评估产品改进效果
- 发现用户行为变化

### 3. RFM 分析

| 维度 | 说明 | 指标 |
|------|------|------|
| Recency | 最近一次购买时间 | 天数 |
| Frequency | 购买频率 | 次数/月 |
| Monetary | 购买金额 | 金额 |

**用户分层**：

| 类型 | R | F | M | 策略 |
|------|---|---|---|------|
| 重要价值 | 高 | 高 | 高 | VIP服务 |
| 重要发展 | 高 | 低 | 高 | 提升频次 |
| 重要保持 | 低 | 高 | 高 | 激活召回 |
| 一般客户 | 中 | 中 | 中 | 常规维护 |

### 4. 留存分析

```
留存率表格

           Day1  Day7  Day14  Day30
1月1日     100%   40%   25%    15%
1月2日     100%   42%   27%    -
1月3日     100%   45%   -      -
1月4日     100%   -     -      -
```

**关键指标**：
- 次日留存：40-60%
- 7日留存：20-40%
- 30日留存：10-20%

### 5. 用户分群分析

**分群维度**：
- 人口属性：年龄、性别、地域
- 行为特征：活跃度、使用频次
- 价值分层：付费、免费、高价值

**应用**：
- 精准营销
- 个性化推荐
- 差异化服务

## 常见陷阱

### 1. 幸存者偏差

**问题**：只分析"幸存"的数据

**示例**：只分析活跃用户，忽视流失用户

**解决**：考虑完整的数据样本

### 2. 相关不等于因果

**问题**：看到相关性就认为是因果关系

**示例**：冰淇淋销量和溺水事故正相关，但不是因果关系

**解决**：
- 进行控制实验
- 寻找因果机制
- 考虑混淆变量

### 3. 数据选择偏差

**问题**：选择性使用支持假设的数据

**解决**：
- 使用完整数据
- 承认反面数据
- 进行交叉验证

### 4. 过度依赖数据

**问题**：忽视直觉和经验

**解决**：
- 数据 + 直觉 = 更好的决策
- 数据提供依据，直觉提供判断
- 两者结合使用
