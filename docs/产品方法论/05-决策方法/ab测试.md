# A/B 测试

## 概述

A/B 测试是一种对比实验方法，通过将用户随机分组，对比不同方案的表现，以数据驱动的方式做出决策。

## 基本原理

### 核心概念

```
用户流量
    │
    ├── 随机分流
    │       │
    │       ├──→ A组（对照组）──→ 方案A ──→ 数据收集
    │       │
    │       └──→ B组（实验组）──→ 方案B ──→ 数据收集
    │                                        │
    └────────────────────────────────────────┘
                                        结果对比
```

### 关键术语

| 术语 | 定义 | 说明 |
|------|------|------|
| 对照组 | 使用现有方案的组 | 作为基准对比 |
| 实验组 | 使用新方案的组 | 测试新方案效果 |
| 样本量 | 参与实验的用户数 | 影响统计显著性 |
| 转化率 | 完成目标动作的比例 | 核心评估指标 |
| 统计显著性 | 结果不是随机产生的概率 | 通常要求 > 95% |

## A/B 测试流程

### 完整流程

```
1. 提出假设
     ↓
2. 确定指标
     ↓
3. 设计实验
     ↓
4. 计算样本量
     ↓
5. 实施分流
     ↓
6. 收集数据
     ↓
7. 分析结果
     ↓
8. 得出结论
     ↓
9. 发布决策
```

### 详细步骤

#### 步骤1：提出假设

```markdown
## 假设模板

**观察**：[当前问题/现象]

**假设**：如果我们 [做什么改变]

**预期**：那么 [会发生什么结果]

**原因**：因为 [为什么会有这个结果]

## 示例

**观察**：注册转化率只有 30%

**假设**：如果我们简化注册流程，从 5 步减少到 3 步

**预期**：注册转化率会提升到 40%

**原因**：因为减少了用户流失点，降低了完成门槛
```

#### 步骤2：确定指标

**指标类型**：

| 类型 | 说明 | 示例 |
|------|------|------|
| 核心指标 | 直接衡量假设的指标 | 转化率 |
| 辅助指标 | 支持分析的指标 | 停留时长 |
| 护栏指标 | 确保没有负面影响 | 退货率 |

**选择原则**：
- 可衡量
- 可归因
- 灵敏度
- 稳定性

#### 步骤3：设计实验

```markdown
## 实验设计

### 实验名称
注册流程优化实验_v1

### 实验方案
- A组（对照组）：5步注册流程
- B组（实验组）：3步注册流程

### 分流方式
- 随机分流
- A:B = 50:50

### 实验周期
- 预计 2 周

### 目标样本量
- 每组 10,000 用户
```

#### 步骤4：计算样本量

**公式**：

```
样本量 = (Z_α/2 + Z_β)² × 2 × p(1-p) / d²

其中：
- Z_α/2：显著性水平对应的Z值（95%置信度 ≈ 1.96）
- Z_β：统计功效对应的Z值（80%功效 ≈ 0.84）
- p：基准转化率
- d：预期提升（最小检测效应）
```

**快速估算**：

| 基准转化率 | 预期提升 | 所需样本/组 |
|-----------|---------|------------|
| 10% | +10% (相对) | ~15,000 |
| 10% | +20% (相对) | ~4,000 |
| 5% | +10% (相对) | ~30,000 |
| 5% | +20% (相对) | ~8,000 |

#### 步骤5：实施分流

**分流方法**：

```
分流逻辑：
if hash(user_id + experiment_id) % 100 < 50:
    group = "A"
else:
    group = "B"
```

**分流原则**：
- 随机性：用户被分到各组的概率相等
- 一致性：同一用户始终分到同一组
- 独立性：不同实验的分流相互独立

#### 步骤6：收集数据

**数据收集**：

```markdown
## 数据收集清单

### 用户数据
- [ ] 用户ID
- [ ] 分组信息
- [ ] 设备信息
- [ ] 时间戳

### 行为数据
- [ ] 页面访问
- [ ] 按钮点击
- [ ] 表单提交
- [ ] 转化事件

### 业务数据
- [ ] 订单数据
- [ ] 支付数据
- [ ] 客服数据
```

#### 步骤7：分析结果

**统计检验**：

```markdown
## 结果分析模板

### 数据汇总
| 指标 | A组 | B组 | 差异 |
|------|-----|-----|------|
| 样本量 | 10,000 | 10,000 | - |
| 转化数 | 3,000 | 3,500 | +500 |
| 转化率 | 30.0% | 35.0% | +5.0% |

### 统计检验
- p-value: 0.001
- 置信区间: [3.2%, 6.8%]
- 统计显著性: 显著（p < 0.05）

### 结论
实验组转化率显著高于对照组，建议全量发布。
```

## 统计基础

### 1. p 值（p-value）

**定义**：在原假设为真的情况下，观察到当前结果或更极端结果的概率

**解释**：
- p < 0.05：结果显著
- p < 0.01：结果非常显著
- p > 0.05：结果不显著

### 2. 置信区间

**定义**：参数真实值的估计范围

**示例**：
```
转化率差异：5.0%
95% 置信区间：[3.2%, 6.8%]

解释：我们有 95% 的信心，真实差异在 3.2% 到 6.8% 之间
```

### 3. 统计功效

**定义**：当存在真实差异时，检测到差异的概率

**建议**：
- 目标功效：80% 或更高
- 功效越高，所需样本量越大

### 4. 第一类错误 vs 第二类错误

| 类型 | 定义 | 风险 |
|------|------|------|
| 第一类错误（α） | 假阳性：没有差异但认为有差异 | 错误地采用无效方案 |
| 第二类错误（β） | 假阴性：有差异但没检测到 | 错过有效方案 |

## 实验类型

### 1. 简单 A/B 测试

**描述**：两组对比

**适用**：单一变量变化

```
A组：原方案
B组：新方案
```

### 2. A/B/N 测试

**描述**：多组对比

**适用**：多个方案比较

```
A组：原方案
B组：新方案1
C组：新方案2
N组：新方案N
```

### 3. 多变量测试（MVT）

**描述**：同时测试多个变量的组合

**适用**：多个元素同时变化

```
元素1: A/B
元素2: X/Y

组合:
A+X, A+Y, B+X, B+Y
```

### 4. 分层实验

**描述**：在不同用户群体中分别实验

**适用**：不同群体可能有不同反应

```
新用户: A/B测试
老用户: A/B测试
VIP用户: A/B测试
```

## 实验设计最佳实践

### 1. 单一变量原则

**原则**：每次只改变一个变量

**原因**：能够清晰归因

**示例**：

```
✓ 正确：
实验1：改变按钮颜色
实验2：改变按钮文案
实验3：改变按钮位置

✗ 错误：
实验1：同时改变按钮颜色、文案、位置
```

### 2. 随机化

**原则**：确保用户随机分配

**方法**：
- 基于用户ID哈希
- 使用随机数生成器

### 3. 同期对照

**原则**：对照组和实验组同时运行

**原因**：避免时间因素影响

### 4. 样本量充足

**原则**：确保样本量足够检测到差异

**风险**：
- 样本量不足：可能漏掉真实差异
- 样本量过大：浪费时间资源

### 5. 避免污染

**原则**：确保各组之间不相互影响

**风险**：
- 用户分享信息
- 网络效应
- 跨设备使用

## 常见问题

### 1. 过早下结论

**问题**：样本量不足就结束实验

**解决**：
- 达到预设样本量再分析
- 使用顺序检验方法

### 2. 多重比较问题

**问题**：同时检验多个指标，增加假阳性

**解决**：
- 使用 Bonferroni 校正
- 聚焦核心指标

### 3. 新奇效应

**问题**：用户因为新鲜感而改变行为

**解决**：
- 延长实验时间
- 观察长期效果

### 4. 辛普森悖论

**问题**：分组结果与总体结果矛盾

**示例**：

```
总体转化率：A组 > B组
但每个细分群体：A组 < B组

原因：A组用户构成不同
```

**解决**：
- 分层分析
- 控制混淆变量

## 实验案例

### 案例1：注册流程优化

```markdown
## 实验背景
当前注册转化率：30%
目标：提升到 40%

## 实验设计
- A组：5步注册
- B组：3步注册

## 实验结果
| 指标 | A组 | B组 | 差异 |
|------|-----|-----|------|
| 样本量 | 10,000 | 10,000 | - |
| 转化率 | 30.0% | 38.0% | +8.0% |
| p-value | 0.001 | 显著 |

## 结论
3步注册流程显著提升转化率，建议全量发布
```

### 案例2：商品详情页改版

```markdown
## 实验背景
用户反馈：商品信息不够直观

## 实验设计
- A组：原设计
- B组：新设计（突出关键信息）

## 实验结果
| 指标 | A组 | B组 | 差异 |
|------|-----|-----|------|
| 样本量 | 20,000 | 20,000 | - |
| 加购率 | 15.0% | 18.0% | +3.0% |
| 跳出率 | 45.0% | 40.0% | -5.0% |
| p-value | 0.002 | 显著 |

## 结论
新设计显著提升加购率，降低跳出率，建议发布
```

## 实验文化

### 1. 实验思维

**原则**：用实验验证假设

**方法**：
- 不凭直觉决策
- 提出可验证的假设
- 用数据说话

### 2. 失败宽容

**原则**：接受实验失败

**态度**：
- 失败也是学习
- 快速迭代
- 持续优化

### 3. 数据驱动

**原则**：基于数据做决策

**方法**：
- 收集充分数据
- 正确分析结果
- 客观评估结论

## 总结

A/B 测试的关键：

1. **明确假设**：清楚要验证什么
2. **科学设计**：遵循实验设计原则
3. **充足样本**：确保统计有效性
4. **正确分析**：使用合适的统计方法
5. **谨慎决策**：基于数据做决策
6. **持续优化**：不断迭代改进
