# 设计思维

## 概述

设计思维（Design Thinking）是一种以人为本的创新方法论，通过深入理解用户需求，创造性地解决问题。

## 五阶段模型

### 流程图

```
同理心 → 定义 → 创意 → 原型 → 测试
   ↑                                  │
   └──────────────────────────────────┘
```

### 阶段详解

#### 1. 同理心（Empathize）

**目标**：深入理解用户

**方法**：
- 用户访谈
- 用户观察
- 用户体验地图
- 沉浸式体验

**关键问题**：
- 用户是谁？
- 用户在做什么？
- 用户在想什么？
- 用户感觉如何？

#### 2. 定义（Define）

**目标**：明确要解决的问题

**方法**：
- 问题陈述
- POV（Point of View）
- HMW（How Might We）

**POV 模板**：

```markdown
## POV 陈述

**用户**：[用户是谁]

**需求**：[用户需要什么]

**洞察**：[我们发现了什么]

## 示例

**用户**：都市白领

**需求**：需要在通勤时高效获取信息

**洞察**：现有方案要么太长要么太浅，无法满足碎片化学习需求
```

**HMW 提问**：

```markdown
## HMW（How Might We）

**格式**：我们如何能够...？

**示例**：
- 我们如何能够帮助白领在5分钟内获取有价值的信息？
- 我们如何能够让信息获取更加个性化？
- 我们如何能够让学习更有趣味性？

**原则**：
- 不要太窄：限制创意
- 不要太宽：无从下手
- 刚刚好：激发创意
```

#### 3. 创意（Ideate）

**目标**：产生大量创意方案

**方法**：
- 头脑风暴
- 最差点子
- 类比思维
- SCAMPER

**头脑风暴规则**：

```markdown
## 头脑风暴规则

1. **延迟判断**：不批评任何点子
2. **鼓励疯狂**：欢迎不切实际的想法
3. **追求数量**：点子越多越好
4. **组合改进**：在他人点子上构建
5. **一个对话**：不同时多人发言
6. **可视化**：把点子写下来
```

**SCAMPER 方法**：

| 字母 | 含义 | 问题 |
|------|------|------|
| S | Substitute（替代） | 能用什么替代？ |
| C | Combine（合并） | 能和什么组合？ |
| A | Adapt（调整） | 能调整什么？ |
| M | Modify（修改） | 能修改什么？ |
| P | Put to other uses（其他用途） | 还能做什么？ |
| E | Eliminate（去除） | 能去除什么？ |
| R | Reverse（重组） | 能重组什么？ |

#### 4. 原型（Prototype）

**目标**：快速验证创意

**类型**：

| 类型 | 特点 | 适用场景 |
|------|------|---------|
| 低保真 | 快速、粗糙 | 早期验证 |
| 中保真 | 有一定细节 | 交互验证 |
| 高保真 | 接近真实 | 最终验证 |

**原则**：
- 快速制作
- 成本低廉
- 足够验证假设
- 允许失败

**方法**：
- 纸质原型
- 线框图
- 可点击原型
- 视频原型

#### 5. 测试（Test）

**目标**：验证原型效果

**方法**：
- 用户测试
- A/B 测试
- 可用性测试
- 焦点小组

**测试清单**：

```markdown
## 可用性测试

### 准备
- [ ] 定义测试任务
- [ ] 准备测试环境
- [ ] 招募测试用户

### 执行
- [ ] 解释测试目的
- [ ] 让用户执行任务
- [ ] 观察用户行为
- [ ] 记录问题和反馈

### 分析
- [ ] 整理问题清单
- [ ] 分类问题优先级
- [ ] 提出改进建议
```

## 设计思维原则

### 1. 以人为本

**原则**：从用户需求出发

**实践**：
- 深入了解用户
- 观察真实行为
- 理解潜在需求

### 2. 拥抱不确定性

**原则**：接受未知和模糊

**实践**：
- 不急于下结论
- 保持开放心态
- 探索多种可能

### 3. 快速迭代

**原则**：快速尝试，快速学习

**实践**：
- 快速原型
- 早期测试
- 持续优化

### 4. 协作共创

**原则**：多元化团队协作

**实践**：
- 跨职能团队
- 共同创造
- 集体智慧

### 5. 行动导向

**原则**：想法要变成行动

**实践**：
- 做中学
- 原型驱动
- 验证假设

## 案例

### 案例1：银行APP 重新设计

```markdown
## 同理心阶段

**用户访谈发现**：
- 用户觉得操作复杂
- 担心安全问题
- 需要更快速的服务

## 定义阶段

**POV**：
作为银行用户，我需要简单安全的操作，
因为我不想在银行事务上花太多时间

**HMW**：
我们如何能让银行APP操作更简单？
我们如何能让用户更信任APP？

## 创意阶段

**解决方案**：
- 简化操作流程
- 生物识别登录
- 智能客服
- 个性化首页

## 原型阶段

**原型类型**：高保真可交互原型

## 测试阶段

**测试结果**：
- 操作时间缩短50%
- 用户满意度提升30%
```

### 案例2：医疗服务优化

```markdown
## 同理心阶段

**观察发现**：
- 患者等待时间长
- 流程不清晰
- 信息不透明

## 定义阶段

**HMW**：
我们如何能减少患者等待时间？
我们如何能让流程更透明？

## 创意阶段

**解决方案**：
- 预约系统
- 实时排队信息
- 电子导诊
- 移动支付

## 测试阶段

**测试结果**：
- 等待时间减少40%
- 患者满意度提升
```

## 总结

设计思维的核心：

1. **同理心**：深入理解用户
2. **定义问题**：明确要解决的问题
3. **发散思维**：产生大量创意
4. **原型验证**：快速验证假设
5. **迭代优化**：持续改进
6. **协作共创**：团队共同创造
