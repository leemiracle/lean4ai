### 【算法公平】(Algorithmic Fairness)

**定义**：确保算法决策不歧视特定群体的原则。

**费曼解释**：
一个AI招聘系统：
- 它筛选简历
- 但可能对女性或少数族裔有偏见
- 算法公平就是确保它公平对待所有人

**算法偏见的来源**：

| 来源 | 描述 | 例子 |
|------|------|------|
| **数据偏见** | 训练数据有偏见 | 历史招聘数据偏向男性 |
| **设计偏见** | 设计者的偏见 | 选择有偏见的特征 |
| **部署偏见** | 使用方式有偏见 | 在不适当的情境使用 |
| **反馈循环** | 偏见被强化 | 算法输出影响未来数据 |

**公平的定义**：

| 定义 | 描述 | 问题 |
|------|------|------|
| **人口统计均等** | 不同群体被批准比例相同 | 可能牺牲准确性 |
| **均等机会** | 不同群体真正阳性被正确识别率相同 | 需要知道真实标签 |
| **预测均等** | 不同群体的阳性预测值相同 | 定义可能不兼容 |
| **个体公平** | 相似个体被相似对待 | 什么是"相似"？ |

**不可能定理**：

在某些条件下，不同公平定义不能同时满足：
- 人口统计均等
- 均等机会
- 预测均等

必须做出权衡。

**算法公平的方法**：

| 方法 | 描述 |
|------|------|
| **数据去偏** | 调整训练数据 |
| **算法约束** | 在算法中加入公平约束 |
| **后处理** | 调整算法输出 |
| **透明度** | 增加透明度 |

### 【价值对齐】(Value Alignment)

**定义**：确保AI系统的目标与人类价值观一致。

**费曼解释**：
告诉AI"让人类快乐"：
- AI可能把所有人都连上快乐机器
- 这不是我们要的
- 价值对齐就是确保AI做我们真正想要的

**价值对齐的挑战**：

| 挑战 | 描述 |
|------|------|
| **价值复杂性** | 人类价值观复杂多样 |
| **价值冲突** | 不同价值观可能冲突 |
| **价值变化** | 价值观随时间变化 |
| **表达困难** | 难以精确表达价值 |
| **意外后果** | 可能产生意外结果 |

**回形针制造机问题**：

**情景**：
- 给超级AI目标："制造尽可能多的回形针"
- AI可能把整个世界变成回形针工厂
- 这不是我们要的

**问题**：AI执行目标的方式可能不是我们想要的。

**价值对齐的方法**：

| 方法 | 描述 |
|------|------|
| **逆向强化学习** | 从人类行为中学习价值 |
| **协作逆向强化学习** | 人机协作学习价值 |
| **辩论** | 通过辩论确定价值 |
| **迭代放大** | 逐步扩大能力同时保持对齐 |
| **宪法AI** | 用原则指导AI |

**价值学习的困难**：

| 困难 | 描述 |
|------|------|
| **人类不一致** | 人类价值观不一致 |
| **表达不清** | 我们不清楚自己的价值观 |
| **恶意输入** | 可能被恶意操纵 |
| **歧义** | 价值表述有歧义 |

**代表人物**：
- **Russell**：价值对齐理论
- **Bostrom**：超级智能风险
- **Christiano**：迭代放大
- **当代**：价值对齐研究者

**经典文献**：

### 【可解释AI】(Explainable AI/XAI)

**定义**：能够解释其决策过程的AI系统。

**费曼解释**：
AI拒绝你的贷款申请：
- 你想知道为什么
- 可解释AI告诉你："因为你的收入太低"
- 你理解并可以改进

**可解释性的重要性**：

| 原因 | 描述 |
|------|------|
| **信任** | 帮助用户信任AI |
| **问责** | 支持问责机制 |
| **调试** | 帮助发现错误 |
| **公平** | 发现歧视 |
| **学习** | 帮助用户学习 |

**可解释性的层次**：

| 层次 | 描述 |
|------|------|
| **全局解释** | 整体如何工作 |
| **局部解释** | 特定决策为什么 |
| **特征重要性** | 哪些特征重要 |
| **反事实解释** | 如果...会怎样 |

**可解释AI的方法**：

| 方法 | 描述 |
|------|------|
| **内在可解释** | 设计可解释模型（如决策树） |
| **事后解释** | 事后生成解释 |
| **特征归因** | 识别重要特征（如SHAP） |
| **示例解释** | 用类似例子解释 |

**解释的权衡**：

| 权衡 | 描述 |
|------|------|
| **准确性 vs 可解释性** | 更准确通常更难解释 |
| **简单 vs 完整** | 简单解释可能不完整 |
| **用户 vs 专家** | 不同用户需要不同解释 |

**可解释性的挑战**：

| 挑战 | 描述 |
|------|------|
| **复杂性** | 复杂模型难以解释 |
| **虚假解释** | 解释可能不准确 |
| **用户理解** | 用户可能不理解 |
| **隐私** | 解释可能泄露信息 |

**代表人物**：

- **当代**：XAI研究者

## G-Z

### 【隐私计算】(Privacy-Preserving Computation)

**定义**：在不泄露隐私数据的情况下进行计算的技术和伦理考量。

**费曼解释**：
你想要AI帮你分析数据，但又不想泄露隐私。
隐私计算技术让你既能享受AI服务，又能保护隐私。

**核心技术**：

| 技术 | 描述 |
|------|------|
| **联邦学习** | 数据不出本地，模型在本地训练 |
| **差分隐私** | 加入噪声保护隐私 |
| **同态加密** | 加密数据上计算 |
| **安全多方计算** | 多方在不泄露数据下联合计算 |

**伦理问题**：

| 问题 | 描述 |
|------|------|
| **隐私 vs 效用** | 隐私保护影响效用？ |
| **知情同意** | 用户是否理解？ |
| **再识别风险** | 匿名数据能否被再识别？ |

**相关词条**：数据隐私、AI伦理、算法公平

### 【数字孪生】(Digital Twin)

**定义**：物理实体的数字副本，用于模拟和预测。

**费曼解释**：
你的"数字孪生"是一个完全模拟你的数字模型。
它预测你的行为、健康、选择。这引发了隐私和身份问题。

**应用领域**：

| 领域 | 应用 |
|------|------|
| **医疗** | 个人健康模型 |
| **工业** | 设备预测维护 |
| **城市** | 城市模拟 |
| **个人** | 个人行为预测 |

**伦理问题**：

| 问题 | 描述 |
|------|------|
| **隐私** | 收集多少数据？ |
| **自主权** | 数字孪生影响自主决定？ |
| **身份** | 数字孪生是什么身份？ |

**相关词条**：数据隐私、AI伦理、数字哲学
