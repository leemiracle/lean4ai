# 职业词典使用指南

## 📖 快速开始

### 第一次使用？
1. 先读 **README.md** - 了解词典整体框架
2. 再看 **职业索引.md** - 找到您感兴趣的职业
3. 然后打开对应模块文件 - 查看详细信息

---

## 🎯 四大使用场景

### 场景一：职业规划

**适用人群**：学生、职场新人、想提升的人

**使用步骤**：

1. **自我评估**
   - 你的兴趣是什么？（喜欢与人打交道 vs 喜欢技术）
   - 你的能力如何？（数学好 vs 语言好 vs 动手能力强）
   - 你的学历如何？（高中、大专、本科、硕士、博士）

2. **查找职业**
   - 打开职业索引.md
   - 按领域浏览（技术、金融、医疗、教育...）
   - 按薪资筛选（<10万、10-20万、20-50万、50-100万、100万+）

3. **深度了解**
   - 查看目标职业的详细模块
   - 看"费曼学习法解释" - 理解工作是啥
   - 看"核心技能" - 需要学会什么
   - 看"薪资范围" - 能赚多少钱
   - 看"发展路径" - 未来能去哪

4. **对比选择**
   - 对比2-3个感兴趣的职业
   - 看技能要求、薪资、发展前景
   - 选择最适合自己的

5. **制定计划**
   - 根据技能要求制定学习计划
   - 看需要什么学历、证书
   - 设定短期（1年）、中期（3年）、长期（5年）目标

**举例**：
```
我是计算机专业本科生，喜欢编程，想找高薪工作。
步骤：
1. 查看"技术"领域的职业索引
2. 筛选薪资50-100万以上的岗位
3. 发现：机器学习工程师（90-150万）、Go后端工程师（65-120万）、前端架构师（80-150万）
4. 对比后发现机器学习工程师薪资最高，但需要数学好；Go后端需求大，发展好
5. 决定：先从Go后端工程师开始，学习Go语言，3年后考虑转型机器学习
```

### 场景二：职业转型

**适用人群**：想换行业、想换岗位的人

**使用步骤**：

1. **分析当前**
   - 你现在的职业是什么？
   - 你掌握的技能有哪些？
   - 哪些技能可以迁移到新职业？

2. **找目标职业**
   - 打开职业索引.md
   - 找到你想去的行业/领域
   - 查看该领域的所有职业

3. **对比分析**
   - 看"核心技能" - 新职业需要什么技能
   - 对比你现有技能 - 你已经有了什么
   - 找差距 - 你还需要学什么

4. **制定转型计划**
   - 列出需要补齐的技能
   - 确定学习顺序（先学什么、后学什么）
   - 设定时间表（3个月、6个月、1年）
   - 准备作品集/项目证明

5. **开始行动**
   - 学习新技能（在线课程、书籍、实践）
   - 做项目（GitHub、博客、demo）
   - 建立人脉（行业交流、LinkedIn）
   - 投递简历（根据新职业要求优化简历）

**举例**：
```
现状：Java后端工程师，5年经验，想转型AI
步骤：
1. 当前技能：Java、Spring、MySQL、Redis
2. 目标：机器学习工程师
3. 差距分析：
   - 已有：编程能力、项目经验
   - 缺少：Python、机器学习算法、深度学习框架
4. 转型计划（6个月）：
   - 1-2月：学习Python + 数学基础
   - 3-4月：学习机器学习算法 + Scikit-learn
   - 5-6月：学习深度学习 + TensorFlow + 做项目
5. 行动：每天学习2小时，周末做项目，6个月后投递ML工程师岗位
```

### 场景三：教育培训

**适用人群**：老师、培训师、HR、教育机构

**使用步骤**：

1. **课程设计**
   - 确定培训领域（技术、销售、管理等）
   - 查看该领域模块的职业列表
   - 选择目标职业群（如：整个前端开发职业群）

2. **技能拆解**
   - 看"核心技能" - 列出所有需要教授的技能
   - 分优先级 - 哪些是必备、哪些是进阶
   - 设计课程大纲 - 按技能分类设计课程

3. **内容准备**
   - 每个技能点：
     - 理论部分（讲什么）
     - 实践部分（练什么）
     - 考核部分（考什么）
   - 准备案例和项目
   - 准备学习资源

4. **教学实施**
   - 讲"费曼学习法解释" - 让学生理解职业是啥
   - 讲"核心技能" - 告诉学生要学什么
   - 讲"工作内容" - 告诉学生将来干什么
   - 讲"发展路径" - 告诉学生将来能去哪

5. **效果评估**
   - 根据技能要求设计考核
   - 根据薪资范围设定目标
   - 根据发展路径做职业指导

**举例**：
```
设计"前端开发工程师"培训课程：
步骤：
1. 目标职业：前端开发工程师
2. 技能拆解：
   - 必备：HTML5、CSS3、JavaScript、框架（React/Vue）
   - 进阶：TypeScript、工程化、性能优化
3. 课程大纲（12周）：
   - 1-2周：HTML+CSS
   - 3-6周：JavaScript基础+进阶
   - 7-10周：React框架
   - 11-12周：项目实践
4. 教学：
   - 第1课：前端开发是啥（费曼解释）
   - 第2课：学习路线图（核心技能）
   - 第3课：每天干什么（工作内容）
   - 第4课：未来去向（发展路径）
5. 评估：
   - 理论考试（技能掌握）
   - 项目作品（实践能力）
   - 模拟面试（职业准备）
```

### 场景四：薪资调研

**适用人群**：求职者、HR、企业管理者

**使用步骤**：

1. **确定调研范围**
   - 行业领域（技术、金融、医疗等）
   - 职业层级（初级、中级、高级、专家）
   - 地区（一线城市、二三线城市）

2. **收集数据**
   - 查看对应模块的薪资范围
   - 查看薪资汇总表
   - 记录多个职业的薪资数据

3. **数据对比**
   - 同一职业不同层级的对比
   - 同一层级不同职业的对比
   - 不同行业同一职业的对比

4. **影响因素分析**
   - 学历影响（本科 vs 硕士）
   - 经验影响（3年 vs 5年）
   - 地域影响（北京 vs 成都）
   - 行业影响（互联网 vs 传统企业）

5. **得出结论**
   - 求职者：我的目标薪资是否合理
   - HR：公司薪资是否具竞争力
   - 管理者：团队薪酬是否合理

**举例**：
```
调研北京地区Java后端工程师薪资：
步骤：
1. 查看模块：06-商业银行与保险、01-软件开发与工程
2. 收集数据：
   - 互联网公司：初级20-40万、中级40-70万、高级70-120万
   - 金融科技公司：初级25-45万、中级50-90万、高级90-150万
   - 传统银行：初级18-35万、中级35-55万、高级55-90万
3. 对比分析：
   - 同层级：金融科技 > 互联网 > 传统银行（高10-30%）
   - 同公司：高级 > 中级 > 初级（翻倍增长）
4. 结论：
   - 5年Java经验，在北京互联网公司应目标：70-120万
   - 金融科技公司溢价：可目标90-150万
```

## 💡 使用技巧

### 技巧1：善用搜索
- Windows/Linux：Ctrl+F
- Mac：Cmd+F
- 搜索职业名称、技能关键词

### 技巧2：交叉验证
- 多看几个相似职业，对比信息
- 结合实际招聘网站验证薪资
- 查询行业报告验证趋势

### 技巧3：动态更新
- 职业词典持续更新
- 关注最新模块
- 职业趋势会变化

### 技巧4：个性化调整
- 薪资范围是一线城市参考
- 二三线城市需打折（7-8折）
- 小城市需更大折扣（5-6折）

### 技巧5：结合实际
- 理论+实践结合
- 不要只看薪资，看兴趣
- 考虑公司规模和文化

## ⚠️ 注意事项

### 薪资说明
- 薪资范围是2025年一线城市参考
- 实际薪资因公司、能力、经济环境变化
- 高薪伴随高压力和高要求

### 技能要求
- "核心技能"是最基本的
- 实际工作可能需要更多技能
- 持续学习是必要的

### 学历要求
- 学历是门槛，不是天花板
- 能力和经验更重要
- 证书是加分项

### 职业前景
- "职业前景"评级是主观的
- 个人情况会影响前景
- 市场变化会影响前景
