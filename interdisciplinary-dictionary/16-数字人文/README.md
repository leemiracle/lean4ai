# 数字人文 (Digital Humanities)

## 学科概述：当人文学科遇上计算机

### 什么是数字人文？

想象一下，你是一个历史学家，面前堆着10000封18世纪的书信。传统的做法是：一封封读，手工记录，靠记忆找规律。

但如果有工具能帮你：
- 自动识别这些手写信的内容
- 标记出谁和谁在通信
- 在地图上画出信件往来的路线
- 统计最常出现的词汇

这就是数字人文——**用计算机的"超能力"来研究人的问题**。

### 三个学科的碰撞

```
┌─────────────┐
│  人文学科   │  → 提出问题：历史意义、文化价值、审美体验
│ (人文素养)  │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│   计算机    │  → 提供方法：算法、编程、系统开发
│ (技术能力)  │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  数据科学   │  → 处理数据：统计分析、可视化、挖掘
│ (分析思维)  │
└─────────────┘
```

### 为什么需要数字人文？

**传统人文学科的困境**：
- 文献太多，读不完
- 关系太复杂，靠脑子理不清
- 规律太隐蔽，肉眼看不见

**数字人文的解决方案**：
- 计算机能读百万本书
- 算法能理清复杂关系网络
- 可视化能让规律"显形"

## 文本挖掘与自然语言处理

### 核心概念：让计算机"读"文本

就像教小孩子认字：
1. **识字** → 分词、词性标注
2. **理解** → 语义分析、情感分析
3. **归纳** → 主题建模、关键词提取

### 关键技术

#### 命名实体识别 (NER)

**问题**：从文本中识别出人名、地名、机构名等

**例子**：
```
原文：李白在长安遇到了杜甫，两人一起游历洛阳。

识别结果：
- 人名：李白、杜甫
- 地名：长安、洛阳
```

**应用场景**：
- 史料中的人物关系梳理
- 文学作品中的地理标注
- 档案文献的信息抽取

#### 词频分析

**简单理解**：统计词汇出现次数，找出关键词

**进阶应用**：
- **TF-IDF**：找出文档中最有代表性的词
- **历时分析**：追踪词汇在不同时期的使用变化

**实际案例**：
```
研究《红楼梦》各回中"林黛玉"和"薛宝钗"的出现频率
→ 发现叙事重心的转移
→ 理解小说结构的安排
```

#### 主题建模 (LDA)

**类比理解**：像是给一堆文档自动分类，找出它们各自讨论什么主题

**工作原理**：
1. 假设每篇文档是多个主题的混合
2. 每个主题是一组词汇的概率分布
3. 算法自动学习这些分布

**应用**：
- 分析100年间报纸社论的主题演变
- 识别古典小说的叙事母题
- 梳理学术文献的研究脉络

#### 情感分析

**目标**：判断文本表达的是正面还是负面情感

**技术路线**：
```
基于词典的方法 → 用情感词典匹配
机器学习方法 → 训练分类模型
深度学习方法 → 使用BERT等预训练模型
```

**人文学术应用**：
- 分析文学作品中的情感走向
- 研究历史人物的态度变化
- 探索公众舆论的演变

### 实践工具

| 任务 | 常用工具 |
|------|----------|
| 中文分词 | jieba, pkuseg |
| NER | spaCy, HanLP |
| 主题建模 | Gensim, MALLET |
| 情感分析 | SnowNLP, transformers |
| 词向量 | Word2Vec, FastText |

## 社会网络分析

### 核心思想：关系即数据

传统研究关注"个体"，网络分析关注"关系"。

**类比**：
- 传统方法：研究一个个演员
- 网络方法：研究演员之间的互动关系

### 网络的基本要素

```
节点 (Node) = 实体（人、地点、概念）
边 (Edge) = 关系（通信、引用、共现）
权重 (Weight) = 关系的强弱
方向 (Direction) = 关系的方向性
```

### 关键指标

#### 中心性分析

识别网络中的重要节点：

| 类型 | 含义 | 应用 |
|------|------|------|
| 度中心性 | 连接数量多 | 社交活跃人物 |
| 中介中心性 | 桥接不同群体 | 信息传播枢纽 |
| 接近中心性 | 到各节点距离短 | 信息传播中心 |
| 特征向量中心性 | 连接重要节点 | 幕后影响力 |

#### 社区发现

**问题**：在网络中找出紧密连接的群体

**算法**：
- 模块度优化 (Louvain)
- 标签传播
- 随机游走

**应用**：
- 识别文学圈子的派系
- 分析学术共同体
- 发现贸易网络的区域结构

### 可视化方法

**力导向布局**：节点像磁铁一样相互吸引或排斥，自然形成布局

**工具选择**：
- Gephi：交互式探索
- NetworkX：Python分析
- Palladio：历史网络可视化
- Cytoscape：生物网络，也适用于人文

### 案例研究

**文艺复兴时期的书信网络**：
1. 数据：收集名人书信的收发记录
2. 建网：发信人→收信人
3. 分析：谁是信息枢纽？存在哪些圈子？
4. 发现：地理距离与通信频率的关系

## 地理信息系统与空间分析

### 核心概念：把数据放到地图上

**GIS = 地图 + 数据库 + 分析工具**

### 空间数据类型

```
矢量数据：
- 点 (Point)：城市、事件地点
- 线 (Line)：道路、河流、迁徙路线
- 面 (Polygon)：行政区、文化区域

栅格数据：
- 格网化的连续数据
- 如：地形高程、人口密度热力图
```

### 关键操作

#### 地理编码 (Geocoding)

**任务**：把地名转换为坐标

**例子**：
```
输入："北宋都城汴京"
输出：坐标 (34.7971°N, 114.3074°E)
```

**挑战**：
- 古地名与现代地名不对应
- 同名异地的歧义
- 地名随历史变化

#### 空间聚合

**操作**：把点数据汇总到区域

**应用**：
- 统计各县进士数量
- 汇总各地区文献产出
- 分析文化现象的地理分布

#### 距离分析

**问题类型**：
- 最近邻分析：点是聚集还是分散？
- 缓冲区分析：某范围内有什么？
- 网络分析：沿着道路的最短路径

### 时空可视化

**静态地图**： choropleth（分区着色）、比例符号图

**动态地图**：时间滑块展示变化过程

**故事地图**：结合地图与叙事

### 工具与平台

| 工具 | 特点 |
|------|------|
| QGIS | 开源、功能全面 |
| ArcGIS | 商业、行业标准 |
| Leaflet | Web交互地图 |
| Mapbox | 定制化Web地图 |
| Kepler.gl | 大数据时空可视化 |
| Recogito | 古籍地名标注 |

## 数据可视化

### 为什么可视化很重要

**人的大脑**：
- 处理图像比处理数字快60000倍
- 善于识别模式、异常、趋势

**数字人文的需求**：
- 让复杂数据变得直观
- 发现隐藏的模式
- 传播研究成果

### 可视化类型与选择

#### 比较类

| 图表 | 用途 |
|------|------|
| 柱状图 | 分类比较 |
| 条形图 | 分类比较（类别名称长） |
| 雷达图 | 多维度比较 |
| 小多图 | 多组数据的并置比较 |

#### 趋势类

| 图表 | 用途 |
|------|------|
| 折线图 | 时间序列 |
| 面积图 | 累积趋势 |
| 地平线图 | 多序列趋势对比 |

#### 关系类

| 图表 | 用途 |
|------|------|
| 散点图 | 两个变量的关系 |
| 气泡图 | 三个变量的关系 |
| 网络图 | 节点间关系 |
| 桑基图 | 流向与转化 |

#### 分布类

| 图表 | 用途 |
|------|------|
| 直方图 | 单变量分布 |
| 箱线图 | 多组分布比较 |
| 小提琴图 | 分布密度 |
| 热力图 | 二维分布密度 |

#### 构成类

| 图表 | 用途 |
|------|------|
| 饼图 | 简单比例（慎用） |
| 堆叠柱状图 | 时序构成变化 |
| 树状图 | 层级构成 |

#### 人文特色可视化

- **词云**：词汇频率的艺术化展示
- **时间轴**：历史事件的时序呈现
- **故事地图**：地理与叙事的结合
- **网络图**：关系的结构展示

### 可视化设计原则

1. **数据-墨水比**：每滴墨水都要传达信息
2. **避免误导**：坐标轴从零开始，比例一致
3. **颜色使用**：考虑色盲，有意义的配色
4. **标注清晰**：标题、图例、数据来源

### 工具生态

**编程方式**：
- Python: Matplotlib, Seaborn, Plotly
- R: ggplot2, Shiny
- JavaScript: D3.js, ECharts

**无代码/低代码**：
- Tableau：商业智能
- Power BI：微软生态
- RawGraphs：开源快速生成

## 数字档案与文化遗产

### 从纸质到数字的转变

**传统档案**：
- 实体保存，空间有限
- 检索困难，依赖人工
- 访问受限，需亲临现场

**数字档案**：
- 无限存储，易于复制
- 全文检索，秒级响应
- 远程访问，随时随地

### 数字化工作流程

```
1. 评估与选择
   ↓ 哪些文献需要数字化？优先级？

2. 扫描与采集
   ↓ 高分辨率图像采集

3. 图像处理
   ↓ 裁剪、矫正、增强

4. OCR识别
   ↓ 图像转文字

5. 元数据著录
   ↓ 标题、日期、作者、主题...

6. 质量控制
   ↓ 校对、修正

7. 存储与发布
   → 长期保存、提供访问
```

### OCR技术

**OCR (Optical Character Recognition)**：图像文字识别

**挑战**：
- 手写体识别困难
- 古籍排版复杂
- 墨迹褪色、纸张污损

**解决方案**：
- 专用古籍OCR模型
- 人工校对与众包
- 结合人工标注的机器学习

### 元数据标准

**Dublin Core**：15个核心元素

| 元素 | 说明 |
|------|------|
| Title | 题名 |
| Creator | 创建者 |
| Subject | 主题 |
| Description | 描述 |
| Date | 日期 |
| Type | 类型 |
| Format | 格式 |
| Identifier | 标识符 |
| ... | ... |

### 数字保存

**三大威胁**：
1. 技术过时：格式、软件不再支持
2. 载体老化：硬盘、光盘损坏
3. 灾难事件：火灾、水灾

**策略**：
- OAIS参考模型
- 多副本存储（3-2-1原则）
- 格式迁移与仿真
- 定期校验与刷新

### 虚拟博物馆

**技术实现**：
- 3D扫描与建模
- 虚拟展厅设计
- 交互式导览
- 增强现实叠加

**价值**：
- 突破物理空间限制
- 保护脆弱文物
- 提供沉浸式体验
- 促进教育普及

## 虚拟现实与增强现实

### 概念区分

**VR (虚拟现实)**：完全沉浸的虚拟世界
**AR (增强现实)**：在现实上叠加虚拟信息
**MR (混合现实)**：虚实融合与交互

### 在人文中的应用

#### 虚拟重建

**场景**：重建已消失的历史场景

**案例**：
- 古罗马建筑的虚拟复原
- 毁损古迹的数字重建
- 历史城市的3D再现

**技术流程**：
```
考古资料 → 3D建模 → 纹理贴图 → 
光照渲染 → 交互开发 → 发布体验
```

#### 沉浸式体验

**应用方向**：
- 历史场景的"穿越"体验
- 文学作品的场景还原
- 博物馆的虚拟参观

#### 增强现实导览

**场景**：用手机扫描实物，叠加虚拟信息

**案例**：
- 博物馆展品的信息叠加
- 历史遗迹的古今对比
- 艺术作品的创作过程展示

#### 空间人文

**概念**：将虚拟内容嵌入物理空间

**应用**：
- 文学地理的实地体验
- 历史事件的地点重现
- 文化记忆的空间标注

### 技术栈

**开发平台**：
- Unity：游戏引擎，跨平台
- Unreal Engine：高端渲染
- A-Frame：WebVR开发

**硬件设备**：
- VR：Oculus, HTC Vive
- AR：手机ARKit/ARCore, HoloLens

## 计算史学

### 什么是计算史学？

**定义**：用计算方法研究历史问题

**核心理念**：从"读历史"到"计算历史"

### 研究范式

#### 计量史学 (Cliometrics)

**起源**：1950-60年代，用统计方法研究经济史

**方法**：
- 历史数据的量化
- 统计分析与假设检验
- 经济计量模型

#### 数字史学

**扩展**：不仅计量，还包括：
- 文本挖掘
- 网络分析
- 空间分析
- 可视化

### 典型研究

#### 历史人口学

**数据**：户籍、族谱、教区记录

**问题**：
- 人口结构变化
- 婚姻模式
- 迁徙路线

#### 史料数据库

**项目类型**：
- 人物数据库（如CBDB）
- 史籍全文库
- 历史地理信息系统

#### 历史网络分析

**应用**：
- 政治精英的社会网络
- 贸易网络的演变
- 知识传播路径

### 史料的数字化处理

**流程**：
```
原始史料 → 数字化 → OCR → 标注 → 
结构化 → 数据库 → 分析可视化
```

**挑战**：
- 史料的不完整性
- 数据的时间覆盖不均
- 历史概念的界定

## 数字文学研究

### 文学的"远读"

**概念** (Franco Moretti)：
- 近读：精读少量文本
- 远读：宏观把握大量文本

**方法**：
- 统计分析大规模文学语料
- 发现文学史的模式与规律
- 用数据挑战文学史常识

### 研究方向

#### 作者归属研究

**问题**：确定匿名作品的作者

**方法**：
- 文体计量学 (Stylometry)
- 词频统计、句法特征
- 机器学习分类

**经典案例**：
- 《联邦党人文集》作者判定
- J.K.罗琳笔名作品识别

#### 文体分析

**维度**：
- 词汇丰富度
- 句子长度分布
- 标点使用模式
- 词性序列

**应用**：
- 分析作家风格演变
- 比较不同作家的文体
- 识别流派特征

#### 叙事分析

**方法**：
- 情节结构的形式化
- 人物网络的构建
- 时空叙事的映射

#### 情感曲线

**概念**：追踪叙事中的情感起伏

**方法**：
- 分段情感分析
- 绘制情感时序图
- 识别叙事模式

**发现**：
- 常见的情感弧线类型
- 与读者接受度的关系

### 文学语料库建设

**关键问题**：
- 文本选择与代表性
- 版本校勘与标准化
- 元数据著录
- 版权与访问

## 结语：数字人文的未来

数字人文不是要取代传统人文研究，而是提供新的工具和视角。

**核心在于**：
- 问题驱动，技术为用
- 保持人文关怀与批判精神
- 在定量与定性间平衡
- 推动开放协作的学术生态

**未来的数字人文研究者**：
- 既是人文学者，也是技术实践者
- 既能提出深刻问题，也能用工具探索
- 既懂方法，也懂局限
- 既能独立研究，也能跨学科协作

数字人文的大门向所有对人文学科有热情、愿意学习新技术的人敞开。
