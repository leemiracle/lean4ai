# 06-嵌入式软件

## 定义

嵌入式软件（Embedded Software）是专为特定硬件设备设计的计算机程序，运行在嵌入式系统（如微控制器、单片机）中，具有特定的功能要求，通常要求高可靠性、低功耗、实时性，广泛应用于物联网、汽车、家电、工业控制等领域。

---

## 核心特征

1. **硬件专用** - 为特定硬件设备设计
2. **资源受限** - 处理器、内存、存储有限
3. **实时性要求** - 需要在规定时间内响应
4. **高可靠性** - 要求稳定运行，不易出错
5. **低功耗** - 电池供电设备要求节能
6. **专用功能** - 执行特定任务，功能单一

---

## 费曼解释

想象一下，电脑软件就像是一个全能管家，什么都能做，但需要大房子（大内存、强处理器）。

嵌入式软件就像是家里的小家电里的"小管家"，比如空调、微波炉、洗衣机里的程序。它们不需要做很多事，只需要做好一件或几件事：
- 空调的嵌入式软件：控制温度、风速、定时
- 洗衣机的嵌入式软件：控制洗涤程序、脱水、漂洗
- 汽车的嵌入式软件：控制发动机、刹车、转向

这些小管家（嵌入式软件）虽然不如电脑管家功能强大，但它们更专注、更可靠、更省电。比如你开车时，汽车的嵌入式软件必须在毫秒级别响应刹车信号，否则就危险了。

嵌入式软件就像是一个"专用工具"，虽然功能单一，但精准、高效、可靠。

## 产品分类

### 1. 按应用领域分类

#### 汽车电子
- **ECU（电子控制单元）** - 发动机控制、变速箱控制
- **ADAS（高级驾驶辅助系统）** - 自动刹车、车道保持、自适应巡航
- **信息娱乐系统** - 导航、音乐、蓝牙
- **车身控制** - 车窗、车灯、座椅控制
- **OTA（在线升级）** - 远程升级软件

#### 家电智能
- **智能冰箱** - 温控、食材管理
- **智能洗衣机** - 智能洗涤程序
- **智能空调** - 智能温控、节能
- **智能电视** - 智能系统、语音控制
- **扫地机器人** - 路径规划、避障

#### 工业控制
- **PLC（可编程逻辑控制器）** - 工业自动化控制
- **SCADA** - 数据采集与监控系统
- **机器人控制** - 工业机器人控制
- **数控系统** - 机床控制
- **传感器采集** - 工业传感器数据采集

#### 医疗设备
- **心电监护仪** - 心率监测、数据处理
- **血压计** - 血压测量、数据记录
- **胰岛素泵** - 胰岛素精准注射
- **起搏器** - 心脏起搏
- **医疗影像** - 图像处理、显示

#### 通信设备
- **路由器** - 数据转发、路由表
- **交换机** - 数据交换、VLAN
- **基站** - 信号处理、调度
- **调制解调器** - 信号调制解调
- **卫星通信** - 信号处理、通信协议

#### 物联网设备
- **智能家居** - 灯光控制、门锁、传感器
- **可穿戴设备** - 智能手表、健康监测
- **智能传感器** - 温度、湿度、光照传感器
- **智能门锁** - 人脸识别、指纹识别
- **智能音箱** - 语音识别、音频处理

#### 消费电子
- **智能手表** - 健康、运动监测
- **游戏机** - 游戏运行、图形处理
- **电子书** - 电子书显示、翻页
- **相机** - 图像处理、对焦、曝光
- **无人机** - 飞行控制、图像传输

### 2. 按实时性要求分类

#### 硬实时系统
- **特点** - 必须在严格规定时间内响应，否则后果严重
- **代表** - 汽车刹车、飞行控制、医疗监护
- **要求** - 可预测性、高可靠性

#### 软实时系统
- **特点** - 希望在规定时间内响应，但偶尔延迟可接受
- **代表** - 多媒体播放、游戏、用户界面
- **要求** - 流畅体验、低延迟

#### 非实时系统
- **特点** - 没有严格的时间要求
- **代表** - 数据记录、日志系统
- **要求** - 功能正确即可

### 3. 按系统复杂度分类

#### 简单系统
- **特点** - 单一功能，资源极有限
- **代表** - 微波炉控制器、LED驱动
- **技术** - 无操作系统，裸机程序

#### 中等系统
- **特点** - 多功能，资源有限
- **代表** - 智能家电、传感器节点
- **技术** - 实时操作系统（RTOS）

#### 复杂系统
- **特点** - 功能复杂，资源相对充足
- **代表** - 汽车ECU、路由器
- **技术** - 嵌入式Linux、VxWorks

## 技术架构

### 硬件平台
1. **微控制器（MCU）** - STM32、AVR、PIC、MSP430
2. **微处理器（MPU）** - ARM Cortex-A、RISC-V
3. **DSP（数字信号处理器）** - TI DSP、ADI DSP
4. **FPGA** - Xilinx、Altera
5. **SoC（片上系统）** - 高度集成，如高通骁龙、联发科

### 操作系统
1. **RTOS（实时操作系统）**
   - FreeRTOS - 开源RTOS
   - RT-Thread - 国产RTOS
   - Zephyr - Linux基金会RTOS
   - μC/OS - 商业RTOS

2. **嵌入式Linux**
   - Yocto - 构建系统
   - Buildroot - 轻量级构建
   - OpenWrt - 路由器系统

3. **专用操作系统**
   - VxWorks - 风河，商业RTOS
   - QNX - 黑莓，汽车OS
   - Android - 谷歌，移动OS

### 开发语言
1. **C/C++** - 主要语言，性能高
2. **汇编语言** - 底层驱动、性能关键代码
3. **Rust** - 内存安全，嵌入式领域增长快
4. **Python** - 高层应用、快速原型
5. **Ada** - 高可靠性要求系统（航空、航天）

### 开发工具
1. **IDE** - Keil、IAR、Eclipse、VS Code
2. **编译器** - GCC、LLVM/Clang、Arm Compiler
3. **调试器** - JTAG/SWD调试器
4. **仿真器** - QEMU、硬件仿真
5. **烧录工具** - ST-Link、J-Link

### 通信协议
1. **串口通信** - UART、USART
2. **I2C** - 两线串行总线
3. **SPI** - 串行外设接口
4. **CAN** - 汽车总线
5. **USB** - 通用串行总线
6. **以太网** - TCP/IP协议栈
7. **无线** - Wi-Fi、Bluetooth、Zigbee、LoRa、NB-IoT

## 开发嵌入式软件的考虑因素

### 硬件选型
1. **性能要求** - 处理器性能、内存大小
2. **功耗要求** - 电池供电还是市电
3. **成本预算** - 硬件成本限制
4. **可靠性** - 工业级、车规级、军规级
5. **生态支持** - 开发工具、文档、社区

### 操作系统选择
1. **实时性要求** - 是否需要RTOS
2. **资源限制** - RAM、Flash大小
3. **开发效率** - 开发周期要求
4. **商业支持** - 是否需要商业RTOS
5. **开源需求** - 是否需要开源系统

### 开发流程
1. **需求分析** - 功能、性能、功耗要求
2. **系统设计** - 硬件、软件架构设计
3. **编码实现** - C/C++、Rust开发
4. **测试验证** - 单元测试、集成测试
5. **现场测试** - 实际环境测试

### 调试与测试
1. **硬件调试** - 示波器、逻辑分析仪
2. **软件调试** - JTAG/SWD调试
3. **性能测试** - 功耗、响应时间测试
4. **可靠性测试** - 长时间运行测试
5. **安全测试** - 漏洞扫描、渗透测试
