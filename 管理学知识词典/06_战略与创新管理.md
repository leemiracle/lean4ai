## 第一部分：战略管理基础

---

### 战略管理

**学科归属**：工商管理 > 战略管理  
**英文名称**：Strategic Management  
**关联概念**：[竞争优势] [战略规划] [战略执行]

#### 核心定义（一句话）
确定企业长期发展方向，并协调资源实现目标的综合性管理过程。

#### 费曼式解释（说人话）

**什么是战略管理？**

简单说：**决定企业往哪走、怎么走**。

**三个层次的战略**：

| 层次 | 问题 | 例子 |
|------|------|------|
| 公司战略 | 做什么业务？ | 阿里：电商、金融、物流、文娱 |
| 业务战略 | 怎么竞争？ | 低价？差异化？ |
| 职能战略 | 各部门怎么配合？ | 营销、生产、人力资源策略 |

**战略管理过程**：
```
战略分析 → 战略制定 → 战略实施 → 战略评估
    ↑                               ↓
    ←←←←← 反馈与调整 ←←←←←←←←←←←
```

**为什么重要？**
- 没有战略 = 没有方向
- 资源有限，需要聚焦
- 环境变化快，需要提前规划

---

### 战略分析

**学科归属**：战略管理 > 战略分析  
**英文名称**：Strategic Analysis  
**关联概念**：[外部环境] [内部资源] [SWOT]

#### 核心定义（一句话）
分析企业内外部环境，为战略制定提供依据。

#### 费曼式解释（说人话）

**分析什么？**

**外部环境分析**：
- 宏观环境（PEST）
- 行业环境（波特五力）
- 竞争对手

**内部环境分析**：
- 资源和能力
- 核心竞争力
- 价值链

**SWOT分析**：

| | 正面 | 负面 |
|---|------|------|
| 内部 | Strengths 优势 | Weaknesses 劣势 |
| 外部 | Opportunities 机会 | Threats 威胁 |

**组合策略**：
- SO：发挥优势，抓住机会
- WO：利用机会，弥补劣势
- ST：发挥优势，规避威胁
- WT：减少劣势，规避威胁

---

### PEST分析

**学科归属**：战略管理 > 战略分析  
**英文名称**：PEST Analysis  
**关联概念**：[宏观环境] [外部环境]

#### 核心定义（一句话）
分析政治、经济、社会、技术四个宏观环境要素的工具。

#### 费曼式解释（说人话）

**PEST四个维度**：

| 维度 | 内容 | 例子 |
|------|------|------|
| P - Political 政治 | 政策、法规、政治稳定性 | 环保政策、贸易战 |
| E - Economic 经济 | GDP、利率、通胀、汇率 | 经济增长、消费能力 |
| S - Social 社会 | 人口、文化、生活方式 | 老龄化、消费习惯 |
| T - Technological 技术 | 技术发展、创新 | AI、5G、新能源 |

**使用方法**：
1. 列出每个维度的重要因素
2. 分析对企业的影响（正面/负面）
3. 确定优先级
4. 制定应对策略

**延伸**：PESTEL（增加环境Environment、法律Legal）

### 波特五力模型

**学科归属**：战略管理 > 行业分析  
**提出者**：迈克尔·波特  
**英文名称**：Porter's Five Forces  
**关联概念**：[行业竞争] [利润潜力]

#### 核心定义（一句话）
分析行业竞争结构的五个力量。

#### 费曼式解释（说人话）

**五种力量**：

```
         潜在进入者威胁
              ↓
供应商议价能力 → [行业竞争] ← 购买者议价能力
              ↑
         替代品威胁
```

| 力量 | 问题 | 影响因素 |
|------|------|---------|
| 现有竞争者 | 竞争激烈吗？ | 竞争者数量、行业增长、固定成本 |
| 潜在进入者 | 新对手容易进吗？ | 进入壁垒、资本需求 |
| 替代品 | 有其他选择吗？ | 替代品数量、转换成本 |
| 供应商议价能力 | 供应商强势吗？ | 供应商数量、替代投入 |
| 购买者议价能力 | 客户强势吗？ | 客户数量、转换成本 |

**结论**：
- 五力越强，行业利润越低
- 理想情况：五力都弱

### 核心竞争力

**学科归属**：战略管理 > 资源基础观  
**提出者**：普拉哈拉德、哈默尔  
**英文名称**：Core Competence  
**关联概念**：[竞争优势] [资源]

#### 核心定义（一句话）
企业独有的、难以模仿的能力，是竞争优势的来源。

#### 费曼式解释（说人话）

**什么是核心竞争力？**

简单说：**你会别人不会的、难以学会的本领**。

**四个标准**（VRIO）：

| 标准 | 问题 |
|------|------|
| V - Value 价值 | 能创造价值吗？ |
| R - Rarity 稀缺性 | 别人有吗？ |
| I - Imitability 难模仿 | 别人能学会吗？ |
| O - Organization 组织化 | 企业能利用吗？ |

**例子**：
- 苹果：设计+生态系统
- 沃尔玛：供应链管理
- 华为：研发能力
- 海底捞：服务文化

**核心竞争力的特点**：
1. 能打开多个市场
2. 对终端产品贡献大
3. 难以模仿

---

### 竞争战略

**学科归属**：战略管理 > 业务战略  
**提出者**：迈克尔·波特  
**英文名称**：Competitive Strategy  
**关联概念**：[成本领先] [差异化] [聚焦]

#### 核心定义（一句话）
企业在市场中获得竞争优势的基本方法。

#### 费曼式解释（说人话）

**波特的三种基本战略**：

| 战略 | 核心逻辑 | 例子 |
|------|---------|------|
| 成本领先 | 我比谁都便宜 | 沃尔玛、拼多多 |
| 差异化 | 我和别人不一样 | 苹果、星巴克 |
| 聚焦 | 我只服务一小部分人 | 劳斯莱斯 |

**成本领先战略**：
- 规模经济
- 效率提升
- 成本控制
- 适合：标准化产品

**差异化战略**：
- 品牌
- 设计
- 服务
- 技术创新
- 适合：能创造独特价值

**聚焦战略**：
- 聚焦特定市场
- 可以是成本聚焦或差异化聚焦
- 适合：细分市场专家

**⚠️ 夹在中间**：既不成本领先，也不差异化 = 没有竞争优势

---

### 蓝海战略

**学科归属**：战略管理 > 竞争战略  
**提出者**：金伟灿、莫博涅  
**英文名称**：Blue Ocean Strategy  
**关联概念**：[红海] [价值创新]

#### 核心定义（一句话）
开创无人竞争的市场空间，摆脱红海血腥竞争。

#### 费曼式解释（说人话）

**蓝海 vs 红海**：

| 红海 | 蓝海 |
|------|------|
| 在已知市场里竞争 | 开创新市场 |
| 打败竞争对手 | 让竞争无关紧要 |
| 在价值与成本间权衡 | 同时追求价值和成本 |
| 需求有限 | 创造新需求 |

**核心思想**：价值创新
- 不是技术领先
- 不是差异化定价
- 而是创造新的价值曲线

**四步动作框架**：

| 动作 | 问题 |
|------|------|
| 剔除 | 哪些被竞争认为是标准的要素应该剔除？ |
| 减少 | 哪些要素应该减少到行业标准以下？ |
| 增加 | 哪些要素应该增加到行业标准以上？ |
| 创造 | 哪些要素是行业从未提供过的？ |

**案例**：
- 太阳马戏团：剔除动物表演，增加艺术性
- 任天堂Wii：牺牲画质，增加互动性

## 第二部分：创新管理

---

### 创新

**学科归属**：战略管理 > 创新管理  
**英文名称**：Innovation  
**关联概念**：[发明] [商业化] [价值创造]

#### 核心定义（一句话）
将创意转化为商业价值的过程。

#### 费曼式解释（说人话）

**什么是创新？**

**创新 ≠ 发明**：
- 发明：创造新东西
- 创新：把新东西商业化

**创新的类型**（按程度）：

| 类型 | 特点 | 例子 |
|------|------|------|
| 渐进式创新 | 小改进 | iPhone 13 → 14 |
| 颠覆式创新 | 改变游戏规则 | 智能手机取代功能机 |
| 架构式创新 | 重新组合 | 特斯拉（电动车+软件） |

**创新的类型**（按对象）：

| 类型 | 内容 |
|------|------|
| 产品创新 | 新产品 |
| 流程创新 | 新的生产方法 |
| 商业模式创新 | 新的赚钱方式 |
| 服务创新 | 新的服务方式 |

---

### 破坏性创新

**学科归属**：创新管理 > 创新理论  
**提出者**：克莱顿·克里斯坦森  
**英文名称**：Disruptive Innovation  
**关联概念**：[持续性创新] [在位企业]

#### 核心定义（一句话）
从低端切入，逐步颠覆现有市场的创新模式。

#### 费曼式解释（说人话）

**破坏性创新 vs 持续性创新**：

| 持续性创新 | 破坏性创新 |
|-----------|-----------|
| 提升现有产品性能 | 初期性能更差 |
| 服务主流客户 | 服务被忽视的客户 |
| 在位企业擅长 | 在位企业忽视 |
| 沿着既有轨道 | 开创新轨道 |

**破坏性创新的过程**：
1. 从低端市场切入（大公司看不上）
2. 用低成本、简单技术
3. 逐步改进
4. 最终颠覆主流市场

**例子**：
- 数码相机 vs 胶卷相机
- 智能手机 vs 电脑
- 网飞 vs 传统影视

**为什么大公司会被颠覆？**
- 关注高利润客户
- 忽视低端市场
- 资源配置偏向现有业务

---

### 开放式创新

**学科归属**：创新管理 > 创新模式  
**提出者**：亨利·切萨布鲁夫  
**英文名称**：Open Innovation  
**关联概念**：[封闭式创新] [知识流动]

#### 核心定义（一句话）
企业利用内外部资源进行创新，创新成果也向外部流动。

#### 费曼式解释（说人话）

**封闭式创新 vs 开放式创新**：

| 封闭式创新 | 开放式创新 |
|-----------|-----------|
| 自己研发 | 内外部结合 |
| 自己生产 | 合作生产 |
| 自己销售 | 多渠道销售 |
| 人才在内部 | 人才流动 |
| 知识保密 | 知识流动 |

**开放式创新的做法**：

| 方向 | 做法 | 例子 |
|------|------|------|
| 由外向内 | 引入外部技术 | 收购、专利授权 |
| 由内向外 | 输出内部技术 | 技术转让、分拆 |
| 耦合 | 双向流动 | 合作研发、联盟 |

**好处**：
- 加速创新
- 降低成本
- 分散风险

**挑战**：
- 知识产权保护
- 文化冲突
- 管理复杂性

---

### 创新扩散理论

**学科归属**：创新管理 > 创新传播  
**提出者**：埃弗雷特·罗杰斯  
**英文名称**：Diffusion of Innovation  
**关联概念**：[采纳曲线] [跨越鸿沟]

#### 核心定义（一句话）
描述创新在人群中传播的过程和规律。

#### 费曼式解释（说人话）

**创新的采纳者分类**：

```
创新者 → 早期采用者 → 早期大众 → 晚期大众 → 落后者
 2.5%      13.5%       34%        34%       16%
```

| 类型 | 特点 | 策略 |
|------|------|------|
| 创新者 | 爱尝鲜，敢于冒险 | 吸引他们试用 |
| 早期采用者 | 意见领袖，有影响力 | 赢得口碑 |
| 早期大众 | 实用主义，需要证据 | 证明价值 |
| 晚期大众 | 怀疑论者，跟随大众 | 降低风险 |
| 落后者 | 保守，拒绝新事物 | 基本可以忽略 |

**"跨越鸿沟"**：
- 早期采用者 → 早期大众之间有鸿沟
- 很多创新死在这里
- 需要针对大众需求调整产品

**影响采纳速度的因素**：
1. 相对优势
2. 兼容性
3. 复杂性
4. 可试用性
5. 可观察性

## 第三部分：商业模式

---

### 商业模式

**学科归属**：战略管理 > 商业模式  
**英文名称**：Business Model  
**关联概念**：[价值主张] [盈利模式]

#### 核心定义（一句话）
企业创造、传递和获取价值的方式。

#### 费曼式解释（说人话）

**什么是商业模式？**

简单说：**企业怎么赚钱**。

**商业模式画布**（9个要素）：

| 类别 | 要素 | 问题 |
|------|------|------|
| 产品 | 价值主张 | 给客户提供什么价值？ |
| 客户 | 客户细分 | 服务谁？ |
| | 渠道 | 怎么触达客户？ |
| | 客户关系 | 和客户什么关系？ |
| 资产 | 核心资源 | 需要什么资源？ |
| | 关键业务 | 需要做什么？ |
| | 重要伙伴 | 谁是合作伙伴？ |
| 财务 | 成本结构 | 成本在哪里？ |
| | 收入来源 | 怎么赚钱？ |

**常见商业模式**：
1. 订阅模式（Netflix）
2. 平台模式（淘宝）
3. 免费增值模式（印象笔记）
4. 长尾模式（亚马逊）
5. 多边平台模式（Uber）

---

### 商业模式画布

**学科归属**：战略管理 > 商业模式  
**提出者**：亚历山大·奥斯特瓦德  
**英文名称**：Business Model Canvas  
**关联概念**：[商业模式] [价值主张]

#### 核心定义（一句话）
用九个要素系统描述商业模式的可视化工具。

#### 费曼式解释（说人话）

**画布结构**：

```
┌─────────┬─────────┬─────────┬──────────┬─────────┐
│关键合作伙伴│关键业务  │价值主张  │客户关系   │客户细分  │
│         │核心资源  │        │渠道通路   │        │
├─────────┴─────────┴─────────┴──────────┴─────────┤
│                  成本结构                  收入来源  │
└──────────────────────────────────────────────────┘
```

**使用步骤**：
1. 确定客户细分
2. 定义价值主张
3. 设计渠道和客户关系
4. 确定需要的资源和活动
5. 识别合作伙伴
6. 分析成本和收入

**好处**：
- 可视化
- 系统性
- 便于沟通
- 易于调整
