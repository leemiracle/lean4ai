### 运筹学

**学科归属**：管理科学与工程 > 运筹学  
**英文名称**：Operations Research (OR)  
**关联概念**：[优化] [数学模型] [决策]

#### 核心定义（一句话）
运用数学模型和算法，为复杂决策问题提供最优解决方案的科学方法。

#### 费曼式解释（说人话）

**什么是运筹学？**

简单说：**用数学帮你做最好的决定**。

**来源**：二战时，英国军队请科学家帮他们"运筹帷幄"——如何布置雷达、如何调度船只。战后，这套方法被用于企业管理。

**运筹学做什么？**
1. **建模**：把现实问题变成数学公式
2. **求解**：用算法找出最优解
3. **应用**：把数学结果变成实际决策

**典型问题**：
- **怎么走最快？** → 最短路径
- **怎么分配资源最省？** → 线性规划
- **排队多长合适？** → 排队论

**应用场景**：
- 物流：配送路线优化
- 生产：排程优化
- 金融：投资组合优化
- 航空：机组排班

### 线性规划

**学科归属**：管理科学与工程 > 运筹学  
**英文名称**：Linear Programming (LP)  
**关联概念**：[优化] [约束条件] [目标函数]

#### 核心定义（一句话）
在一组线性约束条件下，求线性目标函数最大值或最小值的数学方法。

#### 费曼式解释（说人话）

**什么是线性规划？**

**一个例子**：
- 你有100亩地
- 可以种小麦（每亩利润100元）或玉米（每亩利润150元）
- 小麦需要2个工人/亩，玉米需要3个工人/亩
- 你有240个工人
- 怎么种最赚钱？

**转化为数学**：
- 目标：最大化利润 = 100×小麦 + 150×玉米
- 约束：
  - 土地：小麦 + 玉米 ≤ 100
  - 劳力：2×小麦 + 3×玉米 ≤ 240
  - 非负：小麦 ≥ 0，玉米 ≥ 0

**求解**：用单纯形法、图解法或软件（Excel、Python）

**应用场景**：
- 生产计划
- 资源分配
- 运输问题
- 人员排班

---

### 单纯形法

**学科归属**：管理科学与工程 > 运筹学  
**提出者**：乔治·丹齐格（1947）  
**英文名称**：Simplex Method  
**关联概念**：[线性规划] [迭代]

#### 核心定义（一句话）
求解线性规划问题的经典算法，通过迭代在可行域的顶点间移动找到最优解。

#### 费曼式解释（说人话）

**什么是单纯形法？**

**核心思想**：
- 线性规划的最优解一定在"顶点"上
- 从一个顶点开始
- 不断移到更好的顶点
- 直到找不到更好的

**打个比方**：
- 你在山上（可行域）
- 要爬到最高点（最优解）
- 每次往更高的方向走
- 直到周围都比你现在低

**步骤**：
1. 找一个初始可行解
2. 判断是否最优
3. 如果不是，选一个更好的方向
4. 移动到新点
5. 重复2-4直到最优

**现在**：
- 手算已经很少了
- 主要用软件求解

### 整数规划

**学科归属**：管理科学与工程 > 运筹学  
**英文名称**：Integer Programming (IP)  
**关联概念**：[线性规划] [0-1规划]

#### 核心定义（一句话）
决策变量必须取整数的规划问题。

#### 费曼式解释（说人话）

**什么是整数规划？**

**与线性规划的区别**：
- 线性规划：变量可以是小数（如生产3.5个产品）
- 整数规划：变量必须是整数（如生产3个或4个产品）

**为什么需要整数规划？**
- 不能建2.5个工厂
- 不能派1.3个人
- 要么选A，要么选B（0-1规划）

**例子**：
- 工厂选址：建还是不建？→ 0-1规划
- 机器数量：买几台？→ 整数规划

**求解难度**：
- 比线性规划难得多
- 常用方法：分支定界法、割平面法

### 运输问题

**学科归属**：管理科学与工程 > 运筹学  
**英文名称**：Transportation Problem  
**关联概念**：[线性规划] [网络流]

#### 核心定义（一句话）
确定从多个供应点到多个需求点的最优运输方案，使总成本最小。

#### 费曼式解释（说人话）

**什么是运输问题？**

**典型场景**：
- 3个工厂生产产品
- 4个仓库需要产品
- 每个工厂到每个仓库的运费不同
- 如何运输使总运费最低？

**特点**：
- 特殊的线性规划问题
- 有专门的解法（表上作业法）
- 可以用软件快速求解

**应用**：
- 物流配送
- 生产调度
- 人员分配

### 排队论

**学科归属**：管理科学与工程 > 运筹学  
**英文名称**：Queuing Theory  
**关联概念**：[等待时间] [服务率]

#### 核心定义（一句话）
研究排队系统中顾客等待和服务过程的数学理论。

#### 费曼式解释（说人话）

**什么是排队论？**

**生活中的排队**：
- 超市收银
- 银行柜台
- 医院挂号
- 客服电话

**排队论解决的问题**：
- 平均等多久？
- 队伍多长？
- 需要多少服务窗口？

**核心指标**：
| 指标 | 含义 |
|------|------|
| 到达率(λ) | 单位时间来了多少人 |
| 服务率(μ) | 单位时间能服务多少人 |
| 利用率(ρ) | 服务台的繁忙程度 |
| 平均等待时间 | 顾客等多久 |
| 平均队长 | 队伍多长 |

**经典模型：M/M/1**
- M：到达服从泊松分布
- M：服务时间服从指数分布
- 1：1个服务台

**管理启示**：
- 服务能力 vs 成本的权衡
- 太少：顾客等太久
- 太多：资源浪费

### 库存管理

**学科归属**：管理科学与工程 > 运筹学  
**英文名称**：Inventory Management  
**关联概念**：[EOQ] [安全库存] [订货点]

#### 核心定义（一句话）
确定最优库存水平，平衡库存成本与缺货风险的管理活动。

#### 费曼式解释（说人话）

**什么是库存管理？**

**核心问题**：存货多少才合适？
- 太少：缺货，损失销售
- 太多：积压，占用资金

**库存成本**：
1. **订货成本**：每次订货的费用
2. **持有成本**：仓储、资金占用
3. **缺货成本**：缺货造成的损失

**经典模型：EOQ（经济订货量）**

```
EOQ = √(2×D×S/H)

D：年需求量
S：每次订货成本
H：单位持有成本
```

**实际应用**：
- 零售：超市补货
- 制造：原料库存
- 电商：仓储管理

---

### EOQ模型

**学科归属**：管理科学与工程 > 运筹学  
**全称**：Economic Order Quantity  
**关联概念**：[库存管理] [订货成本]

#### 核心定义（一句话）
使总库存成本最小的最优订货数量。

#### 费曼式解释（说人话）

**EOQ解决什么问题？**

**场景**：
- 你开一家店，卖某种商品
- 每年卖1000个
- 每次订货成本50元
- 每个商品每年持有成本10元
- 每次订多少最划算？

**两种极端**：
- 一年订1次，一次订1000个 → 持有成本高
- 一年订1000次，每次1个 → 订货成本高

**EOQ找到平衡点**：
```
EOQ = √(2×1000×50/10) = √10000 = 100个
```

**订货频率**：1000/100 = 10次/年

**假设条件**（现实中可能不完全成立）：
- 需求恒定
- 提前期确定
- 不允许缺货
- 瞬时补货

### 博弈论

**学科归属**：管理科学与工程 > 运筹学  
**英文名称**：Game Theory  
**关联概念**：[纳什均衡] [囚徒困境] [零和博弈]

#### 核心定义（一句话）
研究决策主体在相互影响情况下如何做出最优决策的理论。

#### 费曼式解释（说人话）

**什么是博弈论？**

简单说：**你做决定时要考虑别人怎么做决定**。

**博弈的要素**：
1. **参与者**：谁在做决定？
2. **策略**：能选择什么？
3. **收益**：每种结果得到什么？

**经典案例：囚徒困境**

| | A沉默 | A背叛 |
|---|---|---|
| B沉默 | 各判1年 | A判10年，B释放 |
| B背叛 | A释放，B判10年 | 各判5年 |

**结论**：虽然都沉默最好，但理性选择是都背叛——这就是"纳什均衡"。

**应用**：
- 商业竞争（价格战）
- 国际关系
- 拍卖设计
- 谈判策略

---

### 纳什均衡

**学科归属**：管理科学与工程 > 博弈论  
**提出者**：约翰·纳什  
**英文名称**：Nash Equilibrium  
**关联概念**：[博弈论] [策略]

#### 核心定义（一句话）
在给定其他人策略的情况下，每个参与者都没有动机改变自己策略的状态。

#### 费曼式解释（说人话）

**什么是纳什均衡？**

简单说：**谁都不想改了，改了也没好处**。

**三个条件**：
1. 每个人都选择了自己的最优策略
2. 这个"最优"是在别人策略不变的前提下
3. 没有人单方面改变会更好

**囚徒困境中的纳什均衡**：
- 两人都背叛
- 虽然不是最好的结果
- 但没人愿意单方面改变

**为什么重要？**
- 预测博弈的结果
- 分析竞争策略
- 理解社会现象

### 囚徒困境

**学科归属**：管理科学与工程 > 博弈论  
**英文名称**：Prisoner's Dilemma  
**关联概念**：[纳什均衡] [合作]

#### 核心定义（一句话）
个体理性导致集体非理性的经典博弈模型。

#### 费曼式解释（说人话）

**囚徒困境的故事**：

两个小偷被抓了，警察分开审问：
- 如果都沉默：各判1年
- 如果都坦白：各判5年
- 如果一个坦白、一个沉默：坦白的释放，沉默的判10年

**分析**：
- 如果对方沉默：我坦白（释放）> 沉默（1年）
- 如果对方坦白：我坦白（5年）> 沉默（10年）
- **结论**：无论对方怎样，坦白都是最好的

**结果**：两人都坦白，各判5年
- 但如果都沉默，只要各判1年！
- **个体理性 → 集体非理性**

**现实意义**：
- 价格战：都降价，都受伤
- 环境问题：都排放，都受害
- 军备竞赛：都增加军费，都不安全

**如何打破？**
- 重复博弈：建立信任
- 第三方惩罚
- 建立合作机制

### 系统动力学

**学科归属**：管理科学与工程 > 系统工程  
**提出者**：杰伊·福瑞斯特  
**英文名称**：System Dynamics  
**关联概念**：[反馈回路] [因果图]

#### 核心定义（一句话）
研究复杂系统中反馈回路和动态行为的方法论。

#### 费曼式解释（说人话）

**什么是系统动力学？**

**核心概念**：
- **反馈**：结果反过来影响原因
- **延迟**：行动和结果之间有时间差
- **非线性**：不是简单的一加一等于二

**两种反馈**：

1. **正反馈（增强回路）**
   - 原因 → 结果 → 更强的原因
   - 例：口碑传播、复利

2. **负反馈（平衡回路）**
   - 原因 → 结果 → 抑制原因
   - 例：恒温器、库存调节

**经典案例：啤酒游戏**
- 啤酒销量增加
- 零售商多订货
- 批发商多订货
- 工厂多生产
- 结果：库存积压（牛鞭效应）

---

### 牛鞭效应

**学科归属**：管理科学与工程 > 供应链管理  
**英文名称**：Bullwhip Effect  
**关联概念**：[供应链] [信息扭曲]

#### 核心定义（一句话）
供应链中需求信息从下游向上游传递时被逐级放大的现象。

#### 费曼式解释（说人话）

**什么是牛鞭效应？**

**现象**：
- 消费者需求波动10%
- 零售商订单波动20%
- 批发商订单波动40%
- 制造商订单波动80%

**像鞭子一样**：手腕动一点，鞭子末端动很大。

**原因**：
1. **需求预测修正**：每个环节都加"安全边际"
2. **批量订货**：攒一批再订
3. **价格波动**：打折时囤货
4. **短缺博弈**：怕缺货就多订

**后果**：
- 库存积压
- 成本上升
- 资源浪费

**解决方案**：
- 信息共享
- 减少中间环节
- 稳定价格
- JIT生产

### 项目管理

**学科归属**：管理科学与工程 > 项目管理  
**英文名称**：Project Management  
**关联概念**：[WBS] [甘特图] [关键路径]

#### 核心定义（一句话）
在限定时间、成本和质量约束下，实现项目目标的计划和控制活动。

#### 费曼式解释（说人话）

**什么是项目管理？**

**项目 vs 日常运营**：
- 项目：临时的、独特的（如建造一座桥）
- 日常运营：重复的、持续的（如工厂生产）

**项目的三重约束（铁三角）**：
```
        质量
       /   \
      /     \
   时间 —— 成本
```
- 要快，就得花钱
- 要便宜，就得慢
- 要好，就得花时间花钱

**项目管理五大过程**：

| 过程 | 内容 |
|------|------|
| 启动 | 确定项目，获得授权 |
| 规划 | 制定计划 |
| 执行 | 按计划实施 |
| 监控 | 跟踪进度，调整偏差 |
| 收尾 | 交付成果，总结经验 |

**常用工具**：
- WBS（工作分解结构）
- 甘特图
- PERT/CPM

### WBS工作分解结构

**学科归属**：管理科学与工程 > 项目管理  
**英文名称**：Work Breakdown Structure  
**关联概念**：[项目范围] [分解]

#### 核心定义（一句话）
将项目按层次分解为可管理的工作单元的结构化方法。

#### 费曼式解释（说人话）

**什么是WBS？**

**核心思想**：把大任务分解成小任务，直到可以管理。

**例子：建造房屋**

```
1. 建造房屋
   1.1 地基
       1.1.1 挖掘
       1.1.2 浇筑
   1.2 结构
       1.2.1 墙体
       1.2.2 屋顶
   1.3 装修
       1.3.1 内装
       1.3.2 外装
   1.4 收尾
       1.4.1 清洁
       1.4.2 验收
```

**WBS的作用**：
- 明确工作范围
- 便于分配任务
- 便于估算成本和时间
- 便于跟踪进度

**100%原则**：WBS必须包含100%的工作，不能多也不能少。

### 甘特图

**学科归属**：管理科学与工程 > 项目管理

**英文名称**：Gantt Chart  
**关联概念**：[进度管理] [可视化]

#### 核心定义（一句话）
用条形图表示项目任务及其时间安排的可视化工具。

#### 费曼式解释（说人话）

**什么是甘特图？**

**样子**：横轴是时间，纵轴是任务，用条形表示任务的时间跨度。

**示例**：
```
任务        1月  2月  3月  4月  5月
─────────────────────────────────
需求分析    ████

开发            ████████
测试                ████████
```

**甘特图的优点**：

**甘特图的缺点**：

### 关键路径法

**学科归属**：管理科学与工程 > 项目管理  
**英文名称**：Critical Path Method (CPM)  
**关联概念**：[网络图] [最短工期]

#### 核心定义（一句话）
找出项目中决定最短工期的任务序列的方法。

#### 费曼式解释（说人话）

**什么是关键路径？**

**核心思想**：有些任务可以并行，有些必须串行。串行任务的链条决定了最短工期。

**例子**：
- A任务：3天（无依赖）
- B任务：5天（无依赖）
- C任务：4天（依赖A完成）
- D任务：2天（依赖B、C完成）

**分析**：
- 路径1：A→C→D = 3+4+2 = 9天
- 路径2：B→D = 5+2 = 7天

**关键路径**：A→C→D（9天）
- 这是项目最短工期
- 这些任务不能延迟

**B任务的浮动时间**：2天（9-7=2）
- B任务可以延迟2天，不影响项目完成

**管理启示**：
- 重点关注关键路径上的任务
- 非关键任务有"缓冲时间"

---

### PERT技术

**学科归属**：管理科学与工程 > 项目管理  
**全称**：Program Evaluation and Review Technique  
**关联概念**：[关键路径] [概率]

#### 核心定义（一句话）
考虑任务时间不确定性的项目进度管理技术。

#### 费曼式解释（说人话）

**PERT vs CPM**：
- CPM：任务时间确定
- PERT：任务时间不确定（用三点估计）

**三点估计**：
- **乐观时间(O)**：最顺利的情况下
- **悲观时间(P)**：最不顺利的情况下
- **最可能时间(M)**：正常情况下

**计算期望时间**：
```
期望时间 = (O + 4M + P) / 6
```

**例子**：
- 乐观：5天
- 最可能：10天
- 悲观：20天
- 期望 = (5 + 4×10 + 20) / 6 = 10.83天

**应用**：
- 研发项目（不确定性高）
- 大型工程
- 军事项目

### PDCA循环

**学科归属**：管理科学与工程 > 质量管理  
**提出者**：戴明  
**英文名称**：PDCA Cycle  
**关联概念**：[持续改进] [戴明环]

#### 核心定义（一句话）
计划-执行-检查-处理的持续改进循环。

#### 费曼式解释（说人话）

**什么是PDCA？**

**四个步骤**：

1. **Plan（计划）**
   - 确定目标
   - 制定计划

2. **Do（执行）**
   - 按计划实施
   - 收集数据

3. **Check（检查）**
   - 评估结果
   - 找出差距

4. **Act（处理）**
   - 标准化成功的做法
   - 改进失败的地方
   - 进入下一轮循环

**打个比方**：
- P：决定减肥，制定计划
- D：执行计划（运动、控制饮食）
- C：检查效果（体重下降了吗？）
- A：调整计划，继续

**特点**：
- 循环往复
- 螺旋上升
- 持续改进

---

### 六西格玛

**学科归属**：管理科学与工程 > 质量管理  
**英文名称**：Six Sigma  
**发起企业**：摩托罗拉  
**关联概念**：[DMAIC] [缺陷率]

#### 核心定义（一句话）
将缺陷率控制在百万分之3.4以下的质量改进方法。

#### 费曼式解释（说人话）

**什么是六西格玛？**

**西格玛(σ)**：统计学中的标准差

**六西格玛水平**：
- 意味着99.99966%的合格率
- 每百万次只有3.4次缺陷

**对比**：

| 西格玛水平 | 合格率 | 每百万缺陷数 |
|------------|--------|--------------|
| 1σ | 69% | 690,000 |
| 2σ | 93% | 308,000 |
| 3σ | 93.3% | 66,800 |
| 4σ | 99.4% | 6,210 |
| 5σ | 99.98% | 230 |
| 6σ | 99.99966% | 3.4 |

**DMAIC改进流程**：
1. **Define（定义）**：确定问题
2. **Measure（测量）**：收集数据
3. **Analyze（分析）**：找出原因
4. **Improve（改进）**：实施改进
5. **Control（控制）**：维持成果

**角色体系**：
- 黄带：入门
- 绿带：中级
- 黑带：高级
- 黑带大师：顶级

---

### 全面质量管理

**学科归属**：管理科学与工程 > 质量管理  
**英文名称**：Total Quality Management (TQM)  
**关联概念**：[持续改进] [顾客满意]

#### 核心定义（一句话）
全员参与的、以顾客满意为导向的持续质量改进活动。

#### 费曼式解释（说人话）

**什么是TQM？**

**核心特点**：

1. **全面**：
   - 全员参与
   - 全过程管理
   - 全面质量

2. **以顾客为中心**：
   - 顾客定义质量
   - 满足顾客需求

3. **持续改进**：
   - 没有最好，只有更好
   - PDCA循环

4. **过程导向**：
   - 管理过程，不只是结果

5. **数据驱动**：
   - 用数据说话
   - 统计工具

**实践工具**：
- 因果图（鱼骨图）
- 帕累托图
- 控制图
- 检查表

### 鱼骨图

**学科归属**：管理科学与工程 > 质量管理  
**别名**：因果图、石川图  
**英文名称**：Fishbone Diagram  
**关联概念**：[根因分析] [5M1E]

#### 核心定义（一句话）
分析问题根本原因的可视化工具。

#### 费曼式解释（说人话）

**什么是鱼骨图？**

**样子**：像鱼骨头
- 鱼头：问题（结果）
- 鱼骨：可能的原因

**分析维度（5M1E）**：

```
        人(Man)
           \
            \
    机(Machine)——[问题]——方法(Method)
            /
           /
  环境(Environment)    料(Material)    测(Measurement)
```

**使用步骤**：
1. 确定问题
2. 画鱼头和主骨
3. 按类别列出可能原因
4. 深入分析
5. 找出根本原因

**例子**：
- 问题：产品缺陷率高
- 原因分析：
  - 人：培训不足、疲劳
  - 机：设备老化、精度不够
  - 料：原料质量差
  - 法：工艺不合理
  - 环：温度不稳定
  - 测：测量标准不一致

### 管理信息系统

**学科归属**：管理科学与工程 > 管理信息系统  
**英文名称**：Management Information System (MIS)  
**关联概念**：[ERP] [决策支持系统]

#### 核心定义（一句话）
支持组织管理决策的人机系统，整合信息收集、处理、存储和传递功能。

#### 费曼式解释（说人话）

**什么是MIS？**

**三个要素**：
- **人**：使用者、管理者
- **信息**：数据、报表、分析
- **系统**：技术平台

**信息系统的层次**：

| 层次 | 系统 | 服务对象 |
|------|------|---------|
| 操作层 | 事务处理系统(TPS) | 一线员工 |
| 管理层 | 管理信息系统(MIS) | 中层管理 |
| 决策层 | 决策支持系统(DSS) | 高层管理 |
| 战略层 | 高管信息系统(EIS) | 高管 |

**作用**：
- 提高效率
- 支持决策
- 增强竞争力

---

### ERP系统

**学科归属**：管理科学与工程 > 管理信息系统  
**全称**：Enterprise Resource Planning  
**关联概念**：[信息化] [集成]

#### 核心定义（一句话）
整合企业所有资源的信息管理系统。

#### 费曼式解释（说人话）

**什么是ERP？**

**核心思想**：把企业的所有业务整合在一个系统里。

**整合的模块**：
- 财务管理
- 人力资源管理
- 生产管理
- 采购管理
- 销售管理
- 库存管理
- ……

**好处**：
1. **信息共享**：各部门看到相同数据
2. **流程标准化**：统一流程
3. **效率提升**：减少重复劳动
4. **决策支持**：实时数据

**挑战**：
1. **实施难度大**：流程改造、员工培训
2. **成本高**：软件、硬件、咨询费
3. **失败率高**：很多企业实施失败

**主流厂商**：
- SAP
- Oracle
- 用友
- 金蝶

---

### 决策支持系统

**学科归属**：管理科学与工程 > 管理信息系统  
**英文名称**：Decision Support System (DSS)  
**关联概念**：[商业智能] [数据分析]

#### 核心定义（一句话）
利用数据和模型帮助管理者进行决策的计算机系统。

#### 费曼式解释（说人话）

**什么是DSS？**

**核心功能**：
- 收集数据
- 分析数据
- 提供决策建议

**DSS vs 传统报表**：

| 传统报表 | DSS |
|---------|-----|
| 静态 | 动态 |
| 定期 | 即时 |
| 历史数据 | 预测分析 |
| 固定格式 | 灵活查询 |

**组成**：
1. 数据库
2. 模型库（分析模型）
3. 知识库（专家知识）
4. 用户界面

**应用场景**：
- 销售预测
- 投资决策
- 生产计划
- 风险评估

### 🆕 商业智能

**学科归属**：管理科学与工程 > 管理信息系统  
**英文名称**：Business Intelligence (BI)  
**关联概念**：[数据仓库] [数据可视化]

#### 核心定义（一句话）
将企业数据转化为有价值信息，支持决策的技术和方法。

#### 费曼式解释（说人话）

**什么是BI？**

简单说：**把数据变成可用的信息**。

**BI的功能**：
1. **数据整合**：从不同来源收集数据
2. **数据分析**：发现规律、趋势
3. **可视化展示**：仪表盘、图表
4. **决策支持**：为决策提供依据

**典型应用**：
- 销售分析：哪个产品卖得好？
- 客户分析：谁是高价值客户？
- 运营分析：哪个环节效率低？

**BI工具**：
- Tableau
- Power BI
- QlikView
- FineBI（帆软）

**与大数据的关系**：
- BI：处理结构化数据，支持决策
- 大数据：处理海量、复杂数据

---

### 🆕 数据仓库

**学科归属**：管理科学与工程 > 管理信息系统  
**英文名称**：Data Warehouse  
**关联概念**：[ETL] [OLAP]

#### 核心定义（一句话）
面向主题的、集成的、相对稳定的、反映历史变化的数据集合。

#### 费曼式解释（说人话）

**什么是数据仓库？**

**与传统数据库的区别**：

| 数据库 | 数据仓库 |
|--------|----------|
| 支持日常业务 | 支持分析决策 |
| 当前数据 | 历史数据 |
| 频繁更新 | 相对稳定 |
| 详细数据 | 汇总数据 |

**四个特点**（ACID原则）：
1. **面向主题**：按业务主题组织（如销售、客户）
2. **集成性**：数据来自多个系统，统一格式
3. **非易失性**：一旦写入，很少修改
4. **时变性**：保存历史数据，支持趋势分析

**ETL过程**：
- **Extract（抽取）**：从源系统获取数据
- **Transform（转换）**：清洗、格式化
- **Load（加载）**：加载到数据仓库
