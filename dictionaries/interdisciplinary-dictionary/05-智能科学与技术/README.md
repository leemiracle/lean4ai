# 智能科学与技术

> 用最简单的话，讲清楚"智能"是怎么被科学理解和实现的。

## 学科概述：智能科学与AI是什么关系？

### 想象一下

你有一台电脑。如果只是打字、存文件，它只是工具。但如果它能看懂你的表情、听懂你的话、帮你做决策，这就叫"智能"。

**智能科学与技术**就是研究"智能"这门技术的学科。

### 和AI的区别

| 视角 | 人工智能(AI) | 智能科学与技术 |
|------|-------------|---------------|
| 关注点 | 怎么做出智能程序 | 智能的本质是什么 |
| 方法 | 算法优先 | 理论+工程+认知 |
| 来源 | 计算机科学 | 多学科交叉 |
| 目标 | 解决实际问题 | 理解智能并创造智能 |

打个比方：
- AI工程师问：怎么让机器下围棋赢过人类？
- 智能科学家问：人类是怎么学会下围棋的？机器能学会吗？为什么能？

### 学科定位

```
                    智能科学与技术
                         |
          ┌──────────────┼──────────────┐
          |              |              |
       理论基础       技术实现       应用领域
    ┌───┴───┐      ┌───┴───┐      ┌───┴───┐
   信息论  控制论  机器学习  机器人  医疗  交通
   系统论  认知论  模式识别  智能控制 教育  安防
```

## 核心理论：智能的三大支柱

### 信息论：智能需要"知道"

**想象你收到一封信。**

信里有信息吗？有多少？

信息论告诉我们：**信息=消除的不确定性**。

```
例子：
- 你掷骰子前，有6种可能（不确定性高）
- 我告诉你结果是3（消除不确定性）
- 你获得了信息
```

**为什么智能需要信息论？**

1. **感知**：传感器采集的原始数据，信息量是多少？
2. **传输**：信息从传感器传到大脑（或计算机），会丢失吗？
3. **压缩**：能不能用更少的数据表达同样的信息？

**核心概念**

| 概念 | 通俗解释 | 公式 |
|------|---------|------|
| 熵 | 不确定性的大小 | H = -∑p·log(p) |
| 信息量 | 一个事件带来的信息 | I = -log(p) |
| 互信息 | 两个事物相关程度 | I(X;Y) = H(X) - H(X\|Y) |

### 控制论：智能需要"反馈"

**想象你在骑自行车。**

眼睛看路→大脑判断→手调方向→眼睛再确认→调整

这个循环就叫**反馈控制**。

控制论的核心：**系统通过反馈来适应环境**。

```
        ┌─────────────────────────────┐
        |                             |
        ↓                             |
    [传感器] → [控制器] → [执行器] → [环境]
        |                             |
        └──────── 反馈信号 ────────────┘
```

**为什么智能需要控制论？**

1. **智能体必须能调整自己的行为**
2. **必须能感知自己行为的效果**
3. **必须能在变化的环境中保持目标**

**核心概念**

| 概念 | 通俗解释 |
|------|---------|
| 反馈 | 根据结果调整行动 |
| 稳定性 | 系统不乱跑的能力 |
| 鲁棒性 | 抗干扰的能力 |
| 自适应 | 自动调节参数 |

### 系统论：智能是"整体"

**想象你买了一辆车。**

把发动机、轮子、方向盘拆开，每个都是好零件。但组装起来才能"跑"。

系统论的核心：**整体大于部分之和**。

```
智能系统 = 感知 + 处理 + 行为 + 环境
    ↓
涌现（Emergence）：整体出现了部分没有的特性
```

**为什么智能需要系统论？**

1. 智能不是单一功能，是多个子系统协同
2. 单个神经元不聪明，但千亿神经元组成的大脑很聪明
3. 优化局部≠优化整体

**系统层次**

```
    环境
      |
  社会系统（多人协作）
      |
  认知系统（思维/决策）
      |
  感知运动系统（感官+动作）
      |
  神经系统（信号传递）
      |
  细胞/硬件
```

## 智能感知：让机器"看见"和"听见"

### 传感器：机器的眼睛和耳朵

**你用眼睛看、用耳朵听。机器用什么？**

| 人类感官 | 传感器 | 检测量 | 应用 |
|---------|--------|-------|------|
| 视觉 | 摄像头 | 光线 | 人脸识别、自动驾驶 |
| 听觉 | 麦克风 | 声波 | 语音助手 |
| 触觉 | 压力传感器 | 压力 | 机器人抓取 |
| 嗅觉 | 气体传感器 | 化学成分 | 环境监测 |
| 前庭 | 加速度计/陀螺仪 | 运动 | 手机横竖屏 |

**传感器的问题**

1. **噪声**：数据不干净，像收音机的沙沙声
2. **维度高**：一张图片=几百万个像素
3. **不完整**：被遮挡了、光线暗了
4. **不同步**：摄像头和雷达的时间对不上

### 模式识别：从数据到意义

**想象教孩子认"苹果"。**

你给他看很多苹果的图片，告诉他"这是苹果"。慢慢的，他能认出新苹果。

这就是**模式识别**：

```
原始数据 → 特征提取 → 分类/匹配 → 结果
   |           |           |        |
 图片      颜色/形状    对比模板   "苹果"
```

**传统方法 vs 深度学习**

| 传统模式识别 | 深度学习 |
|-------------|---------|
| 人工设计特征 | 自动学习特征 |
| 需要领域专家 | 只需要数据 |
| 可解释性强 | 黑盒 |
| 数据需求少 | 需要大量数据 |

**经典流程**

```
1. 预处理：去噪、归一化
2. 特征提取：找有代表性的属性
3. 特征选择：去掉没用的特征
4. 分类/聚类：给数据贴标签或分组
5. 后处理：修正结果
```

**典型任务**

- **分类**：这张图是猫还是狗？
- **检测**：图里有几个人？在哪？
- **分割**：哪个像素属于人，哪个属于背景？
- **识别**：这个人是谁？

## 智能处理：让机器"思考"

### 机器学习：从数据到模型

**想象你在学骑自行车。**

没人给你公式。你就是骑、摔、调整、再骑。最终你"学会"了。

机器学习就是：**让机器从数据中自动学会规律**。

```
传统编程：
  规则 + 数据 → 答案

机器学习：
  数据 + 答案 → 规则（模型）
```

**学习范式**

| 类型 | 怎么学 | 例子 |
|------|-------|------|
| 监督学习 | 有老师告诉你对错 | 给图片标"猫/狗" |
| 无监督学习 | 自己找规律 | 把客户分群 |
| 强化学习 | 试错，做得好有奖励 | AlphaGo下棋 |
| 半监督学习 | 少量有标签+大量无标签 | 医疗诊断 |
| 自监督学习 | 自己给自己出题 | GPT预测下一个词 |

**模型选择**

```
数据量少 + 特征简单 → 逻辑回归、SVM
数据量少 + 特征复杂 → 决策树、随机森林
数据量大 + 有结构   → 深度神经网络
```

### 知识表示：让机器"记忆"

**你知道"鸟会飞"。但企鹅是鸟，却不会飞。**

怎么把这种知识告诉机器？

这就是**知识表示**的问题。

**常见方法**

| 方法 | 怎么表示 | 优点 | 缺点 |
|------|---------|------|------|
| 逻辑规则 | 如果是鸟且不是企鹅→会飞 | 可解释 | 难处理例外 |
| 语义网络 | 鸟—(是)→企鹅 | 直观 | 难推理 |
| 本体(Ontology) | 层次化的概念体系 | 标准化 | 构建成本高 |
| 知识图谱 | 实体—关系→实体 | 易查询 | 需要大规模数据 |
| 向量表示 | king - man + woman ≈ queen | 可计算 | 可解释性差 |

**知识图谱示例**

```
[李白] --(朝代)--> [唐朝]
  |
 (作品)
  |
[静夜思] --(体裁)--> [五言诗]
```

### 推理与决策

**有知识，还要会推理。**

```
已知：鸟会飞，麻雀是鸟
推理：麻雀会飞

这是演绎推理（从一般到特殊）
```

**推理类型**

1. **演绎**：从一般到特殊（数学证明）
2. **归纳**：从特殊到一般（科学发现）
3. **类比**：从A像B推出B像A（创意）
4. **概率推理**：在不确定情况下推理（贝叶斯）

**决策过程**

```
感知 → 建模 → 推理 → 规划 → 执行
                  ↑          |
                  └── 反馈 ──┘
```

## 智能行为：让机器"行动"

### 机器人学：从虚拟到实体

**软件里的智能，还是虚拟的。机器人把智能带进物理世界。**

**机器人的组成**

```
┌────────────────────────────────┐
│           控制器（大脑）         │
│    规划 | 决策 | 学习 | 通信      │
└────────────────────────────────┘
         |                    |
    ┌────┴────┐          ┌────┴────┐
    │ 传感器   │          │ 执行器   │
    │ (感知)   │          │ (行动)   │
    └─────────┘          └─────────┘
```

**关键技术**

| 技术 | 解决什么问题 |
|------|------------|
| 运动学 | 怎么让机械臂到指定位置 |
| 动力学 | 怎么考虑力和质量 |
| 路径规划 | 怎么避开障碍物 |
| 定位与建图(SLAM) | 我在哪？周围是什么？ |
| 人机交互 | 怎么和人配合 |

**机器人分类**

- 工业机器人：工厂里焊接、装配
- 服务机器人：送餐、打扫
- 医疗机器人：手术辅助
- 特种机器人：排爆、深海探测

### 智能控制：比PID更聪明

**传统控制（如PID）像条件反射：有误差就修正。**

**智能控制像有大脑：能学习、能适应、能推理。**

**对比**

| 传统控制 | 智能控制 |
|---------|---------|
| 固定参数 | 自适应参数 |
| 精确模型 | 不需要精确模型 |
| 处理不了复杂非线性 | 能处理复杂系统 |
| 不能学习 | 可以在线学习 |

**智能控制方法**

1. **模糊控制**：用"大、中、小"代替精确数值
2. **神经网络控制**：用网络学习控制策略
3. **专家控制**：把专家经验编码成规则
4. **学习控制**：从历史数据中学会控制

**模糊控制示例**

```
传统控制：如果温度低于25.3°C，加热功率=60%
模糊控制：如果温度"偏低"，加热功率"较大"

优点：更接近人的思维方式，鲁棒性强
```

## 脑启发计算：向大脑学习

### 为什么研究大脑？

**人脑功耗约20瓦，却能完成超级计算机都难做的任务。**

| | 人脑 | 超级计算机 |
|---|------|----------|
| 功耗 | ~20W | ~数MW |
| 神经元 | ~860亿 | - |
| 连接 | ~100万亿突触 | - |
| 并行度 | 极高 | 高 |
| 容错性 | 很强 | 一般 |

大脑一定有值得我们学习的"设计模式"。

### 人工神经网络

**灵感来源：神经元**

```
生物神经元：
树突(输入) → 细胞体(处理) → 轴突(输出)

人工神经元：
x₁ ─┐
x₂ ─┼→ [Σ + 激活] → y
x₃ ─┘
```

**发展历程**

```
1943: MP神经元模型
1958: 感知机（单层）
1986: 反向传播（多层）
1998: 卷积神经网络
2014: GAN、注意力机制
2017: Transformer
2020+: 大模型时代
```

### 脉冲神经网络(SNN)

**传统神经网络：用连续数值传递信息。**

**脉冲神经网络：像大脑一样，用"脉冲"传递信息。**

```
传统ANN神经元输出：0.73（一个数字）
SNN神经元输出：██░░█░██░（一串脉冲）
```

**特点**

- 更接近生物真实
- 可以用神经形态芯片实现低功耗
- 天然适合处理时序信息
- 训练更困难（不可微）

### 类脑计算硬件

**软件模拟大脑还不够，硬件也要像大脑。**

| 芯片 | 特点 |
|------|------|
| IBM TrueNorth | 100万神经元，2.56亿突触，70mW |
| Intel Loihi | 支持在线学习 |
| Google TPU | 矩阵运算加速（非类脑） |
| 国产：天机芯 | 融合ANN和SNN |

**存算一体**

传统计算机：存储和计算分开（冯·诺依曼瓶颈）

大脑：存储和计算在一起（突触既存又算）

```
传统：CPU ←(数据传输)→ 内存
类脑：存储单元就是计算单元
```

## 认知科学基础：智能的本质

### 认知是什么？

**认知=获取知识、处理知识、使用知识的全过程。**

包括：感知、注意、记忆、语言、思维、决策、意识

### 认知架构

**人类大脑是怎么组织的？**

科学家提出了各种"认知架构"来解释：

| 架构 | 核心思想 |
|------|---------|
| SOAR | 基于规则的问题求解 |
| ACT-R | 记忆是核心，有不同模块 |
| CLARION | 内隐(无意识)+外显(有意识)双系统 |
| NARS | 非公理推理系统，处理不确定性 |

**ACT-R示例**

```
[视觉模块] ──┐
[听觉模块] ──┼→ [中央执行器] ←→ [陈述记忆]
[运动模块] ──┘        ↑
                      |
               [程序记忆(技能)]
```

### 注意机制

**你不可能同时关注所有信息。大脑会选择。**

这就是**注意**。

**注意的类型**

1. **选择性注意**：只看重要信息
2. **分配性注意**：同时做两件事
3. **持续性注意**：长时间专注

**计算模型**

```
输入 → 注意力机制 → 加权输入 → 处理
          |
    哪些更重要？(学习得到)
```

深度学习中的Attention机制就借鉴了认知科学。

### 记忆系统

**人的记忆不是一个大仓库，是多个系统。**

| 类型 | 存什么 | 持续时间 | 容量 |
|------|-------|---------|------|
| 感觉记忆 | 原始感觉 | <1秒 | 大 |
| 工作记忆 | 当前处理的信息 | ~20秒 | 7±2项 |
| 长期记忆 | 知识、技能 | 终身 | 无限 |

**对AI的启发**

```
RNN/LSTM：模拟工作记忆
知识图谱：模拟语义记忆
经验回放(强化学习)：模拟情景记忆
```

### 学习理论

**人是怎么学习的？**

| 理论 | 核心观点 | AI应用 |
|------|---------|--------|
| 行为主义 | 刺激-反应，强化 | 强化学习 |
| 认知主义 | 内部心理过程 | 认知架构 |
| 建构主义 | 主动构建知识 | 主动学习 |
| 联结主义 | 神经网络连接 | 深度学习 |

### 意识问题

**最难的问题：什么是意识？**

- 机器能有意识吗？
- 意识是计算的还是物理的？
- 如何测试机器是否有意识？

**相关理论**

1. **全局工作空间理论**：意识是信息在大脑各模块间广播
2. **整合信息理论**：意识=信息整合度(Φ)
3. **高阶理论**：意识是对心理状态的高阶表征

目前没有定论，是哲学+神经科学+AI的交叉前沿。

## 总结

智能科学与技术是一门**交叉学科**，它：

- 用**信息论**理解智能处理的本质
- 用**控制论**实现智能系统的自适应
- 用**系统论**把握智能的整体性
- 用**认知科学**理解智能的来源
- 用**机器学习**实现从数据到智能
- 用**机器人学**把智能带进物理世界

**未来的智能系统**将是：
- 更低功耗（类脑芯片）
- 更具通用性（大模型→AGI）
- 更好解释（可解释AI）
- 更安全可控（对齐人类价值观）
