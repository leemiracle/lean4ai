# 设计学 - 用费曼学习法理解

## 学科概述：设计是什么？

想象你是一个原始人，需要过河。你可以：
- 游过去（可能会淹死）
- 砍树搭桥（需要力气和技巧）
- 找浅滩（可能绕很远）

**设计**就是那座桥——它是有目的的创造活动，连接"问题"和"解决方案"。

### 设计的本质

用最简单的话说：**设计 = 有目的的创造**

就像做饭：
- 材料 = 资源和约束
- 菜谱 = 设计方法和流程
- 美味佳肴 = 设计成果
- 食客满意 = 用户体验

### 四大学科的交叉

设计学像一块披萨，由四种"食材"组成：

```
        艺术（表达与美感）
              ↑
              |
商业 ←------- 设计 -------→ 工程（技术与实现）
  (价值与市场)    |
              ↓
        心理学（认知与行为）
```

| 学科 | 贡献 | 例子 |
|------|------|------|
| 艺术 | 美学、形式、表达 | 产品外观造型 |
| 工程 | 技术、结构、功能 | 电子产品的电路设计 |
| 心理学 | 认知、行为、情感 | 按钮的颜色和位置 |
| 商业 | 市场、品牌、战略 | 产品定价和定位 |

## 设计理论：设计的底层逻辑

### 什么是好设计？

想象你在评价一把椅子：
- 能坐吗？（功能性）
- 坐着舒服吗？（舒适性）
- 好看吗？（美学性）
- 结实吗？（耐用性）
- 买得起吗？（经济性）

**迪特·拉姆斯的好设计十原则**就像"好男友/女友标准"：

1. **创新** - 不要照搬，要有新意
2. **实用** - 有用才行，不能光好看
3. **美观** - 丑东西没人爱用
4. **易懂** - 看了就知道怎么用
5. **克制** - 别花里胡哨
6. **诚实** - 别装成它不是的东西
7. **耐用** - 用很久不坏
8. **细致** - 每个细节都要考虑
9. **环保** - 别糟蹋地球
10. **少即是多** - 简单就是美

### 设计三层次理论

诺曼把设计分成三个层次，就像人的三种感受：

| 层次 | 英文 | 就像... | 设计重点 |
|------|------|---------|----------|
| 本能层 | Visceral | 一见钟情 | 外观、感官 |
| 行为层 | Behavioral | 相处愉快 | 功能、操作 |
| 反思层 | Reflective | 回味无穷 | 意义、故事 |

例子：iPhone
- 本能层：好看，摸着舒服
- 行为层：好用，流畅
- 反思层：用苹果代表我的品味

## 视觉传达设计：让眼睛"说话"

### 核心概念

视觉传达就像"不用张嘴的对话"——通过图像、颜色、字体传递信息。

想象你在国外餐厅点餐：
- 看不懂菜单文字
- 但看到图片上的汉堡
- 你就知道了：这是汉堡，大概多少钱

这就是视觉传达！

### 四大要素

1. **文字（Typography）**
   - 字体有性格：宋体=传统，黑体=现代，手写体=亲切
   - 像人的穿着打扮，传达不同气质

2. **图像（Image）**
   - 插画 vs 照片 vs 图标
   - 就像讲故事用不同方式：画画、拍照、符号

3. **色彩（Color）**
   - 红色=热情/警告，蓝色=冷静/信任，绿色=自然/健康
   - 颜色是情感的语言

4. **版式（Layout）**
   - 信息如何排列
   - 就像整理房间，东西放哪很重要

### 设计原则

| 原则 | 解释 | 例子 |
|------|------|------|
| 对比 | 让重要信息突出 | 标题比正文大 |
| 对齐 | 东西不要乱放 | 文字左对齐 |
| 重复 | 建立统一感 | 全用同一种字体 |
| 亲密性 | 相关的东西放一起 | 联系方式聚在一起 |

## 产品设计：创造有用之物

### 什么是产品？

产品就是"被设计出来满足需求的东西"：
- 实体产品：手机、椅子、水杯
- 数字产品：APP、网站、游戏
- 服务产品：外卖、打车、教育

### 产品设计流程

想象你在发明一个新水杯：

```
发现问题 → 研究用户 → 生成概念 → 制作原型 → 测试改进 → 生产上市
    ↓           ↓           ↓           ↓           ↓          ↓
 水杯漏水    用户调研    画草图     3D打印     找人试用    工厂生产
```

### 产品三要素

| 要素 | 问题 | 例子（手机） |
|------|------|-------------|
| 功能性 | 能做什么？ | 打电话、上网、拍照 |
| 可用性 | 好用吗？ | 操作简单、反应快 |
| 愉悦性 | 喜欢用吗？ | 外观好看、手感好 |

### 产品生命周期

产品像人一样有生命周期：
1. **导入期** - 刚出生，没人认识
2. **成长期** - 越来越多人用
3. **成熟期** - 用的人最多
4. **衰退期** - 被新产品替代

## 环境设计：营造空间体验

### 核心概念

环境设计就是"设计人活动的场所"——让空间有目的、有情感。

想象三种房间：
- 医院：白色、简洁、消毒水味 → 感觉专业、干净
- 咖啡馆：暖色、沙发、咖啡香 → 感觉放松、舒适
- 图书馆：安静、整齐、书香 → 感觉专注、学习

同样的空间，设计不同，感受完全不同！

### 设计领域

| 领域 | 对象 | 关键词 |
|------|------|--------|
| 室内设计 | 室内空间 | 功能布局、材料选择 |
| 景观设计 | 室外环境 | 植物、地形、水体 |
| 城市设计 | 城市空间 | 街道、广场、建筑群 |
| 展示设计 | 展览空间 | 信息传递、参观流线 |

### 空间设计原则

1. **功能性** - 空间要满足使用目的
2. **安全性** - 不能有危险
3. **舒适性** - 用着舒服
4. **美观性** - 看着舒服
5. **文化性** - 体现地域和时代特征

## 交互设计：设计行为与反馈

### 什么是交互？

交互就是"一来一回的对话"：
- 你按门铃 → 门铃响 → 你知道它工作了
- 你点按钮 → 页面变化 → 你知道操作成功了

没有反馈 = 没有交互 = 用户困惑

### 交互设计五维度

| 维度 | 内容 | 例子 |
|------|------|------|
| 1D 文字 | 语言文字 | 按钮上的"确定" |
| 2D 视觉 | 图形元素 | 按钮的形状和颜色 |
| 3D 空间 | 物理或虚拟空间 | 屏幕上的位置 |
| 4D 时间 | 时序变化 | 动画效果 |
| 5D 行为 | 用户动作 | 点击、滑动、语音 |

### 交互设计原则

**尼尔森十大可用性原则**（用大白话解释）：

1. **状态可见** - 让用户知道发生了什么
2. **环境贴切** - 用用户熟悉的语言
3. **用户可控** - 给用户"后悔药"
4. **一致性** - 同样的事情同样做
5. **防错** - 在出错前阻止
6. **识别好过回忆** - 直接看到，不用想
7. **灵活高效** - 新手专家都能用
8. **审美简约** - 别太复杂
9. **容错恢复** - 出错后帮用户解决
10. **帮助文档** - 实在不会用有说明书

## 服务设计：设计完整体验

### 什么是服务？

服务 = 一系列创造价值的活动

想象去餐厅吃饭：
1. 进门被招呼
2. 看菜单点菜
3. 等菜时喝茶
4. 菜上来了吃
5. 结账离开
6. 可能还有会员积分

这一整条链路就是"服务"，服务设计就是优化这条链路！

### 服务设计要素

| 要素 | 解释 | 例子 |
|------|------|------|
| 人物 | 谁参与？ | 顾客、服务员、厨师 |
| 道具 | 用什么？ | 菜单、桌子、POS机 |
| 场景 | 在哪？ | 餐厅环境 |
| 流程 | 怎么做？ | 从点菜到上菜的步骤 |
| 规则 | 什么规则？ | 先付款还是后付款 |

### 服务触点

触点 = 服务与用户接触的每一个点：

```
服务前          服务中          服务后
  ↓               ↓               ↓
看到广告 → 进店 → 点单 → 等待 → 用餐 → 结账 → 离开 → 收到优惠券
```

每个触点都需要设计！

### 服务蓝图

服务蓝图就像"服务的设计图纸"：

| 层级 | 内容 | 可见性 |
|------|------|--------|
| 顾客行为 | 顾客做什么 | 完全可见 |
| 前台服务 | 员工在顾客面前做什么 | 部分可见 |
| 后台服务 | 员工在背后做什么 | 不可见 |
| 支持系统 | 系统和资源 | 不可见 |

## 设计思维：解决问题的方法论

### 什么是设计思维？

设计思维 = 像设计师一样思考

核心心态：
- **以人为本** - 从人的需求出发
- **发散收敛** - 先多想，再筛选
- **快速原型** - 做出来试试
- **迭代改进** - 不求一次完美

### 五步法（斯坦福d.school模型）

```
同理心(Empathize)
    ↓ 理解用户
定义(Define)
    ↓ 明确问题
构思(Ideate)
    ↓ 产生想法
原型(Prototype)
    ↓ 做出模型
测试(Test)
    ↓ 验证改进
    ↓
  循环迭代
```

### 具体怎么做？

**第一步：同理心**
- 观察：看用户怎么做事
- 访谈：问用户怎么想
- 体验：自己走一遍流程

**第二步：定义问题**
- 用"我们如何可能..."句式
- 例如："我们如何可能让老年人更容易使用智能手机？"

**第三步：构思**
- 头脑风暴：越多越好
- 不批评：先想再说
- 组合想法：1+1>2

**第四步：原型**
- 低保真：纸上画、便利贴
- 中保真：线框图、纸板模型
- 高保真：接近真实的模型

**第五步：测试**
- 让真实用户试用
- 观察他们的行为
- 听取他们的反馈
- 修改再测试

## 人机工程学：让产品适应人

### 核心概念

人机工程学 = 研究人、机器、环境之间的关系

简单说：**让产品来适应人，而不是人去适应产品**

反面例子：
- 椅子太高 → 脚悬空 → 腿麻
- 屏幕太亮 → 眼睛累 → 近视
- 按钮太小 → 点不准 → 挫败

### 三个层次

| 层次 | 内容 | 目标 |
|------|------|------|
| 物理层 | 身体尺寸、力量 | 匹配人体尺寸 |
| 认知层 | 感知、记忆、决策 | 匹配认知能力 |
| 情感层 | 情绪、体验 | 匹配情感需求 |

### 人体测量数据

设计要考虑人的身体尺寸：
- 第5百分位：小个子（5%的人比这小）
- 第50百分位：中等个子
- 第95百分位：大个子（95%的人比这小）

设计原则：**满足5%-95%的人群**

### 应用示例

| 产品 | 人机工程考虑 |
|------|-------------|
| 椅子 | 坐高、靠背角度、材质 |
| 键盘 | 按键大小、力度、排列 |
| 手机 | 握持尺寸、屏幕亮度 |
| 汽车 | 座椅调节、仪表盘位置 |

## 设计心理学：理解用户心智

### 为什么需要设计心理学？

因为用户的反应不是完全理性的：
- 同样的功能，红色按钮比蓝色按钮更让人紧张
- 同样的价格，99元比100元更让人想买
- 同样的等待，有进度条的比没进度条的更让人安心

### 核心概念

**1. 诺曼门问题**

有些门你不知道该推还是拉——这就是糟糕的设计！

解决方案：用**示能（Affordance）**
- 门把手 = 拉
- 金属板 = 推
- 设计要让操作方式显而易见

**2. 心理模型**

用户对产品怎么工作有自己的"心理模型"。

例子： thermostat（恒温器）
- 很多人以为调得越高，温度升得越快
- 实际上只是"开/关"到目标温度
- 设计应该匹配用户的心理模型，或者教育用户

**3. 错误 vs 失误**

| 类型 | 原因 | 例子 |
|------|------|------|
| 失误(Slip) | 无意识做错 | 想按A按了B |
| 错误(Mistake) | 有意识做错决定 | 买了不需要的东西 |

设计策略：
- 防止失误：按钮分开，确认操作
- 防止错误：提供信息，帮助决策

**4. 认知负荷**

人脑的处理能力有限。

设计要减少认知负荷：
- 别让用户记太多东西
- 信息分组呈现
- 提供视觉线索

**5. 峰终定律**

用户对体验的记忆取决于：
- 最强烈的高峰时刻
- 结束时的感受

应用：
- 让核心体验足够好
- 结束体验要令人愉快

## 可持续设计：为地球而设计

### 为什么需要可持续设计？

因为地球资源有限，垃圾无处堆放。

数据：
- 全球每年产生20亿吨垃圾
- 电子垃圾增长最快
- 很多产品设计时就注定被抛弃

### 设计策略

| 策略 | 解释 | 例子 |
|------|------|------|
| 减量化 | 用更少材料 | 轻量化包装 |
| 再利用 | 可以多次使用 | 可换刷头牙刷 |
| 再循环 | 材料可以回收 | 用回收塑料做产品 |
| 可拆解 | 方便拆开回收 | 模块化设计 |
| 耐用性 | 做得结实持久 | 高质量产品 |

### 全生命周期思考

从"摇篮到坟墓"考虑产品的影响：

```
原材料 → 生产 → 运输 → 使用 → 废弃
   ↓        ↓       ↓       ↓       ↓
 开采污染  能源消耗  碳排放  耗能耗材  垃圾污染
```

### 循环设计

从"线性经济"到"循环经济"：

- 线性：取 → 造 → 用 → 扔
- 循环：取 → 造 → 用 → 回收 → 再造

### 情感可持续

让用户爱上产品，不愿意扔掉：
- 耐看的设计
- 可以升级换代
- 陪伴用户成长的故事

## 学习路径

### 阶段一：建立基础认知（1-3个月）

**目标**：理解设计是什么，培养设计眼光

**学习内容**：
- 《设计心理学》（唐纳德·诺曼）
- 《写给大家看的设计书》
- 《设计中的设计》（原研哉）

**实践任务**：
- 观察身边的好设计和坏设计
- 每天收集一个喜欢的设计案例
- 尝试分析为什么这个设计好

### 阶段二：掌握核心技能（3-6个月）

**目标**：学会设计的基本方法和工具

**学习内容**：
- 设计软件：Figma/Sketch（UI）、Photoshop（图像）、Rhino（3D）
- 视觉设计基础：排版、配色、字体
- 用户研究方法：访谈、问卷、可用性测试

**实践任务**：
- 完成一个小型设计项目（如个人名片、APP界面）
- 参与一个完整的用户研究
- 学会制作低保真原型

### 阶段三：深化专业方向（6-12个月）

**目标**：在某一领域深入发展

**可选方向**：

| 方向 | 重点学习 | 推荐资源 |
|------|----------|----------|
| 视觉传达 | 品牌设计、排版 | 《网格系统》 |
| 产品设计 | 材料工艺、制造 | 《产品设计与制造》 |
| 交互设计 | 用户研究、原型 | 《About Face》 |
| 服务设计 | 系统思维、蓝图 | 《服务设计思维》 |

**实践任务**：
- 完成一个完整的设计项目
- 参加设计比赛或实习
- 建立个人作品集

### 阶段四：综合提升（1年以上）

**目标**：成为独立的设计思考者

**学习内容**：
- 设计战略与商业思维
- 跨学科知识（心理学、社会学、商业）
- 设计领导力

**实践任务**：
- 主导一个设计项目
- 参与跨学科团队合作
- 形成自己的设计方法论

### 推荐资源

**经典书籍**：
- 《设计心理学》- 诺曼
- 《情感化设计》- 诺曼
- 《设计思维》- 蒂姆·布朗
- 《交互设计精髓》- 库珀
- 《服务设计思维》- 斯蒂克多恩

**在线资源**：
- Coursera: 设计思维课程
- YouTube: The Futur、Design Course
- 网站: Dribbble、Behance、Pinterest

**工具软件**：
- 界面设计：Figma、Sketch
- 原型制作：ProtoPie、Framer
- 3D建模：Rhino、Blender
- 协作工具：Miro、Notion

## 结语

设计学是一门"让人、物、环境和谐相处"的学问。

记住三个核心：
1. **以人为本** - 设计最终是为人服务的
2. **系统思考** - 看到整体，理解关系
3. **迭代改进** - 好设计是改出来的

正如迪特·拉姆斯所说：
> "好的设计是尽可能少的设计。"

大道至简，设计亦然。
