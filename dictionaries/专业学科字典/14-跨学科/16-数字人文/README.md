# 数字人文 (Digital Humanities)

## 学科概述：当人文学科遇上计算机

### 什么是数字人文？

想象你是一个历史学家，面前堆着10000封18世纪的书信。传统做法是：一封封读，手工记录，靠记忆找规律。这个工作量可能需要数年时间。

数字人文就是**用计算机的"超能力"来研究人的问题**——自动识别手写信内容、标记通信关系、在地图上画往来路线、统计高频词汇。原本需要数年的工作，现在可能几周就能完成初步分析。

### 三个学科的碰撞

```
人文学科 → 提出问题（历史意义、文化价值、审美体验）
   ↓
计算机 → 提供方法（算法、编程、系统开发）
   ↓
数据科学 → 处理数据（统计分析、可视化、挖掘）
```

**类比理解**：如果把做研究比作做饭，人文学科决定做什么菜，计算机提供厨具，数据科学提供食材处理方法。

**传统人文学科的困境**：文献太多读不完，关系复杂理不清，规律隐蔽看不见。

**数字人文的解法**：计算机能读百万本书，算法能理清关系网络，可视化能让规律"显形"。

### 数字人文能回答什么问题？

- 某作家一生中写作风格如何变化？
- 历史上某群体的人际网络是什么样的？
- 文化现象在地理上是如何传播的？
- 大规模文本中隐藏着什么主题和模式？

## 文本挖掘与自然语言处理

### 让计算机"读"文本

就像教小孩子认字：**识字**（分词）→ **理解**（语义分析）→ **归纳**（主题建模）

计算机"读"书不是真的理解，而是通过数学方法找出文本中的模式。

### 关键技术

**命名实体识别 (NER)**：从文本中识别出人名、地名、机构名

```
原文：李白在长安遇到了杜甫，两人一起游历洛阳。
识别：人名[李白、杜甫] 地名[长安、洛阳]
```

应用：从史料中自动抽取人物关系、给文学作品标注地理位置。

**词频分析**：统计词汇出现次数，找出关键词。

进阶应用：
- **TF-IDF**：找出最能代表某文档的词（比如"爱因斯坦"之于物理学论文）
- **历时分析**：追踪词汇在不同时期的使用变化（比如"科学"一词在清末民初的含义演变）

**主题建模 (LDA)**：像给一堆文档自动分类，找出各自讨论什么主题。

类比：你有一堆没有标签的新闻，算法自动告诉你"这组讲政治，那组讲体育"。

工作原理：假设每篇文档是多个主题的混合，每个主题是一组词汇的概率分布，算法自动学习这些分布。

**情感分析**：判断文本表达的是正面还是负面情感。

方法演进：
- 基于词典 → 机器学习 → 深度学习(BERT)
- 准确率逐步提升，但对讽刺、隐喻仍有困难

**应用场景**：史料人物关系梳理、文学作品地理标注、分析情感走向、研究历史人物态度变化。

### 常用工具

| 任务 | 工具 |
|------|------|
| 中文分词 | jieba, pkuseg |
| NER | spaCy, HanLP |
| 主题建模 | Gensim, MALLET |
| 情感分析 | SnowNLP, transformers |

## 社会网络分析

### 关系即数据

传统研究关注"个体"，网络分析关注"关系"。

类比：传统方法研究一个个演员，网络方法研究演员之间的互动，发现谁认识谁，谁是社交中心。

### 网络基本要素

```
节点(Node)=实体（人、地点、概念）
边(Edge)=关系（通信、引用、共现）
权重=关系的强弱（通信次数、引用次数）
方向=关系是否单向（老师→学生是单向的）
```

### 中心性分析

识别网络中的重要节点，但"重要"有多种定义：

| 类型 | 含义 | 识别对象 | 生活类比 |
|------|------|----------|----------|
| 度中心性 | 连接数量多 | 社交活跃人物 | 派对上认识最多人的人 |
| 中介中心性 | 桥接不同群体 | 信息传播枢纽 | 两个圈子唯一的共同好友 |
| 接近中心性 | 到各节点距离短 | 信息传播中心 | 谣言从他这里传得最快 |
| 特征向量中心性 | 连接重要节点 | 幕后影响力 | 和大佬做朋友的人 |

### 社区发现

在网络中找出紧密连接的群体。

类比：在社交网络中自动识别"高中同学群"、"大学同学群"、"工作同事群"。

常用算法：Louvain（最快）、标签传播（简单）、随机游走（直观）。

应用：识别文学圈子派系、分析学术共同体、发现贸易网络区域结构。

### 可视化工具

- **Gephi**：交互式探索，适合中等规模网络
- **NetworkX**：Python分析，适合编程用户
- **Palladio**：历史网络可视化，无需编程

### 实践案例

研究《红楼梦》人物网络：
1. 提取人物共现（同一回中出现）
2. 构建网络，边权重为共现次数
3. 分析中心性，发现贾宝玉和王熙凤是关键节点
4. 社区发现，识别各房人物群体

## 地理信息系统与空间分析

### GIS = 地图 + 数据库 + 分析工具

类比：GIS就像一个智能地图，不仅能看位置，还能查询、分析、展示与位置相关的数据。

### 空间数据类型

- **矢量数据**：点（城市、事件）、线（道路、迁徙路线）、面（行政区、文化区域）
- **栅格数据**：格网化连续数据（地形高程、人口密度、气温分布）

### 关键操作

**地理编码**：把地名转换为坐标。
- 输入："北宋都城汴京"
- 输出：坐标 (34.7971°N, 114.3074°E)

挑战：古地名与现代地名不对应、同名异地歧义、地名随历史变化。

**空间聚合**：把点数据汇总到区域，如统计各县进士数量、汇总各地区文献产出。

类比：把散落各地的数据点"装进"行政区域的"篮子"里统计。

**距离分析**：
- 最近邻分析：点是聚集还是分散？
- 缓冲区分析：某范围内有什么？（比如城墙周围5公里内的聚落）
- 网络分析：沿着道路的最短路径

### 时空可视化

- **静态**：分区着色图、比例符号图
- **动态**：时间滑块展示变化过程（如王朝疆域演变）
- **叙事**：故事地图（结合地图与文字叙事）

### 工具选择

| 工具 | 特点 | 适合场景 |
|------|------|----------|
| QGIS | 开源、功能全面 | 日常分析 |
| ArcGIS | 商业、行业标准 | 专业项目 |
| Leaflet | Web交互地图 | 在线发布 |
| Kepler.gl | 大数据时空可视化 | 数据探索 |

## 数据可视化

### 为什么重要

人脑处理图像比处理数字快60000倍，善于识别模式、异常、趋势。

费曼学习法的核心：能画出图来，说明真懂了。

### 可视化类型

**比较类**：柱状图、条形图、雷达图、小多图

**趋势类**：折线图、面积图、地平线图

**关系类**：散点图、气泡图、网络图、桑基图

**分布类**：直方图、箱线图、小提琴图、热力图

**构成类**：堆叠柱状图、树状图

**人文特色**：词云、时间轴、故事地图、网络图

### 如何选择

```
想展示什么？
├─ 比较 → 柱状图/条形图
├─ 趋势 → 折线图
├─ 关系 → 散点图/网络图
├─ 分布 → 直方图/箱线图
└─ 构成 → 堆叠图/树状图
```

### 设计原则

1. **数据-墨水比**：每滴墨水都要传达信息，删掉无用的装饰
2. **避免误导**：坐标轴从零开始，比例一致
3. **颜色使用**：考虑色盲，有意义的配色（不用彩虹色）
4. **标注清晰**：标题、图例、数据来源

### 工具生态

**编程**：Python (Matplotlib, Seaborn, Plotly)、R (ggplot2)、JavaScript (D3.js)

**无代码**：Tableau、RawGraphs

## 数字档案与文化遗产

### 从纸质到数字

**传统档案**：实体保存空间有限、检索困难、访问受限

**数字档案**：无限存储、全文检索秒级响应、远程访问

### 数字化流程

```
评估选择 → 扫描采集 → 图像处理 → OCR识别 → 
元数据著录 → 质量控制 → 存储发布
```

每一步都需要标准化和质量把控。

### OCR技术

**OCR (Optical Character Recognition)**：把图像中的文字转成可搜索的文本。

**挑战**：手写体识别困难、古籍排版复杂、墨迹褪色污损

**方案**：专用古籍OCR模型、人工校对与众包、结合人工标注的机器学习

### 元数据标准

**Dublin Core**：15个核心元素（Title、Creator、Subject、Date、Type...）

类比：就像图书馆的目录卡片，但更结构化、机器可读。

### 数字保存

**威胁**：技术过时（格式打不开）、载体老化（硬盘损坏）、灾难事件

**策略**：OAIS参考模型、3-2-1原则（3副本2种介质1份异地）、格式迁移、定期校验

## 数字学术出版

### 传统 vs 数字出版

| 维度 | 传统 | 数字 |
|------|------|------|
| 周期 | 月/年 | 实时 |
| 容量 | 有限 | 无限 |
| 互动 | 单向 | 双向 |
| 链接 | 无 | 超链接 |
| 多媒体 | 图片为主 | 图音视交互 |

### 新形态

**多模态学术写作**：结合文本、图像、音频、视频、数据、代码

**交互式论文**：可调参数图表、可探索数据集、可运行代码

**开放获取**：
- 绿色OA：自存档到机构库
- 金色OA：期刊直接开放
- 钻石OA：无作者费、无读者费

**数据论文**：专门发表数据集的论文，包含数据描述、采集方法、处理流程、质量评估。

### 协作工具

**Git工作流**：追踪修改历史、多人协作、随时回退

**学术标识**：DOI（论文）、ORCID（研究者）、ROR（机构）、DataCite（数据集）

## 虚拟现实与增强现实

### 概念区分

- **VR (虚拟现实)**：完全沉浸的虚拟世界，戴头显
- **AR (增强现实)**：在现实上叠加虚拟信息，用手机/眼镜
- **MR (混合现实)**：虚实融合与交互

### 人文应用

**虚拟重建**：古罗马建筑复原、毁损古迹数字重建、历史城市3D再现

**沉浸式体验**：历史场景"穿越"、文学作品场景还原、博物馆虚拟参观

**增强现实导览**：博物馆展品信息叠加、历史遗迹古今对比

**空间人文**：文学地理实地体验、历史事件地点重现、文化记忆空间标注

### 技术栈

**开发**：Unity、Unreal Engine、A-Frame (WebVR)

**硬件**：Oculus、HTC Vive (VR)；手机ARKit/ARCore、HoloLens (AR)

## 计算史学

### 从"读历史"到"计算历史"

**计量史学 (Cliometrics)**：1950-60年代，用统计方法研究经济史

**数字史学**：扩展到文本挖掘、网络分析、空间分析、可视化

### 典型研究

**历史人口学**：分析户籍、族谱、教区记录，研究人口结构、婚姻模式、迁徙路线

**史料数据库**：人物数据库（CBDB）、史籍全文库、历史地理信息系统

**历史网络分析**：政治精英社会网络、贸易网络演变、知识传播路径

### 史料数字化处理

```
原始史料 → 数字化 → OCR → 标注 → 结构化 → 数据库 → 分析
```

**挑战**：史料不完整、数据时间覆盖不均、历史概念界定模糊

## 数字文学研究

### 文学的"远读"

**概念** (Moretti)：近读是精读少量文本，远读是宏观把握大量文本。

类比：近读像用显微镜看细胞，远读像用卫星看地貌。

**方法**：统计分析大规模语料、发现文学史模式、用数据挑战常识

### 研究方向

**作者归属研究**：确定匿名作品作者，使用文体计量学、词频统计、机器学习分类

经典案例：识别《联邦党人文集》争议文章的作者。

**文体分析**：词汇丰富度、句子长度、标点模式、词性序列

**叙事分析**：情节结构形式化、人物网络构建、时空叙事映射

**情感曲线**：分段情感分析、绘制情感时序图、识别叙事模式

研究发现小说的情感弧线有几种基本类型（如"从悲剧到幸福"、"从幸福到悲剧"）。

### 语料库建设

**关键**：文本选择代表性、版本校勘标准化、元数据著录、版权访问

## 学习路径

### 第一阶段：基础准备 (1-3个月)

**人文**：确定领域（历史/文学/艺术...）、了解研究问题、阅读经典案例

**技术**：Python基础、Pandas数据处理、Git操作

**数学**：统计学概念、线性代数入门、概率论基础

建议：先从一个感兴趣的小问题开始，边做边学。

### 第二阶段：核心技能 (3-6个月)

**文本**：jieba分词、词频统计可视化、主题建模、情感分析

**网络**：NetworkX、中心性指标、Gephi可视化

**空间**：QGIS、空间数据类型、地理编码

**可视化**：Matplotlib/Seaborn、Plotly交互图表

建议：每个技能做一个实践项目。

### 第三阶段：项目实践 (6-12个月)

**小型项目**：小数据集、完整流程（获取→处理→分析→可视化）、写报告

**中型项目**：更大数据、数据清洗整合、多种方法结合

### 第四阶段：深入研究 (1年+)

**方法论**：阅读方法论文献、理解局限、批判性思考

**原创研究**：发现问题、设计方案、执行研究、撰写论文

### 推荐资源

**著作**：《数字人文导论》、《远读》(Moretti)

**课程**：Coursera Digital Humanities、B站Python数据分析

**期刊**：Digital Humanities Quarterly、Journal of Digital History

**项目**：CBDB（人物传记）、CHGIS（历史地理）、Project Gutenberg

**社区**：DH年会、中国数字人文年会

## 结语

数字人文不是取代传统人文，而是提供新的工具和视角。

**核心理念**：
- 问题驱动，技术为用
- 保持人文关怀与批判精神
- 定量与定性平衡
- 推动开放协作的学术生态

**未来的数字人文研究者**：既是人文学者也是技术实践者，能提出深刻问题也能用工具探索，懂方法也懂局限，能独立研究也能跨学科协作。

数字人文的大门向所有对人文学科有热情、愿意学习新技术的人敞开。
