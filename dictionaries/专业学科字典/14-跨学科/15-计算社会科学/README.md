# 计算社会科学 (Computational Social Science)

## 学科概述：当社会学遇上计算机和统计学

想象你是一个社会学家，以前你只能通过问卷调查几百个人来研究社会现象。现在，突然有人给了你一个超级望远镜——你可以同时观察几亿人在社交媒体上的行为、手机移动轨迹、消费记录。这就是计算社会科学。

**核心公式**：
```
计算社会科学 = 社会学问题 + 计算机方法 + 统计思维 + 大数据
```

**为什么现在才出现？**
- 以前：数据稀缺、计算昂贵
- 现在：人类行为数字化产生海量数据、云计算让分析成为可能

**三个学科各贡献什么？**
| 学科 | 贡献 |
|------|------|
| 社会学 | 提出有意义的问题，理解社会结构和意义 |
| 计算机科学 | 提供处理大规模数据的工具和算法 |
| 统计学 | 帮助从数据中推断因果关系，避免伪相关 |

## 社会网络分析 (Social Network Analysis)

### 核心思想

把社会看成一张网：人是节点，关系是边。

就像分析蜘蛛网的结构——哪些丝承重、哪些节点是关键连接点、整张网怎么传递震动。

### 关键概念

**1. 节点的重要性**
- 度中心性：认识多少人（朋友数量）
- 介数中心性：多少最短路径经过你（信息中介）
- 接近中心性：到其他所有人的平均距离（消息传播速度）
- 特征向量中心性：你的朋友有多重要（名人效应）

```
度中心性高的节点：班级里的社交达人
介数中心性高的节点：连接不同圈子的桥梁人物
```

**2. 网络结构特征**
- 聚类系数：你的朋友互为朋友的概率（物以类聚）
- 平均路径长度：任意两人之间的平均距离（六度分隔）
- 度分布：网络中度数的分布情况（幂律分布=少数枢纽节点）

**3. 小世界网络**
- 高聚类 + 短路径 = 小世界
- 解释为什么信息传播如此之快

**4. 无标度网络**
- 度分布服从幂律：少数节点连接极多，大多数节点连接很少
- 现实例子：互联网、社交网络、引文网络

### 分析方法

```
1. 网络可视化：Gephi, NetworkX, igraph
2. 社区发现：找到紧密连接的子群体
3. 影响力传播：信息如何在网络中扩散
4. 链接预测：预测谁可能与谁建立联系
```

### 应用场景
- 谣言传播追踪
- 关键意见领袖识别
- 组织内部沟通分析
- 疾病传播建模

## 计算文本分析 (Computational Text Analysis)

### 核心思想

文本是人类思想和行为的记录。计算文本分析就是教计算机"读懂"海量文本，发现人类读不完的模式。

### 主要方法

**1. 词典方法**
- 最简单直接：数关键词出现次数
- 情感分析：用情感词典判断文本正面/负面
- 限制：依赖词典质量，难以处理讽刺和语境

**2. 主题模型 (Topic Modeling)**
- LDA（潜在狄利克雷分配）：自动发现文档集合中的主题
- 想象：把一堆文章自动分类成"政治"、"体育"、"经济"等主题
- 每个文档是多个主题的混合

```
文档 = 主题1(30%) + 主题2(50%) + 主题3(20%)
主题 = 词1(高概率) + 词2(高概率) + ...
```

**3. 词嵌入 (Word Embeddings)**
- 把词语变成数字向量
- 语义相似的词在向量空间中距离近
- Word2Vec, GloVe, BERT

```
king - man + woman ≈ queen
```

**4. 大语言模型**
- GPT, BERT 等
- 可以做更复杂的理解任务
- 情感分析、实体识别、关系抽取

### 应用场景
- 舆情监测：分析社交媒体对某事件的反应
- 政策文本分析：追踪政策关注点的变化
- 历史文献挖掘：发现历史叙事的演变
- 谣言检测：识别虚假信息

## 社会模拟 (Social Simulation)

### 核心思想

在计算机里创建一个"虚拟社会"，观察它如何演化。就像玩《模拟城市》，但目的是理解真实社会规律。

### 主要方法

**1. 基于主体的建模 (Agent-Based Modeling, ABM)**

想象你在模拟一个市场：
- 每个消费者是一个"主体"(Agent)
- 主体有自己的规则（买便宜货、跟风买、只买品牌等）
- 主体之间会互动（交流、模仿、竞争）
- 观察整体市场价格的演化

```python
class Consumer(Agent):
    def decide(self):
        if self.is_trend_follower:
            buy_popular_product()
        else:
            buy_cheapest_product()
```

**2. 为什么需要模拟？**
- 很多社会现象难以做实验（伦理、成本）
- 复杂系统中，个体行为≠集体结果
- 可以测试政策干预的效果

**3. 经典案例**
- 谢林模型：解释种族隔离如何从微小的个体偏好涌现
- 糖域模型：研究财富不平等的涌现
- 人工股市：理解市场泡沫和崩盘

### 关键原则
- KISS (Keep It Simple, Stupid)：从简单规则开始
- 验证：模拟结果是否符合已知的真实数据
- 敏感性分析：参数变化如何影响结果

## 大数据与社会 (Big Data and Society)

### 核心思想

大数据改变了我们认识社会的方式——从"问问题收集数据"到"从数据中发现问题"。

### 大数据的4V特征

| V | 含义 | 社会研究意义 |
|---|------|-------------|
| Volume | 数据量大 | 可以研究小数据看不到的模式 |
| Velocity | 数据产生快 | 可以实时追踪社会变化 |
| Variety | 数据类型多 | 多角度理解同一现象 |
| Veracity | 数据质量参差 | 需要谨慎处理偏差 |

### 数据来源

```
1. 数字痕迹：社交媒体、搜索记录、GPS轨迹
2. 行政数据：政府记录、医疗数据、教育数据
3. 传感器数据：手机信号、交通监控、环境监测
4. 交易数据：信用卡记录、电商行为
```

### 研究范式转变

| 传统方法 | 大数据方法 |
|---------|-----------|
| 样本调查 → | 全量数据 |
| 横截面 → | 纵向追踪 |
| 问你想什么 → | 看你做什么 |
| 假设驱动 → | 模式发现 |

### 挑战与陷阱

**1. 选择偏差**
- 社交媒体用户≠全体人口
- 只分析"发声"的人

**2. 算法偏差**
- 数据收集本身受算法影响
- 平台推荐改变用户行为

**3. 虚假相关**
- 大数据容易发现"显著"但无意义的相关
- 需要理论指导解释

## 数字社会研究 (Digital Society Research)

### 核心思想

研究数字技术如何重塑社会，以及社会如何塑造数字技术。

### 研究主题

**1. 数字不平等**
- 数字鸿沟：接入、技能、使用的差异
- 算法歧视：AI如何放大社会不平等

**2. 在线社区**
- 虚拟社区的形成与演化
- 身份认同与在线自我呈现

**3. 数字劳动**
- 平台经济与零工劳动者
- 数据作为劳动产品

**4. 数字极化**
- 回音室与过滤气泡
- 社交媒体与政治极化

**5. 数字治理**
- 平台权力与监管
- 数字威权主义

### 研究方法

```
1. 数字民族志：深入在线社区观察
2. API数据采集：从平台获取数据
3. 网络爬虫：系统性收集网页数据
4. 实验：A/B测试、在线实验
5. 混合方法：定量+定性结合
```

## 计算政治学 (Computational Politics)

### 核心思想

用计算方法研究政治现象：选举、舆论、政策、冲突。

### 主要研究领域

**1. 选举预测**
- 用社交媒体情绪预测投票意向
- 搜索趋势与候选人支持率
- 限制：难以预测选举中的意外事件

**2. 政治文本分析**
- 议会发言的自动分类
- 政党宣言的比较
- 政治框架的识别

**3. 政治传播**
- 假新闻的传播机制
- 政治广告的精准投放
- 社交媒体与政治动员

**4. 冲突预测**
- 用新闻文本预测暴力事件
- 社交媒体信号与抗议活动

### 经典问题
- 社交媒体如何影响民主？
- 算法推荐是否导致政治极化？
- 机器人账号如何操纵舆论？

## 计算经济学 (Computational Economics)

### 核心思想

用计算方法解决经济学问题：复杂市场、策略互动、机制设计。

### 主要方向

**1. 代理人基础计算经济学 (ACE)**
- 用ABM模拟市场
- 研究异质性主体的互动
- 理解市场动态和泡沫

**2. 算法博弈论**
- 计算纳什均衡
- 拍卖设计
- 机制设计的计算复杂性

**3. 计量经济学与机器学习**
- 因果推断与预测的结合
- 高维数据的处理
- 处理效应的异质性

**4. 平台经济学**
- 双边市场的定价
- 网络效应的建模
- 平台竞争与反垄断

### 应用场景
- 电力市场模拟
- 交通网络定价
- 在线广告拍卖
- 共享经济平台

## 社会计算 (Social Computing)

### 核心思想

设计支持社会互动的计算系统，同时用计算方法理解社会行为。

### 两个方向

**1. 系统设计方向**
- 如何设计促进协作的系统
- 在线社区的设计原则
- 人机协作的机制

**2. 分析研究方向**
- 从用户行为数据中理解社会规律
- 预测用户行为
- 优化系统设计

### 典型系统

| 系统类型 | 例子 | 核心问题 |
|---------|------|---------|
| 协作系统 | Wikipedia | 如何激励贡献、保证质量 |
| 推荐系统 | Netflix | 如何平衡个性化与发现 |
| 众包平台 | Amazon MTurk | 如何设计激励机制 |
| 社交媒体 | Twitter | 如何促进良性互动 |

### 核心挑战
- 搭便车问题：如何激励贡献
- 信息过载：如何筛选和推荐
- 信任建立：陌生人协作
- 集体智慧：如何汇聚分散信息

## 伦理与隐私 (Ethics and Privacy)

### 核心问题

我们有能力分析任何人，但我们应该这样做吗？

### 主要伦理议题

**1. 知情同意**
- 用户知道数据被用于研究吗？
- 服务条款≠知情同意

**2. 隐私保护**
- 匿名化够吗？（通常不够）
- 差分隐私：在数据中加入可控噪声
- k-匿名、l-多样性

**3. 数据正义**
- 谁从数据中获益？
- 谁被数据排斥？

**4. 算法公平**
- 算法如何继承和放大偏见
- 如何定义和测量公平

### 研究伦理实践

```
1. 伦理审查：IRB/伦理委员会审批
2. 数据最小化：只收集必要数据
3. 保护措施：加密、匿名化、访问控制
4. 透明度：公开研究方法和数据使用
5. 公众参与：让受影响者参与设计
```

### 关键问题清单
- [ ] 数据来源是否合法合规？
- [ ] 研究是否尊重研究对象？
- [ ] 研究结果是否可能造成伤害？
- [ ] 是否采取了隐私保护措施？
- [ ] 研究是否透明可复现？

## 核心思维模式

### 1. 数据不是中立的
数据由人产生、被人收集、用算法处理——每一步都有偏见。

### 2. 相关不等于因果
大数据擅长发现相关，但因果需要研究设计和理论。

### 3. 个体与集体不同层次
"涌现"：个体行为的集体后果无法从个体规则直接推断。

### 4. 技术与社会共同演化
技术改变社会，社会也塑造技术——两者不可分离。

### 5. 伦理贯穿始终
能力越大，责任越大。伦理不是事后的装饰，是研究的底色。
