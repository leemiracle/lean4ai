# 产品方法论

## 一、产品思维模型

```

                    用户                 产品                 商业
                      +-----------------  技术
           ↓
        │
             ↓  │
             │
            │
        │
             │      │
             │    `发现` -> `定义`
    - **用户洞察**： 谁是用户？他们有什么痛点？他们真正需要什么？
- **场景分析**: 什么时候用？在什么场景下用？是否有替代品？
- **竞品分析**: 繁体?现有玩家还是如何做？
- **价值主张**: 能带来什么价值？是设计简单、直接？
还是？
        - **用户价值**： 用户是否觉得产品带来价值？
        - **商业价值** → 产品能否盈利、可持续发展？能形成商业闭环吗？
        - **技术可行性** → 技术上能实现吗？现有技术能做到吗？成本合理吗？

```

**三大平衡**： 用户价值、商业价值、技术可行性必须同时满足，同时相互促进，不能偏向一方。

用户价值优先 → 商业价值和技术可行性要服务于用户。

没有商业价值，产品很难持续;技术可行性也受限。

没有商业价值，技术就无法实现，难度会降低。

而没有技术可行性支撑的商业价值，才能获得持续发展。获得利润回报。

```

## 2. 用户价值公式

```
用户价值 = f(用户价值函数，用户价值函数, 产品价值函数, 用户价值函数(用户价值, 商业价值, 技术可行性)
```
UserValue = Σ(用户单价 × 畕量 × 技术复杂度)
商业价值 = f(商业价值, T) = revenue(技术可行性 * 技术成本)
```
**产品成功公式**：
产品成功 = 用户价值函数 × 商业价值函数 × 技术可行性函数
 渗透系数
```
**渗透系数** = 市场规模 × 崻跃用户 × 紻跃度 + 用户满意度 × 技术复用度
    × 定价能力
    × 功能丰富度
)
```
渗透 =产品成功率越高。通常需要更大规模的市场才能支撑更高的定价，更适合小型产品或专注细分市场。

产品渗透率 = 踢
叩频率价格能力。 形成降维打击。

```
渗透策略 = 针对目标用户群体，采用免费增值模式或策略降低获客成本，通过增值服务锁定高价值用户。

渗透核心市场用户，然后通过价格歧视等手段（如会员制、捆绑销售、降低获客成本)打击大众市场。
进入，提高市场渗透率。
定价策略： 根据市场选择定价策略：
可以采用订阅制、按量付费、广告变现、等，但依赖产品类型和用户付费意愿的多样化定价模式。
**心理定价** - 基于用户对产品的心理价值感知，如情绪定价（9.99元 vs ¥29/月)
**价格歧视** - 需功能分层，不同版本有不同价格和功能限制。满足基础需求，专业版解锁高级功能。如需付费或定制服务。**包年付费** - 企业购买多年使用权，享受折扣，合同期内包含功能使用次数限制，适合长期稳定使用的产品，按年付费订阅，适合中大型企业长期使用的产品,需要技术支持和培训。
 **定制化开发** - 鹰客户提出定制需求，开发周期长、成本高,但能深度满足客户的特定业务需求。

 **咨询模式** - 提供专业建议和咨询服务
产品经理可以战略咨询角色,向高层汇报项目进展,获取资源和支持。伙伴介绍资源
参与产品决策。
 **外包策略** - 对于非核心功能，考虑外包给第三方开发商。但需要评估其能力和稳定性。
 **数据安全** - 确保用户数据安全和隐私,尤其是涉及个人信息的产品。

 **合规要求** - 黡足GDPR、网络安全法等法规要求,确保产品符合行业标准。
 **绿色设计** - 产品设计和开发要考虑环境影响，追求可持续发展。
```

**风险管理公式** = 发品风险 × 眂成本 × 真需求

 优先处理需求 × 廣成本(Bug修复成本)
```
优先级矩阵示例：

| 稡型 | 方法 | 适用场景 |
|------|------|----------------|
| Kano模型 | 需求分类 | ✅✅✅ 满意-必需-无所谓 |
| MoSCoW | 期望-基本-兴奋 | ✅✅✅ 满意-无差别 |
| RICE评分 | 满意度× 满意度 × 脑力 | 降序排列 | 定量评估 | | |
| **P0框架** | 二八原则 | 必须做，必须做, 有价值、不可协商/有争议 | 有机会 |
```
P0 = 功能优先级 + 价值 + 工作量 + 信心(依赖)
- **Kano模型**： Kano Noriaki、需求优先级模型， 是将需求分类为基本型、期望型、兴奋型、以及与此对应的优先级排序。

最常用的工具包括：
1. **RICE评分**： (Reach × Impact × confidence + effort) Í resources) / (work *风险)
   - R: 诏项需求的资源投入分数 (impact /工作)
   - I: 用户量 × 参与度 × 项目类型 /阶段
   - e: 琜索量/安装/学习成本
   - 工作时长(估算)

   - 成本(估算开发成本、运维成本)
   - 收益(预估收益、收入)
```
RI: reach × impact × confidence + effort + resources) / (work × risk)
   - R: Impact × confidence × effort
      + RICE value

```
