# 设计思维

## 一、定义

**设计思维(Design Thinking)**是一种以人为本的创新方法论,通过同理心理解用户需求,创新解决方案。

## 二、五阶段模型

```
设计思维五阶段

1. 共情(Empathize)
   ┌──────────┐
   │ 理解用户  │
   │ 观察      │
   │ 访谈      │
   └─────┬────┘
         ↓
2. 定义(Define)
   ┌──────────┐
   │ 明确问题  │
   │ 用户洞察  │
   │ POV      │
   └─────┬────┘
         ↓
3. 构思(Ideate)
   ┌──────────┐
   │ 发散思维  │
   │ 头脑风暴  │
   │ 方案生成  │
   └─────┬────┘
         ↓
4. 原型(Prototype)
   ┌──────────┐
   │ 快速原型  │
   │ 低保真    │
   │ 可测试    │
   └─────┬────┘
         ↓
5. 测试(Test)
   ┌──────────┐
   │ 用户测试  │
   │ 收集反馈  │
   │ 迭代优化  │
   └──────────┘
         ↓
      持续迭代
```

## 三、核心方法

### 3.1 共情方法
- 用户观察
- 深度访谈
- 同理心地图
- 用户旅程图

### 3.2 定义工具
- POV(Point of View)
- How Might We
- 问题陈述

### 3.3 构思技术
- 头脑风暴
- 最坏主意
- 类比思维
- SCAMPER

### 3.4 原型类型
- 纸上原型
- 线框图
- 可交互原型
- 角色扮演

### 3.5 测试方法
- 可用性测试
- A/B测试
- 用户反馈

## 四、最佳实践

DO:
✓ 深度共情
✓ 聚焦问题
✓ 发散思维
✓ 快速原型
✓ 用户验证

DON'T:
✗ 跳过共情
✗ 过早收敛
✗ 追求完美
✗ 不做测试

记住: 设计思维是过程,不是结果
