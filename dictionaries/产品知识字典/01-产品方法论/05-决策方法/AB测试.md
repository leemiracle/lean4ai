# AB测试

## 一、定义

**AB测试**是一种对比实验方法,通过将用户随机分配到不同版本,比较各版本的表现,以数据驱动决策。

## 二、AB测试流程

```
AB测试流程

1. 提出假设
   ┌────────┐
   │ 问题   │
   └───┬────┘
       ↓
2. 设计实验
   ┌────────┐
   │ 方案A  │ vs │ 方案B │
   └────┬───┘    └───┬──┘
        └────┬────────┘
             ↓
3. 流量分配
   ┌────────────┐
   │ 随机分流   │
   │ A组 50%   │
   │ B组 50%   │
   └──────┬─────┘
          ↓
4. 收集数据
   ┌────────────┐
   │ 转化率对比 │
   └──────┬─────┘
          ↓
5. 分析结果
   ┌────────────┐
   │ 显著性检验 │
   └──────┬─────┘
          ↓
6. 决策实施
   胜出方案全量发布
```

## 三、AB测试要素

### 3.1 假设设计

示例:
- 假设: 红色按钮比绿色按钮转化率高
- 指标: 点击率、转化率
- 预期: 提升10%

### 3.2 样本量计算

```
样本量影响因素:
1. 显著性水平(α): 通常0.05
2. 统计功效(1-β): 通常0.8
3. 基准转化率
4. 期望提升幅度

计算公式:
n = 16 * σ² / Δ²
```

### 3.3 显著性检验

- t检验
- 卡方检验
- 置信区间

## 四、AB测试最佳实践

DO:
✓ 只测试一个变量
✓ 样本量足够
✓ 运行时间足够
✓ 同时测试多个指标

DON'T:
✗ 同时测试多个变量
✗ 样本量不足
✗ 过早下结论
✗ 忽视用户体验

记住: AB测试是工具,不是目的
