# 决策框架

## 一、定义

**决策框架**是帮助做出系统化、结构化决策的方法和工具集，确保决策过程的科学性和一致性。

```
决策框架的作用

输入              处理              输出
┌──────┐      ┌──────────┐      ┌──────┐
│信息  │ ───→ │ 决策框架 │ ───→ │ 决策 │
│选项  │      │          │      │ 行动 │
│约束  │      └──────────┘      └──────┘
└──────┘            ↑
                  降低认知偏差
                  提高决策效率
                  确保决策质量
```

## 二、SWOT分析

### 2.1 基本概念

**SWOT分析**是一种战略分析工具，通过分析内部优势劣势和外部机会威胁来制定策略。

```
SWOT矩阵

              内部因素
         ┌────────┬────────┐
         │Strength│Weakness│
         │ 优势   │ 劣势   │
─────────┼────────┼────────┤
外 Opportunity│  SO    │  WO   │
部  机会     │增长型  │ 扭转型 │
因 ─────────┼────────┼────────┤
素  Threat │  ST    │  WT   │
    威胁   │ 多元化 │ 防御型 │
         └────────┴────────┘
```

### 2.2 SWOT分析步骤

```
步骤1: 识别优势(Strengths)
┌────────────────────┐
│ 我们擅长什么？      │
│ 有什么独特资源？    │
│ 用户为什么选择我们？│
└────────────────────┘

步骤2: 识别劣势(Weaknesses)
┌────────────────────┐
│ 哪里可以改进？      │
│ 缺乏什么资源？      │
│ 用户抱怨什么？      │
└────────────────────┘

步骤3: 识别机会(Opportunities)
┌────────────────────┐
│ 市场有什么趋势？    │
│ 技术有什么突破？    │
│ 竞争对手有什么弱点？│
└────────────────────┘

步骤4: 识别威胁(Threats)
┌────────────────────┐
│ 竞争对手在做什么？  │
│ 政策有什么变化？    │
│ 用户需求如何变化？  │
└────────────────────┘

步骤5: 制定策略
- SO策略：发挥优势抓住机会
- WO策略：克服劣势抓住机会
- ST策略：发挥优势应对威胁
- WT策略：减少劣势回避威胁
```

### 2.3 SWOT案例

```
案例：某内容平台SWOT分析

优势(S):
- 优质内容创作者资源
- 强大的推荐算法
- 用户粘性高

劣势(W):
- 变现能力弱
- 海外市场经验不足
- 内容审核成本高

机会(O):
- 短视频市场增长
- 知识付费兴起
- 海外市场空间大

威胁(T):
- 竞争对手增加
- 内容监管趋严
- 用户增长放缓

策略制定：
SO：利用算法优势拓展短视频
WO：引入知识付费提升变现
ST：加强内容审核应对监管
WT：优化成本结构提升效率
```

## 三、决策矩阵

### 3.1 基本概念

**决策矩阵**（又称选择矩阵）是一种多标准决策工具，用于在多个选项中做出最优选择。

```
决策矩阵结构

         评价标准
         ↓
    ┌─────────────────────┐
    │ 权重 │ C1 │ C2 │ C3 │ ← 标准权重
    ├──────┼────┼────┼────┤
选项│ A    │    │    │    │
    ├──────┼────┼────┼────┤
    │ B    │    │    │    │
    ├──────┼────┼────┼────┤
    │ C    │    │    │    │
    └──────┴────┴────┴────┘
    
    加权总分 → 最优选择
```

### 3.2 决策矩阵应用步骤

```
步骤1: 明确决策目标
示例：选择技术方案

步骤2: 列出所有选项
- 方案A：自研
- 方案B：采购
- 方案C：外包

步骤3: 确定评价标准
- 成本（权重30%）
- 时间（权重25%）
- 质量（权重25%）
- 风险（权重20%）

步骤4: 设定权重
总权重 = 100%

步骤5: 评分（1-5分）
方案A: 成本3分，时间2分，质量5分，风险3分
方案B: 成本4分，时间5分，质量3分，风险4分
方案C: 成本2分，时间4分，质量2分，风险2分

步骤6: 计算加权总分
方案A: 0.3×3 + 0.25×2 + 0.25×5 + 0.2×3 = 3.2
方案B: 0.3×4 + 0.25×5 + 0.25×3 + 0.2×4 = 4.0
方案C: 0.3×2 + 0.25×4 + 0.25×2 + 0.2×2 = 2.5

步骤7: 选择最优方案
方案B得分最高，选择采购方案
```

### 3.3 决策矩阵示例

| 方案 | 成本(30%) | 时间(25%) | 质量(25%) | 风险(20%) | 加权总分 |
|-----|-----------|-----------|-----------|-----------|----------|
| 自研 | 3(0.9) | 2(0.5) | 5(1.25) | 3(0.6) | 3.25 |
| 采购 | 4(1.2) | 5(1.25) | 3(0.75) | 4(0.8) | 4.0 ✓ |
| 外包 | 2(0.6) | 4(1.0) | 2(0.5) | 2(0.4) | 2.5 |

## 四、加权评分法

### 4.1 基本概念

**加权评分法**是一种量化决策方法，通过对各评价因素赋予权重，计算综合得分来做决策。

```
加权评分公式

总分 = Σ (因素得分 × 因素权重)

其中：
- 因素得分：1-5或1-10分
- 因素权重：反映因素重要性
- 权重之和 = 1 (或100%)
```

### 4.2 加权评分法应用

```
案例：产品功能优先级排序

功能1: AI推荐
功能2: 社交分享
功能3: 会员体系

评价因素及权重：
- 用户价值（40%）
- 实现成本（30%）
- 技术难度（20%）
- 战略重要性（10%）

评分表：

| 功能 | 用户价值 | 成本 | 难度 | 战略 | 总分 |
|-----|---------|------|------|------|------|
| AI推荐 | 5×0.4=2.0 | 2×0.3=0.6 | 2×0.2=0.4 | 5×0.1=0.5 | 3.5 |
| 社交 | 3×0.4=1.2 | 4×0.3=1.2 | 4×0.2=0.8 | 3×0.1=0.3 | 3.5 |
| 会员 | 4×0.4=1.6 | 4×0.3=1.2 | 5×0.2=1.0 | 4×0.1=0.4 | 4.2 ✓ |

优先级：会员体系 > AI推荐 = 社交分享
```

### 4.3 权重确定方法

```
1. 专家打分法
   多位专家独立评分，取平均值

2. 层次分析法(AHP)
   两两比较确定相对重要性

   比较矩阵：
        A    B    C
   A [  1   3    5  ]  A比B重要3倍
   B [ 1/3  1    3  ]  A比C重要5倍
   C [ 1/5 1/3  1  ]  B比C重要3倍

   计算权重：A=0.63, B=0.26, C=0.11

3. 德尔菲法
   多轮专家意见收敛

4. 数据驱动
   基于历史数据或统计分析
```

## 五、其他决策工具

### 5.1 决策平衡单

```
决策平衡单

考虑因素      方案A    方案B
───────────────────────────
收益因素：
- 经济收益    +5      +3
- 个人成长    +4      +5
- 工作满足    +3      +4
───────────────────────────
成本因素：
- 时间成本    -3      -2
- 经济成本    -4      -2
- 风险成本    -2      -3
───────────────────────────
机会因素：
- 未来机会    +4      +5
- 人脉扩展    +3      +4
───────────────────────────
总分          +10     +14 ✓

适用：个人职业选择、重大决策
```

### 5.2 六顶思考帽

```
六顶思考帽 - 平行思维工具

⚪ 白帽：数据和事实
   - 我们有哪些信息？
   - 还需要什么信息？

🔴 红帽：情感和直觉
   - 我对这个方案感觉如何？
   - 我的直觉是什么？

⚫ 黑帽：谨慎和风险
   - 可能出什么问题？
   - 有什么风险？

🟡 黄帽：乐观和价值
   - 有什么好处？
   - 为什么可行？

🟢 绿帽：创造和可能性
   - 还有其他选择吗？
   - 能否创新？

🔵 蓝帽：控制和过程
   - 我们在哪个阶段？
   - 下一步做什么？

应用：团队讨论、复杂决策
```

### 5.3 力场分析

```
力场分析 - 分析变革的推动力和阻碍力

        推动力                阻碍力
        (→)                  (←)
    ┌─────────┐           ┌─────────┐
    │市场机会 │           │资金不足 │
    │技术成熟 │           │团队能力 │
    │用户需求 │           │竞争对手 │
    │政策支持 │           │风险担忧 │
    └─────────┘           └─────────┘
    
    推动力总和 = 4
    阻碍力总和 = 3
    
    净力 = 4 - 3 = 1 (向右，可以推进)
    
决策：
- 增强推动力
- 减弱阻碍力
- 或两者结合
```

### 5.4 成本效益分析

```
成本效益分析

项目：____________________

成本项：
┌──────────────┬───────┐
│ 开发成本     │ 100万 │
│ 运营成本     │ 50万  │
│ 机会成本     │ 30万  │
├──────────────┼───────┤
│ 总成本       │ 180万 │
└──────────────┴───────┘

收益项：
┌──────────────┬───────┐
│ 直接收益     │ 200万 │
│ 间接收益     │ 100万 │
│ 品牌价值     │ 50万  │
├──────────────┼───────┤
│ 总收益       │ 350万 │
└──────────────┴───────┘

净收益 = 350 - 180 = 170万
收益率 = 170/180 = 94%

决策：收益率>0，可行
```

## 六、决策框架选择指南

```
决策场景 → 推荐框架

战略决策 → SWOT分析
         → 波特五力模型
         → PEST分析

选择决策 → 决策矩阵
         → 加权评分法
         → 成本效益分析

复杂决策 → 六顶思考帽
         → 力场分析

风险决策 → 风险矩阵
         → 敏感性分析

团队决策 → 德尔菲法
         → 头脑风暴
         → 投票决策

快速决策 → 2x2矩阵
         → 决策树
         → 规则决策
```

## 七、决策框架案例

### 7.1 案例：产品路线规划决策

```
决策问题：下季度优先开发哪个功能？

SWOT分析：
S: 技术能力强
W: 资源有限
O: 市场需求大
T: 竞品迭代快

决策矩阵：

| 功能 | 用户价值(40%) | 开发成本(30%) | 战略契合(30%) | 总分 |
|-----|--------------|--------------|--------------|------|
| A功能 | 4 | 3 | 5 | 4.0 |
| B功能 | 5 | 2 | 3 | 3.4 |
| C功能 | 3 | 5 | 4 | 3.9 |

决策：优先开发A功能
理由：综合得分最高，战略契合度高
```

### 7.2 案例：技术选型决策

```
决策问题：选择云服务供应商？

评价标准与权重：
1. 成本（30%）
2. 性能（25%）
3. 稳定性（25%）
4. 服务支持（20%）

加权评分：

| 供应商 | 成本 | 性能 | 稳定性 | 服务 | 总分 |
|-------|------|------|-------|------|------|
| AWS | 3 | 5 | 5 | 4 | 4.2 |
| 阿里云 | 4 | 4 | 4 | 5 | 4.2 |
| 腾讯云 | 5 | 3 | 4 | 4 | 4.0 |

六顶思考帽讨论：
白帽：三家的价格、性能数据
红帽：团队更熟悉阿里云
黑帽：AWS可能受政策影响
黄帽：阿里云服务响应快
绿帽：可否多云部署？
蓝帽：综合评估，选择阿里云

最终决策：选择阿里云
```

## 八、决策框架最佳实践

DO:
✓ 根据决策类型选择合适框架
✓ 多框架交叉验证
✓ 团队参与决策
✓ 记录决策依据
✓ 定期回顾决策效果

DON'T:
✗ 盲目套用框架
✗ 忽视直觉和经验
✗ 分析瘫痪
✗ 忽视决策执行
✗ 不复盘不改进

记住：框架是工具，不是目的。好的决策=合适的框架+充分的思考+果断的行动
