# OKR管理

## 一、定义

**OKR(Objectives and Key Results)**即目标与关键结果,是一套目标管理工具和方法。

```
OKR结构

Objective(目标)
  - 定性的
  - 有挑战性
  - 激励人心
      ↓
Key Results(关键结果)
  - 定量的
  - 可衡量
  - 有时间限制

示例:
O: 提升产品用户体验
KR1: 用户留存率从30%提升到45%
KR2: NPS从20提升到40
KR3: 用户投诉率降低50%
```

## 二、OKR制定

### 2.1 制定原则

1. 对齐(Alignment)
   - 公司OKR
   - 部门OKR
   - 个人OKR

2. 聚焦(Focus)
   - 3-5个目标
   - 每个3-5个KR
   - 不超过15个KR

3. 透明(Transparency)
   - 全员可见
   - 定期review
   - 结果公开

4. 挑战(Stretch)
   - 60-70%完成度为佳
   - 鼓励挑战性目标

### 2.2 制定步骤

1. 设定Objective
   - 我想去哪里?
   - 什么最重要?
   - 有挑战性吗?

2. 定义Key Results
   - 如何衡量?
   - 具体数字是什么?
   - 时间期限是?

3. 对齐与沟通
   - 与上级对齐
   - 与团队对齐
   - 与协作者对齐

4. 定期回顾
   - 周检查
   - 月评估
   - 季度复盘

## 三、OKR vs KPI

| 维度 | OKR | KPI |
|-----|-----|-----|
| 导向 | 目标导向 | 指标导向 |
| 特征 | 挑战性 | 可达成 |
| 用途 | 管理工具 | 考核工具 |
| 灵活性 | 季度调整 | 年度固定 |
| 透明度 | 全员公开 | 层级控制 |

## 四、OKR最佳实践

DO:
✓ 目标有挑战性
✓ KR可量化
✓ 定期检查
✓ 团队参与

DON'T:
✗ 与奖金直接挂钩
✗ 目标太多
✗ 设置太容易
✗ 只看数字不看本质

记住: OKR是管理工具,不是考核工具
