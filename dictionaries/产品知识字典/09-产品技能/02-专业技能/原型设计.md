# 原型设计

## 概述

原型设计是将产品概念可视化的重要手段，通过快速构建产品模型，验证想法、沟通需求、指导开发。

## 原型类型

### 按保真度分类

#### 1. 低保真原型
**特点**
- 快速制作
- 成本低
- 易于修改
- 概念验证

**适用场景**
- 早期创意探索
- 快速讨论想法
- 用户测试概念
- 内部沟通

**制作方式**
- 手绘草图
- 纸面原型
- 白板绘制
- 简单线框图

#### 2. 中保真原型
**特点**
- 有基本布局
- 包含交互逻辑
- 部分视觉元素
- 功能流程完整

**适用场景**
- 需求评审
- 设计讨论
- 可用性测试
- 开发参考

**制作工具**
- Axure
- Sketch
- Figma
- 墨刀

#### 3. 高保真原型
**特点**
- 接近最终效果
- 完整交互
- 动效演示
- 真实数据

**适用场景**
- 投资人演示
- 用户测试
- 开发参照
- 上线前确认

**制作要求**
- 完整UI设计
- 交互细节
- 动效实现
- 数据模拟

### 按用途分类

#### 功能原型
展示产品功能和流程

#### 概念原型
验证产品想法和概念

#### 演示原型
向他人展示产品

#### 测试原型
用于用户测试验证

## 原型设计流程

### 1. 需求理解
```
阅读PRD → 梳理功能 → 理解用户场景 → 明确设计目标
```

### 2. 信息架构
```
内容梳理 → 分类组织 → 层级规划 → 导航设计
```

### 3. 流程设计
```
用户流程 → 操作路径 → 异常处理 → 状态转换
```

### 4. 界面设计
```
布局设计 → 元素摆放 → 交互添加 → 细节优化
```

### 5. 评审迭代
```
原型评审 → 收集反馈 → 修改优化 → 确认定稿
```

## 设计规范

### 移动端设计规范

#### 尺寸规范
```
屏幕尺寸：
- iOS：375x812 (iPhone X)
- Android：360x640 (通用)

设计倍率：
- @1x：基础尺寸
- @2x：2倍图
- @3x：3倍图

常用单位：
- pt (iOS)
- dp (Android)
- px (Web)
```

#### 字体规范
```
iOS：
- 标题：17-34pt
- 正文：15-17pt
- 辅助：12-14pt
- 字体：SF Pro

Android：
- 标题：16-32sp
- 正文：14-16sp
- 辅助：12sp
- 字体：Roboto
```

#### 颜色规范
```
主色：#007AFF (iOS蓝) / #6200EE (Android紫)
成功：#34C759
警告：#FF9500
错误：#FF3B30
文字：#000000 (87%透明度)
辅助文字：#000000 (54%透明度)
```

#### 间距规范
```
基础间距：4px
常用间距：8px、12px、16px、24px、32px
边距：16px (iOS) / 16dp (Android)
```

### Web端设计规范

#### 栅格系统
```
常见栅格：
- 12列栅格
- 24列栅格

列宽：60-80px
间距：20-30px
边距：适应屏幕
```

#### 布局规范
```
内容区宽度：1200-1440px
左侧导航：200-240px
顶部导航：48-64px
卡片间距：16-24px
```

#### 对齐原则
```
✓ 元素对齐
✓ 视觉平衡
✓ 网格系统
✓ 一致性

✗ 随意摆放
✗ 不对齐
✗ 缺乏规律

```

## 原型评审

### 评审准备
```
1. 完成原型制作
2. 准备演示流程
3. 预判可能问题
4. 准备备选方案
```

### 评审要点
```
1. 功能完整性
   - 所有功能是否覆盖
   - 流程是否闭环
   - 异常是否处理

2. 交互合理性
   - 操作是否流畅
   - 逻辑是否清晰
   - 反馈是否及时

3. 体验友好性
   - 是否易于理解
   - 是否符合习惯
   - 是否有歧义

4. 技术可行性
   - 是否能实现
   - 难度如何
   - 成本多少
```

### 评审后跟进
```
1. 整理评审意见
2. 明确修改内容
3. 更新原型版本
4. 确认修改结果
5. 通知相关人员
```
