# 用户研究

## 概述

用户研究是通过系统化方法深入了解用户需求、行为和态度，为产品决策提供依据的专业技能。

## 研究价值

### 产品决策
- 验证产品假设
- 发现用户需求
- 优化产品设计
- 降低开发风险

### 体验优化
- 发现体验问题
- 理解用户行为
- 提升满意度
- 增加用户粘性

### 战略规划
- 市场机会识别
- 用户细分
- 竞争优势分析
- 创新方向探索

## 研究方法分类

### 按数据类型

#### 定性研究
**特点**
- 深度理解
- 探索性强
- 样本量小
- 主观性强

**方法**
- 用户访谈
- 焦点小组
- 可用性测试
- 田野调查

#### 定量研究
**特点**
- 数据量化
- 验证性强
- 样本量大
- 客观性强

**方法**
- 问卷调查
- A/B测试
- 数据分析
- 眼动测试

### 按研究目的

#### 探索性研究
**目的**：发现问题、理解用户
**方法**：访谈、观察、日记研究

#### 描述性研究
**目的**：描述现象、了解现状
**方法**：问卷、数据分析

#### 因果性研究
**目的**：验证因果关系
**方法**：A/B测试、实验

## 核心研究方法

### 1. 用户访谈

#### 访谈类型

**深度访谈**
```
时长：30-60分钟
人数：1对1
形式：面对面/视频
目的：深入了解用户想法
```

**半结构化访谈**
```
提纲：有核心问题，可灵活调整
优点：有针对性，又有弹性
适用：大多数场景
```

#### 访谈流程

**准备阶段**
```markdown
1. 明确访谈目标
2. 制定访谈提纲
3. 招募访谈对象
4. 准备访谈环境
5. 预访谈测试
```

**执行阶段**
```markdown
1. 开场寒暄（2-3分钟）
2. 说明目的和规则
3. 按提纲展开访谈
4. 灵活追问
5. 总结确认
6. 致谢结束
```

**分析阶段**
```markdown
1. 整理访谈记录
2. 提取关键信息
3. 归纳共性问题
4. 形成洞察结论
5. 输出访谈报告
```

#### 访谈技巧

**提问技巧**
```
✓ 开放式问题
✓ 中立态度
✓ 追问细节
✓ 鼓励表达

✗ 引导性问题
✗ 诱导性表述
✗ 评判用户
✗ 替用户回答
```

**好的问题示例**
```
✓ "能跟我讲讲您上次使用的经历吗？"
✓ "当时是什么让您决定这样做？"
✓ "您觉得哪里不太顺手？"

✗ "您喜欢这个功能吗？"
✗ "这个设计是不是很棒？"
✗ "您应该会想用这个吧？"
```

**倾听技巧**
```
1. 专注倾听
2. 适时反馈
3. 观察表情
4. 记录要点
5. 不打断用户
```

#### 访谈提纲模板

```markdown
# 用户访谈提纲

## 基本信息
- 姓名/代号
- 年龄/性别
- 职业
- 使用产品时间

## 开场（2分钟）
"感谢您参加访谈，今天主要想了解..."
"访谈大约需要30分钟，您的所有回答..."
"访谈会被记录用于..."

## 话题1：使用背景（10分钟）
1. 请介绍一下您使用[产品]的背景
2. 您通常在什么情况下使用？
3. 您使用的主要目的是什么？

## 话题2：使用行为（15分钟）
1. 请详细描述您上一次使用的过程
2. 使用过程中遇到了什么问题？
3. 您是如何解决的？
4. 有哪些让您印象深刻的地方？

## 话题3：需求和期望（10分钟）
1. 您觉得产品还有哪些可以改进的地方？
2. 您最希望增加什么功能？
3. 理想的产品应该是什么样的？

## 结束（3分钟）
1. 还有什么想补充的吗？
2. 感谢您的参与

## 备注
- 追问要点
- 注意事项
```

### 2. 问卷调查

#### 问卷设计原则

**清晰简洁**
```
✓ 问题表述清晰
✓ 避免专业术语
✓ 简短精炼
✓ 重点突出

✗ 模糊表述
✗ 专业名词
✗ 冗长复杂
✗ 主题分散
```

**逻辑合理**
```
1. 从简单到复杂
2. 从一般到具体
3. 相关问题集中
4. 敏感问题后置
```

#### 问卷结构

```markdown
# 问卷标题

## 1. 开场白（1段）
尊敬的用户：
感谢您参与本次调研。本问卷旨在...

## 2. 筛选问题（3-5题）
Q1. 您是否使用过我们的产品？
○ 是
○ 否（结束问卷）

## 3. 核心问题（15-20题）
[按主题分组]

## 4. 人口统计学问题（3-5题）
年龄、性别、职业等

## 5. 结束语
感谢您的参与！
```

#### 问题类型

**单选题**
```
Q1. 您使用产品的频率是？
○ 每天
○ 每周几次
○ 每月几次
○ 偶尔使用
```

**多选题**
```
Q2. 您使用产品的场景有哪些？（多选）
□ 工作中
□ 学习中
□ 休闲娱乐
□ 其他___
```

**量表题**
```
Q3. 您对产品的整体满意度如何？
1 2 3 4 5
非常不满意 ← → 非常满意
```

**开放题**
```
Q4. 您对产品有什么建议？
[文本框]
```

#### 问卷设计技巧

**避免的问题**
```
1. 双重否定
   ✗ "您不认为不应该..."
   ✓ "您认为应该..."

2. 引导性问题
   ✗ "我们的产品很好，您觉得呢？"
   ✓ "请评价我们的产品"

3. 术语问题
   ✗ "您对UI/UX满意吗？"
   ✓ "您对界面设计满意吗？"

4. 双重问题
   ✗ "您对产品的功能和价格满意吗？"
   ✓ 拆分成两个问题
```

**题量控制**
```
- 总题数：15-25题
- 完成时间：5-10分钟
- 避免中途放弃
```

#### 问卷投放

**投放渠道**
- 产品内推送
- 邮件邀请
- 社交媒体
- 第三方平台

**样本要求**
```
- 代表性：覆盖目标用户
- 足够量：至少300份有效问卷
- 质量控制：剔除无效问卷
```

#### 数据分析

**描述统计**
```
- 频次分析
- 百分比
- 平均值
- 标准差
```

**交叉分析**
```
- 不同用户群体差异
- 相关性分析
- 回归分析
```

### 3. 可用性测试

#### 测试类型

**形成性测试**
- 时机：设计阶段
- 目的：发现问题
- 样本：5-8人
- 方法：边做边说

**总结性测试**
- 时机：上线前后
- 目的：评估效果
- 样本：10-20人
- 方法：任务测试+问卷

#### 测试流程

**准备阶段**
```markdown
1. 确定测试目标
2. 设计测试任务
3. 准备测试原型
4. 招募测试用户
5. 准备测试环境
```

**执行阶段**
```
1. 欢迎和介绍（5分钟）
2. 前置访谈（5分钟）
3. 任务测试（20-30分钟）
4. 后置访谈（5分钟）
5. 问卷填写（5分钟）
6. 感谢和奖励
```

**分析阶段**
```markdown
1. 整理测试记录
2. 统计任务完成率
3. 分析问题严重程度
4. 提出改进建议
5. 输出测试报告
```

#### 测试任务设计

**任务示例**
```markdown
任务1：购买一件商品
场景：您想买一件白色T恤，请找到并加入购物车
难度：简单
预期时间：2分钟

任务2：查询订单状态
场景：您想查看上周订单的配送状态
难度：中等
预期时间：3分钟

任务3：申请退款
场景：您对购买的商品不满意，请申请退款
难度：困难
预期时间：5分钟
```

**任务设计原则**
```
✓ 真实场景
✓ 具体任务
✓ 有明确目标
✓ 不给操作提示

✗ 假设场景
✗ 模糊任务
✗ 引导性描述
```

#### 测试指导语

```markdown
# 可用性测试指导语

"感谢您参加今天的测试。"

"我想让您知道：
- 我们测试的是产品，不是您
- 过程中可能会遇到问题，这是正常的
- 请按照您的真实想法操作"

"测试过程中：
- 请边做边说出您的想法
- 不用担心犯错
- 可以随时提问"

"让我们开始吧..."
```

#### 测试指标

**定量指标**
```
任务完成率 = 完成任务数 / 总任务数
任务完成时间 = 开始时间 - 结束时间
错误率 = 错误次数 / 操作次数
求助率 = 求助次数 / 任务数
```

**定性指标**
```
用户满意度（SUS量表）
主观难度评分
情感曲线
口头反馈
```

#### 行为数据分析

**用户分群**
```
维度：
- 新老用户
- 活跃度
- 付费情况
- 使用功能
- 渠道来源
```

**行为路径**
```
- 用户流分析
- 转化漏斗
- 流失分析
- 留存分析
```

**事件分析**
```
- 事件触发量
- 事件用户数
- 事件属性分布
- 事件趋势变化
```

#### 用户反馈分析

**反馈来源**
```
- 应用商店评论
- 客服工单
- 社交媒体
- 用户社区
- 问卷反馈
```

**分析方法**
```
1. 分类归档
2. 频次统计
3. 情感分析
4. 优先级排序
5. 改进建议
```

- 共识用户认知
- 指导产品决策
- 统一团队语言
- 聚焦目标用户

### 画像要素

```markdown
# 用户画像模板

## 基本信息
- 姓名/代号
- 年龄/性别
- 职业/行业
- 收入水平
- 地理位置
- 家庭状况

## 标签
- 用户类型
- 使用场景
- 核心需求

## 个人简介
[1-2段描述性文字]

## 目标与动机
- 主要目标
- 次要目标
- 使用动机

## 痛点与需求
- 核心痛点
- 次要痛点
- 未被满足的需求

## 行为特征
- 使用频率
- 使用时段
- 使用设备
- 使用场景

## 态度与偏好
- 技术熟练度
- 品牌偏好
- 决策因素

## 引用语
"原话1"
"原话2"
```

### 画像案例

```markdown
# 用户画像：职场白领-小美

## 基本信息
- 姓名：小美
- 年龄：28岁
- 性别：女
- 职业：互联网运营
- 收入：15K/月
- 地点：北京

## 个人简介
小美在互联网公司工作3年，平时工作繁忙，经常加班。她注重效率，喜欢尝试新工具提升工作效率。业余时间喜欢健身和看剧。

## 核心需求
1. 提高工作效率
2. 合理管理时间
3. 保持工作生活平衡

## 痛点
1. 任务多，容易遗漏
2. 协作沟通成本高
3. 时间碎片化

## 行为特征
- 使用频率：每天多次
- 使用时段：工作时间+通勤
- 使用设备：手机+电脑
- 使用场景：办公室+家

## 引用语
"我需要一款能帮我理清工作思路的工具"
"希望能快速记录想法，不然一会儿就忘了"
"最好能和同事协作，这样效率更高"
```
