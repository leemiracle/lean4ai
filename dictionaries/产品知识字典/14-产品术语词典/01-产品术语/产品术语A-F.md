# 产品术语 A-F

## A

| 术语 | 英文 | 定义 | 示例 |
|------|------|------|------|
| A/B测试 | A/B Testing | 通过对比两个版本来测试哪个效果更好的实验方法 | 将50%用户看到新版按钮，50%看到旧版，比较点击率 |
| API | Application Programming Interface | 应用程序接口，允许不同软件系统之间进行通信 | 开放API让第三方开发者接入支付功能 |
| ARR | Annual Recurring Revenue | 年度经常性收入，SaaS产品的核心指标 | 公司ARR达到1000万美元 |
| ACE | Average Check/Call/Conversation | 平均每次交互的价值 | 客服ACE为每次15元 |
| ARPU | Average Revenue Per User | 每用户平均收入，衡量用户价值的关键指标 | 游戏ARPU为每月50元 |
| Acquisition | User Acquisition | 用户获取，吸引新用户使用产品的过程 | 通过广告投放实现用户获取 |
| Active User | Active User | 活跃用户，在一定时间内使用产品的用户 | DAU（日活）达到100万 |
| Adoption | Product Adoption | 产品采纳，用户开始使用并接受产品的过程 | 新功能上线后采纳率达到30% |
| Attribution | Attribution | 归因，确定用户行为来源的分析方法 | 通过归因分析发现70%用户来自搜索引擎 |
| Accessibility | Accessibility | 无障碍设计，确保残障人士也能使用产品 | 为图片添加alt文本提升无障碍性 |

## B

| 术语 | 英文 | 定义 | 示例 |
|------|------|------|------|
| BRD | Business Requirement Document | 商业需求文档，描述产品商业价值的文档 | BRD中明确了市场规模和盈利模式 |
| B2B | Business to Business | 企业对企业的商业模式 | 阿里巴巴是典型的B2B平台 |
| B2C | Business to Consumer | 企业对消费者的商业模式 | 京东是典型的B2C平台 |
| Bounce Rate | Bounce Rate | 跳出率，用户只访问一个页面就离开的比例 | 首页跳出率高达60%需要优化 |
| Backlog | Product Backlog | 产品待办列表，待开发功能的优先级队列 | 将用户反馈加入产品backlog |
| Beta Testing | Beta Testing | Beta测试，产品正式发布前的公开测试阶段 | 开放1000个Beta测试名额 |
| Burn Rate | Burn Rate | 资金消耗率，初创公司每月的资金支出 | 公司burn rate为每月50万 |
| Benchmark | Benchmark | 基准，用于比较的参考标准 | 将转化率与行业benchmark对比 |
| Business Case | Business Case | 商业案例，论证项目可行性的分析报告 | 通过business case获得管理层批准 |
| Buyer Persona | Buyer Persona | 买方画像，理想客户的详细描述 | 创建3个核心buyer persona指导营销 |

## C

| 术语 | 英文 | 定义 | 示例 |
|------|------|------|------|
| CAC | Customer Acquisition Cost | 客户获取成本，获得一个客户所需的费用 | CAC为200元，低于LTV才能盈利 |
| CLV/LTV | Customer Lifetime Value | 客户终身价值，一个客户带来的总收益 | CLV为1000元，CAC/CLV比为1:5 |
| CTR | Click-Through Rate | 点击率，点击次数与展示次数的比值 | 广告CTR为2.5% |
| CTA | Call to Action | 行动号召，引导用户采取特定行动的元素 | "立即注册"按钮是典型的CTA |
| Churn Rate | Churn Rate | 流失率，用户停止使用产品的比例 | 月流失率控制在5%以内 |
| Cohort Analysis | Cohort Analysis | 同类群组分析，跟踪特定用户群行为变化的方法 | 通过cohort分析发现新用户留存率下降 |
| Conversion | Conversion | 转化，用户完成目标行为的过程 | 注册转化率为15% |
| Cross-selling | Cross-selling | 交叉销售，向客户推荐相关产品 | 购买手机后推荐手机壳 |
| Customer Journey | Customer Journey | 客户旅程，用户与产品互动的完整过程 | 绘制customer journey map识别痛点 |
| Core User | Core User | 核心用户，产品最忠实的高价值用户群 | 核心用户占比20%贡献80%收入 |

## D

| 术语 | 英文 | 定义 | 示例 |
|------|------|------|------|
| DAU | Daily Active Users | 日活跃用户数，每天使用产品的用户数量 | DAU突破500万 |
| Data-driven | Data-driven | 数据驱动，基于数据做决策的方法论 | 采用data-driven方法优化产品功能 |
| Demo | Demo/Demonstration | 演示，展示产品功能的原型或样品 | 给客户做产品demo |
| Design Thinking | Design Thinking | 设计思维，以用户为中心的创新方法论 | 运用design thinking重构用户体验 |
| Discovery | Discovery | 发现阶段，产品开发前期探索需求的过程 | 在discovery阶段进行用户访谈 |
| Drop-off Rate | Drop-off Rate | 流失率，用户在流程中放弃的比例 | 注册流程第二步drop-off rate达40% |
| Documentation | Documentation | 文档，记录产品信息的各种文件 | 完善API documentation |
| DV | Daily Visitor | 日访问量，每天访问网站的用户数 | 网站DV达到100万 |
| Deliverable | Deliverable | 可交付成果，项目阶段性的产出物 | 本周deliverable是PRD文档 |
| Dependency | Dependency | 依赖关系，任务之间的前后关系 | 识别项目中的dependency |

## E

| 术语 | 英文 | 定义 | 示例 |
|------|------|------|------|
| Engagement | Engagement | 参与度，用户与产品互动的深度和频率 | 提升用户engagement是本季度目标 |
| EOL | End of Life | 产品生命周期结束，停止维护的时间点 | 产品将于2024年EOL |
| Ecosystem | Ecosystem | 生态系统，产品相关的完整环境 | 构建开放的开发者ecosystem |
| Experiment | Experiment | 实验，验证假设的测试活动 | 进行产品定价experiment |
| Exit Rate | Exit Rate | 退出率，用户从特定页面离开的比例 | 购物车页面exit rate过高 |
| Early Adopter | Early Adopter | 早期采用者，最先接受新产品的用户群体 | 邀请100名early adopter试用 |
| Ecosystem Map | Ecosystem Map | 生态图谱，展示产品环境的可视化图表 | 绘制竞争ecosystem map |
| Experiment Design | Experiment Design | 实验设计，规划测试方案的流程 | 完善experiment design确保结果可靠 |
| Experimentation | Experimentation | 实验化，持续测试验证的文化 | 建立experimentation文化 |
| MVP Error Rate | Error Rate | 错误率，系统出错的频率 | API error rate需控制在0.1%以下 |

## F

| 术语 | 英文 | 定义 | 示例 |
|------|------|------|------|
| Feature | Feature | 功能特性，产品的具体能力 | 新feature上线后用户好评 |
| Funnel | Funnel | 漏斗，用户完成目标的步骤序列 | 分析注册funnel找出流失点 |
| Feedback | Feedback | 反馈，用户对产品的意见和建议 | 收集用户feedback优化产品 |
| FAB | Features-Advantages-Benefits | 特性-优势-利益，产品营销框架 | 用FAB框架撰写产品介绍 |
| First-time User | First-time User | 首次用户，第一次使用产品的用户 | 优化first-time user onboarding |
| Flow | User Flow | 用户流程，用户完成任务的路径 | 简化购买flow提升转化 |
| Focus Group | Focus Group | 焦点小组，收集用户意见的小组访谈 | 组织focus group讨论新功能 |
| Framework | Framework | 框架，系统化的方法论或结构 | 使用RICE framework排序优先级 |
| Freemium | Freemium | 免费增值，基础免费高级收费的模式 | 采用freemium模式获取用户 |
| Feature Flag | Feature Flag | 功能开关，控制功能上线状态的技术 | 使用feature flag逐步发布新功能 |
