## 📋 使用说明

本Checklist用于上线前后的检查，确保上线顺利。

### 上线流程

```
上线准备 → 预发布验证 → 正式上线 → 上线验证 → 监控观察
```

### 职责分工

| 角色 | 职责 |
|------|------|
| 产品经理 | 验收功能、用户通知 |
| 开发负责人 | 代码部署、技术保障 |
| 测试负责人 | 质量把关、回归测试 |
| 运维负责人 | 环境部署、监控配置 |
| 运营负责人 | 用户沟通、数据监控 |

## ⚠️ 注意事项

1. **充分测试**：上线前必须完成充分测试
2. **备份第一**：任何变更前先备份
3. **小步快跑**：优先灰度发布，降低风险
4. **保持在线**：上线后相关人员保持在线待命
5. **及时沟通**：任何异常及时沟通处理
