## 📋 使用说明

本清单用于需求评审前的准备和评审过程的检查，确保需求质量。

### 评审流程

```
评审前准备 → 评审会议 → 评审记录 → 评审结论 → 需求确认
```

### 评审角色

| 角色 | 职责 |
|------|------|
| 产品经理 | 需求讲解，回答问题 |
| 开发代表 | 技术可行性评估 |
| 测试代表 | 测试要点确认 |
| 设计代表 | 设计方案确认 |
| 业务方 | 业务价值确认 |
| 评审主持人 | 组织评审，记录结论 |

## ⚠️ 注意事项

1. **提前准备**：文档需提前发送，给评审人足够时间阅读
2. **控制时间**：评审会议不宜过长，建议1-2小时
3. **聚焦重点**：重点讨论有疑问的地方，不要重复说明
4. **记录完整**：所有讨论结论要记录清楚
5. **跟进落实**：评审后的行动项要跟踪完成
