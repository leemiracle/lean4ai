## 📋 使用说明

本清单用于设计评审前的准备和评审过程的检查，确保设计质量。

### 评审流程

```
设计自检 → 评审前准备 → 评审会议 → 评审记录 → 设计修改 → 设计确认
```

### 评审角色

| 角色 | 职责 |
|------|------|
| 设计师 | 设计讲解，回答问题 |
| 产品经理 | 业务逻辑确认 |
| 开发代表 | 实现可行性评估 |
| 测试代表 | 交互逻辑确认 |
| 设计主管 | 设计质量把控 |
| 评审主持人 | 组织评审，记录结论 |

## ⚠️ 注意事项

1. **设计先行**：设计评审应在开发前完成
2. **多角色参与**：产品、开发、测试都应参与
3. **关注细节**：交互细节和异常场景要重点关注
4. **记录完整**：所有修改意见要记录并跟踪
5. **迭代优化**：设计可以持续优化，不必追求完美
