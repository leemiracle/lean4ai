# 数据安全法

## 核心摘要
《数据安全法》是中国数据安全领域的基础性法律，确立了数据分类分级、数据安全保护、数据安全审查等制度，产品需建立完善的数据安全管理体系。

## 法律概述

### 基本信息
- **全称**：《中华人民共和国数据安全法》
- **通过时间**：2021年6月10日
- **实施时间**：2021年9月1日
-**立法机关**：全国人民代表大会常务委员会
- **法律层级**：法律

### 立法目的
1. 规范数据处理活动
2. 保障数据安全
3. 促进数据开发利用
4. 保护公民、组织合法权益
5. 维护国家主权、安全、发展利益

### 适用范围
- **地域范围**：中华人民共和国境内
- **主体范围**：所有开展数据处理活动的组织、个人
- **行为范围**：数据的收集、存储、使用、加工、传输、提供、公开等

## 核心制度

### 数据分类分级制度

#### 分类维度
| 分类维度 | 具体类别 | 示例 |
|---------|---------|------|
| 按重要性 | 一般数据、重要数据、核心数据 | 见下表 |
| 按行业 | 金融、医疗、交通、电信等 | 各行业数据 |
| 按主体 | 个人数据、组织数据、国家数据 | 不同主体 |
| 按敏感度 | 公开、内部、敏感、机密 | 不同级别 |

#### 分级标准
| 级别 | 定义 | 保护要求 | 典型数据 |
|-----|------|---------|---------|
| 一般数据 | 一旦泄露影响较小 | 基本保护 | 普通业务数据 |
| 重要数据 | 一旦泄露可能危害国家安全、公共利益 | 严格保护 | 人口、经济、地理信息 |
| 核心数据 | 关系国家安全、国民经济命脉、重要民生、重大公共利益 | 最严格保护 | 国家秘密、关键基础设施 |

#### 分类分级流程
```
1. 数据资产盘点 → 识别所有数据资产
2. 数据分类 → 按维度分类
3. 数据分级 → 评估数据级别
4. 目录制定 → 形成数据目录
5. 动态调整 → 定期评估调整
```

### 数据安全保护制度

#### 总体要求
1. **建立健全数据安全管理制度**
2. **组织开展数据安全教育培训**
3. **采取相应技术措施保障数据安全**
4. **重要数据处理者需定期开展风险评估**

#### 技术措施
| 措施类型 | 具体内容 | 适用场景 |
|---------|---------|---------|
| 访问控制 | 身份认证、权限管理 | 所有数据 |
| 加密存储 | 敏感数据加密 | 敏感数据 |
| 传输加密 | HTTPS、VPN | 数据传输 |
| 脱敏处理 | 数据脱敏、匿名化 | 测试、分析 |
| 备份恢复 | 数据备份、灾备 | 重要数据 |
| 审计日志 | 操作日志、访问日志 | 所有数据 |

#### 管理措施
1. **组织架构**：设立数据安全负责人、数据安全管理机构
2. **制度规范**：制定数据安全管理制度
3. **人员管理**：背景审查、保密协议、安全培训
4. **应急响应**：建立数据安全事件应急预案
5. **合规审计**：定期开展数据安全审计

### 数据安全审查制度

#### 审查对象
1. 影响或可能影响国家安全的数据处理活动
2. 关键信息基础设施运营者采购网络产品和服务
3. 大量收集、使用个人信息的平台
4. 跨境数据传输

#### 审查内容
- 数据安全风险评估
- 数据处理合法性审查
- 数据安全保护措施审查
- 对国家安全的影响评估

#### 审查程序
```
发起审查 → 材料提交 → 审查评估 → 审查决定 → 复议（如需）
```

## 重点条款解读

### 数据处理者义务（第27条）
**条款内容**：开展数据处理活动应当依照法律、法规的规定，建立健全全流程数据安全管理制度，组织开展数据安全教育培训，采取相应的技术措施和其他必要措施，保障数据安全。

**合规要点**
1. 建立全流程数据安全管理制度
2. 开展数据安全教育培训
3. 采取技术措施保障安全
4. 定期开展风险评估

### 重要数据处理者义务（第30条）
**条款内容**：重要数据的处理者应当按照规定对其数据处理活动定期开展风险评估，并向有关主管部门报送风险评估报告。

**合规要点**
1. 识别重要数据
2. 定期开展风险评估（至少每年一次）
3. 编制风险评估报告
4. 向主管部门报送报告

### 数据交易管理（第33条）
**条款内容**：法律、行政法规规定提供数据处理相关服务应当取得行政许可的，服务提供者应当依法取得许可。

**合规要点**
1. 数据交易需取得相应资质
2. 数据来源合法合规
3. 数据质量符合要求
4. 数据交易双方签订协议

### 跨境数据传输（第31条）
**条款内容**：关键信息基础设施的运营者在中华人民共和国境内运营中收集和产生的关键数据应当在境内存储。因业务需要，确需向境外提供的，应当通过国家网信部门会同国务院有关部门组织的安全评估。

**合规要点**
1. 关键数据境内存储
2. 跨境传输需安全评估
3. 通过国家网信部门评估
4. 符合数据出境管理规定

### 数据安全管理制度体系
| 制度类型 | 主要内容 | 适用对象 |
|---------|---------|---------|
| 数据分类分级管理制度 | 分类标准、分级标准、目录管理 | 全员 |
| 数据采集管理制度 | 采集原则、范围、方式、授权 | 产品、技术 |
| 数据存储管理制度 | 存储位置、加密、备份、保留期 | 技术 |
| 数据使用管理制度 | 使用范围、权限控制、审计 | 全员 |
| 数据共享管理制度 | 共享原则、审批流程、协议 | 业务 |
| 数据销毁管理制度 | 销毁标准、方式、记录 | 技术 |
| 数据安全事件应急预案 | 响应流程、处置措施、报告 | 全员 |

### 风险评估要点
| 评估维度 | 评估内容 | 评估方法 |
|---------|---------|---------|
| 数据资产 | 数据资产识别、分类分级 | 资产盘点 |
| 安全风险 | 数据安全威胁、脆弱性 | 风险分析 |
| 管理措施 | 制度、流程、人员 | 制度审查 |
| 技术措施 | 技术措施有效性 | 技术测试 |
| 应急能力 | 应急预案、演练 | 演练评估 |

### 数据安全事件处置
```
事件发现 → 初步研判 → 应急响应 → 事件处置 → 影响评估 → 报告备案 → 改进优化
   │          │          │          │          │          │          │
 立即报告   启动预案   成立小组   技术处置   评估影响   监管报告   完善措施
```

## 常见违规案例

### 典型案例
| 案例 | 违规行为 | 处罚 | 教训 |
|-----|---------|------|------|
| 某出行公司 | 违规收集、存储、使用个人信息 | 下架、罚款 | 数据采集需合规 |
| 某互联网公司 | 未经安全评估跨境传输数据 | 罚款、整改 | 跨境传输需评估 |
| 某科技公司 | 数据安全管理制度不健全 | 警告、罚款 | 制度建设是基础 |
| 某APP | 过度收集个人信息 | 下架、罚款 | 最小必要原则 |

### 合规红线
1. **禁止非法收集、使用数据**
2. **禁止未经评估跨境传输重要数据**
3. **禁止泄露、篡改、损毁数据**
4. **禁止利用数据从事违法犯罪活动**

## 产品合规建议

### 产品设计阶段
1. **数据最小化**：只收集必要数据
2. **隐私设计**：内置隐私保护
3. **用户授权**：获取用户同意
4. **数据分类**：明确数据分类分级

### 产品开发阶段
1. **安全编码**：遵循安全编码规范
2. **加密存储**：敏感数据加密
3. **访问控制**：最小权限原则
4. **审计日志**：记录关键操作

### 产品运营阶段
1. **安全监控**：实时安全监控
2. **定期审计**：定期安全审计
3. **应急响应**：快速响应事件
4. **合规审查**：定期合规审查
