# 服务设计 (Service Design)

## 定义

服务设计是规划和组织人员、基础设施、沟通和物料组件的过程，目的是提高服务质量、服务提供者与客户之间的互动以及客户的体验。

## 服务设计核心

### 1. 服务设计定义

```
服务 = 触点 + 流程 + 人员 + 物料

- 触点：用户与服务的接触点
- 流程：服务提供的顺序和逻辑
- 人员：提供服务的角色
- 物料：支持服务的工具和环境
```

### 2. 服务设计 vs 用户体验设计

| 维度 | 服务设计 | 用户体验设计 |
|------|----------|--------------|
| 范围 | 全渠道体验 | 单一触点/产品 |
| 视角 | 系统性 | 用户侧 |
| 时间 | 服务全生命周期 | 产品使用阶段 |
| 利益相关者 | 用户+员工+组织 | 主要用户 |

## 服务设计原则

### 1. 设计原则

**以用户为中心**
- 理解用户需求
- 设计服务体验
- 持续优化改进

**整体性**
- 考虑全触点
- 端到端体验
- 系统性思考

**共创**
- 多方参与
- 跨部门协作
- 利益相关者共创

**迭代**
- 快速原型
- 持续测试
- 不断优化

### 2. Stanmark原则

| 原则 | 说明 |
|------|------|
| User-centered | 以用户为中心 |
| Co-creative | 协作共创 |
| Sequencing | 服务是序列的 |
| Evidencing | 服务需要可视化 |
| Holistic | 整体性思考 |

## 服务设计工具

### 1. 服务蓝图 (Service Blueprint)

**定义**
服务蓝图是描述服务过程的可视化图表，展示前台和后台活动、支持过程以及各环节之间的关系。

**基本结构**
```
┌─────────────────────────────────────────────────────────────┐
│                      服务蓝图                                │
├─────────────────────────────────────────────────────────────┤
│                    │  阶段1  │  阶段2  │  阶段3  │          │
├─────────────────────────────────────────────────────────────┤
│ 用户行为           │  ...    │  ...    │  ...    │          │
├────────────────────┼─────────┼─────────┼─────────┤          │
│                    │  ← 可视线 →       │          │          │
├────────────────────┼─────────┼─────────┼─────────┤          │
│ 前台员工行为       │  ...    │  ...    │  ...    │          │
├────────────────────┼─────────┼─────────┼─────────┤          │
│ 后台员工行为       │  ...    │  ...    │  ...    │          │
├────────────────────┼─────────┼─────────┼─────────┤          │
│ 支持过程           │  ...    │  ...    │  ...    │          │
└────────────────────┴─────────┴─────────┴─────────┘
```

**组成要素**
| 层级 | 说明 | 示例 |
|------|------|------|
| 用户行为 | 用户看得见的行动 | 下单、支付 |
| 前台行为 | 直接面对用户的服务 | 客服接待 |
| 后台行为 | 不直接面对用户的支持 | 库存检查 |
| 支持过程 | 支持服务的基础设施 | 系统维护 |

## 服务触点

### 1. 触点类型

| 触点类型 | 说明 | 示例 |
|----------|------|------|
| 人际触点 | 人与人互动 | 客服、销售 |
| 数字触点 | 数字渠道 | APP、网站 |
| 物理触点 | 实体环境 | 门店、包装 |
| 品牌触点 | 品牌接触 | 广告、口碑 |

### 2. 触点设计

**触点分析**
```
触点识别 → 触点评估 → 触点优化 → 效果追踪

1. 识别所有触点
2. 评估触点重要性
3. 设计触点体验
4. 持续优化改进
```

## 服务设计流程

### 1. 设计流程

```
1. 探索 (Explore)
   ├── 用户研究
   ├── 员工访谈
   ├── 现状分析
   └── 发现机会
        ↓
2. 创造 (Create)
   ├── 构思方案
   ├── 服务概念
   ├── 原型设计
   └── 内部验证
        ↓
3. 反思 (Reflect)
   ├── 用户测试
   ├── 员工反馈
   ├── 方案优化
   └── 确定方案
        ↓
4. 实施 (Implement)
   ├── 落地执行
   ├── 培训推广
   ├── 效果追踪
   └── 持续优化
```

### 2. 双钻模型应用

```
第一钻：发现服务机会
┌─────────────────────────────────────────┐
│ 发散 → 收敛 → 发散 → 收敛              │
│  ↓       ↓       ↓       ↓              │
│ 探索    定义    开发    交付            │
└─────────────────────────────────────────┘

第二钻：设计服务方案
┌─────────────────────────────────────────┐
│ 发散 → 收敛 → 发散 → 收敛              │
│  ↓       ↓       ↓       ↓              │
│ 构思    原型    测试    实施            │
└─────────────────────────────────────────┘
```

## 服务触点优化

### 1. 优化方法

**峰终定律**
```
人们对体验的评价主要取决于：
1. 体验的高峰时刻（最好或最差的感受）
2. 体验的结束时刻

优化策略：
- 创造峰值体验
- 优化结束体验
- 淡化低谷体验
```

**关键时刻 (MOT)**
```
MOT = Moment of Truth

关键触点识别：
1. 第一印象（首次接触）
2. 核心体验（核心服务）
3. 最终印象（离开时刻）

优化重点：
- 强化正向MOT
- 消除负向MOT
```

### 2. 触点设计原则

- **一致性**：各触点体验统一
- **连贯性**：触点之间自然衔接
- **个性化**：针对不同用户需求
- **情感化**：创造情感连接

## 服务设计最佳实践

### 1. 设计建议

- 全渠道一致性
- 员工体验同样重要
- 持续迭代优化
- 数据驱动决策

### 2. 常见错误

- ❌ 忽视后台服务
- ❌ 各触点体验不统一
- ❌ 忽视员工体验
- ❌ 缺乏系统性思考

### 3. 成功要素

- ✅ 高层支持
- ✅ 跨部门协作
- ✅ 用户研究支撑
- ✅ 持续迭代优化
