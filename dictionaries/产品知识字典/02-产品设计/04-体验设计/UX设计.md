# UX设计方法

## 定义

用户体验设计（User Experience Design，简称UX设计）是以用户为中心的设计过程，关注用户在使用产品过程中的整体感受，包括可用性、愉悦性、效率等维度。

## UX设计核心

### 1. UX定义

```
用户体验 = 用户 + 产品 + 情境

- 用户：需求、目标、行为、限制
- 产品：功能、形式、内容、行为
- 情境：环境、情感、时间、社会因素
```

### 2. UX vs UI

| 维度 | UX设计 | UI设计 |
|------|--------|--------|
| 关注点 | 整体体验 | 视觉表现 |
| 内容 | 流程、结构、功能 | 颜色、字体、布局 |
| 方法 | 研究分析、原型 | 视觉设计、细节 |
| 目标 | 解决用户问题 | 美观、吸引力 |

## UX设计流程

### 设计思维五阶段

```
1. 同理心 (Empathize)
   ├── 用户访谈
   ├── 观察
   └── 问卷调查
        ↓
2. 定义 (Define)
   ├── 问题陈述
   ├── 用户画像
   └── 用户旅程
        ↓
3. 构思 (Ideate)
   ├── 头脑风暴
   ├── 思维导图
   └── 概念设计
        ↓
4. 原型 (Prototype)
   ├── 低保真原型
   ├── 高保真原型
   └── 交互原型
        ↓
5. 测试 (Test)
   ├── 可用性测试
   ├── 用户反馈
   └── 迭代优化
```

### 双钻模型

```
第一钻：发现问题
┌─────────────────────────────────────────┐
│ 发散 → 收敛 → 发散 → 收敛              │
│  ↓       ↓       ↓       ↓              │
│ 发现    定义    开发    交付            │
│                                          │
│ 用户研究 问题定义 方案设计 验证迭代     │
└─────────────────────────────────────────┘
```

## 用户研究方法

### 1. 研究方法分类

| 类型 | 方法 | 用途 |
|------|------|------|
| 定性 | 访谈、观察 | 深度理解用户 |
| 定量 | 问卷、数据分析 | 验证假设、量化问题 |
| 探索性 | 用户访谈、田野调查 | 发现新洞察 |
| 验证性 | A/B测试、可用性测试 | 验证设计方案 |

### 2. 用户访谈

**访谈类型**
| 类型 | 特点 | 适用场景 |
|------|------|----------|
| 结构化 | 固定问题 | 收集标准化数据 |
| 半结构化 | 有框架，可发散 | 平衡深度和广度 |
| 非结构化 | 开放式对话 | 探索性研究 |

**访谈流程**
```
1. 准备
   ├── 确定研究目标
   ├── 设计访谈提纲
   └── 招募受访者
   
2. 执行
   ├── 建立关系
   ├── 按提纲引导
   ├── 深入追问
   └── 记录整理
   
3. 分析
   ├── 整理访谈记录
   ├── 提取关键洞察
   └── 形成研究结论
```

### 3. 问卷调查

**问卷设计原则**
- 问题简洁明了
- 避免引导性问题
- 选项互斥且完整
- 控制问卷长度

**常用问卷类型**
```
单选题：
○ 选项A
○ 选项B
○ 选项C

多选题：
☑ 选项A
☐ 选项B
☑ 选项C

量表题：
非常不满意 ○—○—○—○—○ 非常满意

开放题：
[请填写您的建议...              ]
```

### 4. 可用性测试

**测试类型**
| 类型 | 特点 | 用途 |
|------|------|------|
| 模态测试 | 按任务测试 | 验证可用性 |
| 探索性测试 | 开放式使用 | 发现问题 |
| 对比测试 | 对比多个方案 | 选择最优方案 |

**测试流程**
```
1. 准备
   ├── 确定测试目标
   ├── 设计测试任务
   ├── 准备测试原型
   └── 招募测试用户

2. 执行
   ├── 欢迎和说明
   ├── 预访谈
   ├── 任务执行（出声思考）
   └── 后访谈

3. 分析
   ├── 整理测试记录
   ├── 分析问题
   └── 提出改进建议
```

## 用户画像

### 1. 画像定义

**用户画像是基于研究创建的典型用户模型，代表目标用户群体的特征。**

### 2. 画像要素

```markdown
# 用户画像示例

## 基本信息
- 姓名：张小明
- 年龄：28岁
- 职业：产品经理
- 城市：北京

## 目标
- 提高工作效率
- 快速获取行业信息
- 提升专业技能

## 痛点
- 信息过载，难以筛选
- 工具分散，效率不高
- 学习时间有限

## 行为特征
- 每天使用手机3小时+
- 偏好简洁高效的工具
- 习惯碎片化阅读

## 使用场景
- 通勤路上浏览资讯
- 工作间隙快速查资料
- 周末集中学习提升
```

### 3. 画像类型

| 类型 | 说明 |
|------|------|
| 目标导向画像 | 基于用户目标分类 |
| 角色基于画像 | 基于用户角色分类 |
| 习惯基于画像 | 基于使用习惯分类 |

## 原型设计

### 1. 原型类型

| 类型 | 保真度 | 用途 |
|------|--------|------|
| 低保真 | 低 | 概念验证 |
| 中保真 | 中 | 流程验证 |
| 高保真 | 高 | 交互测试 |

### 2. 低保真原型

```
手绘草图 / 线框图

┌─────────────────────────────────┐
│ Logo              登录 | 注册   │
├─────────────────────────────────┤
│                                 │
│        [大Banner区域]           │
│                                 │
├─────────────────────────────────┤
│  ┌────┐  ┌────┐  ┌────┐        │
│  │    │  │    │  │    │        │
│  │    │  │    │  │    │        │

```

### 3. 高保真原型

- 接近真实产品外观
- 完整的交互逻辑
- 丰富的视觉效果

## 可用性评估

### 1. 评估方法

| 方法 | 类型 | 用途 |
|------|------|------|
| 可用性测试 | 用户参与 | 发现可用性问题 |
| 启发式评估 | 专家评审 | 快速识别问题 |
| 认知走查 | 模拟用户 | 发现流程问题 |

### 2. 启发式评估

**尼尔森十大可用性原则**
1. 系统状态可见性
2. 系统与现实世界匹配
3. 用户控制与自由
4. 一致性与标准化
5. 错误预防
6. 识别而非回忆
7. 灵活性与效率
8. 美观与极简设计
9. 帮助用户识别和恢复错误
10. 帮助与文档

### 3. 评估指标

| 指标 | 说明 |
|------|------|
| 任务完成率 | 完成任务的用户比例 |
| 任务完成时间 | 完成任务所需时间 |
| 错误率 | 操作错误的比例 |
| 满意度 | 用户主观满意度 |
| 净推荐值(NPS) | 用户推荐意愿 |

## UX设计方法

### 1. 设计冲刺 (Design Sprint)

**5天流程**
```
Day 1: 理解 (Understand)
       - 分享知识
       - 定义目标

Day 2: 发散 (Diverge)
       - 头脑风暴
       - 概念设计

Day 3: 决策 (Decide)
       - 方案筛选
       - 故事板

Day 4: 原型 (Prototype)
       - 制作原型

Day 5: 验证 (Validate)
       - 用户测试
```

### 2. 最小可行产品 (MVP)

**定义**
用最少的资源开发包含核心功能的产品，快速验证假设。

**MVP策略**
```
1. 识别核心假设
2. 定义最小功能集
3. 快速开发
4. 用户测试
5. 迭代优化
```

### 3. A/B测试

**流程**
```
1. 确定测试假设
   ↓
2. 设计A/B方案
   ↓
3. 分配流量
   ↓
4. 收集数据
   ↓
5. 分析结果
   ↓
6. 选择最优方案
```

## UX设计原则

### 1. 核心原则

- **以用户为中心**：理解用户需求
- **简洁**：减少认知负担
- **一致性**：保持统一体验
- **反馈**：及时响应用户操作
- **容错**：允许撤销和恢复
- **效率**：优化操作流程

### 2. 常见错误

- ❌ 忽视用户研究
- ❌ 功能堆砌
- ❌ 设计不一致
- ❌ 缺乏反馈
- ❌ 忽视可用性

## 最佳实践

### 1. 设计建议

- 始终以用户为中心
- 持续进行用户研究
- 快速原型，快速验证
- 数据驱动设计决策

### 2. 流程建议

- 建立完整的UX流程
- 跨部门协作
- 持续迭代优化
- 建立设计规范
