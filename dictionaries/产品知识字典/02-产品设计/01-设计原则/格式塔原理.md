# 格式塔原理

## 定义

格式塔原理（Gestalt Principles）是20世纪初德国心理学家提出的视觉感知理论，解释了人类如何将视觉元素组织成有意义的整体。核心观点是"整体大于部分之和"。

## 核心原理

### 1. 接近性原则 (Proximity)

**定义**
距离相近的元素被视为一组或相关联。

**视觉示例**
```
●●●●     ●  ●  ●  ●
●●●●  vs ●  ●  ●  ●
●●●●     ●  ●  ●  ●
成组      独立元素
```

**设计应用**
- 相关内容靠近排列
- 不相关内容增加间距
- 分组信息使用适当间距

**案例**
```
✓ 表单标签靠近对应输入框
✓ 列表项之间间距小于列表组之间
✗ 相关元素距离过远导致误解
```

**间距建议**
| 场景 | 推荐间距 |
|------|----------|
| 同组元素 | 4-8px |
| 不同组元素 | 16-24px |
| 章节之间 | 32-48px |

### 2. 相似性原则 (Similarity)

**定义**
外观相似的元素被视为一组或相关联。

**视觉示例**
```
●○●○●○    ●●●●●●
●○●○●○ vs ○○○○○○
●○●○●○    ●●●●●●
交替分组   横向分组
```

**设计应用**
- 使用颜色、形状、大小进行分组
- 一致的视觉样式表示同类内容
- 区分不同类型的元素

**案例**
```
✓ 所有可点击按钮使用相同样式
✓ 不同状态使用不同颜色区分
✗ 相同功能按钮样式不一致
```

**相似元素对比**
| 维度 | 示例 |
|------|------|
| 颜色 | 主要按钮用品牌色 |
| 形状 | 图标风格统一 |
| 大小 | 同级标题字号相同 |
| 字体 | 正文使用统一字体 |

### 3. 连续性原则 (Continuity)

**定义**
眼睛倾向于沿着平滑的线条或曲线移动，而不是识别突然的改变。

**视觉示例**
```
(   )
 \ /    眼睛会跟随曲线方向
  )
 / \
(   )
```

**设计应用**
- 引导用户视线流动
- 创建清晰的阅读路径
- 使用线条引导注意力

**案例**
```
✓ 时间线使用连续线条
✓ 表单字段对齐形成连续线
✓ 列表项边缘对齐
```

**应用场景**
- 视觉引导路径
- 信息流设计
- 页面布局对齐

### 4. 闭合性原则 (Closure)

**定义**
大脑倾向于将不完整的形状识别为完整的形状。

**视觉示例**
```
(    )    (•••)
      →    
[    ]    [•••]
不完整    完整感知
```

**设计应用**
- 使用不完整图形暗示完整
- 减少视觉噪音
- 创建现代简约设计

**案例**
```
✓ IBM 标志使用不连续线条
✓ 图标设计中暗示形状
✓ 卡片布局中元素不封闭但感知完整
```

**设计技巧**
- 部分形状可以暗示整体
- 负空间可以形成图形
- 不需要完整的边框

### 5. 图形与背景 (Figure & Ground)

**定义**
眼睛会将视觉区域分为图形（焦点）和背景。

**视觉示例**
```
 Rubin's Vase
   | |      看到:
  (   )     - 花瓶
   | |      或
  /   \     - 两张脸
```

**设计应用**
- 突出重要内容
- 模态对话框（背景暗化）
- 卡片设计

**案例**
```
✓ 弹窗背景变暗突出前景
✓ 卡片阴影区分层级
✗ 图形与背景对比度不足
```

**层级区分方式**
| 方式 | 效果 |
|------|------|
| 颜色对比 | 深色背景+浅色内容 |
| 阴影 | 提升元素层级 |
| 模糊 | 背景虚化突出前景 |
| 遮罩 | 半透明遮罩 |

### 6. 对称性原则 (Symmetry)

**定义**
对称的元素被视为一组，给人稳定和有序的感觉。

**视觉示例**
```
  ◆      ◆
 ◆◆◆    ◆◆
◆◆◆◆◆  ◆◆◆
 ◆◆◆    ◆◆◆
  ◆      ◆
对称    不对称
```

**设计应用**
- 创建平衡的布局
- 表达稳定和专业
- 对称性用于重要信息

**案例**
```
✓ 网页整体布局对称
✓ Logo居中对称
✓ 表格内容对齐
```

### 7. 共同区域 (Common Region)

**定义**
位于同一封闭区域的元素被视为一组。

**视觉示例**
```
┌─────┐ ┌─────┐
│ ● ● │ │ ○ ○ │
│ ● ● │ │ ○ ○ │
└─────┘ └─────┘
 明显的两组
```

**设计应用**
- 使用卡片分组
- 背景色区分区域
- 边框包围相关内容

**案例**
```
✓ 使用卡片组织内容
✓ 不同区域使用不同背景色
✓ 表单分组使用分隔线
```

### 8. 同域性原则 (Uniform Connectedness)

**定义**
通过线条或颜色连接的元素被视为一组。

**视觉示例**
```
●—●—●    分组：
         1-2-3 一组
|  |  |  4-5-6 一组
●  ●  ●  
4  5  6
```

**设计应用**
- 连接线表示关系
- 颜色编码分组
- 列表连接符号

### 9. 共同命运 (Common Fate)

**定义**
朝相同方向运动的元素被视为一组。

**视觉示例**
```
→ → →    向右的箭头
         被视为一组
← ← ←    向左的箭头
         被视为另一组
```

**设计应用**
- 动画方向一致
- 滑动方向分组
- 运动状态指示

## 设计应用

### 界面布局

```
┌─────────────────────────────────┐
│  Logo        Nav1  Nav2  Nav3   │ ← 接近性:导航成组
├─────────────────────────────────┤
│  ┌─────┐ ┌─────┐ ┌─────┐       │
│  │     │ │     │ │     │       │ ← 共同区域:卡片
│  │  A  │ │  B  │ │  C  │       │ ← 相似性:相同样式
│  │     │ │     │ │     │       │
│  └─────┘ └─────┘ └─────┘       │
│                                 │
│  ●─────────────●                │ ← 连续性:时间线
│  2024    2025    2026           │ ← 接近性:日期靠近节点
└─────────────────────────────────┘
```

### 表单设计

**接近性应用**
```
┌────────────────────────┐
│ 个人信息               │ ← 分组标题
│ ────────────────────   │
│ 姓名                   │
│ [输入框              ] │ ← 标签靠近输入框
│                        │
│ 邮箱                   │
│ [输入框              ] │
└────────────────────────┘
```

### 列表设计

**相似性应用**
```
┌────────────────────────┐
│ ○ 任务1                │
│ ○ 任务2                │ ← 相同样式
│ ○ 任务3                │
│ ────────────────────   │
│ ● 已完成1              │
│ ● 已完成2              │ ← 不同样式区分状态
└────────────────────────┘
```

## 最佳实践

### 1. 布局设计

- 使用间距和分组创造视觉层级
- 相关内容靠近放置
- 使用卡片组织内容块

### 2. 视觉层次

- 通过对比突出重点
- 使用颜色区分不同类型
- 保持一致的视觉样式

### 3. 交互设计

- 动画方向保持一致
- 相关元素同步响应
- 状态变化使用过渡

### 4. 常见错误

- ❌ 不相关元素距离太近
- ❌ 同类元素样式不一致
- ❌ 图形与背景对比不足
- ❌ 过度复杂的分组方式
