# 导航设计

## 定义

导航设计是规划和设计用户在产品中移动和定位的方式，帮助用户理解当前所在位置、可以去哪里以及如何到达。

## 导航的作用

### 核心功能

1. **定位**：告诉用户"我在哪里"
2. **指引**：告诉用户"可以去哪里"
3. **路径**：告诉用户"如何到达"
4. **上下文**：帮助用户理解内容关系

## 导航类型

### 1. 全局导航 (Global Navigation)

**定义**
出现在网站/应用所有页面，覆盖主要功能模块。

**位置示例**
```
┌─────────────────────────────────┐
│ Logo  首页  产品  服务  关于我们 │ ← 顶部全局导航
├─────────────────────────────────┤
│                                 │
│           内容区域              │
│                                 │
└─────────────────────────────────┘

┌─────────────────────────────────┐
│                                 │
│           内容区域              │
│                                 │
├─────────────────────────────────┤
│  🏠   🔍   ➕   ❤️   👤        │ ← 底部全局导航
└─────────────────────────────────┘
```

**设计要点**
- 限制项目数量（5-9项）
- 保持位置一致
- 当前位置高亮

### 2. 局部导航 (Local Navigation)

**定义**
在特定内容区域内，提供该区域内部的导航。

**类型**
```
┌──────────┬──────────────────────┐
│ 产品A    │                      │
│ ────     │     内容详情         │ ← 侧边局部导航
│ 产品B    │                      │
│ ────     │                      │
│ 产品C    │                      │
│          │                      │
└──────────┴──────────────────────┘

┌─────────────────────────────────┐
│ 产品详情 | 规格参数 | 用户评价   │ ← 标签页局部导航
├─────────────────────────────────┤
│                                 │
│           内容区域              │
│                                 │
└─────────────────────────────────┘
```

### 3. 面包屑导航 (Breadcrumb)

**定义**
显示用户在网站层级中的位置路径。

**类型**

| 类型 | 示例 |
|------|------|
| 位置型 | 首页 > 产品 > 电子产品 > 手机 |
| 路径型 | 首页 > 搜索结果 > 手机 |
| 属性型 | 首页 > 手机 > 品牌:苹果 > 价格:5000+ |

**设计规范**
```
✓ 首页 > 产品 > 手机 > iPhone 15
✓ 首页 / 产品 / 手机 / iPhone 15
✗ 首页 - 产品 - 手机 - iPhone 15
```

### 4. 上下文导航 (Contextual Navigation)

**定义**
根据当前内容相关的导航链接。

**示例**
```
┌─────────────────────────────────┐
│                                 │
│  当前文章内容...                │
│                                 │
│  相关文章：                     │ ← 上下文导航
│  • 文章A                        │
│  • 文章B                        │
│  • 文章C                        │
│                                 │
│  上一篇：文章X  下一篇：文章Y   │
└─────────────────────────────────┘
```

### 5. 辅助导航 (Utility Navigation)

**定义**
提供辅助功能的导航入口。

**常见元素**
```
┌─────────────────────────────────┐
│ 登录 | 注册 | 帮助 | 语言 ▼ | 🛒│
├─────────────────────────────────┤
```

**包含**
- 登录/注册
- 搜索
- 购物车
- 语言切换
- 帮助中心

### 6. 抽屉导航 (Drawer Navigation)

**定义**
隐藏在侧边的导航，点击展开。

**适用场景**
```
┌──────────────┐
│ ☰           │ ← 汉堡菜单图标
│              │
│   内容区     │
│              │
└──────────────┘

点击展开：
┌──────┬───────────┐
│ 菜单1│           │
│ 菜单2│  内容区   │
│ 菜单3│           │
│ ...  │           │
└──────┴───────────┘
```

## 导航设计原则

### 1. 可见性

**原则**
- 导航元素清晰可见
- 当前位置明确标识

**示例**
```
✓ 当前页高亮
┌─────────────────────────────────┐
│ 首页 [产品] 服务 关于           │
└─────────────────────────────────┘

✗ 无当前位置指示
┌─────────────────────────────────┐
│ 首页 产品 服务 关于             │
└─────────────────────────────────┘
```

### 2. 一致性

**原则**
- 位置一致
- 样式一致
- 行为一致

### 3. 清晰性

**原则**
- 标签简洁明了
- 图标含义清晰
- 避免歧义

### 4. 效率

**原则**
- 减少点击次数
- 提供快捷方式
- 支持搜索

### 5. 反馈

**原则**
- 点击状态反馈
- 加载状态提示
- 错误状态处理

## 导航设计流程

```
1. 信息架构分析
   ├── 内容分类
   ├── 层级关系
   └── 用户任务
        ↓
2. 导航结构设计
   ├── 确定导航类型
   ├── 规划导航层级
   └── 设计导航路径
        ↓
3. 视觉设计
   ├── 布局位置
   ├── 样式风格
   └── 交互状态
        ↓
4. 可用性测试
   ├── 用户测试
   ├── 问题收集
   └── 迭代优化
```

## 常见导航模式

### 1. 顶部导航

```
┌─────────────────────────────────┐
│ Logo    产品  服务  关于  联系  │
├─────────────────────────────────┤
│                                 │
└─────────────────────────────────┘
```

**适用**：内容层级较少的网站

### 2. 侧边导航

```
┌──────┬──────────────────────────┐
│ 菜单1│                          │
│ 菜单2│       内容区域           │
│ 菜单3│                          │
│ 菜单4│                          │
└──────┴──────────────────────────┘
```

**适用**：后台管理系统、复杂应用

### 3. 混合导航

```
┌─────────────────────────────────┐
│ Logo    产品  服务  关于        │ ← 主导航
├──────┬──────────────────────────┤
│ 子项1│                          │
│ 子项2│       内容区域           │ ← 子导航
│ 子项3│                          │
└──────┴──────────────────────────┘
```

### 4. 响应式导航

```
桌面端：
┌─────────────────────────────────┐
│ Logo  导航A 导航B 导航C 导航D  │
└─────────────────────────────────┘

移动端：
┌─────────────────────────────────┐
│ Logo                          ☰ │
└─────────────────────────────────┘
```

## 导航设计检查清单

- [ ] 导航项目数量合理（5-9项）
- [ ] 当前位置有明确指示
- [ ] 导航标签清晰易懂
- [ ] 导航位置一致
- [ ] 支持返回操作
- [ ] 有搜索入口
- [ ] 响应式适配良好
- [ ] 符合用户心智模型

## 最佳实践

### 1. 设计建议

- 限制导航项数量
- 使用用户熟悉的标签
- 提供多种导航方式
- 保持导航可见性

### 2. 常见错误

- ❌ 导航层级过深
- ❌ 标签含义不清
- ❌ 位置不固定
- ❌ 缺少当前位置指示

### 3. 优化方法

- 定期分析用户路径
- A/B测试导航方案
- 收集用户反馈
- 持续迭代优化
