# 表单设计

## 定义

表单是用户与系统进行数据交换的主要界面，用于收集、编辑和提交信息。良好的表单设计能显著提升用户体验和转化率。

## 表单类型

### 按用途分类

| 类型 | 用途 | 示例 |
|------|------|------|
| 注册表单 | 创建账户 | 用户注册 |
| 登录表单 | 身份验证 | 用户登录 |
| 信息收集 | 收集用户数据 | 问卷调查 |
| 交易表单 | 完成交易 | 支付表单 |
| 搜索表单 | 查找内容 | 搜索框 |

### 按复杂度分类

**简单表单**
```
┌─────────────────────────────────┐
│ 邮箱                           │
│ [                            ] │
│                                │
│ 密码                           │
│ [                            ] │
│                                │
│         [登录]                 │
└─────────────────────────────────┘
```

**复杂表单**
```
┌─────────────────────────────────┐
│ 基本信息                        │
│ ────────────────────────        │
│ 姓名 [        ]  性别 [ ]      │
│ 电话 [        ]  邮箱 [ ]      │
│                                │
│ 详细地址                        │
│ ────────────────────────        │
│ 省份 [    ▼]  城市 [    ▼]     │
│ 详细地址 [                    ]│
│                                │
│        [上一步] [下一步]        │
└─────────────────────────────────┘
```

## 表单结构

### 1. 基本组成元素

```
┌─────────────────────────────────┐
│ 表单标题                        │ ← 标题
│ 表单描述信息                    │ ← 描述
│ ────────────────────────────    │
│ 字段标签 *                      │ ← 标签 + 必填标记
│ [输入框                      ] │ ← 输入框
│ 辅助提示文字                    │ ← 帮助文本
│ ────────────────────────────    │
│         [提交] [取消]           │ ← 操作按钮
└─────────────────────────────────┘
```

### 2. 标签位置

**顶部标签**
```
┌─────────────────────────────────┐
│ 用户名                          │
│ [                            ] │
│                                │
│ 密码                            │
│ [                            ] │
└─────────────────────────────────┘
```

优点：移动端友好，标签清晰
缺点：占用垂直空间

**左对齐标签**
```
┌─────────────────────────────────┐
│ 用户名 [                      ]│
│ 密码   [                      ]│
└─────────────────────────────────┘
```

优点：节省垂直空间
缺点：标签长度受限

**右对齐标签**
```
┌─────────────────────────────────┐
│  用户名 [                      ]│
│    密码 [                      ]│
└─────────────────────────────────┘
```

优点：标签与输入框关联强
缺点：标签不易扫描

### 3. 输入字段类型

| 类型 | 用途 | 示例 |
|------|------|------|
| text | 单行文本 | 姓名 |
| textarea | 多行文本 | 留言 |
| password | 密码 | 登录密码 |
| email | 邮箱 | user@example.com |
| number | 数字 | 数量 |
| tel | 电话 | 手机号 |
| date | 日期 | 出生日期 |
| select | 下拉选择 | 省份 |
| checkbox | 多选 | 兴趣爱好 |
| radio | 单选 | 性别 |
| file | 文件上传 | 头像 |

## 表单设计原则

### 1. 简化原则

**减少字段数量**
```
✓ 只收集必要信息
✗ 收集"可能需要"的信息

示例：
✓ 姓名、邮箱、密码（注册）
✗ 姓名、邮箱、密码、性别、生日、职业、公司...（注册）
```

**使用智能默认值**
```
✓ 根据定位自动填充城市
✓ 根据历史记录自动补全
```

### 2. 清晰原则

**标签清晰**
```
✓ "手机号码"
✗ "手机" / "联系方式"

✓ "收货地址"
✗ "地址"
```

**必填标记明确**
```
方式1：必填项标 *
┌─────────────────────────────────┐
│ 用户名 *                        │
│ [                            ] │
│                                │
│ 昵称（选填）                   │
│ [                            ] │
└─────────────────────────────────┘

方式2：选填项标注"选填"
```

### 3. 反馈原则

**即时校验**
```
┌─────────────────────────────────┐
│ 手机号码 *                      │
│ [13800138000                 ] │ ✓ 格式正确
└─────────────────────────────────┘

┌─────────────────────────────────┐
│ 手机号码 *                      │
│ [13800138                    ] │ ✗ 请输入11位手机号
└─────────────────────────────────┘
```

**状态反馈**
```
默认状态：
┌─────────────────────────────────┐
│ [                            ] │
└─────────────────────────────────┘

聚焦状态：
┌─────────────────────────────────┐
│ [|                           ] │ ← 蓝色边框
└─────────────────────────────────┘

成功状态：
┌─────────────────────────────────┐
│ [有效内容                    ] │ ← 绿色边框+勾
└─────────────────────────────────┘

错误状态：
┌─────────────────────────────────┐
│ [错误内容                    ] │ ← 红色边框+提示
│ 请输入正确的格式               │
└─────────────────────────────────┘
```

## 表单设计最佳实践

### 1. 布局设计

**单列布局**
```
✓ 推荐
┌─────────────────────────────────┐
│ 姓名                            │
│ [                            ] │
│                                │
│ 邮箱                            │
│ [                            ] │
│                                │
│ 电话                            │
│ [                            ] │
└─────────────────────────────────┘
```

**多列布局（慎用）**
```
仅用于逻辑相关的短字段
┌─────────────────────────────────┐
│ 姓名 [      ]  性别 [ ]        │
│ 电话 [      ]  邮箱 [ ]        │
└─────────────────────────────────┘
```

### 2. 分组设计

**视觉分组**
```
┌─────────────────────────────────┐
│ 联系信息                        │
│ ───────────────────────────     │
│ 姓名                            │
│ [                            ] │
│ 电话                            │
│ [                            ] │
│                                │
│ 配送地址                        │
│ ───────────────────────────     │
│ 省份                            │
│ [                            ] │
│ 地址                            │
│ [                            ] │
└─────────────────────────────────┘
```

**步骤分组**
```
┌─────────────────────────────────┐
│ ① 基本信息 ─ ② 详细信息 ─ ③ 确认│
│                                 │
│ 当前步骤：基本信息               │
│ ───────────────────────────     │
│ 姓名                            │
│ [                            ] │
│                                │
│      [上一步]    [下一步]       │
└─────────────────────────────────┘
```

### 3. 输入控制

**格式限制**
```
手机号：只允许输入数字，最大11位
日期：使用日期选择器
金额：限制小数位数
```

**自动格式化**
```
手机号：138 0013 8000
银行卡：6222 0202 0000 0000
金额：1,000.00
```

### 4. 帮助信息

**占位符提示**
```
┌─────────────────────────────────┐
│ 手机号码                        │
│ [请输入11位手机号            ] │
└─────────────────────────────────┘
```

**帮助文本**
```
┌─────────────────────────────────┐
│ 密码                            │
│ [                            ] │
│ 至少8位，包含字母和数字         │ ← 帮助文本
└─────────────────────────────────┘
```

**图标提示**
```
┌─────────────────────────────────┐
│ 密码                     [👁]   │ ← 显示/隐藏密码
│ [●●●●●●●●                   ] │
└─────────────────────────────────┘
```

### 5. 错误处理

**即时错误提示**
```
┌─────────────────────────────────┐
│ 邮箱                            │
│ [user@                      ] │
│ ✗ 请输入有效的邮箱地址         │
└─────────────────────────────────┘
```

**提交后错误提示**
```
┌─────────────────────────────────┐
│ ⚠️ 表单提交失败，请检查以下错误  │
│ • 邮箱格式不正确                │
│ • 密码长度不足                  │
└─────────────────────────────────┘
```

**错误提示位置**
- 输入框下方（推荐）
- 输入框右侧
- 表单顶部汇总

### 6. 操作按钮

**按钮层级**
```
┌─────────────────────────────────┐
│    [提交]（主要） [取消]（次要） │
└─────────────────────────────────┘
```

**按钮位置**
```
✓ 右对齐或居中
✗ 左对齐（不符合阅读顺序）
```

**按钮状态**
```
默认：[提交]
悬停：[提交] 颜色加深
禁用：[提交] 灰色不可点击
加载：[○ 提交中...] 显示加载状态
```

## 表单设计检查清单

### 布局
- [ ] 标签位置一致
- [ ] 对齐方式统一
- [ ] 间距合理
- [ ] 分组清晰

### 内容
- [ ] 字段数量精简
- [ ] 标签清晰明确
- [ ] 必填项标记清晰
- [ ] 帮助文本恰当

### 交互
- [ ] Tab键顺序正确
- [ ] 即时校验反馈
- [ ] 错误提示清晰
- [ ] 提交状态明确

### 可访问性
- [ ] 键盘可操作
- [ ] 屏幕阅读器友好
- [ ] 颜色对比度足够
- [ ] 焦点状态可见

## 案例

### 案例1：登录表单优化

**优化前**
```
┌─────────────────────────────────┐
│ 用户名                          │
│ [                            ] │
│ 密码                            │
│ [                            ] │
│ 验证码                          │

**优化后**
```
┌─────────────────────────────────┐
│ 欢迎回来                        │
│                                 │
│ 手机号/邮箱                     │

## 最佳实践总结

1. **简化**：只收集必要信息
2. **清晰**：标签和帮助信息明确
3. **反馈**：即时校验和状态反馈
4. **容错**：允许修改和撤销
5. **可访问**：支持所有用户
