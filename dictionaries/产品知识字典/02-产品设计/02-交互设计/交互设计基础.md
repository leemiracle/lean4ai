# 交互设计基础

## 定义

交互设计（Interaction Design，简称 IxD）是设计和规划人与产品、服务或系统之间交互方式的学科，关注用户如何与产品交互以及产品如何响应用户操作。

## 核心概念

### 1. 交互设计五维度

| 维度 | 说明 | 内容 |
|------|------|------|
| 1D - 文字 | 语言交流 | 标签、按钮文字、提示信息 |
| 2D - 视觉表现 | 图形元素 | 图标、图像、排版、色彩 |
| 3D - 物理对象/空间 | 硬件设备 | 触屏、键盘、鼠标、VR设备 |
| 4D - 时间 | 时序变化 | 动画、视频、音频、进度 |
| 5D - 行为 | 交互动作 | 点击、滑动、语音、手势 |

### 2. 交互设计要素

**用户 (User)**
- 目标
- 需求
- 行为
- 限制

**产品 (Product)**
- 功能
- 形式
- 内容
- 行为

**情境 (Context)**
- 环境
- 情感
- 时间
- 社会因素

## 交互设计流程

### 标准流程

```
1. 研究分析
   ├── 用户研究
   ├── 竞品分析
   └── 需求分析
        ↓
2. 概念设计
   ├── 信息架构
   ├── 交互框架
   └── 用户流程
        ↓
3. 原型设计
   ├── 低保真原型
   ├── 高保真原型
   └── 交互原型
        ↓
4. 评估迭代
   ├── 可用性测试
   ├── 迭代优化
   └── 最终交付
```

### 设计活动

**分析阶段**
- 用户访谈
- 问卷调查
- 任务分析
- 场景分析

**设计阶段**
- 线框图设计
- 交互流程设计
- 原型制作
- 规格文档

**评估阶段**
- 可用性测试
- 启发式评估
- A/B测试
- 用户反馈

## 交互设计原则

### 基础原则

#### 1. 可见性 (Visibility)

**定义**
用户能够看到可用的功能和当前状态。

**应用**
```
✓ 按钮明显可见
✓ 功能入口清晰
✓ 状态指示明确
✗ 隐藏的重要功能
```

#### 2. 反馈 (Feedback)

**定义**
系统对用户操作做出响应。

**反馈类型**
| 类型 | 示例 |
|------|------|
| 即时反馈 | 按钮点击效果 |
| 过程反馈 | 加载进度条 |
| 结果反馈 | 操作成功提示 |

#### 3. 约束 (Constraints)

**定义**
限制用户可能的操作，减少错误。

**约束类型**
```
物理约束：USB接口只能一个方向插入
逻辑约束：已售商品不能再次购买
文化约束：红色表示危险/删除
```

#### 4. 映射 (Mapping)

**定义**
控制与效果之间的关系。

**示例**
```
✓ 音量上滑增大，下滑减小
✓ 开关向上是开，向下是关
✗ 控制与效果不匹配
```

#### 5. 一致性 (Consistency)

**定义**
相似的操作有相似的行为。

**应用**
- 平台一致性
- 内部一致性
- 外部一致性

#### 6. 可供性 (Affordance)

**定义**
物品的外观暗示其使用方式。

**示例**
```
按钮的立体感暗示可点击
输入框暗示可输入
滑块暗示可拖动
```

## 交互模式

### 1. 导航模式

**层级导航**
```
首页 → 列表 → 详情
返回: 详情 → 列表 → 首页
```

**标签导航**
```
┌─────────────────────────────────┐
│                                 │
│           内容区                │
│                                 │
├─────────────────────────────────┤
│  🏠   🔍   ➕   ❤️   👤        │
│  首页 搜索 添加 收藏 我的       │
└─────────────────────────────────┘
```

**抽屉导航**
```
┌──────────────────┬──────────────┐
│                  │              │
│   菜单项1        │    内容区    │
│   菜单项2        │              │
│   菜单项3        │              │
│                  │              │
└──────────────────┴──────────────┘
```

### 2. 输入模式

**表单输入**
```
┌─────────────────────────────────┐
│ 标签                            │
│ [输入框                      ] │
│ 提示文字/错误信息               │
└─────────────────────────────────┘
```

**选择输入**
```
单选: ○ 选项1  ● 选项2  ○ 选项3
多选: ☑ 选项1  ☐ 选项2  ☑ 选项3
下拉: [请选择 ▼]
```

### 3. 反馈模式

**模态反馈**
```
┌─────────────────────────────────┐
│         操作确认               │
│                                 │
│   确定要删除此项目吗？         │
│                                 │
│      [取消]    [确定]          │
└─────────────────────────────────┘
```

**非模态反馈**
```
┌─────────────────────────────────┐
│                    ┌──────────┐ │
│   主要内容        │ 提示信息 │ │
│   主要内容        └──────────┘ │
│   主要内容                      │
└─────────────────────────────────┘
```

### 4. 内容展示模式

**列表模式**
```
┌─────────────────────────────────┐
│ ┌─────────────────────────────┐ │
│ │ 项目A              操作    │ │
│ └─────────────────────────────┘ │
│ ┌─────────────────────────────┐ │
│ │ 项目B              操作    │ │
│ └─────────────────────────────┘ │
└─────────────────────────────────┘
```

**卡片模式**
```
┌────────┐ ┌────────┐ ┌────────┐
│ 图片   │ │ 图片   │ │ 图片   │
│ 标题   │ │ 标题   │ │ 标题   │
│ 描述   │ │ 描述   │ │ 描述   │
│ 操作   │ │ 操作   │ │ 操作   │
└────────┘ └────────┘ └────────┘
```

## 交互状态

### 状态类型

| 状态 | 说明 | 视觉表现 |
|------|------|----------|
| 默认 | 初始状态 | 基础样式 |
| 悬停 | 鼠标移上 | 颜色变化/阴影 |
| 聚焦 | 键盘聚焦 | 外边框 |
| 激活 | 点击瞬间 | 按下效果 |
| 禁用 | 不可操作 | 灰色/半透明 |
| 加载 | 处理中 | 加载动画 |
| 成功 | 操作成功 | 绿色提示 |
| 错误 | 操作失败 | 红色提示 |

### 状态示例

**按钮状态**
```
默认:   ┌──────────┐
        │ 提交     │ 品牌色背景
        └──────────┘

悬停:   ┌──────────┐
        │ 提交     │ 颜色加深
        └──────────┘

点击:   ┌──────────┐
        │ 提交     │ 按下效果
        └──────────┘

禁用:   ┌──────────┐
        │ 提交     │ 灰色/不可点击
        └──────────┘

加载:   ┌──────────┐
        │ ○ 提交   │ 加载动画
        └──────────┘
```

## 交互设计方法

### 1. 任务分析

**步骤**
```
1. 识别用户目标
2. 分解任务步骤
3. 分析每步需求
4. 优化任务流程
```

**示例：购买商品任务分析**
```
目标：完成购买

任务分解：
1. 浏览商品 → 需要商品展示
2. 选择商品 → 需要选择操作
3. 加入购物车 → 需要反馈确认
4. 确认订单 → 需要订单信息
5. 支付 → 需要支付方式
6. 完成反馈 → 需要成功提示
```

### 2. 用户流程图

**符号**
```
○ 开始/结束
□ 操作步骤
◇ 判断/决策
→ 流程方向
```

**示例**
```
○ 开始
 ↓
□ 打开应用
 ↓
◇ 已登录？
 ↙    ↘
否      是
 ↓       ↓
□ 登录  □ 浏览内容
 ↓       ↓
□ 浏览内容
 ↓
○ 结束
```

### 3. 线框图

**目的**
- 展示页面结构
- 确定信息层级
- 规划交互区域

**低保真线框图**
```
┌─────────────────────────────────┐
│ Logo              登录 | 注册   │
├─────────────────────────────────┤
│                                 │
│        [大Banner区域]           │
│                                 │
├─────────────────────────────────┤
│  ┌────┐  ┌────┐  ┌────┐        │
│  │    │  │    │  │    │        │
│  │    │  │    │  │    │        │
│  └────┘  └────┘  └────┘        │
│                                 │
├─────────────────────────────────┤
│           页脚信息              │
└─────────────────────────────────┘
```

## 交互设计评估

### 评估方法

**1. 可用性测试**
- 观察用户操作
- 收集使用数据
- 发现问题点

**2. 启发式评估**
- 专家评审
- 对照原则检查
- 识别问题

**3. 认知走查**
- 模拟用户视角
- 逐步检查流程
- 发现障碍

### 评估指标

| 指标 | 说明 |
|------|------|
| 任务完成率 | 完成任务的用户比例 |
| 完成时间 | 完成任务所需时间 |
| 错误率 | 操作错误的比例 |
| 满意度 | 主观满意度评分 |
| 学习性 | 新用户上手难度 |

## 最佳实践

### 1. 设计原则应用

- 始终保持一致性
- 提供清晰的反馈
- 减少用户认知负担
- 支持撤销和恢复

### 2. 常见错误

- ❌ 过度复杂的交互
- ❌ 缺乏必要的反馈
- ❌ 违反平台规范
- ❌ 忽视可访问性

### 3. 优化建议

- 简化操作流程
- 提供清晰的视觉引导
- 支持多种输入方式
- 考虑边缘情况
