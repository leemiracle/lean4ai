# 信息架构 (Information Architecture)

## 定义

信息架构（Information Architecture，简称IA）是组织和结构化信息的艺术与科学，帮助用户更有效地找到和理解内容。

## 核心概念

### 1. IA定义

**目的**
- 帮助用户找到信息
- 帮助用户理解信息
- 帮助用户使用信息

**组成部分**
```
信息架构
├── 组织系统 (Organization Systems)
├── 标签系统 (Labeling Systems)
├── 导航系统 (Navigation Systems)
└── 搜索系统 (Search Systems)
```

### 2. 信息生态模型

```
        用户 (Users)
           ▲
          /│\
         / │ \
        /  │  \
       /   │   \
      /    │    \
     /     │     \
内容 ←─────┼─────→ 情境
(Content)  │  (Context)
```

## 组织系统

### 1. 组织结构类型

#### 层级结构 (Hierarchy)

```
                首页
                 │
        ┌───────┼───────┐
        │       │       │
      产品     服务    关于我们
        │
   ┌────┼────┐
   │    │    │
 分类A 分类B 分类C
```

**特点**
- 自顶向下
- 清晰的层级关系
- 适合大多数网站

#### 矩阵结构 (Matrix)

```
        │ 产品A │ 产品B │ 产品C │
────────┼───────┼───────┼───────┤
 功能1  │   ●   │   ○   │   ●   │
 功能2  │   ○   │   ●   │   ○   │
 功能3  │   ●   │   ●   │   ●   │
```

**特点**
- 多维度组织
- 多种浏览路径
- 适合复杂产品

#### 线性结构 (Linear)

```
步骤1 → 步骤2 → 步骤3 → 步骤4 → 完成
```

**特点**
- 有序流程
- 适合引导式任务
- 如：注册流程、购物流程

#### 网络结构 (Network)

```
    ┌─────┐
    │  A  │
    └──┬──┘
   ┌───┼───┐
┌──┴──┐ ┌──┴──┐
│  B  │─│  C  │
└──┬──┘ └──┬──┘
   └───┼───┘
    ┌──┴──┐
    │  D  │
    └─────┘
```

**特点**
- 非线性连接
- 内容关联丰富
- 如：Wiki系统

### 2. 组织方法

#### 按主题 (Topic)

```
科技
├── 人工智能
├── 区块链
└── 物联网

财经
├── 股票
├── 基金
└── 理财
```

#### 按任务 (Task)

```
账户管理
├── 注册
├── 登录
├── 修改密码
└── 绑定手机
```

#### 按用户 (Audience)

```
个人用户
├── 个人理财
└── 信用卡

企业用户
├── 企业贷款
└── 代发工资
```

#### 按时间 (Chronological)

```
2024年
├── 1月
├── 2月
└── 3月
```

#### 按地点 (Geographical)

```
中国
├── 北京
├── 上海
└── 广州
```

### 3. 精确性 vs 模糊性

| 精确性组织 | 模糊性组织 |
|------------|------------|
| 按字母顺序 | 按主题 |
| 按时间 | 按用户群 |
| 按地点 | 按任务 |

## 标签系统

### 1. 标签设计原则

**简洁**
```
✓ 购物车
✗ 用于存放想要购买的商品
```

**清晰**
```
✓ 关于我们
✗ 关于 / 关于公司 / 公司介绍
```

**一致**
```
✓ 购物车 → 结账 → 支付
✗ 购物车 → 收银台 → 付款
```

**用户语言**
```
✓ 会员中心 (用户语言)
✗ 用户管理后台 (技术语言)
```

### 2. 标签类型

| 类型 | 示例 |
|------|------|
| 导航标签 | 首页、产品、关于 |
| 索引标签 | A-Z、按时间 |
| 图标标签 | 🏠→首页、🔍→搜索 |
| 上下文标签 | "相关推荐" |
| 标题标签 | 页面标题、章节标题 |

## 导航系统

### 1. 导航类型

#### 全局导航

```
┌─────────────────────────────────┐
│ Logo │ 首页 │ 产品 │ 服务 │ 联系 │ ← 全局导航
├─────────────────────────────────┤
│                                 │
│           页面内容              │
│                                 │
└─────────────────────────────────┘
```

**特点**
- 始终可见
- 覆盖全站主要内容

#### 局部导航

```
┌─────────────────────────────────┐
│ 产品                            │
│ ├── 产品A                       │
│ ├── 产品B    │   产品详情      │ ← 局部导航
│ └── 产品C    │   内容...       │
│              │                  │
└─────────────────────────────────┘
```

#### 上下文导航

```
┌─────────────────────────────────┐
│                                 │
│  文章内容...                    │
│                                 │
│  相关推荐：文章A / 文章B         │ ← 上下文导航
│                                 │
│  上一篇 / 下一篇                 │
└─────────────────────────────────┘
```

#### 面包屑导航

```
首页 > 产品 > 电子产品 > 手机
```

**作用**
- 显示当前位置
- 提供返回路径
- 理解网站结构

#### 辅助导航

```
┌─────────────────────────────────┐
│ 登录 | 注册 | 帮助 | 语言 ▼     │ ← 辅助导航
├─────────────────────────────────┤
│                                 │
```

### 2. 导航设计原则

- **可见性**：让用户知道"在哪"
- **可理解**：标签清晰易懂
- **一致性**：全站风格统一
- **效率**：快速到达目标

## 信息架构方法

### 1. 卡片分类法 (Card Sorting)

**开放式**
- 用户自由分类
- 自己命名类别

**封闭式**
- 给定类别
- 用户分配卡片

**流程**
```
1. 准备卡片（内容项）
   ↓
2. 邀请用户
   ↓
3. 执行分类
   ↓
4. 分析结果
   ↓
5. 优化IA
```

### 2. 站点地图 (Sitemap)

```
网站地图
│
├── 首页
│   ├── 产品
│   │   ├── 分类A
│   │   └── 分类B
│   ├── 服务
│   │   ├── 服务A
│   │   └── 服务B
│   └── 关于我们
│       ├── 公司介绍
│       └── 联系方式
```

### 3. 用户流程图

```
○ 进入网站
 ↓
□ 浏览首页
 ↓
◇ 有明确目标？
 ↙        ↘
是          否
 ↓           ↓
□ 搜索     □ 浏览分类
 ↓           ↓
□ 查看结果  □ 浏览详情
 ↓           ↓
□ 购买/离开 ←───┘
```

## IA 设计流程

```
1. 研究
   ├── 内容审计
   ├── 用户研究
   └── 竞品分析
        ↓
2. 策略
   ├── 确定结构
   ├── 定义标签
   └── 规划导航
        ↓
3. 设计
   ├── 站点地图
   ├── 线框图
   └── 原型设计
        ↓
4. 评估
   ├── 卡片分类
   ├── 可用性测试
   └── 迭代优化
```

## 最佳实践

### 1. IA设计原则

- 以用户为中心
- 保持简洁清晰
- 考虑未来扩展
- 定期评估优化

### 2. 常见错误

- ❌ 结构过于深层
- ❌ 标签不清晰
- ❌ 导航混乱
- ❌ 搜索功能缺失

### 3. 优化建议

- 控制层级深度（3-4层为宜）
- 提供多种浏览路径
- 持续收集用户反馈
- 定期审计和优化
