# 手势设计

## 定义

手势设计是指设计和定义用户在触摸屏设备上通过手指动作与界面交互的方式，包括点击、滑动、缩放等操作。

## 手势类型

### 1. 基础手势

| 手势 | 操作 | 用途 |
|------|------|------|
| 点击 (Tap) | 单指轻触 | 选择、激活 |
| 长按 (Long Press) | 单指长按 | 菜单、编辑 |
| 双击 (Double Tap) | 快速双击 | 缩放、点赞 |
| 滑动 (Swipe) | 单指滑动 | 滚动、切换 |
| 拖拽 (Drag) | 单指拖动 | 移动、排序 |
| 捏合 (Pinch) | 双指捏合 | 缩放 |
| 旋转 (Rotate) | 双指旋转 | 旋转 |
| 侧滑 (Edge Swipe) | 边缘滑动 | 返回、菜单 |

### 2. 手势图示

```
点击 (Tap)：
    ● → (轻触)

长按 (Long Press)：
    ● ────→ (保持)

双击 (Double Tap)：
    ● ● (快速两次)

滑动 (Swipe)：
    ──────→ (快速滑动)

拖拽 (Drag)：
    ● ────→ (按住移动)

捏合 (Pinch)：
    \   /     \   /
     \ /   →   \ /
      ●        ○
     / \   →   / \
    /   \     /   \
    
旋转 (Rotate)：
      ↑          ↗
    ←   →  →  ↙   ↘
      ↓          ↙
```

## 常见手势应用

### 1. 点击手势

**应用场景**
```
- 按钮激活
- 列表项选择
- 链接跳转
- 开关切换
```

**设计要点**
```
触摸目标：
- 最小尺寸：44×44pt (iOS) / 48×48dp (Android)
- 推荐尺寸：48×48pt以上
- 间距：至少8pt
```

### 2. 长按手势

**应用场景**
```
┌─────────────────────────────────┐
│ 消息内容                        │
│ ┌───────────────────────────┐  │
│ │ 长按显示菜单               │  │
│ │ • 复制                    │  │
│ │ • 转发                    │  │
│ │ • 删除                    │  │
│ └───────────────────────────┘  │
└─────────────────────────────────┘
```

**设计要点**
- 响应时间：约500ms
- 触觉反馈：震动提示
- 视觉反馈：高亮或放大

### 3. 双击手势

**应用场景**
```
图片查看器：
- 双击放大/缩小

社交媒体：
- 双击点赞

地图：
- 双击放大
```

## 平台手势规范

### 1. iOS手势规范

**系统手势**
| 手势 | 功能 |
|------|------|
| 从底部上滑 | 回到主屏幕 |
| 从底部上滑停顿 | 多任务 |
| 左右滑动 | 切换应用 |
| 右上角下滑 | 控制中心 |
| 左侧/中间下滑 | 通知中心 |

**返回手势**
```
┌─────────────────────────────────┐
│←────────────────────            │
│ 从屏幕左边缘向右滑动            │
│ 返回上一页                      │
└─────────────────────────────────┘
```

### 2. Android手势规范

**系统手势**
| 手势 | 功能 |
|------|------|
| 从底部上滑 | 主屏幕 |
| 从底部上滑停顿 | 多任务 |
| 从左右边缘滑动 | 返回 |
| 从底部两侧上滑 | 切换应用 |

**返回手势**
```
┌─────────────────────────────────┐
│←────────────────────            │
│ 从屏幕左/右边缘向内滑动         │
│ 返回上一页                      │
└─────────────────────────────────┘
```

## 手势设计原则

### 1. 可发现性

**原则**
- 关键功能不应只依赖手势
- 提供视觉提示
- 支持多种操作方式

**设计方法**
```
1. 提供替代操作
   手势操作 + 按钮操作

2. 视觉提示
   滑动提示、手势引导

3. 首次使用引导
   新手教程、手势提示
```

### 2. 一致性

**原则**
- 使用通用手势习惯
- 同类应用手势一致
- 平台规范遵循

**通用手势习惯**
| 手势 | 通用含义 |
|------|----------|
| 下拉 | 刷新 |
| 左滑 | 删除/返回 |
| 右滑 | 更多选项/下一项 |
| 双指捏合 | 缩放 |
| 双击 | 放大/点赞 |

### 3. 反馈性

**原则**
- 即时视觉反馈
- 触觉反馈
- 操作结果确认

**反馈类型**
```
视觉反馈：
- 按钮高亮
- 动画效果
- 状态变化

触觉反馈：
- 轻触：轻震动
- 长按：重震动
- 成功：双震动

听觉反馈：
- 点击音效
- 成功音效
```

### 4. 容错性

**原则**
- 避免误触发
- 提供撤销机会
- 渐进式触发

**设计方法**
```
1. 确认区域
   重要操作需要确认

2. 触发阈值
   滑动距离达到阈值才触发

3. 撤销功能
   支持操作撤销
```

## 手势冲突处理

### 1. 常见冲突

| 冲突场景 | 解决方案 |
|----------|----------|
| 滚动 vs 侧滑 | 优先滚动，横向位移大则侧滑 |
| 缩放 vs 双击 | 双击执行固定缩放，捏合自由缩放 |
| 点击 vs 长按 | 短按时点击，长按时长按操作 |

### 2. 冲突解决策略

```
1. 优先级规则
   - 系统手势 > 应用内手势
   - 页面手势 > 元素手势
   - 横向手势 > 纵向手势

2. 方向判定
   - 主要方向决定手势类型
   - 设置角度阈值

3. 距离判定
   - 设置触发距离阈值
   - 渐进式触发
```

## 手势设计最佳实践

### 1. 设计建议

- 使用通用手势
- 提供视觉提示
- 支持多种操作方式
- 即时反馈

### 2. 常见错误

- ❌ 手势过于复杂
- ❌ 关键功能只依赖手势
- ❌ 与系统手势冲突
- ❌ 缺乏反馈

### 3. 检查清单

- [ ] 手势是否可发现？
- [ ] 是否符合平台规范？
- [ ] 反馈是否及时？
- [ ] 是否有误触问题？
- [ ] 是否支持多种操作方式？
