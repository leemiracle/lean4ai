# 移动性能与体验

## 定义

移动性能体验是指移动应用在加载速度、响应时间、流畅度、资源消耗等方面的表现，直接影响用户的使用体验和满意度。

## 性能指标

### 1. 核心性能指标

| 指标 | 说明 | 标准 |
|------|------|------|
| 首屏时间 (FCP) | 首次内容绘制 | < 1.8s |
| 最大内容绘制 (LCP) | 最大内容绘制 | < 2.5s |
| 首次输入延迟 (FID) | 首次交互延迟 | < 100ms |
| 累积布局偏移 (CLS) | 视觉稳定性 | < 0.1 |
| 交互时间 (TTI) | 可交互时间 | < 3.8s |

### 2. 移动端特有指标

| 指标 | 说明 | 标准 |
|------|------|------|
| 启动时间 | 冷启动时间 | < 3s |
| 帧率 (FPS) | 动画流畅度 | ≥ 60fps |
| 内存占用 | 运行内存 | < 100MB |
| 电量消耗 | 功耗 | 低 |
| 流量消耗 | 数据传输 | 优化 |

## 性能优化策略

### 1. 加载优化

#### 首屏优化

```
优化策略：
1. 关键路径渲染
   - 优先加载首屏内容
   - 延迟加载非首屏内容

2. 骨架屏
   - 即时显示骨架
   - 内容逐步填充

3. 预加载
   - 预加载关键资源
   - 预连接域名
```

**骨架屏示例**
```
加载前：
┌─────────────────────────────────┐
│ ▓▓▓▓▓▓▓▓                        │
│ ▓▓▓▓▓▓                          │
│ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓            │
│ ▓▓▓▓▓▓                          │
│ ▓▓▓▓▓▓▓▓▓▓▓▓                   │
└─────────────────────────────────┘

加载后：
┌─────────────────────────────────┐
│ 标题                            │
│ 作者 · 日期                     │
│ 正文内容正文内容正文内容...     │
│ 标签                            │
│ 相关推荐                        │
└─────────────────────────────────┘
```

#### 图片优化

```
优化策略：
1. 图片压缩
   - WebP格式
   - 适当质量压缩

2. 响应式图片
   - 根据屏幕尺寸加载
   - srcset / picture

3. 懒加载
   - 图片进入视口再加载
   - 占位图展示

4. 图片缓存
   - 本地缓存
   - CDN加速
```

**懒加载示例**
```html
<img 
  src="placeholder.jpg" 
  data-src="real-image.jpg"
  loading="lazy"
  alt="懒加载图片">
```

#### 代码优化

```
优化策略：
1. 代码压缩
   - JS/CSS压缩
   - 代码混淆

2. 代码分割
   - 按路由分割
   - 按功能分割

3. Tree Shaking
   - 移除未使用代码
   - 减少包体积

4. 缓存策略
   - 静态资源缓存
   - API响应缓存
```

### 2. 渲染优化

#### 动画优化

```
原则：
- 使用 transform 和 opacity
- 避免改变布局属性
- 使用 will-change 提示

优化前：
element.style.left = '100px';  // 触发重排

优化后：
element.style.transform = 'translateX(100px)';  // 不触发重排
```

**动画性能对比**
| 属性 | 性能 | 说明 |
|------|------|------|
| transform | ✅ 好 | GPU加速 |
| opacity | ✅ 好 | GPU加速 |
| left/top | ❌ 差 | 触发重排 |
| width/height | ❌ 差 | 触发重排 |

#### 列表优化

```
优化策略：
1. 虚拟列表
   - 只渲染可见区域
   - 动态创建/销毁元素

2. 列表分页
   - 分批加载
   - 无限滚动

3. 图片懒加载
   - 滚动到时加载
   - 预加载下一屏
```

**虚拟列表示例**
```
┌─────────────────────────────────┐
│          ↑ 滚动区域             │
├─────────────────────────────────┤
│  项目 1                         │ ← 可见区域
│  项目 2                         │ ← 可见区域
│  项目 3                         │ ← 可见区域
│  项目 4                         │ ← 可见区域
├─────────────────────────────────┤
│          ↓ 滚动区域             │
│  （虚拟渲染，不实际创建DOM）    │
└─────────────────────────────────┘
```

### 3. 网络优化

#### 请求优化

```
优化策略：
1. 减少请求数
   - 合并资源
   - 图标合并（Sprite）

2. 请求压缩
   - Gzip/Brotli压缩
   - 图片压缩

3. 并发请求
   - HTTP/2多路复用
   - 合理并发控制

4. 缓存策略
   - 强缓存
   - 协商缓存
```

#### 离线优化

```
优化策略：
1. Service Worker
   - 离线缓存
   - 后台同步

2. 本地存储
   - IndexedDB
   - LocalStorage

3. 离线提示
   - 网络状态检测
   - 离线模式UI
```

### 4. 内存优化

#### 内存管理

```
优化策略：
1. 及时释放
   - 清除定时器
   - 移除事件监听
   - 释放大对象

2. 避免内存泄漏
   - 闭包问题
   - DOM引用
   - 事件监听

3. 图片内存
   - 及时释放图片
   - 限制图片数量
```

#### 列表内存

```
虚拟列表：
- 只保留可见区域的DOM
- 滚动时回收不可见元素
- 避免大量DOM节点
```

## 体验优化策略

### 1. 感知优化

**即时反馈**
```
点击反馈：
- 按钮状态变化
- 触觉反馈
- 微小动画

加载状态：
- 立即显示加载指示
- 进度条/骨架屏
```

**预加载**
```
预加载策略：
1. 预加载下一页内容
2. 预加载可能访问的页面
3. 预连接需要的域名
```

### 2. 流畅度优化

**60fps原则**
```
帧时间：1000ms / 60 = 16.67ms

每帧任务：
1. JavaScript执行
2. 样式计算
3. 布局
4. 绘制
5. 合成

总时间 < 16.67ms
```

**滚动优化**
```
优化策略：
1. 使用硬件加速
2. 避免滚动事件重计算
3. 使用 passive 事件
4. 防抖/节流

示例：
element.addEventListener('scroll', handler, {
  passive: true
});
```

### 3. 电量优化

**优化策略**
```
1. 减少定时器
   - 不必要的轮询
   - 页面不可见时暂停

2. 减少动画
   - 低电量时简化动画
   - 用户偏好设置

3. 网络优化
   - 减少请求
   - 批量传输

4. GPS优化
   - 按需定位
   - 降低精度
```

### 4. 流量优化

**优化策略**
```
1. 图片压缩
   - WebP格式
   - 适当尺寸

2. 数据压缩
   - Gzip/Brotli
   - 数据结构优化

3. 增量更新
   - 差量传输
   - 增量同步

4. 省流模式
   - 用户可控
   - 减少图片加载
```
