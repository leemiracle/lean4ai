# 移动设计规范 (iOS/Android)

## 定义

移动设计规范是指在iOS和Android平台上进行应用设计时需要遵循的设计准则和最佳实践，确保应用符合平台习惯并提供良好的用户体验。

## iOS设计规范

### 1. Human Interface Guidelines (HIG)

**设计原则**
| 原则 | 说明 |
|------|------|
| 清晰 | 文字清晰、图标精确、功能明确 |
| 依从 | 界面让位内容，流畅动效 |
| 深度 | 层次分明、视觉层级清晰 |

**设计主题**
- 清晰的视觉层次
- 流畅的交互反馈
- 精致的视觉细节
- 暗黑模式支持

### 2. iOS界面元素

#### 导航栏 (Navigation Bar)

```
┌─────────────────────────────────┐
│ ← 返回    标题           编辑   │
├─────────────────────────────────┤
│                                 │
│           内容区域              │
│                                 │
└─────────────────────────────────┘

规范：
- 高度：44pt (标准) / 96pt (大标题)
- 返回按钮：左侧，带<箭头
- 操作按钮：右侧
```

#### 标签栏 (Tab Bar)

```
┌─────────────────────────────────┐
│                                 │
│           内容区域              │
│                                 │
├─────────────────────────────────┤

│  首页  搜索  发布  消息  我的   │
└─────────────────────────────────┘

规范：
- 高度：49pt
- 项目数：3-5个
- 图标：SF Symbols或自定义
```

#### 工具栏 (Toolbar)

```
┌─────────────────────────────────┐
│                                 │
│           内容区域              │
│                                 │
├─────────────────────────────────┤
│   🗑️    📝    ↗️    ❤️        │
└─────────────────────────────────┘

规范：
- 高度：44pt
- 作用于当前页面
```

## Android设计规范

### 1. Material Design

**设计原则**
| 原则 | 说明 |
|------|------|
| Material隐喻 | 现实世界的物理特性 |
| 大胆醒目 | 鲜明的颜色、排版 |
| 意义动效 | 有目的的动画 |

- 标准高度：56dp

## iOS vs Android差异

### 1. 导航差异

| 元素 | iOS | Android |
|------|-----|---------|
| 返回 | 左上角<返回 | 顶部/底部返回 |
| 导航栏 | 顶部 | 顶部 |
| 主要导航 | 底部Tab | 底部导航 |
| 抽屉菜单 | 较少使用 | 常用 |

### 2. 交互差异

| 交互 | iOS | Android |
|------|-----|---------|
| 返回手势 | 左滑返回 | 各厂商不同 |
| 多任务 | 上滑停顿 | 方形键/手势 |
| 通知 | 下拉 | 下拉 |
| 控制中心 | 右上角下滑 | 下拉两次 |

### 3. 视觉差异

| 元素 | iOS | Android |
|------|-----|---------|
| 按钮 | 文字为主 | 大写+填充 |
| 图标 | 线性为主 | 填充为主 |
| 阴影 | 柔和 | 明显 |
| 圆角 | 持续圆角 | 超椭圆/圆角 |

### 4. 组件差异

```
iOS:
- Segmented Control (分段控制)
- Page Control (页面指示器)
- Stepper (步进器)
- Picker (选择器)

Android:
- Chips (标签)
- Snackbar (轻提示)
- Bottom Sheet (底部抽屉)
- Side Sheet (侧边抽屉)
```

## 跨平台设计策略

### 1. 统一策略

**视觉统一**
- 品牌色统一
- 图标风格统一
- 字体选择相似

**体验统一**
- 核心流程一致
- 信息架构统一
- 交互逻辑一致

### 2. 平台适配

**保持平台特性**
- 遵循各自规范
- 保持平台习惯
- 优化平台体验

**适配要点**
| 方面 | 适配内容 |
|------|----------|
| 尺寸 | 安全区域、状态栏 |
| 导航 | 返回按钮、导航模式 |
| 字体 | 系统字体、字号 |
| 图标 | 平台图标风格 |
| 手势 | 平台手势习惯 |

## 移动设计最佳实践

### 1. 通用原则

- **触摸友好**：最小点击区域44×44pt
- **性能优先**：快速加载、流畅动效
- **省电**：减少耗电操作
- **离线可用**：支持离线功能

### 2. 适配建议

- 适配各种屏幕尺寸
- 支持横竖屏切换
- 考虑刘海屏/挖孔屏
- 适配暗黑模式

### 3. 常见错误

- ❌ 违背平台规范
- ❌ 触摸目标过小
- ❌ 忽视安全区域
- ❌ 手势冲突
