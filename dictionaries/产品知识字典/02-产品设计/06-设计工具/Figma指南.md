# Figma指南

## 定义

Figma是一款基于云端的协作设计工具，支持UI设计、原型制作、设计系统管理等功能，是目前最流行的设计工具之一。

## Figma核心优势

### 1. 核心特点

| 特点 | 说明 |
|------|------|
| 云端协作 | 实时多人协作 |
| 跨平台 | 基于浏览器运行 |
| 组件化 | 强大的组件系统 |
| 自动布局 | 响应式设计支持 |
| 原型交互 | 内置原型功能 |
| 开发交付 | 自动生成代码 |

### 2. 适用场景

- UI/UX设计
- 原型制作
- 设计系统
- 团队协作
- 设计交付

## 组件系统

### 1. 组件创建

**创建组件**
```
1. 选择元素
2. 右键 → Create Component
   或快捷键 Cmd+Opt+K (Mac) / Ctrl+Alt+K (Win)
3. 组件创建后，可创建实例
```

**组件 vs 实例**
```
组件（Component）：
┌──────────┐
│  主按钮  │ ← 修改会影响所有实例
└──────────┘

实例（Instance）：
┌──────────┐ ┌──────────┐ ┌──────────┐
│  主按钮  │ │  主按钮  │ │  主按钮  │ ← 可覆盖部分属性
└──────────┘ └──────────┘ └──────────┘
```

### 2. 组件属性

**属性类型**
| 属性 | 说明 |
|------|------|
| Text | 文本内容 |
| Boolean | 显示/隐藏 |
| Instance Swap | 切换组件 |
| Variant | 变体 |

**设置属性**
```
1. 选中组件
2. 右侧面板 → Component properties
3. 添加属性
```

### 3. 组件变体

**创建变体**
```
组件变体示例：

按钮组件变体：
┌────────────────────────────────────┐
│ Button                             │
│ ┌──────────┐ ┌──────────┐         │
│ │ Primary  │ │ Primary  │         │
│ │ Default  │ │ Hover    │         │
│ └──────────┘ └──────────┘         │
│ ┌──────────┐ ┌──────────┐         │
│ │ Secondary│ │ Secondary│         │
│ │ Default  │ │ Hover    │         │
│ └──────────┘ └──────────┘         │
└────────────────────────────────────┘
```

**变体属性**
```
Type: Primary / Secondary
State: Default / Hover / Active / Disabled
Size: Small / Medium / Large
```

## 自动布局 (Auto Layout)

### 1. 基本概念

**自动布局**
```
普通Frame：
┌──────────────────┐
│ 内容随容器变化    │
└──────────────────┘

自动布局：
┌──────────┐
│ 内容     │ ← 容器随内容变化
└──────────┘
```

### 2. 设置自动布局

**快捷键**：Shift + A

**属性设置**
```
Direction（方向）：
- Horizontal（水平）
- Vertical（垂直）
- Wrap（换行）

Spacing（间距）：
- Gap between items（元素间距）
- Padding（内边距）

Alignment（对齐）：
- 主轴对齐
- 交叉轴对齐
```

### 3. 自动布局示例

**按钮自动布局**
```
┌─────────────────────────────────┐
│ [图标]  按钮文字  [图标]        │
└─────────────────────────────────┘

设置：
- Direction: Horizontal
- Gap: 8px
- Padding: 12px 16px
```

**卡片自动布局**
```
┌─────────────────────────────────┐
│ ┌─────────────────────────────┐ │
│ │                             │ │
│ │         图片                │ │
│ │                             │ │
│ └─────────────────────────────┘ │
│ 标题                            │
│ 描述文字...                     │
│ [按钮]                          │
└─────────────────────────────────┘

设置：
- Direction: Vertical
- Gap: 16px
- Padding: 16px
```

## 约束 (Constraints)

### 1. 约束类型

| 约束 | 说明 |
|------|------|
| Left | 左边距固定 |
| Right | 右边距固定 |
| Left & Right | 左右边距固定 |
| Top | 上边距固定 |
| Bottom | 下边距固定 |
| Top & Bottom | 上下边距固定 |
| Center | 居中 |
| Scale | 等比缩放 |

### 2. 约束示例

```
固定宽度按钮：
┌─────────────────────────────────┐
│                                 │
│         [居中按钮]              │ ← Center
│                                 │
└─────────────────────────────────┘

固定两侧边距：
┌─────────────────────────────────┐
│ [导航栏                      ]  │ ← Left & Right
└─────────────────────────────────┘

固定底部：
┌─────────────────────────────────┐
│                                 │
│           内容                  │
│                                 │
├─────────────────────────────────┤
│        [底部操作栏]             │ ← Bottom
└─────────────────────────────────┘
```

## 样式 (Styles)

### 1. 样式类型

| 类型 | 说明 |
|------|------|
| Color | 颜色 |
| Text | 文字 |
| Effect | 效果（阴影等） |
| Grid | 栅格 |
| Layout Grid | 布局栅格 |

### 2. 创建样式

```
1. 选择元素
2. 点击属性面板中的四个点图标
3. 选择 "Create Style"
4. 命名样式
```

### 3. 命名规范

```
推荐命名规范：

颜色：
- primary/default
- primary/hover
- gray/100
- gray/200

文字：
- heading/1
- heading/2
- body/default

效果：
- shadow/sm
- shadow/md
- shadow/lg
```

## 原型设计

### 1. 创建原型

```
1. 选择元素
2. 右侧面板切换到 Prototype
3. 点击元素边缘的 + 号
4. 拖拽连接到目标Frame
5. 设置交互类型
```

### 2. 交互类型

| 类型 | 说明 |
|------|------|
| On Click | 点击 |
| On Drag | 拖拽 |
| While Hovering | 悬停 |
| While Pressing | 按下 |
| Keyboard | 键盘 |
| Mouse Enter | 鼠标进入 |
| Mouse Leave | 鼠标离开 |

### 3. 过渡效果

| 效果 | 说明 |
|------|------|
| Instant | 瞬间 |
| Dissolve | 溶解 |

## 团队协作

### 1. 文件组织

```
团队结构：
Team
├── Project 1
│   ├── 文件 1
│   ├── 文件 2
│   └── ...
├── Project 2
│   └── ...
└── Team Library
    ├── 基础样式
    ├── 组件库
    └── 设计规范
```

### 2. 共享库

```
发布组件库：
1. 打开组件库文件
2. 点击右上角 "Publish"
3. 添加发布说明
4. 发布到团队

使用组件库：
1. 打开目标文件
2. Assets 面板
3. 点击 "Team Library"
4. 启用需要的库
```

### 3. 评论协作

```
添加评论：
1. 选择评论工具（快捷键 C）
2. 点击画布位置
3. 输入评论内容
4. @提及团队成员

回复评论：
- 直接回复
- 解决评论
```

常用设置：

## Figma最佳实践

### 1. 命名规范

```
文件命名：
- 项目_类型_日期
- 例：App_iOS_20240101

图层命名：
- 使用语义化名称
- 例：Button_Primary
- 例：Card_Product
```

### 2. 组织结构

```
页面结构：
- Cover（封面）
- Design System（设计系统）
- Components（组件）
- Screens（页面）
- Prototype（原型）
```

### 3. 常见错误

- ❌ 命名不规范
- ❌ 不使用组件
- ❌ 缺乏约束设置
- ❌ 文件组织混乱
