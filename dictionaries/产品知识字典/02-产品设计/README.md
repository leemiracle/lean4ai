## 设计思维

### 双钻模型
```
第一钻：发现和定义问题
    发散 → 收敛
     ┌──────────────┐
     │   发现问题   │ → 洞察
     └──────────────┘
          ↓
     ┌──────────────┐
     │   定义问题   │ → 聚焦
     └──────────────┘

第二钻：开发与交付方案
    发散 → 收敛
     ┌──────────────┐
     │   开发方案   │ → 创意
     └──────────────┘
          ↓
     ┌──────────────┐
     │   交付方案   │ → 实现
     └──────────────┘
```

### 用户体验五要素
```
┌─────────────────────────────────────┐
│           表现层 Surface            │  视觉设计
├─────────────────────────────────────┤
│           框架层 Skeleton           │  交互设计、信息架构
├─────────────────────────────────────┤
│           结构层 Structure          │  交互设计、信息架构
├─────────────────────────────────────┤
│           范围层 Scope              │  功能规格、内容需求
├─────────────────────────────────────┤
│           战略层 Strategy           │  用户需求、产品目标
└─────────────────────────────────────┘
```

## 设计原则速查

### 尼尔森十大可用性原则
1. **系统状态可见性** - 让用户知道发生了什么
2. **环境贴切度** - 使用用户熟悉的语言
3. **用户可控性** - 提供撤销和重做
4. **一致性** - 相同功能相同表现
5. **防错设计** - 预防错误发生
6. **识别而非回忆** - 减少记忆负担
7. **灵活高效** - 满足新手和专家
8. **审美与简约** - 界面简洁明了
9. **容错设计** - 帮助用户恢复错误
10. **帮助与文档** - 提供必要帮助

## 设计流程

### 标准设计流程
```
需求分析 → 用户研究 → 信息架构 → 交互设计 → 视觉设计 → 原型测试 → 开发跟进 → 上线验收
```

### 设计评审要点
1. **是否满足用户需求**
2. **是否符合设计规范**
3. **交互是否流畅**
4. **视觉是否统一**
5. **技术是否可实现**
6. **是否考虑边界情况**
