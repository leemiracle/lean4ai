# 色彩理论与色彩心理学

## 定义

色彩理论是研究颜色如何相互作用、如何影响人类感知和情绪的学科。在设计中，理解色彩理论有助于创造有效的视觉体验和情感连接。

## 色彩基础

### 1. 色彩三要素

#### 色相 (Hue)

**定义**
颜色的基本属性，即颜色的名称。

**色相环**
```
         红
        /│\
       / │ \
      /  │  \
   紫红  │   橙
     │   │   │
  紫 ─────┼───── 黄
     │   │   │
   蓝紫  │   黄橙
      \  │  /
       \ │ /
        \│/
         蓝
```

**原色、间色、复色**
| 类型 | 颜色 |
|------|------|
| 原色 (Primary) | 红、黄、蓝 |
| 间色 (Secondary) | 橙、绿、紫 |
| 复色 (Tertiary) | 红橙、黄橙、黄绿、蓝绿、蓝紫、红紫 |

#### 饱和度 (Saturation)

**定义**
颜色的纯度或强度。

```
高饱和度：■■■ 鲜艳、强烈
中饱和度：■■■ 柔和、温和
低饱和度：■■■ 灰暗、沉静
```

#### 明度 (Value/Brightness)

**定义**
颜色的明暗程度。

```
高明度：■■■ 明亮、轻盈
中明度：■■■ 中等
低明度：■■■ 深沉、厚重
```

### 2. 色彩模式

#### RGB模式

**定义**
通过红(R)、绿(G)、蓝(B)三色光叠加显示颜色。

**适用场景**
- 屏幕显示
- Web设计
- 数字产品

```
R(255,0,0) + G(0,255,0) + B(0,0,255) = 白色
R(0,0,0) + G(0,0,0) + B(0,0,0) = 黑色
```

#### CMYK模式

**定义**
通过青(C)、品红(M)、黄(Y)、黑(K)四色油墨混合显示颜色。

**适用场景**
- 印刷品
- 包装设计
- 平面设计

#### HSB/HSV模式

**定义**
通过色相(H)、饱和度(S)、明度(B/V)描述颜色。

**优点**
- 更符合人类对颜色的认知
- 便于调色

### 3. 十六进制颜色码

```
格式：#RRGGBB

示例：
#FF0000 = 红色 (R:255, G:0, B:0)
#00FF00 = 绿色 (R:0, G:255, B:0)
#0000FF = 蓝色 (R:0, G:0, B:255)
#FFFFFF = 白色 (R:255, G:255, B:255)
#000000 = 黑色 (R:0, G:0, B:0)
```

## 配色方案

### 1. 单色配色 (Monochromatic)

**定义**
使用同一色相的不同明度和饱和度变化。

```
示例：
█████████
深蓝、蓝、浅蓝
```

**特点**
- 简洁、和谐
- 易于使用
- 适合需要统一感的场景

### 2. 类似色配色 (Analogous)

**定义**
使用色相环上相邻的颜色。

```
示例：
选择：蓝
类似色：蓝紫、蓝、蓝绿

    █████████
```

**特点**
- 自然、和谐
- 丰富但不杂乱
- 适合自然主题

### 3. 互补色配色 (Complementary)

**定义**
使用色相环上相对的两种颜色。

```
示例：
互补色对：
红 - 绿
蓝 - 橙
黄 - 紫

████  ████
```

**特点**
- 对比强烈
- 引人注目
- 需要谨慎使用比例

### 4. 三角配色 (Triadic)

**定义**
使用色相环上等距的三种颜色。

```
示例：
红 - 黄 - 蓝
橙 - 绿 - 紫

████  ████  ████
```

**特点**
- 丰富、活力
- 需要控制比例
- 主色 + 辅色 + 点缀色

### 5. 分裂互补配色 (Split-Complementary)

**定义**
一个颜色 + 其互补色两侧的颜色。

```
示例：
选择：蓝
配色：蓝 + 黄橙 + 红橙

████  ████  ████
```

**特点**
- 对比有趣
- 比互补色柔和
- 适合初学者

### 6. 四角配色 (Tetradic/Square)

**定义**
使用色相环上等距的四种颜色。

```
示例：
红、黄橙、绿、蓝紫

████  ████  ████  ████
```

**特点**
- 最丰富
- 难以平衡
- 需要一个主色

## 配色比例法则

### 60-30-10 法则

```
60% - 主色（背景、大面积区域）
30% - 辅助色（次要元素）
10% - 强调色（按钮、链接）

示例：
┌─────────────────────────────────┐
│                                 │
│    60% 主色区域                 │ ← 大面积
│                                 │
│   ┌──────────┐  ┌──────────┐   │
│   │  30%     │  │  辅助色  │   │ ← 中等面积
│   └──────────┘  └──────────┘   │
│                                 │
│              [10%]              │ ← 小面积强调
└─────────────────────────────────┘
```

## 色彩心理学

### 1. 颜色情感联想

| 颜色 | 正面联想 | 负面联想 | 典型应用 |
|------|----------|----------|----------|
| 红色 | 热情、能量、爱情 | 危险、警告、愤怒 | 促销、餐饮、警告 |
| 橙色 | 活力、创意、温暖 | 廉价、不稳重 | 创意、年轻品牌 |
| 黄色 | 快乐、希望、智慧 | 警告、欺骗 | 提示、儿童产品 |
| 绿色 | 自然、健康、成长 | 嫉妒、毒性 | 环保、健康、金融 |
| 蓝色 | 信任、专业、冷静 | 忧郁、冷漠 | 科技、金融、企业 |
| 紫色 | 高贵、神秘、创意 | 奢侈、不真实 | 美妆、奢侈品 |
| 粉色 | 浪漫、温柔、女性 | 幼稚、软弱 | 女性产品、婚礼 |
| 黑色 | 高端、权威、神秘 | 死亡、压抑 | 奢侈品、科技 |
| 白色 | 纯洁、简洁、空间 | 空虚、冷漠 | 极简设计、医疗 |
| 灰色 | 中性、专业、平衡 | 沉闷、无趣 | 商务、科技 |

### 2. 行业色彩偏好

| 行业 | 常用色 | 原因 |
|------|--------|------|
| 金融 | 蓝、深蓝 | 信任、稳定 |
| 科技 | 蓝、黑、白 | 专业、创新 |
| 餐饮 | 红、橙、黄 | 食欲、温暖 |
| 医疗 | 蓝、白、绿 | 干净、信任 |
| 环保 | 绿、白、蓝 | 自然、纯净 |
| 奢侈品 | 黑、金、紫 | 高贵、稀缺 |
| 儿童 | 彩色、明快 | 活泼、快乐 |

### 3. 颜色与转化

**按钮颜色测试**
```
常见按钮颜色效果排名：
1. 红色/橙色 - 高对比度，行动导向
2. 绿色 - 积极、继续
3. 蓝色 - 信任、安全
4. 灰色 - 禁用状态
```

**案例：蓝色 vs 绿色按钮**
```
A/B测试结果（示例）：
绿色"立即购买"按钮转化率：3.2%
蓝色"立即购买"按钮转化率：2.8%
结论：绿色更符合"行动"的心理暗示
```

## 产品设计中的色彩应用

### 1. 品牌色

**定义**
代表品牌识别的核心颜色。

**示例**
| 品牌 | 主色 | 色值 |
|------|------|------|
| 微信 | 绿色 | #07C160 |
| 支付宝 | 蓝色 | #1677FF |
| 淘宝 | 橙色 | #FF5000 |
| 京东 | 红色 | #E2231A |
| 美团 | 黄色 | #FFC300 |

### 2. 功能色

**定义**
用于特定功能状态的颜色。

```css
/* 通用功能色 */
--success: #52c41a;   /* 成功 - 绿色 */
--warning: #faad14;   /* 警告 - 黄色 */
--error: #f5222d;     /* 错误 - 红色 */
--info: #1890ff;      /* 信息 - 蓝色 */
```

**应用示例**
```
✓ 成功：保存成功、支付成功、操作完成
⚠ 警告：注意事项、即将过期、风险提示
✗ 错误：操作失败、表单错误、系统异常
ℹ 信息：提示信息、帮助说明、新消息
```

### 3. 中性色

**定义**
用于文字、边框、背景的灰度色系。

```css
/* 中性色系统 */
--gray-1: #ffffff;    /* 白色 */
--gray-2: #fafafa;    /* 背景灰 */
--gray-3: #f5f5f5;    /* 边框灰 */
--gray-4: #e8e8e8;    /* 分割线 */
--gray-5: #d9d9d9;    /* 禁用边框 */
--gray-6: #bfbfbf;    /* 禁用文字 */
--gray-7: #8c8c8c;    /* 辅助文字 */
--gray-8: #595959;    /* 次要文字 */
--gray-9: #262626;    /* 主要文字 */
--gray-10: #000000;   /* 黑色 */
```

### 4. 渐变色

**线性渐变**
```css
background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
```

**径向渐变**
```css
background: radial-gradient(circle, #667eea, #764ba2);
```

**应用场景**
- 按钮
- 背景
- 图标
- 头像边框

## 无障碍色彩设计

### 1. 对比度要求

**WCAG 2.1 标准**

| 级别 | 正文文本 | 大文本 |
|------|----------|--------|
| AA | 4.5:1 | 3:1 |
| AAA | 7:1 | 4.5:1 |

**大文本定义**
- 粗体 ≥ 14px
- 常规 ≥ 18px

### 2. 对比度检查

```css
/* 示例：检查通过 */
background: #ffffff;
color: #262626;
对比度: 13.5:1 ✓ AAA级别

/* 示例：检查不通过 */
background: #ffffff;
color: #bfbfbf;
对比度: 2.3:1 ✗ 不符合标准
```

### 3. 色盲友好设计

**色盲类型**
- 红绿色盲（最常见）
- 蓝黄色盲
- 全色盲

**设计建议**
- 不要仅靠颜色传达信息
- 使用图标 + 颜色
- 使用图案 + 颜色
- 提供高对比模式

```
✓ 使用颜色 + 图标
成功：✓ 绿色勾
错误：✗ 红色叉
警告：⚠ 黄色叹号

✗ 仅使用颜色
成功：绿色文字
错误：红色文字
（色盲用户可能无法区分）
```

## 最佳实践

### 1. 配色建议

- 限制颜色数量（2-3种主色）
- 使用60-30-10法则
- 确保足够的对比度
- 考虑无障碍设计

### 2. 常见错误

- ❌ 颜色过多过杂
- ❌ 对比度不足
- ❌ 忽视色盲用户
- ❌ 违背行业惯例

### 3. 配色流程

```
1. 确定品牌调性
   ↓
2. 选择主色
   ↓
3. 确定配色方案
   ↓
4. 定义功能色和中性色
   ↓
5. 检查对比度
   ↓
6. 建立色彩系统文档
```
