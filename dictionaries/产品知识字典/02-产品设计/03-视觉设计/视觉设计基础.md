# 视觉设计基础

## 定义

视觉设计是通过视觉元素（色彩、形状、文字、图像等）传递信息、创造美感和建立品牌识别的设计活动。在产品设计中，视觉设计是用户体验的重要组成部分。

## 视觉设计要素

### 1. 点、线、面

**点 (Point)**
- 最基本的视觉元素
- 可以是圆点、方块、图标
- 用于标记、装饰、引导

**线 (Line)**
- 连接两个点的路径
- 用于分割、引导、装饰
- 线条类型：实线、虚线、曲线

**面 (Plane)**
- 由线条围合而成的区域
- 用于区块划分、背景、容器
- 如：卡片、按钮、面板

### 2. 形状 (Shape)

| 形状 | 特点 | 应用 |
|------|------|------|
| 圆形 | 友好、柔和、完整 | 头像、按钮、图标 |
| 方形 | 稳定、专业、有序 | 卡片、容器、框架 |
| 三角 | 动感、方向、警示 | 箭头、指示器 |
| 不规则 | 独特、创意、艺术 | 品牌图形、装饰 |

### 3. 颜色 (Color)

**色彩三要素**
- **色相 (Hue)**：颜色种类（红、橙、黄...）
- **饱和度 (Saturation)**：颜色纯度
- **明度 (Value)**：颜色明暗

**色彩心理**
| 颜色 | 情感联想 | 应用场景 |
|------|----------|----------|
| 红色 | 热情、危险、重要 | 警告、促销、强调 |
| 橙色 | 活力、温暖、创意 | 行动号召、年轻品牌 |
| 黄色 | 快乐、警示、希望 | 提示、亮点、品牌 |
| 绿色 | 自然、健康、成功 | 成功状态、环保、健康 |
| 蓝色 | 信任、专业、冷静 | 科技、金融、企业 |
| 紫色 | 高贵、神秘、创意 | 奢侈品、创意行业 |
| 黑色 | 高端、正式、权威 | 奢侈品、科技产品 |
| 白色 | 纯洁、简洁、空间 | 极简设计、留白 |

### 4. 文字 (Typography)

**字体分类**
| 分类 | 特点 | 示例 |
|------|------|------|
| 衬线体 | 传统、正式、优雅 | Times New Roman、宋体 |
| 无衬线体 | 现代、简洁、友好 | Helvetica、苹方、思源黑体 |
| 等宽字体 | 技术感、代码 | Consolas、Monaco |
| 手写体 | 个性、艺术、非正式 | 手写风格字体 |

**字体层级**
```
一级标题：32-48px / Bold
二级标题：24-32px / Semi-bold
三级标题：18-24px / Medium
正文：14-16px / Regular
辅助文字：12-14px / Light
```

### 5. 空间 (Space)

**空间类型**
- **正空间**：实体元素占据的空间
- **负空间（留白）**：元素之间的空白区域

**留白作用**
- 提高可读性
- 突出重点内容
- 创造呼吸感
- 提升品质感

**间距系统**
```
基础单位：8px
├── xs: 4px (0.5单位)
├── sm: 8px (1单位)
├── md: 16px (2单位)
├── lg: 24px (3单位)
├── xl: 32px (4单位)
└── xxl: 48px (6单位)
```

### 6. 图像 (Image)

**图像类型**
- 照片：真实感、情感连接
- 插画：独特风格、品牌调性
- 图标：功能指示、视觉引导
- 背景图：氛围营造、视觉冲击

**图像使用原则**
- 高质量：清晰、无水印
- 相关性：与内容匹配
- 一致性：风格统一
- 优化：压缩大小、适当尺寸

## 视觉设计原则

### 1. 对比 (Contrast)

**定义**
通过差异创造视觉兴趣和层次。

**对比方式**
| 类型 | 示例 |
|------|------|
| 大小对比 | 大标题 + 小正文 |
| 颜色对比 | 深色背景 + 浅色文字 |
| 粗细对比 | 粗体标题 + 细体正文 |
| 形状对比 | 圆形头像 + 方形卡片 |

```
✓ 有对比
┌─────────────────────────────────┐
│                                 │
│      重要大标题                 │ ← 大、粗
│                                 │
│   正文内容正文内容正文内容      │ ← 小、细
│                                 │
└─────────────────────────────────┘

✗ 无对比
┌─────────────────────────────────┐
│                                 │
│      标题                       │
│                                 │
│   正文内容正文内容正文内容      │
│                                 │
└─────────────────────────────────┘
（标题和正文字号太接近）
```

### 2. 重复 (Repetition)

**定义**
在设计中重复使用某些元素，增强一致性和识别度。

**重复元素**
- 颜色
- 字体
- 间距
- 图标风格
- 组件样式

### 3. 对齐 (Alignment)

**定义**
元素之间建立视觉连接，创造秩序感。

**对齐方式**
```
左对齐：
┌─────────────────────────────────┐
│ 标题                            │
│ 正文内容                        │
│ 更多信息                        │
└─────────────────────────────────┘

居中对齐：
┌─────────────────────────────────┐
│              标题               │
│           正文内容              │
│           更多信息              │
└─────────────────────────────────┘

两端对齐：
┌─────────────────────────────────┐
│ 标题                            │
│ 正文内容正文内容正文内容正      │
│ 文内容正文内容正文内容          │
└─────────────────────────────────┘
```

### 4. 亲密性 (Proximity)

**定义**
相关的元素靠近放置，不相关的元素分开。

```
✓ 亲密性良好
┌─────────────────────────────────┐
│ 联系信息                        │
│ ─────────────────────────       │
│ 姓名：张三                      │
│ 电话：13800138000               │
│                                 │ ← 空白分隔
│ 地址信息                        │
│ ─────────────────────────       │
│ 省份：北京                      │
│ 城市：北京市                    │
└─────────────────────────────────┘
```

## 视觉层次

### 1. 层次建立方式

| 方式 | 效果 | 应用 |
|------|------|------|
| 大小 | 越大越重要 | 标题 > 正文 |
| 颜色 | 越显眼越重要 | 强调色 > 灰色 |
| 位置 | 越靠前越重要 | 顶部 > 底部 |
| 粗细 | 越粗越重要 | 粗体 > 细体 |
| 留白 | 周围空白越多越重要 | 孤立元素 |

### 2. 层次示例

```
视觉层次从高到低：
┌─────────────────────────────────┐
│                                 │
│         页面主标题              │ ← 最大、最粗、最突出
│                                 │
│      页面副标题/描述            │ ← 中等大小、次要颜色
│                                 │
│   ─────────────────────         │
│                                 │
│   正文内容正文内容正文内容      │ ← 常规大小、常规颜色
│   正文内容正文内容正文内容      │
│                                 │
│   辅助信息 / 时间 / 作者       │ ← 最小、最浅颜色
│                                 │
└─────────────────────────────────┘
```

## 栅格系统

### 1. 栅格概念

**定义**
使用规律的列和间距来组织内容，确保布局的一致性和灵活性。

**基本组成**
```
┌─────────────────────────────────────────────────────┐
│ Margin │  Column  │ Gutter │  Column  │   Margin   │
│        │          │        │          │            │
└─────────────────────────────────────────────────────┘

Margin: 页边距
Column: 列
Gutter: 列间距
```

### 2. 常见栅格系统

| 栅格 | 适用场景 |
|------|----------|
| 12列 | 灵活性高，最常用 |
| 16列 | 更精细的控制 |
| 24列 | 复杂布局 |

### 3. 响应式断点

```
移动端：      0 - 576px
平板：        576px - 768px
小屏桌面：    768px - 992px
桌面：        992px - 1200px
大屏：        1200px+
```

## 视觉风格

### 1. 风格类型

| 风格 | 特点 | 代表 |
|------|------|------|
| 极简 | 简洁、留白、少元素 | Apple、Google |
| 扁平 | 无阴影、纯色、简洁图标 | Microsoft Metro |
| 拟物 | 模拟真实材质、阴影 | 早期iOS |
| 玻璃态 | 模糊、透明、层次 | iOS/macOS |
| 渐变 | 色彩过渡、动感 | 现代Web设计 |
| 插画风 | 手绘风格、个性 | Notion、Slack |

### 2. 风格选择考量

- 品牌调性
- 目标用户
- 产品类型
- 行业惯例

## 最佳实践

### 1. 设计建议

- 建立清晰的视觉层次
- 保持设计一致性
- 合理使用留白
- 注意颜色对比度（WCAG标准）

### 2. 常见错误

- ❌ 层次不清晰
- ❌ 元素过多过杂
- ❌ 颜色使用不当
- ❌ 对齐不规范
- ❌ 间距不统一

### 3. 设计检查清单

- [ ] 视觉层次是否清晰？
- [ ] 颜色是否协调？
- [ ] 间距是否一致？
- [ ] 对齐是否准确？
- [ ] 字体使用是否规范？
- [ ] 是否有足够的留白？
