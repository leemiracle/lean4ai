# 排版设计 (Typography)

## 定义

排版设计是组织和呈现文字信息的艺术，通过字体、字号、行高、间距等属性的合理运用，使内容易于阅读且视觉美观。

## 字体基础

### 1. 字体分类

#### 衬线体 (Serif)

**特点**
- 笔画末端有装饰（衬线）
- 传统、正式、优雅
- 适合长文本阅读

**常见字体**
- 中文字体：宋体、楷体
- 英文字体：Times New Roman、Georgia

```
衬线体示例：
| 横 | 横  ← 有装饰
└───┘
```

#### 无衬线体 (Sans-serif)

**特点**
- 笔画末端无装饰
- 现代、简洁、友好
- 适合屏幕显示

**常见字体**
- 中文字体：苹方、思源黑体、微软雅黑
- 英文字体：Helvetica、Arial、Roboto

```
无衬线体示例：
| 横 | 横  ← 无装饰
```

#### 等宽字体 (Monospace)

**特点**
- 每个字符宽度相同
- 技术感、代码风格
- 适合代码展示

**常见字体**
- Consolas、Monaco、Courier New

#### 手写体/装饰体

**特点**
- 个性化强
- 艺术感
- 适合标题、品牌

### 2. 中文字体选择

**系统默认字体栈**
```css
/* macOS/iOS */
font-family: -apple-system, BlinkMacSystemFont;

/* Windows */
font-family: "Microsoft YaHei";

/* Android */
font-family: "Roboto", "Noto Sans SC";

/* 通用字体栈 */
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, 
  "Helvetica Neue", Arial, "Noto Sans", "PingFang SC", 
  "Microsoft YaHei", sans-serif;
```

**推荐中文字体**
| 字体 | 类型 | 特点 |
|------|------|------|
| 苹方 | 无衬线 | 苹果系统，简洁现代 |
| 思源黑体 | 无衬线 | 开源免费，多字重 |
| 思源宋体 | 衬线 | 开源免费，优雅 |
| 微软雅黑 | 无衬线 | Windows系统 |
| 阿里巴巴普惠体 | 无衬线 | 开源免费 |

### 3. 英文字体选择

**推荐英文字体**
| 字体 | 类型 | 特点 |
|------|------|------|
| Inter | 无衬线 | 屏幕 optimized，开源 |

| Helvetica | 无衬线 | 经典，中性 |
| Georgia | 衬线 | 屏幕友好 |

## 字体属性

### 1. 字号 (Font Size)

**常用字号系统**
```css
--font-size-xs: 12px;    /* 辅助信息 */
--font-size-sm: 14px;    /* 次要内容 */
--font-size-base: 16px;  /* 正文 */
--font-size-lg: 18px;    /* 强调 */
--font-size-xl: 20px;    /* 小标题 */
--font-size-2xl: 24px;   /* 三级标题 */
--font-size-3xl: 30px;   /* 二级标题 */
--font-size-4xl: 36px;   /* 一级标题 */
--font-size-5xl: 48px;   /* 大标题 */
```

**字号层级**
```
48px ────────────────────── 大标题
36px ────────────────────── 一级标题
30px ────────────────────── 二级标题
24px ────────────────────── 三级标题
20px ────────────────────── 小标题
16px ────────────────────── 正文
14px ────────────────────── 次要文字
12px ────────────────────── 辅助信息
```

### 2. 字重 (Font Weight)

**字重等级**
| 值 | 名称 | CSS |
|----|------|-----|
| 100 | Thin | font-weight: 100 |
| 200 | Extra Light | font-weight: 200 |
| 300 | Light | font-weight: 300 |
| 400 | Regular | font-weight: 400 |
| 500 | Medium | font-weight: 500 |
| 600 | Semi Bold | font-weight: 600 |
| 700 | Bold | font-weight: 700 |
| 800 | Extra Bold | font-weight: 800 |
| 900 | Black | font-weight: 900 |

**常用字重**
```css
--font-weight-light: 300;     /* 辅助文字 */
--font-weight-regular: 400;   /* 正文 */
--font-weight-medium: 500;    /* 强调 */
--font-weight-semibold: 600;  /* 小标题 */
--font-weight-bold: 700;      /* 标题 */
```

### 3. 行高 (Line Height)

**行高比例**
```css
--line-height-tight: 1.25;    /* 标题 */
--line-height-normal: 1.5;    /* 正文 */
--line-height-loose: 1.75;    /* 长文本 */
```

**行高示例**
```
行高1.25（紧凑）：
标题文字标题文字标题文字
标题文字标题文字标题文字

行高1.5（正常）：
正文内容正文内容正文内容
正文内容正文内容正文内容

行高1.75（宽松）：
长文本内容长文本内容长文本内容
长文本内容长文本内容长文本内容
```

**行高建议**
| 内容类型 | 建议行高 |
|----------|----------|
| 大标题 | 1.0 - 1.25 |
| 小标题 | 1.25 - 1.35 |
| 正文 | 1.5 - 1.65 |
| 长文本 | 1.65 - 1.8 |

### 4. 字间距 (Letter Spacing / Tracking)

```css
letter-spacing: -0.05em;  /* 紧凑 */
letter-spacing: 0;        /* 正常 */
letter-spacing: 0.05em;   /* 宽松 */
letter-spacing: 0.1em;    /* 很宽松 */
```

**应用建议**
- 大标题：略微紧凑
- 正文：正常或略宽松
- 全大写文字：略微宽松

### 5. 词间距 (Word Spacing)

```css
word-spacing: normal;   /* 正常 */
word-spacing: 0.1em;    /* 宽松 */
word-spacing: 0.25em;   /* 很宽松 */
```

## 段落排版

### 1. 段落宽度

**最佳阅读宽度**
```
最佳字符数：45-75字符/行（英文）
最佳字符数：35-50字符/行（中文）

┌─────────────────────────────────┐
│ 正好的宽度：                    │
│ 这是一段正文内容，宽度适中，    │
│ 阅读起来比较舒适。每行约       │
│ 35-50个汉字比较合适。          │
└─────────────────────────────────┘
```

**过宽的问题**
```
┌───────────────────────────────────────────────────────────┐
│ 太宽：这是一段正文内容，宽度太宽了，阅读时眼睛需要移动很大  │
│ 的距离，容易造成阅读疲劳，用户可能很难找到下一行的开头。    │
└───────────────────────────────────────────────────────────┘
```

**过窄的问题**
```
┌─────────────┐
│ 太窄：      │
│ 这是正文    │
│ 内容，宽    │
│ 度太窄了    │
└─────────────┘
```

### 2. 段落对齐

**左对齐（推荐）**
```
┌─────────────────────────────────┐
│ 这是一段左对齐的文字，          │
│ 左边对齐，右边自然结束。        │
│ 这是最常用的对齐方式，          │
│ 阅读体验最好。                  │
│ 左对齐更适合大多数场景。        │
└─────────────────────────────────┘
```

**两端对齐**
```
┌─────────────────────────────────┐
│ 这 是 一 段 两 端 对 齐 的 文 字│
│ 左右两边都对齐。两端对齐        │
│ 可能会造成词间距不均匀。        │
└─────────────────────────────────┘
```

**居中对齐**
```
┌─────────────────────────────────┐
│       这是一段居中文字          │
│       适合标题或短文本          │
│       不适合长文本阅读          │
└─────────────────────────────────┘
```

### 3. 段落间距

```css
/* 段落间距 > 行间距 */
p {
  margin-bottom: 1.5em;  /* 段落间距 */
  line-height: 1.6;      /* 行间距 */
}
```

```
正确示例：

这是第一段内容，段落之间
有明显的间距。

这是第二段内容，与上一段
保持适当距离。

错误示例：
这是第一段内容。
这是第二段内容。
（段落间距太小，难以区分）
```

## 排版层级

### 1. 建立层级

**方式**
| 方式 | 效果 |
|------|------|
| 字号 | 大 > 小 |
| 字重 | 粗 > 细 |
| 颜色 | 深 > 浅 |
| 间距 | 宽 > 窄 |

**层级示例**
```
┌─────────────────────────────────┐
│                                 │
│      页面主标题（36px Bold）    │
│                                 │
│   副标题说明文字（18px Medium） │
│                                 │
│   ─────────────────────────     │
│                                 │
│   正文内容正文内容正文内容      │
│   正文内容正文内容正文内容      │
│   （16px Regular）              │
│                                 │
│   辅助信息（14px Light 灰色）   │
│                                 │
└─────────────────────────────────┘
```

### 2. 层级比例

**类型缩放比例 (Type Scale)**
```
1.125 (Major Second)：
14 → 16 → 18 → 20 → 22.5 → 25.3 → 28.5

1.25 (Major Third)：
14 → 17.5 → 21.8 → 27.3 → 34.2

1.333 (Perfect Fourth)：
12 → 16 → 21.3 → 28.4 → 37.9

1.5 (Perfect Fifth)：
12 → 18 → 27 → 40.5 → 60.75
```

## 响应式排版

### 1. 字号适配

```css
/* 使用 clamp() 实现流体字号 */
h1 {
  font-size: clamp(24px, 5vw, 48px);
}

/* 响应式字号 */
h1 {
  font-size: 24px;
}

@media (min-width: 768px) {
  h1 {
    font-size: 36px;
  }
}

@media (min-width: 1024px) {
  h1 {
    font-size: 48px;
  }
}
```

### 2. 行宽控制

```css
/* 限制最大宽度 */
.content {
  max-width: 65ch; /* 约65个字符 */
  margin: 0 auto;
}

/* 或使用固定宽度 */
.content {
  max-width: 600px;
  margin: 0 auto;
}
```

### 3. 断点设计

```css
/* 移动端优先 */
body {
  font-size: 16px;
  line-height: 1.5;
}

/* 平板 */
@media (min-width: 768px) {
  body {
    font-size: 18px;
    line-height: 1.6;
  }
}

/* 桌面 */
@media (min-width: 1024px) {
  body {
    font-size: 18px;
    line-height: 1.65;
  }
}
```

## 特殊排版

### 1. 数字排版

**等宽数字 (Tabular Figures)**
```css
.numbers {
  font-variant-numeric: tabular-nums;
}
```

```
普通数字：
1,234
9,876  （对齐不整齐）

等宽数字：
1,234
9,876  （完美对齐）
```

### 2. 列表排版

```css
ul, ol {
  padding-left: 1.5em;
  line-height: 1.6;
}

li {
  margin-bottom: 0.5em;
}
```

### 3. 引用排版

```css
blockquote {
  padding-left: 1em;
  border-left: 3px solid #ccc;
  font-style: italic;
  color: #666;
}
```

### 4. 代码排版


## 排版规范示例

### 完整排版系统

```css
:root {
  /* 字体 */
  --font-family-base: -apple-system, BlinkMacSystemFont, "PingFang SC", 
    "Microsoft YaHei", sans-serif;
  --font-family-code: "Consolas", "Monaco", monospace;
  
  /* 字号 */
  --font-size-xs: 12px;
  --font-size-sm: 14px;
  --font-size-base: 16px;
  --font-size-lg: 18px;
  --font-size-xl: 20px;
  --font-size-2xl: 24px;
  --font-size-3xl: 30px;
  --font-size-4xl: 36px;
  --font-size-5xl: 48px;
  
  /* 行高 */
  --line-height-tight: 1.25;
  --line-height-base: 1.5;
  --line-height-loose: 1.75;
  
  /* 字重 */
  --font-weight-light: 300;
  --font-weight-regular: 400;
  --font-weight-medium: 500;
  --font-weight-semibold: 600;
  --font-weight-bold: 700;
}

/* 标题 */
h1 { font-size: var(--font-size-5xl); font-weight: var(--font-weight-bold); }
h2 { font-size: var(--font-size-4xl); font-weight: var(--font-weight-bold); }
h3 { font-size: var(--font-size-3xl); font-weight: var(--font-weight-semibold); }
h4 { font-size: var(--font-size-2xl); font-weight: var(--font-weight-semibold); }
h5 { font-size: var(--font-size-xl); font-weight: var(--font-weight-medium); }
h6 { font-size: var(--font-size-lg); font-weight: var(--font-weight-medium); }

/* 正文 */
body {
  font-family: var(--font-family-base);
  font-size: var(--font-size-base);
  line-height: var(--line-height-base);
  color: #333;
}
```

## 排版检查清单

- [ ] 字体选择符合品牌调性
- [ ] 字号层级清晰
- [ ] 行高适合阅读
- [ ] 段落宽度适中
- [ ] 对齐方式一致
- [ ] 间距系统规范
- [ ] 响应式适配良好
- [ ] 对比度足够

## 最佳实践

### 1. 排版建议

- 限制字体数量（2-3种）
- 建立清晰的层级
- 保持一致性
- 优化阅读体验

### 2. 常见错误

- ❌ 字体过多
- ❌ 层级不清晰
- ❌ 行高不合适
- ❌ 段落过宽
- ❌ 对齐不一致
