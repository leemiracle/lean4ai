# 无障碍设计 (Accessibility)

## 定义

无障碍设计（Accessibility，简称A11y）是指让产品、服务和环境能够被尽可能多的人使用，包括残障人士、老年人和临时受限人群。

## 无障碍设计的重要性

### 1. 用户范围

**受益人群**
| 人群 | 类型 | 需求 |
|------|------|------|
| 视觉障碍 | 盲人、低视力 | 屏幕阅读器、高对比度 |
| 听觉障碍 | 聋人、听力受损 | 字幕、视觉提示 |
| 运动障碍 | 肢体残疾 | 键盘操作、语音控制 |
| 认知障碍 | 学习障碍、记忆问题 | 简单清晰、一致性 |
| 临时受限 | 手臂受伤、嘈杂环境 | 替代操作方式 |

### 2. 商业价值

- 扩大用户群体
- 法律合规要求
- 提升品牌形象
- 改善所有用户体验

## WCAG标准

### 1. WCAG原则 (POUR)

| 原则 | 英文 | 说明 |
|------|------|------|
| 可感知 | Perceivable | 信息能被感知 |
| 可操作 | Operable | 界面能被操作 |
| 可理解 | Understandable | 内容能被理解 |
| 健壮 | Robust | 内容兼容性强 |

### 2. WCAG等级

| 等级 | 说明 | 要求 |
|------|------|------|
| A | 最低要求 | 必须满足 |
| AA | 推荐标准 | 大多数法规要求 |
| AAA | 最高标准 | 最佳实践 |

### 3. 核心要求

#### 可感知 (Perceivable)

**1.1 文本替代**
- 图片需要alt属性
- 图标需要文本标签
- 复杂图像需要长描述

```html
<!-- 简单图片 -->
<img src="logo.png" alt="公司Logo">

<!-- 装饰性图片 -->
<img src="decoration.png" alt="">

<!-- 复杂图片 -->
<img src="chart.png" alt="销售数据图表" 
     aria-describedby="chart-desc">
<p id="chart-desc">图表详细描述...</p>
```

**1.2 时基媒体**
- 视频需要字幕
- 音频需要文本
- 提供替代内容

**1.3 适应性**
- 信息不依赖单一感官
- 结构化标记
- 有意义的顺序

**1.4 可辨别**
- 颜色对比度足够
- 文本可调整大小
- 文字图片可替代

#### 可操作 (Operable)

**2.1 键盘可访问**
- 所有功能可键盘操作
- 无键盘陷阱
- 焦点顺序合理

```
Tab顺序：
┌─────────────────────────────────┐
│ [1] [2]          [3]    [4]    │
│                                 │
│        [5]                      │
│                                 │
│ [6] [7] [8]                     │
└─────────────────────────────────┘
```

**2.2 充足时间**
- 提供足够操作时间
- 可暂停/停止动画
- 可延长时限

**2.3 癫痫发作预防**
- 避免闪烁内容
- 限制闪光频率

**2.4 导航性**
- 提供多种导航方式
- 位置指示清晰
- 链接目的明确

#### 可理解 (Understandable)

**3.1 可读性**
- 页面语言明确
- 专业术语解释
- 阅读水平适当

**3.2 可预测性**
- 导航一致
- 交互一致
- 避免意外变化

**3.3 输入辅助**
- 错误识别
- 错误建议
- 错误预防

#### 健壮 (Robust)

**4.1 兼容性**
- 标记语言正确
- 状态变化通知
- 辅助技术兼容

## 视觉无障碍

### 1. 颜色对比度

**对比度要求**
| 内容类型 | AA级 | AAA级 |
|----------|------|-------|
| 正文文本 | 4.5:1 | 7:1 |
| 大文本 | 3:1 | 4.5:1 |
| UI组件 | 3:1 | - |

**对比度计算**
```
示例：
前景色：#595959
背景色：#FFFFFF
对比度：7.03:1 ✓ AAA级别

前景色：#8C8C8C
背景色：#FFFFFF
对比度：3.34:1 ✗ 不符合正文要求
```

### 2. 颜色使用

**原则**
```
✓ 正确：
- 颜色 + 图标/文字
- 多种方式传达信息

✗ 错误：
- 仅用颜色区分
- 红绿色盲无法区分
```

**示例**
```
错误提示方式：

✗ 仅颜色：
┌─────────────────────────────────┐
│ 手机号                          │
│ [13800138                    ] │ ← 红色边框
└─────────────────────────────────┘

✓ 颜色+图标+文字：
┌─────────────────────────────────┐
│ 手机号                          │
│ [13800138                    ] │ ← 红色边框
│ ✗ 请输入11位手机号             │ ← 错误图标+文字
└─────────────────────────────────┘
```

### 3. 文字大小

**最小字号**
```
正文：16px（移动端14px）
次级文字：14px
辅助文字：12px
```

**可调整性**
```css
/* 使用相对单位 */
body {
  font-size: 1rem; /* 相对于用户设置 */
}

/* 或使用 clamp() */
h1 {
  font-size: clamp(1.5rem, 5vw, 2.5rem);
}
```

### 4. 焦点指示

**焦点状态**
```css
/* 明显的焦点样式 */
:focus {
  outline: 3px solid #1890ff;
  outline-offset: 2px;
}

/* 不移除焦点样式 */
:focus {
  outline: none; /* ✗ 错误 */
}
```

## 键盘无障碍

### 1. Tab顺序

**合理的Tab顺序**
```
┌─────────────────────────────────┐
│ [1]Logo  [2]首页  [3]产品       │
│                                 │
│ [4]搜索...              [5]🔍  │
│                                 │
│ [6]卡片    [7]卡片    [8]卡片   │
└─────────────────────────────────┘
```

**实现**
```html
<!-- 自然顺序 -->
<nav>
  <a href="/" tabindex="0">首页</a>
  <a href="/product" tabindex="0">产品</a>
</nav>

<!-- 跳过导航 -->
<a href="#main" class="skip-link">跳到主内容</a>
<nav>...</nav>
<main id="main">...</main>
```

### 2. 键盘操作

**常见键盘操作**
| 按键 | 功能 |
|------|------|
| Tab | 下一元素 |
| Shift+Tab | 上一元素 |
| Enter/Space | 激活 |
| Esc | 关闭/取消 |
| 方向键 | 导航/调整 |

**自定义组件键盘支持**
```html
<!-- 自定义下拉框 -->
<div role="combobox" 
     aria-expanded="false"
     aria-haspopup="listbox"
     tabindex="0">
  选择选项
</div>

<script>
element.addEventListener('keydown', (e) => {
  switch(e.key) {
    case 'Enter':
    case ' ':
      toggleDropdown();
      break;
    case 'ArrowDown':
      selectNext();
      break;
    case 'ArrowUp':
      selectPrev();
      break;
    case 'Escape':
      closeDropdown();
      break;
  }
});
</script>
```

### 3. 无键盘陷阱

**问题示例**
```
键盘陷阱：
┌─────────────────────────────────┐
│ Modal弹窗                       │
│ ┌─────────────────────────────┐│
│ │ 焦点被困在弹窗内，           ││
│ │ 无法Tab到外部               ││
│ └─────────────────────────────┘│
└─────────────────────────────────┘
```

**解决方案**
```javascript
// 弹窗打开时限制焦点在弹窗内
const focusableElements = modal.querySelectorAll(
  'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
);
const firstElement = focusableElements[0];
const lastElement = focusableElements[focusableElements.length - 1];

modal.addEventListener('keydown', (e) => {
  if (e.key === 'Tab') {
    if (e.shiftKey && document.activeElement === firstElement) {
      e.preventDefault();
      lastElement.focus();
    } else if (!e.shiftKey && document.activeElement === lastElement) {
      e.preventDefault();
      firstElement.focus();
    }
  }
});
```

## 屏幕阅读器支持

### 1. ARIA基础

**ARIA角色**
```html
<!-- 地标角色 -->
<nav role="navigation">...</nav>
<main role="main">...</main>
<aside role="complementary">...</aside>

<!-- 组件角色 -->
<button role="button">...</button>
<div role="dialog">...</div>
<div role="alert">...</div>
```

**ARIA属性**
```html
<!-- 标签 -->
<button aria-label="关闭弹窗">×</button>

<!-- 状态 -->
<button aria-pressed="true">加粗</button>
<div aria-expanded="false">...</div>
<div aria-hidden="true">...</div>

<!-- 关系 -->
<div aria-labelledby="title">...</div>
<input aria-describedby="hint">
<div aria-live="polite">...</div>
```

### 2. 语义化HTML

**使用正确的元素**
```html
<!-- ✓ 正确 -->
<nav>
  <ul>
    <li><a href="/">首页</a></li>
    <li><a href="/product">产品</a></li>
  </ul>
</nav>

<!-- ✗ 不推荐 -->
<div onclick="navigate('/')">首页</div>
<div onclick="navigate('/product')">产品</div>
```

### 3. 动态内容

**实时区域**
```html
<!-- 礼貌通知（不打断） -->
<div aria-live="polite">保存成功</div>

<!-- 紧急通知（打断） -->
<div aria-live="assertive">发生错误</div>

<!-- 更新整个区域 -->
<div aria-live="polite" aria-atomic="true">
  购物车: 3件商品
</div>
```

## 认知无障碍

### 1. 简洁清晰

**内容简化**
- 使用简单语言
- 避免专业术语
- 分段使用标题
- 提供摘要

**界面简化**
- 减少干扰元素
- 清晰的视觉层级
- 一致的布局

### 2. 一致性

**导航一致**
- 相同位置
- 相同样式
- 相同顺序

**交互一致**
- 相同功能相同操作
- 相同反馈方式

### 3. 错误预防

**清晰的指示**
```
✓ 清晰的表单提示：
┌─────────────────────────────────┐
│ 手机号 *                        │
│ [请输入11位手机号            ] │
│ 示例：13800138000              │
└─────────────────────────────────┘
```

**错误恢复**
- 清晰的错误说明
- 具体的解决建议
- 保留已输入内容

### 键盘
- [ ] 全部功能可键盘操作
- [ ] Tab顺序合理
- [ ] 无键盘陷阱

### 内容
- [ ] 图片有替代文本
- [ ] 视频有字幕
- [ ] 语言明确
- [ ] 结构化标记

### 交互
- [ ] 表单标签清晰

## 最佳实践

### 1. 设计建议

- 从一开始就考虑无障碍
- 使用语义化HTML
- 保持简洁清晰
- 提供多种交互方式

### 2. 常见错误

- ❌ 移除焦点样式
- ❌ 仅用颜色区分
- ❌ 图片缺少alt
- ❌ 键盘无法操作
