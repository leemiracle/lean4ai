# 同期群分析（Cohort Analysis）

## 1. 定义

同期群分析是将用户按特定时间或特征分组，追踪各组用户在生命周期中的行为变化，发现不同群体的行为差异和趋势。

### 核心概念

```
同期群（Cohort）：具有共同特征的用户群体

分类方式：
├─ 时间同期群：按注册时间分组
│   └─ 例：2024年1月注册的用户
├─ 行为同期群：按首次行为分组
│   └─ 例：首次购买用户
└─ 特征同期群：按用户属性分组
    └─ 例：北京地区的用户

分析价值：
├─ 识别用户行为趋势
├─ 排除总量增长的干扰
├─ 评估产品改进效果
└─ 发现用户群体差异
```

## 2. 同期群分析方法

### 2.1 时间同期群分析

**SQL示例：按月分组的同期群留存**
```sql
-- 同期群留存矩阵
WITH monthly_cohorts AS (
    SELECT 
        user_id,
        DATE_FORMAT(register_time, '%Y-%m') as cohort_month,
        MIN(DATE(register_time)) as first_day
    FROM users
    WHERE register_time >= DATE_SUB(NOW(), INTERVAL 12 MONTH)
    GROUP BY user_id, DATE_FORMAT(register_time, '%Y-%m')
),
user_monthly_activity AS (
    SELECT 
        user_id,
        DATE_FORMAT(activity_time, '%Y-%m') as activity_month
    FROM user_activities
    WHERE activity_time >= DATE_SUB(NOW(), INTERVAL 12 MONTH)
    GROUP BY user_id, DATE_FORMAT(activity_time, '%Y-%m')
),
cohort_retention AS (
    SELECT 
        mc.cohort_month,
        mc.user_id,
        uma.activity_month,
        (YEAR(uma.activity_month) - YEAR(mc.cohort_month)) * 12 + 
        (MONTH(uma.activity_month) - MONTH(mc.cohort_month)) as month_number
    FROM monthly_cohorts mc
    LEFT JOIN user_monthly_activity uma ON mc.user_id = uma.user_id
)
SELECT 
    cohort_month,
    COUNT(DISTINCT user_id) as cohort_size,
    -- 各月留存用户数
    COUNT(DISTINCT CASE WHEN month_number = 0 THEN user_id END) as month_0,
    COUNT(DISTINCT CASE WHEN month_number = 1 THEN user_id END) as month_1,
    COUNT(DISTINCT CASE WHEN month_number = 2 THEN user_id END) as month_2,
    COUNT(DISTINCT CASE WHEN month_number = 3 THEN user_id END) as month_3,
    COUNT(DISTINCT CASE WHEN month_number = 6 THEN user_id END) as month_6,
    -- 各月留存率
    ROUND(COUNT(DISTINCT CASE WHEN month_number = 1 THEN user_id END) * 100.0 
        / COUNT(DISTINCT user_id), 2) as month_1_rate,
    ROUND(COUNT(DISTINCT CASE WHEN month_number = 3 THEN user_id END) * 100.0 
        / COUNT(DISTINCT user_id), 2) as month_3_rate,
    ROUND(COUNT(DISTINCT CASE WHEN month_number = 6 THEN user_id END) * 100.0 
        / COUNT(DISTINCT user_id), 2) as month_6_rate
FROM cohort_retention
GROUP BY cohort_month
ORDER BY cohort_month DESC;
```

### 2.2 行为同期群分析

**SQL示例：按首次购买月份分析**
```sql
-- 首次购买用户同期群分析
WITH first_purchase_cohort AS (
    SELECT 
        user_id,
        DATE_FORMAT(MIN(order_time), '%Y-%m') as first_purchase_month
    FROM orders
    WHERE order_status = 'completed'
    GROUP BY user_id
),
monthly_revenue AS (
    SELECT 
        user_id,
        DATE_FORMAT(order_time, '%Y-%m') as purchase_month,
        SUM(amount) as monthly_revenue
    FROM orders
    WHERE order_status = 'completed'
    GROUP BY user_id, DATE_FORMAT(order_time, '%Y-%m')
),
cohort_revenue AS (
    SELECT 
        fpc.first_purchase_month,
        fpc.user_id,
        mr.purchase_month,
        mr.monthly_revenue,
        (YEAR(mr.purchase_month) - YEAR(fpc.first_purchase_month)) * 12 + 
        (MONTH(mr.purchase_month) - MONTH(fpc.first_purchase_month)) as month_number
    FROM first_purchase_cohort fpc
    JOIN monthly_revenue mr ON fpc.user_id = mr.user_id
)
SELECT 
    first_purchase_month,
    COUNT(DISTINCT user_id) as cohort_size,
    -- 各月人均消费
    ROUND(AVG(CASE WHEN month_number = 0 THEN monthly_revenue END), 2) as avg_revenue_m0,
    ROUND(AVG(CASE WHEN month_number = 1 THEN monthly_revenue END), 2) as avg_revenue_m1,
    ROUND(AVG(CASE WHEN month_number = 2 THEN monthly_revenue END), 2) as avg_revenue_m2,
    ROUND(AVG(CASE WHEN month_number = 3 THEN monthly_revenue END), 2) as avg_revenue_m3,
    -- 复购率
    ROUND(COUNT(DISTINCT CASE WHEN month_number = 1 THEN user_id END) * 100.0 
        / COUNT(DISTINCT user_id), 2) as repurchase_rate_m1,
    ROUND(COUNT(DISTINCT CASE WHEN month_number = 3 THEN user_id END) * 100.0 
        / COUNT(DISTINCT user_id), 2) as repurchase_rate_m3
FROM cohort_revenue
GROUP BY first_purchase_month
ORDER BY first_purchase_month DESC;
```

## 3. 同期群分析可视化

### 3.1 热力图可视化

```python
import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns

def plot_cohort_heatmap(cohort_data):
    """
    绘制同期群留存热力图
    """
    # 准备数据
    cohort_pivot = cohort_data.pivot(
        index='cohort_month',
        columns='month_number',
        values='retention_rate'
    )
    
    # 绘制热力图
    plt.figure(figsize=(15, 8))
    sns.heatmap(
        cohort_pivot,
        annot=True,
        fmt='.1f',
        cmap='YlOrRd',
        cbar_kws={'label': '留存率 (%)'}
    )
    
    plt.title('同期群留存热力图', fontsize=16, fontweight='bold')
    plt.xlabel('月数', fontsize=12)
    plt.ylabel('同期群（注册月份）', fontsize=12)
    
    plt.tight_layout()
    return plt

# 示例数据
cohort_example = pd.DataFrame({
    'cohort_month': ['2024-01', '2024-01', '2024-01', '2024-02', '2024-02', '2024-02'],
    'month_number': [0, 1, 2, 0, 1, 2],
    'retention_rate': [100, 45, 32, 100, 42, 30]
})

plot_cohort_heatmap(cohort_example)
plt.show()
```

### 4.2 渠道质量对比

**SQL示例：不同渠道同期群对比**
```sql
-- 渠道同期群分析
WITH channel_cohorts AS (
    SELECT 
        user_id,
        acquisition_channel,
        DATE_FORMAT(register_time, '%Y-%m') as cohort_month
    FROM users
    WHERE register_time >= DATE_SUB(NOW(), INTERVAL 6 MONTH)
),
channel_retention AS (
    SELECT 
        cc.acquisition_channel,
        cc.cohort_month,
        COUNT(DISTINCT cc.user_id) as cohort_size,
        COUNT(DISTINCT CASE WHEN DATEDIFF(DATE(ua.activity_time), DATE(cc.register_time)) = 7 
            THEN cc.user_id END) as day7_retained,
        COUNT(DISTINCT CASE WHEN DATEDIFF(DATE(ua.activity_time), DATE(cc.register_time)) = 30 
            THEN cc.user_id END) as day30_retained
    FROM channel_cohorts cc
    LEFT JOIN user_activities ua ON cc.user_id = ua.user_id
    GROUP BY cc.acquisition_channel, cc.cohort_month
)
SELECT 
    acquisition_channel,
    cohort_month,
    cohort_size,
    ROUND(day7_retained * 100.0 / cohort_size, 2) as day7_retention,
    ROUND(day30_retained * 100.0 / cohort_size, 2) as day30_retention
FROM channel_retention
ORDER BY acquisition_channel, cohort_month DESC;
```

## 5. 最佳实践

### 5.1 同期群分组原则

```
分组维度选择：

1. 时间维度（最常用）
   ├─ 注册时间
   ├─ 首次购买时间
   └─ 首次使用时间

2. 行为维度
   ├─ 首次行为类型
   ├─ 初始活跃程度
   └─ 首次付费金额

3. 属性维度
   ├─ 获客渠道
   ├─ 地理位置
   └─ 用户画像

分组粒度：
├─ 日：短期分析
├─ 周：中期趋势
└─ 月：长期对比
```

### 5.2 解读同期群数据

```
数据解读要点：

1. 横向对比（同一同期群随时间变化）
   - 留存曲线是否健康
   - 何时趋于稳定
   - 是否有异常波动

2. 纵向对比（不同同期群在同一时期的表现）
   - 新同期群是否优于旧同期群
   - 产品改进是否有效
   - 是否有季节性规律

3. 对角线对比（同一时间不同同期群的表现）
   - 是否受到外部事件影响
   - 市场环境变化
   - 竞品影响
```

## 6. 总结

同期群分析是深入了解用户行为的强大工具：

**核心价值**
- 排除总量增长干扰
- 识别真实用户行为趋势
- 评估产品和运营效果
- 发现不同群体差异

**实施要点**
- 选择合适的分组维度
- 确保数据准确性
- 多维度交叉分析
- 结合业务场景解读

记住：**同期群分析让你看到数据背后的真相，而不是被表面数字迷惑。**
