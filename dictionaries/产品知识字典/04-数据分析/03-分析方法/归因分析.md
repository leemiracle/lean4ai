# 归因分析（Attribution Analysis）

## 1. 定义

归因分析是确定用户转化过程中各触点（广告、渠道、营销活动等）对最终转化的贡献程度，从而科学分配营销预算和优化投放策略。

### 核心概念

```
归因（Attribution）：
将转化结果合理分配给影响用户决策的各个触点

触点（Touchpoint）：
用户在转化路径中接触到的营销接触点
├─ 广告曝光
├─ 广告点击
├─ 搜索引擎
├─ 社交媒体
└─ 邮件营销

转化路径（Conversion Path）：
用户从首次接触到最终转化经历的所有触点序列
```

## 2. 归因模型

### 2.1 常见归因模型

```
1. 末次点击归因（Last Click）
   - 100%归因给最后一个触点
   - 优点：简单易懂
   - 缺点：忽视前期触点价值
   - 适用：短决策周期产品

2. 首次点击归因（First Click）
   - 100%归因给第一个触点
   - 优点：重视获客渠道
   - 缺点：忽视后续影响
   - 适用：品牌认知阶段

3. 线性归因（Linear）
   - 所有触点平分功劳
   - 优点：考虑所有触点
   - 缺点：可能低估关键触点
   - 适用：长期培育场景

4. 时间衰减归因（Time Decay）
   - 越接近转化的触点权重越高
   - 优点：兼顾全链路和时效性
   - 缺点：参数设置主观
   - 适用：较长决策周期

5. 位置归因（Position Based）
   - 首尾触点各40%，中间20%
   - 优点：平衡首尾和中间触点
   - 缺点：权重设置固定
   - 适用：多触点协作场景
```

### 2.2 归因模型对比

```python
# 归因模型对比示例
def compare_attribution_models(touchpoints, conversion_value=100):
    """
    对比不同归因模型
    """
    results = {}
    n = len(touchpoints)
    
    # 1. 末次点击归因
    last_click = [0] * n
    last_click[-1] = conversion_value
    results['末次点击'] = last_click
    
    # 2. 首次点击归因
    first_click = [0] * n
    first_click[0] = conversion_value
    results['首次点击'] = first_click
    
    # 3. 线性归因
    linear = [conversion_value / n] * n
    results['线性'] = linear
    
    # 4. 时间衰减归因（半衰期7天）
    import math
    time_decay = []
    half_life = 7
    for i, tp in enumerate(touchpoints):
        days_to_conversion = n - i - 1
        weight = math.pow(2, -days_to_conversion / half_life)
        time_decay.append(weight)
    
    # 归一化
    total = sum(time_decay)
    time_decay = [v / total * conversion_value for v in time_decay]
    results['时间衰减'] = time_decay
    
    # 5. 位置归因
    position = [0] * n
    if n == 1:
        position[0] = conversion_value
    elif n == 2:
        position[0] = conversion_value / 2
        position[1] = conversion_value / 2
    else:
        position[0] = conversion_value * 0.4
        position[-1] = conversion_value * 0.4
        middle_value = conversion_value * 0.2 / (n - 2)
        for i in range(1, n-1):
            position[i] = middle_value
    results['位置'] = position
    
    return results

# 示例
touchpoints = ['搜索广告', '社交广告', '邮件营销', '直接访问']
models = compare_attribution_models(touchpoints)

import pandas as pd
df = pd.DataFrame(models, index=touchpoints)
print(df)
```

## 3. 归因分析实施

### 3.1 数据收集

**SQL示例：转化路径追踪**
```sql
-- 用户转化路径
WITH user_journey AS (
    SELECT 
        user_id,
        channel,
        campaign,
        event_time,
        ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY event_time) as touchpoint_order
    FROM marketing_touchpoints
    WHERE user_id IN (
        SELECT DISTINCT user_id 
        FROM conversions 
        WHERE conversion_time >= DATE_SUB(NOW(), INTERVAL 30 DAY)
    )
    AND event_time <= (
        SELECT MIN(conversion_time) 
        FROM conversions c 
        WHERE c.user_id = marketing_touchpoints.user_id
    )
),
conversion_paths AS (
    SELECT 
        user_id,
        GROUP_CONCAT(channel ORDER BY touchpoint_order SEPARATOR ' → ') as path,
        COUNT(*) as touchpoint_count,
        MIN(event_time) as first_touch_time,
        MAX(event_time) as last_touch_time
    FROM user_journey
    GROUP BY user_id
)
SELECT 
    path,
    COUNT(*) as conversion_count,
    AVG(touchpoint_count) as avg_touchpoints,
    AVG(TIMESTAMPDIFF(HOUR, first_touch_time, last_touch_time)) as avg_journey_hours
FROM conversion_paths
GROUP BY path
ORDER BY conversion_count DESC
LIMIT 20;
```

### 3.2 归因计算

**SQL示例：多模型归因计算**
```sql
-- 多归因模型计算
WITH touchpoints AS (
    SELECT 
        user_id,
        channel,
        event_time,
        ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY event_time) as tp_order,
        COUNT(*) OVER (PARTITION BY user_id) as total_tps
    FROM marketing_touchpoints
    WHERE event_time <= (SELECT MIN(conversion_time) FROM conversions c WHERE c.user_id = marketing_touchpoints.user_id)
),
conversions_with_value AS (
    SELECT 
        user_id,
        conversion_value
    FROM conversions
    WHERE conversion_time >= DATE_SUB(NOW(), INTERVAL 30 DAY)
),
attribution_calc AS (
    SELECT 
        t.channel,
        t.user_id,
        c.conversion_value,
        -- 末次点击归因
        CASE WHEN t.tp_order = t.total_tps THEN c.conversion_value ELSE 0 END as last_click_value,
        -- 首次点击归因
        CASE WHEN t.tp_order = 1 THEN c.conversion_value ELSE 0 END as first_click_value,
        -- 线性归因
        c.conversion_value / t.total_tps as linear_value
    FROM touchpoints t
    JOIN conversions_with_value c ON t.user_id = c.user_id
)
SELECT 
    channel,
    COUNT(DISTINCT user_id) as users,
    -- 末次点击归因
    SUM(last_click_value) as last_click_revenue,
    -- 首次点击归因
    SUM(first_click_value) as first_click_revenue,
    -- 线性归因
    SUM(linear_value) as linear_revenue
FROM attribution_calc
GROUP BY channel
ORDER BY linear_revenue DESC;
```

## 4. 归因分析应用

### 4.1 营销预算分配

```python
# 基于归因的预算分配
def allocate_budget(attribution_results, total_budget):
    """
    基于归因结果分配营销预算
    """
    total_revenue = sum(attribution_results.values())
    
    budget_allocation = {}
    for channel, revenue in attribution_results.items():
        share = revenue / total_revenue
        budget_allocation[channel] = total_budget * share
    
    return budget_allocation

# 示例
attribution_data = {
    '搜索引擎': 1500000,
    '社交媒体': 800000,
    '邮件营销': 500000,
    '直接访问': 400000,
    '推荐': 300000
}

budget = allocate_budget(attribution_data, 1000000)
print("预算分配：")
for channel, amount in budget.items():
    print(f"{channel}: ¥{amount:,.0f}")
```

```sql
-- 渠道ROI分析（使用线性归因）
WITH channel_attribution AS (
    SELECT 
        t.channel,
        SUM(c.conversion_value / t.total_tps) as attributed_revenue,
        COUNT(DISTINCT t.user_id) as attributed_conversions
    FROM (
        SELECT 
            user_id,
            channel,
            COUNT(*) OVER (PARTITION BY user_id) as total_tps
        FROM marketing_touchpoints mt
        WHERE EXISTS (
            SELECT 1 FROM conversions c 
            WHERE c.user_id = mt.user_id
        )
    ) t
    JOIN conversions c ON t.user_id = c.user_id
    GROUP BY t.channel
),
channel_costs AS (
    SELECT 
        channel,
        SUM(cost) as total_cost
    FROM marketing_spend
    WHERE spend_date >= DATE_SUB(NOW(), INTERVAL 30 DAY)
    GROUP BY channel
)
SELECT 
    ca.channel,
    ca.attributed_revenue,
    cc.total_cost as cost,
    ca.attributed_conversions,
    ROUND(ca.attributed_revenue / cc.total_cost, 2) as roi,
    ROUND(cc.total_cost / ca.attributed_conversions, 2) as cpa
FROM channel_attribution ca
JOIN channel_costs cc ON ca.channel = cc.channel
ORDER BY roi DESC;
```

### 5.2 增量归因

```sql
-- 增量归因（Uplift Modeling）
-- 对比暴露组和对照组的转化差异

WITH test_groups AS (
    SELECT 
        user_id,
        CASE 
            WHEN exposed_to_campaign = 1 AND in_control_group = 0 THEN 'treatment'
            WHEN exposed_to_campaign = 0 AND in_control_group = 1 THEN 'control'
            ELSE 'other'
        END as group_type,
        channel
    FROM marketing_experiment
    WHERE experiment_id = 'exp_001'
),
conversion_rates AS (
    SELECT 
        group_type,
        channel,
        COUNT(*) as total_users,
        SUM(CASE WHEN converted = 1 THEN 1 ELSE 0 END) as conversions,
        AVG(CASE WHEN converted = 1 THEN 1.0 ELSE 0.0 END) as conversion_rate
    FROM test_groups tg
    LEFT JOIN conversions c ON tg.user_id = c.user_id
    WHERE tg.group_type IN ('treatment', 'control')
    GROUP BY group_type, channel
)
SELECT 
    t.channel,
    t.conversion_rate as treatment_rate,
    c.conversion_rate as control_rate,
    (t.conversion_rate - c.conversion_rate) as uplift,
    ROUND((t.conversion_rate - c.conversion_rate) / c.conversion_rate * 100, 2) as uplift_pct,
    (t.conversion_rate - c.conversion_rate) * t.total_users as incremental_conversions
FROM conversion_rates t
JOIN conversion_rates c ON t.channel = c.channel
WHERE t.group_type = 'treatment' AND c.group_type = 'control'
ORDER BY uplift DESC;
```

## 6. 最佳实践

### 6.1 归因模型选择

```
选择原则：

1. 根据决策周期
   - 短周期（<1天）：末次点击归因
   - 中周期（1-7天）：时间衰减归因
   - 长周期（>7天）：位置归因或数据驱动归因

2. 根据业务目标
   - 品牌认知：首次点击归因
   - 直接转化：末次点击归因
   - 全链路优化：多触点归因

3. 根据数据能力
   - 数据基础：简单归因模型
   - 数据完善：数据驱动归因
   - 实验能力：增量归因

4. 行业实践
   - 电商：末次点击/位置归因
   - SaaS：时间衰减归因
   - 游戏：首次点击/线性归因
```

### 6.2 常见问题

```
问题1：数据孤岛
解决：统一用户ID，打通数据

问题2：跨设备归因
解决：使用概率匹配或确定性匹配

问题3：线下转化
解决：问卷调查、优惠券追踪

问题4：归因窗口选择
解决：根据业务周期设置合理的归因窗口

问题5：模型切换
解决：小范围测试，逐步切换
```

## 7. 总结

归因分析是优化营销投放的核心工具：

**核心价值**
- 科学评估渠道价值
- 优化预算分配
- 提升营销ROI

**实施要点**
- 选择合适的归因模型
- 确保数据准确性
- 持续监测和优化
- 结合业务场景解读

记住：**没有完美的归因模型，只有最适合的归因模型。关键是持续优化，不断提升归因准确性。**
