# 漏斗分析

## 1. 定义

漏斗分析是一种通过追踪用户在一系列连续步骤中的转化情况，识别流失环节、优化转化路径的分析方法。

### 核心概念

```
漏斗模型：

总用户 100%
    ↓ 转化率 60%
步骤1 60%
    ↓ 转化率 50%
步骤2 30%
    ↓ 转化率 40%
步骤3 12%
    ↓ 转化率 80%
步骤4 9.6%

关键指标：
├─ 转化率：步骤间转化比例
├─ 流失率：步骤间流失比例
├─ 整体转化率：从开始到结束的转化率
└─ 完成率：完成全部步骤的用户比例
```

## 2. 应用场景

```
1. 电商购物流程
   浏览商品 → 加入购物车 → 提交订单 → 完成支付

2. 用户注册流程
   访问首页 → 点击注册 → 填写信息 → 验证邮箱 → 完成注册

3. 活动参与流程
   活动曝光 → 点击活动 → 参与活动 → 完成任务 → 获得奖励

4. 产品使用流程
   下载APP → 打开应用 → 注册登录 → 核心功能使用 → 成为活跃用户

5. 广告转化流程
   广告曝光 → 广告点击 → 落地页访问 → 留资转化 → 销售跟进 → 成交
```

## 3. 漏斗分析步骤

### 3.1 定义漏斗步骤

```markdown
## 定义漏斗的要点

1. 明确业务目标
   - 最终目标是什么？
   - 成功的定义是什么？

2. 梳理用户路径
   - 用户从哪里开始？
   - 中间经历哪些步骤？
   - 在哪里结束？

3. 确定关键步骤
   - 必经步骤
   - 关键决策点
   - 容易流失的环节

4. 定义转化条件
   - 每个步骤的完成标准
   - 时间窗口限制
   - 顺序要求

## 示例：电商购买漏斗

步骤定义：
1. 商品详情页浏览（30分钟内）
2. 加入购物车（浏览后24小时内）
3. 提交订单（加购后7天内）
4. 完成支付（提交后30分钟内）

时间窗口：从步骤1开始后的7天内完成所有步骤
```

### 3.2 漏斗计算

**SQL示例：基础漏斗分析**
```sql
-- 电商购买漏斗分析
WITH funnel_steps AS (
    SELECT 
        user_id,
        MAX(CASE WHEN event = 'view_product' THEN event_time END) as step1_time,
        MAX(CASE WHEN event = 'add_to_cart' THEN event_time END) as step2_time,
        MAX(CASE WHEN event = 'submit_order' THEN event_time END) as step3_time,
        MAX(CASE WHEN event = 'pay_order' THEN event_time END) as step4_time
    FROM user_events
    WHERE event IN ('view_product', 'add_to_cart', 'submit_order', 'pay_order')
        AND event_time >= DATE_SUB(NOW(), INTERVAL 7 DAY)
    GROUP BY user_id
),
funnel_counts AS (
    SELECT 
        COUNT(*) as step1_users,
        SUM(CASE WHEN step2_time IS NOT NULL AND step2_time >= step1_time THEN 1 ELSE 0 END) as step2_users,
        SUM(CASE WHEN step2_time IS NOT NULL AND step3_time IS NOT NULL 
                AND step3_time >= step2_time THEN 1 ELSE 0 END) as step3_users,
        SUM(CASE WHEN step2_time IS NOT NULL AND step3_time IS NOT NULL AND step4_time IS NOT NULL
                AND step3_time >= step2_time AND step4_time >= step3_time THEN 1 ELSE 0 END) as step4_users
    FROM funnel_steps
    WHERE step1_time IS NOT NULL
)
SELECT 
    '商品浏览' as step_name,
    step1_users as user_count,
    100.0 as conversion_rate,
    step1_users as from_previous
FROM funnel_counts

UNION ALL

SELECT 
    '加入购物车',
    step2_users,
    ROUND(step2_users * 100.0 / step1_users, 2),
    step2_users
FROM funnel_counts

UNION ALL

SELECT 
    '提交订单',
    step3_users,
    ROUND(step3_users * 100.0 / step2_users, 2),
    step3_users
FROM funnel_counts

UNION ALL

SELECT 
    '完成支付',
    step4_users,
    ROUND(step4_users * 100.0 / step3_users, 2),
    step4_users
FROM funnel_counts;
```

**Python漏斗分析**
```python
import pandas as pd
import numpy as np

class FunnelAnalyzer:
    def __init__(self, events_data):
        self.events = events_data
        self.funnel_steps = []
    
    def add_step(self, event_name, time_window=None):
        """
        添加漏斗步骤
        """
        self.funnel_steps.append({
            'event': event_name,
            'time_window': time_window
        })
    
    def calculate_funnel(self):
        """
        计算漏斗转化
        """
        # 获取每个用户的每个步骤时间
        user_steps = self.events.pivot_table(
            index='user_id',
            columns='event',
            values='event_time',
            aggfunc='min'
        )
        
        results = []
        prev_count = None
        
        for i, step in enumerate(self.funnel_steps):
            event = step['event']
            time_window = step['time_window']
            
            # 计算到达该步骤的用户
            if i == 0:
                users = user_steps[event].notna()
            else:
                prev_event = self.funnel_steps[i-1]['event']
                # 确保按顺序发生
                users = (
                    user_steps[event].notna() & 
                    (user_steps[event] >= user_steps[prev_event])
                )
                
                # 如果有时间窗口限制
                if time_window:
                    time_diff = (user_steps[event] - user_steps[prev_event]).dt.total_seconds() / 3600
                    users = users & (time_diff <= time_window)
            
            count = users.sum()
            conversion_rate = (count / prev_count * 100) if prev_count else 100
            prev_count = count
            
            results.append({
                'step': f"步骤{i+1}: {event}",
                'users': count,
                'conversion_rate': round(conversion_rate, 2)
            })
        
        return pd.DataFrame(results)
    
    def plot_funnel(self, results):
        """
        绘制漏斗图
        """
        import matplotlib.pyplot as plt
        
        fig, ax = plt.subplots(figsize=(12, 6))
        
        steps = results['step']
        users = results['users']
        max_users = users.max()
        
        # 绘制漏斗
        for i, (step, user_count) in enumerate(zip(steps, users)):
            width = user_count / max_users
            left = (1 - width) / 2
            
            ax.barh(i, width, left=left, height=0.6, 
                   color='steelblue', alpha=0.8)
            
            # 添加文字
            ax.text(0.5, i, f"{step}\n{user_count:,} ({results.loc[i, 'conversion_rate']}%)",
                   ha='center', va='center', fontsize=10, color='white', fontweight='bold')
        
        ax.set_xlim(0, 1)
        ax.set_yticks(range(len(steps)))
        ax.set_yticklabels([''] * len(steps))
        ax.set_title('Funnel Analysis', fontsize=16, fontweight='bold')
        ax.invert_yaxis()
        ax.axis('off')
        
        plt.tight_layout()
        return fig

# 使用示例
events_df = pd.DataFrame({
    'user_id': [1, 1, 1, 2, 2, 3, 3, 3, 3],
    'event': ['view', 'add_cart', 'pay', 'view', 'add_cart', 
              'view', 'add_cart', 'submit', 'pay'],
    'event_time': pd.to_datetime([
        '2024-01-01 10:00', '2024-01-01 10:05', '2024-01-01 10:30',
        '2024-01-01 11:00', '2024-01-01 11:30',
        '2024-01-01 12:00', '2024-01-01 12:10', '2024-01-01 12:20', '2024-01-01 12:30'
    ])
})

analyzer = FunnelAnalyzer(events_df)
analyzer.add_step('view')
analyzer.add_step('add_cart', time_window=24)
analyzer.add_step('submit', time_window=24)
analyzer.add_step('pay', time_window=1)

results = analyzer.calculate_funnel()
print(results)
```

## 4. 漏斗细分分析

### 4.1 多维度细分

**SQL示例：按渠道细分漏斗**
```sql
-- 按渠道分析漏斗
WITH funnel_by_channel AS (
    SELECT 
        u.acquisition_channel,
        COUNT(DISTINCT CASE WHEN e1.event = 'view_product' THEN u.user_id END) as step1,
        COUNT(DISTINCT CASE WHEN e2.event = 'add_to_cart' THEN u.user_id END) as step2,
        COUNT(DISTINCT CASE WHEN e3.event = 'submit_order' THEN u.user_id END) as step3,
        COUNT(DISTINCT CASE WHEN e4.event = 'pay_order' THEN u.user_id END) as step4
    FROM users u
    LEFT JOIN user_events e1 ON u.user_id = e1.user_id AND e1.event = 'view_product'
    LEFT JOIN user_events e2 ON u.user_id = e2.user_id AND e2.event = 'add_to_cart' 
        AND e2.event_time >= e1.event_time
    LEFT JOIN user_events e3 ON u.user_id = e3.user_id AND e3.event = 'submit_order'
        AND e3.event_time >= e2.event_time
    LEFT JOIN user_events e4 ON u.user_id = e4.user_id AND e4.event = 'pay_order'
        AND e4.event_time >= e3.event_time
    WHERE u.register_time >= DATE_SUB(NOW(), INTERVAL 30 DAY)
    GROUP BY u.acquisition_channel
)
SELECT 
    acquisition_channel,
    step1 as '浏览',
    step2 as '加购',
    ROUND(step2 * 100.0 / NULLIF(step1, 0), 2) as '加购率',
    step3 as '下单',
    ROUND(step3 * 100.0 / NULLIF(step2, 0), 2) as '下单率',
    step4 as '支付',
    ROUND(step4 * 100.0 / NULLIF(step3, 0), 2) as '支付率',
    ROUND(step4 * 100.0 / NULLIF(step1, 0), 2) as '整体转化率'
FROM funnel_by_channel
ORDER BY step1 DESC;
```

### 4.2 时间趋势分析

```sql
-- 漏斗转化率趋势
WITH daily_funnel AS (
    SELECT 
        DATE(e1.event_time) as date,
        COUNT(DISTINCT e1.user_id) as step1,
        COUNT(DISTINCT e2.user_id) as step2,
        COUNT(DISTINCT e3.user_id) as step3,
        COUNT(DISTINCT e4.user_id) as step4
    FROM user_events e1
    LEFT JOIN user_events e2 ON e1.user_id = e2.user_id 
        AND e2.event = 'add_to_cart' AND e2.event_time >= e1.event_time
    LEFT JOIN user_events e3 ON e1.user_id = e3.user_id 
        AND e3.event = 'submit_order' AND e3.event_time >= e2.event_time
    LEFT JOIN user_events e4 ON e1.user_id = e4.user_id 
        AND e4.event = 'pay_order' AND e4.event_time >= e3.event_time
    WHERE e1.event = 'view_product'
        AND e1.event_time >= DATE_SUB(NOW(), INTERVAL 30 DAY)
    GROUP BY DATE(e1.event_time)
)
SELECT 
    date,
    step1,
    ROUND(step2 * 100.0 / NULLIF(step1, 0), 2) as step1_to_2_rate,
    ROUND(step3 * 100.0 / NULLIF(step2, 0), 2) as step2_to_3_rate,
    ROUND(step4 * 100.0 / NULLIF(step3, 0), 2) as step3_to_4_rate,
    ROUND(step4 * 100.0 / NULLIF(step1, 0), 2) as overall_rate
FROM daily_funnel
ORDER BY date DESC;
```

## 5. 漏斗优化策略

### 5.1 识别问题步骤

```
分析方法：

1. 找出流失率最高的步骤
   - 流失率 = 1 - 转化率
   - 优先优化流失率最高的环节

2. 分析流失原因
   - 用户调研
   - 数据分析
   - 可用性测试

3. 制定优化方案
   - A/B测试
   - 迭代优化

## 常见流失原因及对策

步骤1→2（浏览→加购）流失率高
原因：
- 商品信息不够吸引
- 价格不够竞争力
- 库存不足
对策：
- 优化商品详情页
- 提供优惠信息
- 推荐相似商品

步骤2→3（加购→下单）流失率高
原因：
- 运费太高
- 结算流程复杂
- 缺少信任感
对策：
- 满额包邮
- 简化结算流程
- 展示评价和保障

步骤3→4（下单→支付）流失率高
原因：
- 支付方式少
- 支付失败
- 支付犹豫
对策：
- 增加支付方式
- 优化支付体验
- 限时优惠促支付
```

## 6. 高级漏斗分析

### 6.1 漏斗时间分析

```sql
-- 各步骤耗时分析
WITH step_durations AS (
    SELECT 
        user_id,
        e1.event_time as step1_time,
        e2.event_time as step2_time,
        e3.event_time as step3_time,
        e4.event_time as step4_time,
        TIMESTAMPDIFF(MINUTE, e1.event_time, e2.event_time) as step1_to_2_minutes,
        TIMESTAMPDIFF(MINUTE, e2.event_time, e3.event_time) as step2_to_3_minutes,
        TIMESTAMPDIFF(MINUTE, e3.event_time, e4.event_time) as step3_to_4_minutes
    FROM user_events e1
    JOIN user_events e2 ON e1.user_id = e2.user_id AND e2.event = 'add_to_cart'
    JOIN user_events e3 ON e1.user_id = e3.user_id AND e3.event = 'submit_order'
    JOIN user_events e4 ON e1.user_id = e4.user_id AND e4.event = 'pay_order'
    WHERE e1.event = 'view_product'
        AND e2.event_time >= e1.event_time
        AND e3.event_time >= e2.event_time
        AND e4.event_time >= e3.event_time
        AND e1.event_time >= DATE_SUB(NOW(), INTERVAL 7 DAY)
)
SELECT 
    '步骤1→2' as step_transition,
    AVG(step1_to_2_minutes) as avg_minutes,
    PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY step1_to_2_minutes) as median_minutes,
    PERCENTILE_CONT(0.9) WITHIN GROUP (ORDER BY step1_to_2_minutes) as p90_minutes
FROM step_durations

UNION ALL

SELECT 
    '步骤2→3',
    AVG(step2_to_3_minutes),
    PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY step2_to_3_minutes),
    PERCENTILE_CONT(0.9) WITHIN GROUP (ORDER BY step2_to_3_minutes)
FROM step_durations

UNION ALL

SELECT 
    '步骤3→4',
    AVG(step3_to_4_minutes),
    PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY step3_to_4_minutes),
    PERCENTILE_CONT(0.9) WITHIN GROUP (ORDER BY step3_to_4_minutes)
FROM step_durations;
```

### 6.2 漏斗路径分析

```python
# 多路径漏斗分析
def analyze_multiple_paths(events_df):
    """
    分析用户完成漏斗的多条路径
    """
    from collections import Counter
    
    # 找出所有可能的路径
    paths = []
    for user_id, group in events_df.groupby('user_id'):
        path = ' → '.join(group.sort_values('event_time')['event'].tolist())
        paths.append(path)
    
    # 统计路径频次
    path_counts = Counter(paths)
    
    # 返回最常见的路径
    return pd.DataFrame(
        path_counts.most_common(10),
        columns=['path', 'count']
    )
```

## 7. 最佳实践

### 7.1 漏斗设计原则

```
1. 步骤合理
   ├─ 步骤数量适中（4-7步）
   ├─ 每个步骤有意义
   └─ 顺序符合用户行为

2. 定义清晰
   ├─ 每个步骤定义明确
   ├─ 转化条件清晰
   └─ 时间窗口合理

3. 可操作性
   ├─ 识别可优化的环节
   ├─ 能指导具体行动
   └─ 可衡量优化效果

4. 持续监控
   ├─ 定期查看漏斗数据
   ├─ 发现异常及时分析
   └─ 持续优化迭代
```

### 7.2 常见误区

```
误区1：步骤过多
解决：合并相似步骤，聚焦关键环节

误区2：忽视时间窗口
解决：设置合理的时间限制

误区3：不细分分析
解决：按维度细分，找出问题群体

误区4：只看整体转化率
解决：关注每个步骤的转化率

误区5：忽略用户差异
解决：分用户群体分析漏斗
```

## 8. 总结

漏斗分析是优化转化路径的核心工具：

**核心价值**
- 识别流失环节
- 指导优化方向
- 评估优化效果

**实施要点**
- 定义清晰的漏斗步骤
- 设置合理的时间窗口
- 多维度细分分析
- 持续监控和优化

**优化策略**
- 优先优化流失率最高的环节
- 通过A/B测试验证效果
- 结合用户研究深入分析

记住：**漏斗分析的目的不是看数据，而是发现问题、优化转化、提升业务。**
