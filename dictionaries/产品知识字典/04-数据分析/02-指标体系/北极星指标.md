# 北极星指标（North Star Metric）

## 1. 定义

北极星指标（North Star Metric，NSM）是最能体现产品核心价值、反映企业长期增长方向的单一关键指标。它像北极星一样，为整个组织指引方向，统一各部门目标。

### 核心特征

```
北极星指标 = 核心价值 + 可衡量 + 可行动 + 长期导向

特征：
1. 反映用户价值：用户从产品获得的核心价值
2. 可衡量：能够量化追踪
3. 可行动：团队可以通过工作影响
4. 长期导向：不是短期虚荣指标
```

## 2. 北极星指标框架

### 2.1 选择标准

```
评估维度：

用户价值 ✓
├─ 用户是否真的关心？
├─ 是否反映用户获得的价值？
└─ 用户价值↑是否带来业务价值↑？

业务价值 ✓
├─ 是否与长期成功相关？
├─ 是否关联核心商业模式？
└─ 指标增长是否带来收入增长？

可操作性 ✓
├─ 团队是否能够影响？
├─ 是否有明确的优化路径？
└─ 能否分解为可执行的子指标？

稳定性 ✓
├─ 长期是否可持续？
├─ 短期波动是否可接受？
└─ 是否能经受住压力测试？
```

### 2.2 指标层级

```
           北极星指标（1个）
                ↓
         关键指标（3-5个）
                ↓
       支撑指标（10-20个）
                ↓
      过程指标（数十个）
```

```

### 3.3 B2B产品北极星

**SaaS产品**
```
Salesforce
北极星：客户终身价值（LTV）

关键指标：
├─ MRR（月度经常性收入）
├─ 客户流失率（Churn Rate）
└─ 客户获取成本（CAC）

SQL示例：
```sql
-- SaaS北极星监控
WITH monthly_metrics AS (
    SELECT 
        DATE_TRUNC('month', date) as month,
        SUM(mrr) as mrr,
        COUNT(DISTINCT CASE WHEN is_new_customer THEN customer_id END) as new_customers,
        COUNT(DISTINCT CASE WHEN is_churned THEN customer_id END) as churned_customers
    FROM saas_metrics
    WHERE date >= DATE_SUB(NOW(), INTERVAL 12 MONTH)
    GROUP BY DATE_TRUNC('month', date)
)
SELECT 
    month,
    mrr,
    new_customers,
    churned_customers,
    mrr - LAG(mrr) OVER (ORDER BY month) as mrr_growth,
    ROUND((mrr - LAG(mrr) OVER (ORDER BY month)) / LAG(mrr) OVER (ORDER BY month) * 100, 2) as growth_rate
FROM monthly_metrics
ORDER BY month DESC;
```

## 4. 确定北极星指标

### 4.1 方法论

**Step 1：明确产品价值**
```markdown
问题清单：
1. 用户为什么使用我们的产品？
2. 用户从产品获得的核心价值是什么？
3. 用户在什么情况下会推荐产品？
4. 什么行为代表用户真正认可产品？

示例：
产品：在线教育平台
用户价值：学习知识，提升能力
核心行为：完成课程学习
```

**Step 2：候选指标生成**
```markdown
头脑风暴候选指标：

候选1：注册用户数
- 优点：简单易懂
- 缺点：虚荣指标，不代表活跃

候选2：课程完成数
- 优点：反映学习价值
- 缺点：课程难度不一，难比较

候选3：学习时长
- 优点：体现用户投入
- 缺点：时长≠学习效果

候选4：获得证书数
- 优点：体现学习成果
- 缺点：覆盖用户有限
```

**Step 3：评估筛选**
```python
# 北极星指标评估矩阵

### 4.2 实战案例

**案例：确定短视频平台北极星**

**背景**：某短视频平台，日活5000万，需要确定北极星指标

**分析过程**：

```markdown
1. 产品价值分析
   - 核心价值：娱乐消遣
   - 用户价值：观看有趣视频
   - 创作者价值：内容变现

2. 候选指标
   - DAU（日活跃用户数）
   - 视频播放量
   - 用户平均使用时长
   - 视频互动数（点赞+评论+分享）

3. 数据分析
```sql
-- 对比候选指标与留存的关系
WITH user_metrics AS (
    SELECT 
        user_id,
        COUNT(*) as videos_watched,
        SUM(watch_duration) / 60 as total_minutes,
        SUM(likes + comments + shares) as interactions,
        MAX(CASE WHEN login_date BETWEEN '2024-02-01' AND '2024-02-28' THEN 1 ELSE 0 END) as retained
    FROM user_activity
    WHERE login_date >= '2024-01-01'
    GROUP BY user_id
)
SELECT 
    CASE 
        WHEN videos_watched < 10 THEN '0-10'
        WHEN videos_watched < 30 THEN '10-30'
        WHEN videos_watched < 50 THEN '30-50'
        ELSE '50+'
    END as video_bucket,
    COUNT(*) as users,
    AVG(retained) as retention_rate,
    AVG(interactions) as avg_interactions
FROM user_metrics
GROUP BY 
    CASE 
        WHEN videos_watched < 10 THEN '0-10'
        WHEN videos_watched < 30 THEN '10-30'
        WHEN videos_watched < 50 THEN '30-50'
        ELSE '50+'
    END
ORDER BY video_bucket;
```

4. 最终选择：**日视频观看时长**

   理由：
   - 与留存强相关（相关系数0.85）
   - 反映用户真实价值
   - 团队可优化（推荐算法、内容质量）
   - 稳定性好，不易被短期操纵
```

## 5. 北极星指标管理

### 5.1 监控体系

**SQL监控模板**
```sql
-- 北极星指标监控看板
WITH daily_nsm AS (
    SELECT 
        DATE(activity_time) as date,
        COUNT(DISTINCT user_id) as dau,
        SUM(watch_duration) / 3600 as total_hours,
        SUM(watch_duration) / 3600 / COUNT(DISTINCT user_id) as avg_hours_per_user
    FROM user_activity
    WHERE activity_time >= DATE_SUB(NOW(), INTERVAL 90 DAY)
    GROUP BY DATE(activity_time)
)
SELECT 
    date,
    dau,
    total_hours as nsm_value,
    avg_hours_per_user,
    total_hours - LAG(total_hours) OVER (ORDER BY date) as daily_change,
    ROUND((total_hours - LAG(total_hours, 7) OVER (ORDER BY date)) / LAG(total_hours, 7) OVER (ORDER BY date) * 100, 2) as wow_growth
FROM daily_nsm
ORDER BY date DESC
LIMIT 30;
```

**告警规则**
```python
# 北极星指标告警
def check_nsm_alert(current_value, historical_values):
    """
    检查北极星指标是否需要告警
    """
    alerts = []
    
    # 1. 绝对值告警
    if current_value < 1000000:  # 例如：低于100万小时
        alerts.append("⚠️ 北极星指标跌破底线阈值")
    
    # 2. 环比下降告警
    if len(historical_values) >= 7:
        wow_change = (current_value - historical_values[-7]) / historical_values[-7]
        if wow_change < -0.1:
            alerts.append(f"⚠️ 北极星指标周环比下降{abs(wow_change)*100:.1f}%")
    
    # 3. 趋势告警
    if len(historical_values) >= 30:
        recent_avg = np.mean(historical_values[-7:])
        previous_avg = np.mean(historical_values[-30:-7])
        if recent_avg < previous_avg * 0.95:
            alerts.append("⚠️ 北极星指标呈持续下降趋势")
    
    return alerts
```

## 6. 常见误区

### 6.1 错误的北极星

**虚荣指标**
```
❌ 注册用户数
- 不代表真实价值
- 无法指导行动

✓ 活跃用户数
- 反映真实使用
- 可优化增长
```

**短期指标**
```
❌ 日收入
- 短视，可能损害长期价值
- 容易被操纵

✓ 客户终身价值（LTV）
- 关注长期价值
- 引导健康增长
```

## 9. 总结

北极星指标是产品增长的核心指南针：

**核心原则**
- 反映用户真实价值
- 与长期成功相关
- 团队可影响优化
- 稳定且可衡量

**实施要点**
- 谨慎选择，充分验证
- 全员对齐，持续追踪
- 定期复盘，适时调整
- 关注健康度，避免短视

记住：**北极星指标不是一个数字，而是整个组织对"什么最重要"的共识。**
