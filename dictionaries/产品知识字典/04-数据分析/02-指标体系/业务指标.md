# 核心业务指标

## 1. 概述

业务指标是衡量企业经营状况和业务健康度的量化标准，涵盖用户、收入、运营、产品等多个维度。

### 指标体系框架

```
业务指标体系：

用户指标
├─ 规模指标：用户数量
├─ 活跃指标：用户活跃度
├─ 质量指标：用户价值
└─ 增长指标：用户增长

收入指标
├─ 规模指标：收入金额
├─ 效率指标：盈利能力
├─ 健康度指标：收入质量
└─ 增长指标：收入增长

运营指标
├─ 效率指标：运营效率
├─ 质量指标：服务质量
├─ 成本指标：成本控制
└─ 效果指标：活动效果

产品指标
├─ 使用指标：产品使用
├─ 体验指标：用户体验
├─ 质量指标：产品质量
└─ 创新指标：产品迭代
```

## 2. 用户指标

### 2.1 规模指标

**核心指标**

```
累计用户数
├─ 定义：历史累计注册用户总数
├─ 计算：COUNT(DISTINCT user_id)
└─ 用途：了解用户规模总量

新增用户数
├─ 定义：指定周期内新注册用户数
├─ 计算：COUNT(DISTINCT 新注册user_id)
└─ 用途：评估获客能力

活跃用户数
├─ DAU：日活跃用户数
├─ WAU：周活跃用户数
└─ MAU：月活跃用户数

计算公式：
活跃用户 = 在周期内有≥1次有效行为的用户数
```

**SQL示例**
```sql
-- 用户规模指标
SELECT 
    DATE(register_time) as date,
    COUNT(*) as new_users,
    SUM(COUNT(*)) OVER (ORDER BY DATE(register_time)) as total_users
FROM users
WHERE register_time >= DATE_SUB(NOW(), INTERVAL 30 DAY)
GROUP BY DATE(register_time)

UNION ALL

SELECT 
    DATE(activity_time) as date,
    COUNT(DISTINCT user_id) as dau,
    NULL as total_users
FROM user_activities
WHERE activity_time >= DATE_SUB(NOW(), INTERVAL 30 DAY)
GROUP BY DATE(activity_time)
ORDER BY date DESC;
```

### 2.2 活跃指标

**核心指标**

```
DAU/MAU比率（用户粘性）
├─ 定义：日活占月活的比例
├─ 计算：DAU / MAU
├─ 标准：20%以下低，20-50%中等，50%以上高
└─ 用途：衡量用户粘性

人均使用次数
├─ 定义：平均每个用户的使用次数
├─ 计算：总使用次数 / 活跃用户数
└─ 用途：评估用户活跃程度

人均使用时长
├─ 定义：平均每个用户的使用时长
├─ 计算：总使用时长 / 活跃用户数
└─ 用途：评估用户参与度
```

**SQL示例**
```sql
-- 活跃度分析
SELECT 
    DATE(activity_time) as date,
    COUNT(DISTINCT user_id) as dau,
    COUNT(*) as total_sessions,
    COUNT(*) / COUNT(DISTINCT user_id) as avg_sessions,
    AVG(session_duration) / 60 as avg_duration_min
FROM user_activities
WHERE activity_time >= DATE_SUB(NOW(), INTERVAL 30 DAY)
GROUP BY DATE(activity_time)
ORDER BY date DESC;
```

### 2.3 留存指标

**核心指标**

```
留存率
├─ 次日留存：Day 1 Retention
├─ 7日留存：Day 7 Retention
├─ 30日留存：Day 30 Retention
└─ 计算：(N日后回访用户数 / 新增用户数) × 100%

流失率
├─ 定义：用户不再使用产品的比例
├─ 计算：1 - 留存率
└─ 用途：评估用户流失情况

生命周期
├─ 定义：用户从开始使用到流失的平均时间
├─ 计算：平均(流失日期 - 注册日期)
└─ 用途：评估用户价值周期
```

**SQL示例**
```sql
-- 留存率计算
WITH user_first_date AS (
    SELECT 
        user_id,
        MIN(DATE(activity_time)) as first_date
    FROM user_activities
    WHERE activity_time >= DATE_SUB(NOW(), INTERVAL 60 DAY)
    GROUP BY user_id
),
user_retention AS (
    SELECT 
        f.user_id,
        f.first_date,
        DATEDIFF(DATE(a.activity_time), f.first_date) as day_diff
    FROM user_first_date f
    JOIN user_activities a ON f.user_id = a.user_id
    WHERE DATEDIFF(DATE(a.activity_time), f.first_date) IN (0, 1, 7, 30)
)
SELECT 
    first_date,
    COUNT(DISTINCT user_id) as total,
    COUNT(DISTINCT CASE WHEN day_diff = 0 THEN user_id END) as day0,
    COUNT(DISTINCT CASE WHEN day_diff = 1 THEN user_id END) as day1,
    COUNT(DISTINCT CASE WHEN day_diff = 7 THEN user_id END) as day7,
    COUNT(DISTINCT CASE WHEN day_diff = 30 THEN user_id END) as day30,
    ROUND(COUNT(DISTINCT CASE WHEN day_diff = 1 THEN user_id END) * 100.0 
        / COUNT(DISTINCT user_id), 2) as day1_rate,
    ROUND(COUNT(DISTINCT CASE WHEN day_diff = 7 THEN user_id END) * 100.0 
        / COUNT(DISTINCT user_id), 2) as day7_rate
FROM user_retention
GROUP BY first_date
ORDER BY first_date DESC;
```

## 3. 收入指标

### 3.1 规模指标

**核心指标**

```
GMV（Gross Merchandise Volume）
├─ 定义：商品交易总额
├─ 计算：∑(订单金额)
└─ 用途：衡量平台交易规模

Revenue（收入）
├─ 定义：实际收入
├─ 计算：∑(实际收入金额)
└─ 用途：衡量企业收入规模

ARPU（Average Revenue Per User）
├─ 定义：平均每用户收入
├─ 计算：总收入 / 总用户数
└─ 用途：评估用户平均贡献

ARPPU（Average Revenue Per Paying User）
├─ 定义：平均每付费用户收入
├─ 计算：总收入 / 付费用户数
└─ 用途：评估付费用户价值
```

**SQL示例**
```sql
-- 收入规模指标
SELECT 
    DATE_TRUNC('month', order_time) as month,
    SUM(amount) as gmv,
    SUM(CASE WHEN status = 'completed' THEN actual_revenue ELSE 0 END) as revenue,
    COUNT(DISTINCT user_id) as paying_users,
    SUM(amount) / COUNT(DISTINCT user_id) as arppu,
    SUM(CASE WHEN status = 'completed' THEN actual_revenue ELSE 0 END) 
        / (SELECT COUNT(DISTINCT) FROM users) as arpu
FROM orders
WHERE order_time >= DATE_SUB(NOW(), INTERVAL 12 MONTH)
GROUP BY DATE_TRUNC('month', order_time)
ORDER BY month DESC;
```

### 3.2 转化指标

**核心指标**

```
付费转化率
├─ 定义：付费用户占总用户比例
├─ 计算：(付费用户数 / 总用户数) × 100%
└─ 用途：评估变现能力

首次付费转化率
├─ 定义：新用户首次付费的比例
├─ 计算：(首次付费用户数 / 新增用户数) × 100%
└─ 用途：评估新用户变现能力

复购率
├─ 定义：重复购买用户占比
├─ 计算：(购买≥2次的用户数 / 总购买用户数) × 100%
└─ 用途：评估用户忠诚度
```

**SQL示例**
```sql
-- 付费转化分析
WITH user_payment AS (
    SELECT 
        user_id,
        MIN(DATE(order_time)) as first_pay_date,
        COUNT(*) as order_count
    FROM orders
    WHERE status = 'completed'
    GROUP BY user_id
)
SELECT 
    DATE_TRUNC('month', u.register_time) as month,
    COUNT(DISTINCT u.user_id) as new_users,
    COUNT(DISTINCT p.user_id) as paying_users,
    ROUND(COUNT(DISTINCT p.user_id) * 100.0 / COUNT(DISTINCT u.user_id), 2) as conversion_rate,
    COUNT(DISTINCT CASE WHEN p.order_count >= 2 THEN p.user_id END) as repeat_users,
    ROUND(COUNT(DISTINCT CASE WHEN p.order_count >= 2 THEN p.user_id END) * 100.0 
        / NULLIF(COUNT(DISTINCT p.user_id), 0), 2) as repeat_rate
FROM users u
LEFT JOIN user_payment p ON u.user_id = p.user_id
WHERE u.register_time >= DATE_SUB(NOW(), INTERVAL 12 MONTH)
GROUP BY DATE_TRUNC('month', u.register_time)
ORDER BY month DESC;
```

### 3.3 盈利指标

**核心指标**

```
毛利率（Gross Margin）
├─ 定义：销售收入扣除直接成本后的利润率
├─ 计算：(收入 - 成本) / 收入 × 100%
└─ 用途：评估产品盈利能力

净利率（Net Margin）
├─ 定义：净利润占收入的比率
├─ 计算：净利润 / 收入 × 100%
└─ 用途：评估企业盈利能力

ROI（Return on Investment）
├─ 定义：投资回报率
├─ 计算：(收益 - 成本) / 成本 × 100%
└─ 用途：评估投资效益
```

**SQL示例**
```sql
-- 盈利能力分析
SELECT 
    DATE_TRUNC('month', order_time) as month,
    SUM(amount) as revenue,
    SUM(cost) as cost,
    SUM(amount - cost) as profit,
    ROUND((SUM(amount - cost) / SUM(amount)) * 100, 2) as gross_margin,
    ROUND((SUM(amount - cost - operating_cost)) / SUM(amount) * 100, 2) as net_margin,
    ROUND((SUM(amount - cost) / SUM(marketing_cost)) * 100, 2) as roi
FROM orders o
JOIN financial_data f ON DATE_TRUNC('month', o.order_time) = f.month
WHERE o.order_time >= DATE_SUB(NOW(), INTERVAL 12 MONTH)
GROUP BY DATE_TRUNC('month', o.order_time)
ORDER BY month DESC;
```

## 4. 运营指标

### 4.1 获客指标

**核心指标**

```
CAC（Customer Acquisition Cost）
├─ 定义：获取一个用户的成本
├─ 计算：营销总成本 / 新增用户数
└─ 用途：评估获客效率

获客渠道占比
├─ 定义：各渠道用户来源占比
├─ 计算：各渠道用户数 / 总用户数
└─ 用途：评估渠道效果

渠道ROI
├─ 定义：各渠道的投资回报率
├─ 计算：渠道收入 / 渠道成本
└─ 用途：优化渠道投放
```

**SQL示例**
```sql
-- 获客成本分析
WITH channel_cost AS (
    SELECT 
        channel,
        SUM(cost) as total_cost
    FROM marketing_spend
    WHERE date >= DATE_SUB(NOW(), INTERVAL 30 DAY)
    GROUP BY channel
),
channel_users AS (
    SELECT 
        acquisition_channel as channel,
        COUNT(DISTINCT user_id) as new_users
    FROM users
    WHERE register_time >= DATE_SUB(NOW(), INTERVAL 30 DAY)
    GROUP BY acquisition_channel
)
SELECT 
    c.channel,
    c.total_cost,
    u.new_users,
    ROUND(c.total_cost / NULLIF(u.new_users, 0), 2) as cac
FROM channel_cost c
JOIN channel_users u ON c.channel = u.channel
ORDER BY c.total_cost DESC;
```

### 4.2 活动指标

**核心指标**

```
活动参与率
├─ 定义：参与活动用户占比
├─ 计算：(参与用户数 / 目标用户数) × 100%
└─ 用途：评估活动吸引力

活动转化率
├─ 定义：活动带来目标行为转化的比例
├─ 计算：(转化用户数 / 参与用户数) × 100%
└─ 用途：评估活动效果

活动ROI
├─ 定义：活动投入产出比
├─ 计算：活动收益 / 活动成本
└─ 用途：评估活动效益
```

**SQL示例**
```sql
-- 活动效果分析
SELECT 
    campaign_name,
    COUNT(DISTINCT CASE WHEN is_target = 1 THEN user_id END) as target_users,
    COUNT(DISTINCT CASE WHEN is_participated = 1 THEN user_id END) as participated_users,
    COUNT(DISTINCT CASE WHEN is_converted = 1 THEN user_id END) as converted_users,
    ROUND(COUNT(DISTINCT CASE WHEN is_participated = 1 THEN user_id END) * 100.0 
        / NULLIF(COUNT(DISTINCT CASE WHEN is_target = 1 THEN user_id END), 0), 2) as participation_rate,
    ROUND(COUNT(DISTINCT CASE WHEN is_converted = 1 THEN user_id END) * 100.0 
        / NULLIF(COUNT(DISTINCT CASE WHEN is_participated = 1 THEN user_id END), 0), 2) as conversion_rate
FROM campaign_data
WHERE campaign_date >= DATE_SUB(NOW(), INTERVAL 90 DAY)
GROUP BY campaign_name
ORDER BY conversion_rate DESC;
```

### 4.3 服务指标

**核心指标**

```
响应时间
├─ 定义：用户发起请求到首次响应的时间
├─ 计算：平均(首次响应时间 - 请求时间)
└─ 用途：评估服务响应速度

解决率
├─ 定义：问题得到解决的比例
├─ 计算：(已解决问题数 / 总问题数) × 100%
└─ 用途：评估服务质量

满意度（CSAT）
├─ 定义：用户满意度评分
├─ 计算：平均(满意度评分)
└─ 用途：评估用户满意程度

NPS（Net Promoter Score）
├─ 定义：净推荐值
├─ 计算：推荐者比例 - 贬损者比例
└─ 用途：评估用户忠诚度
```

**SQL示例**
```sql
-- 服务质量分析
SELECT 
    DATE_TRUNC('week', ticket_create_time) as week,
    COUNT(*) as total_tickets,
    AVG(TIMESTAMPDIFF(HOUR, ticket_create_time, first_response_time)) as avg_response_hours,
    SUM(CASE WHEN status = 'resolved' THEN 1 ELSE 0 END) as resolved,
    ROUND(SUM(CASE WHEN status = 'resolved' THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2) as resolution_rate,
    ROUND(AVG(satisfaction_score), 2) as avg_csat
FROM customer_tickets
WHERE ticket_create_time >= DATE_SUB(NOW(), INTERVAL 12 WEEK)
GROUP BY DATE_TRUNC('week', ticket_create_time)
ORDER BY week DESC;
```

## 5. 产品指标

### 5.1 使用指标

**核心指标**

```
功能使用率
├─ 定义：使用某功能的用户占比
├─ 计算：(使用功能用户数 / 活跃用户数) × 100%
└─ 用途：评估功能受欢迎程度

功能使用频次
├─ 定义：平均每个用户使用功能的次数
├─ 计算：功能使用总次数 / 使用功能用户数
└─ 用途：评估功能粘性

功能使用时长
├─ 定义：平均每次使用功能的时长
├─ 计算：总使用时长 / 使用总次数
└─ 用途：评估功能吸引力
```

**SQL示例**
```sql
-- 功能使用分析
SELECT 
    feature_name,
    COUNT(DISTINCT user_id) as unique_users,
    COUNT(*) as total_usage,
    COUNT(*) / COUNT(DISTINCT user_id) as avg_usage_per_user,
    AVG(usage_duration) / 60 as avg_duration_min,
    ROUND(COUNT(DISTINCT user_id) * 100.0 
        / (SELECT COUNT(DISTINCT) FROM user_activities WHERE activity_time >= DATE_SUB(NOW(), INTERVAL 30 DAY)), 2) as usage_rate
FROM feature_usage
WHERE usage_time >= DATE_SUB(NOW(), INTERVAL 30 DAY)
GROUP BY feature_name
ORDER BY unique_users DESC;
```

### 5.2 体验指标

**核心指标**

```
页面停留时长
├─ 定义：用户在页面的平均停留时间
├─ 计算：平均(离开时间 - 进入时间)
└─ 用途：评估页面吸引力

跳出率
├─ 定义：只访问一个页面就离开的比例
├─ 计算：(单页访问数 / 总访问数) × 100%
└─ 用途：评估页面相关性

完成率
├─ 定义：完成目标流程的用户比例
├─ 计算：(完成用户数 / 开始用户数) × 100%
└─ 用途：评估流程体验

错误率
├─ 定义：出现错误的请求比例
├─ 计算：(错误请求数 / 总请求数) × 100%
└─ 用途：评估产品稳定性
```

**SQL示例**
```sql
-- 页面体验分析
SELECT 
    page_name,
    COUNT(*) as page_views,
    COUNT(DISTINCT session_id) as sessions,
    AVG(time_on_page) as avg_time_seconds,
    SUM(CASE WHEN is_bounce = 1 THEN 1 ELSE 0 END) as bounces,
    ROUND(SUM(CASE WHEN is_bounce = 1 THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2) as bounce_rate,
    SUM(CASE WHEN has_error = 1 THEN 1 ELSE 0 END) as errors,
    ROUND(SUM(CASE WHEN has_error = 1 THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2) as error_rate
FROM page_views
WHERE view_time >= DATE_SUB(NOW(), INTERVAL 30 DAY)
GROUP BY page_name
ORDER BY page_views DESC;
```

### 5.3 质量指标

**核心指标**

```
崩溃率
├─ 定义：应用崩溃的比例
├─ 计算：(崩溃次数 / 启动次数) × 100%
└─ 用途：评估应用稳定性

加载时长
├─ 定义：页面或功能加载的平均时间
├─ 计算：平均(加载完成时间 - 开始加载时间)
└─ 用途：评估性能体验

Bug数量
├─ 定义：发现的Bug数量
├─ 计算：∑(Bug数)
└─ 用途：评估代码质量

修复时长
├─ 定义：Bug从发现到修复的平均时间
├─ 计算：平均(修复时间 - 发现时间)
└─ 用途：评估研发效率
```

## 7. 指标管理

### 7.1 指标监控

**SQL监控模板**
```sql
-- 指标监控查询
CREATE VIEW metric_monitoring AS
SELECT 
    'DAU' as metric_name,
    DATE(activity_time) as date,
    COUNT(DISTINCT user_id) as value,
    LAG(COUNT(DISTINCT user_id), 1) OVER (ORDER BY DATE(activity_time)) as yesterday,
    ROUND((COUNT(DISTINCT user_id) - LAG(COUNT(DISTINCT user_id), 1) OVER (ORDER BY DATE(activity_time))) 
        * 100.0 / LAG(COUNT(DISTINCT user_id), 1) OVER (ORDER BY DATE(activity_time)), 2) as mom_growth
FROM user_activities
WHERE activity_time >= DATE_SUB(NOW(), INTERVAL 30 DAY)
GROUP BY DATE(activity_time)

UNION ALL

SELECT 
    'GMV',
    DATE(order_time),
    SUM(amount),
    LAG(SUM(amount), 1) OVER (ORDER BY DATE(order_time)),
    ROUND((SUM(amount) - LAG(SUM(amount), 1) OVER (ORDER BY DATE(order_time))) 
        * 100.0 / LAG(SUM(amount), 1) OVER (ORDER BY DATE(order_time)), 2)
FROM orders
WHERE order_time >= DATE_SUB(NOW(), INTERVAL 30 DAY)
GROUP BY DATE(order_time);
```

### 7.2 异常告警

```python
# 指标异常检测
def detect_anomaly(metric_values, threshold=2):
    """
    异常检测：基于统计学方法
    """
    import numpy as np
    
    mean = np.mean(metric_values)
    std = np.std(metric_values)
    
    anomalies = []
    for i, value in enumerate(metric_values):
        z_score = (value - mean) / std if std > 0 else 0
        
        if abs(z_score) > threshold:
            anomalies.append({
                'index': i,
                'value': value,
                'z_score': z_score,
                'type': 'high' if z_score > 0 else 'low'
            })
    
    return anomalies
```

## 8. 最佳实践

### 8.1 指标选择原则

```
好的指标特征：

1. 可衡量
   └─ 能够量化，有明确数值

2. 可行动
   └─ 能够指导行动和决策

3. 可对比
   └─ 可以进行时间、维度对比

4. 可理解
   └─ 易于理解，业务含义清晰

5. 可信赖
   └─ 数据准确，来源可靠

避免的指标：

✗ 虚荣指标
  - 看起来好看但无实际意义
  - 例：累计注册用户数（不关注活跃）

✗ 后视镜指标
  - 只能看过去，无法预测未来
  - 例：历史总收入

✗ 复杂指标
  - 计算复杂，难以理解
  - 例：多因子加权综合指数
```

### 8.2 指标应用场景

```
不同场景关注不同指标：

战略决策
├─ 关注：北极星指标、长期趋势
├─ 频率：季度、年度
└─ 示例：GMV、MAU、LTV

业务运营
├─ 关注：运营指标、短期趋势
├─ 频率：日、周
└─ 示例：DAU、转化率、ROI

产品迭代
├─ 关注：产品指标、功能数据
├─ 频率：周、版本周期
└─ 示例：功能使用率、用户满意度

问题排查
├─ 关注：异常指标、细分数据
├─ 频率：实时
└─ 示例：错误率、响应时间
```

## 9. 总结

业务指标体系是企业数据驱动决策的基础：

**核心原则**
- 少即是多：聚焦关键指标
- 业务导向：服务于业务目标
- 层次分明：建立指标层级
- 持续优化：定期评估更新

**实施要点**
- 明确定义：确保理解一致
- 数据准确：保证数据质量
- 及时更新：保证数据时效
- 有效应用：指导业务决策

记住：**指标本身不是目的，而是辅助决策的工具。好的指标体系能够揭示问题、指导行动、驱动增长。**
