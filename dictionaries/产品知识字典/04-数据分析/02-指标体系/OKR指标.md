# OKR指标

## 1. 定义

OKR（Objectives and Key Results）是目标与关键结果管理方法，通过设定有挑战性的目标（O）和可衡量的关键结果（KR），聚焦重点、对齐方向、提升执行力。

### 核心要素

```
OKR = Objective（目标） + Key Results（关键结果）

Objective（O）：
- 定性的方向
- 有挑战性
- 鼓舞人心
- 时间周期内要达成

Key Results（KR）：
- 定量的指标
- 可衡量
- 可验证
- 支撑目标达成
```

## 2. OKR框架体系

### 2.1 基本原则

```
SMART原则（针对KR）：
S - Specific（具体）
M - Measurable（可衡量）
A - Achievable（可实现）
R - Relevant（相关性）
T - Time-bound（有时限）

OKR特点：
✓ 有挑战性：60%-70%达成率为佳
✓ 透明公开：全员可见
✓ 定期复盘：周/月/季度
✓ 不与薪酬直接挂钩
```

### 2.2 OKR层级

```
公司OKR（Company OKR）
    ↓ 对齐
部门OKR（Team OKR）
    ↓ 对齐
个人OKR（Individual OKR）
```

**对齐示例**
```
公司O：成为行业第一的电商平台
├─ KR1：GMV达到1000亿
├─ KR2：市场份额达到30%
└─ KR3：NPS达到60

├─ KR1：转化率从3%提升至5%（支撑公司KR1）
├─ KR2：用户满意度从4.0提升至4.5（支撑公司KR3）

└─ KR3：支付成功率从95%提升至98%
```

## 3. 制定OKR

### 3.1 目标（O）设定

**好的目标特征**
```
✓ 鼓舞人心
✓ 有挑战性
✓ 可理解
✓ 可行动
✓ 有价值

示例：
❌ 差：提高销售额
✓ 好：成为行业增长最快的电商平台

❌ 差：优化产品
✓ 好：打造让用户惊喜的购物体验

❌ 差：降低成本
✓ 好：实现行业领先的成本效率
```

**目标公式**
```
目标 = 动词 + 对象 + 结果

示例：
- 打造（动词）极致的购物体验（对象）提升用户满意度（结果）
- 建立（动词）数据驱动的增长体系（对象）实现快速增长（结果）
- 优化（动词）核心业务流程（对象）提升运营效率（结果）
```

### 3.2 关键结果（KR）设定

**好的KR特征**
```
✓ 可量化
✓ 可验证
✓ 有挑战性
✓ 有明确时限
✓ 支撑目标达成

类型：
1. 数量型：达到X数量
2. 增长型：提升X%
3. 比率型：达到X%
4. 里程碑：完成X项目
```

**KR公式**
```
KR = 动词 + 指标 + 从X到Y + 时间

示例：
- 提升（动词）7日留存率（指标）从30%到40%（从X到Y）
- 新增（动词）活跃用户（指标）从100万到150万（从X到Y）
- 完成（动词）系统重构（指标）从0到1（从X到Y）
```

**SQL示例：OKR进度追踪**
```sql
-- OKR进度追踪
WITH okr_progress AS (
    SELECT 
        objective_id,
        objective_name,
        key_result_id,
        key_result_name,
        target_value,
        current_value,
        (current_value / target_value) * 100 as completion_rate
    FROM okr_tracking
    WHERE quarter = '2024-Q1'
)
SELECT 
    objective_name,
    key_result_name,
    target_value,
    current_value,
    ROUND(completion_rate, 1) as completion_pct,
    CASE 
        WHEN completion_rate >= 100 THEN '✅ 已完成'
        WHEN completion_rate >= 70 THEN '🟢 进展良好'
        WHEN completion_rate >= 50 THEN '🟡 需要关注'
        ELSE '🔴 存在风险'
    END as status
FROM okr_progress
ORDER BY objective_id, key_result_id;
```

### 4.3 月度复盘

```python
# OKR月度复盘报告生成
def generate_okr_review(objective, key_results, progress_data):
    """
    生成OKR月度复盘报告
    """
    report = f"""
# {objective} - 月度复盘

## 1. 目标进展

### 关键结果完成情况
"""
    
    for kr in key_results:
        current = progress_data[kr['id']]['current']
        target = kr['target']
        rate = (current / target) * 100 if target > 0 else 0
        
        status = '🟢' if rate >= 70 else '🟡' if rate >= 50 else '🔴'
        
        report += f"""
#### {kr['name']}
- 目标：{target}
- 当前：{current}
- 完成率：{rate:.1f}% {status}
- 趋势：{progress_data[kr['id']]['trend']}
"""
    
    report += f"""

## 2. 主要成果
{progress_data['achievements']}

## 3. 遇到的挑战
{progress_data['challenges']}

## 4. 下月计划
{progress_data['next_month_plan']}

## 5. 需要的支持
{progress_data['support_needed']}
"""
    
    return report
```

**SQL示例：指标树查询**
```sql
-- OKR指标树查询
WITH metric_tree AS (
    SELECT 
        'GMV' as metric_name,
        NULL as parent_metric,
        SUM(amount) as current_value,
        100000000 as target_value,
        '2024-Q1' as period
    FROM orders
    WHERE quarter = '2024-Q1'
    
    UNION ALL
    
    SELECT 
        '用户数贡献' as metric_name,
        'GMV' as parent_metric,
        COUNT(DISTINCT user_id) * AVG(order_amount) as current_value,
        60000000 as target_value,
        '2024-Q1' as period
    FROM orders
    WHERE quarter = '2024-Q1'
    
    UNION ALL
    
    SELECT 
        '转化率贡献' as metric_name,
        'GMV' as parent_metric,
        COUNT(DISTINCT user_id) * 0.05 * AVG(order_amount) as current_value,
        20000000 as target_value,
        '2024-Q1' as period
    FROM user_visits v
    LEFT JOIN orders o ON v.user_id = o.user_id
    WHERE quarter = '2024-Q1'
)
SELECT 
    metric_name,
    parent_metric,
    current_value,
    target_value,
    ROUND((current_value / target_value) * 100, 1) as completion_rate
FROM metric_tree
ORDER BY parent_metric NULLS FIRST, metric_name;
```

## 6. OKR常见问题

### 6.1 目标设定问题

**问题1：目标过于保守**
```
❌ 错误：
O：完成日常工作
KR1：按时完成项目
KR2：不出大问题

✓ 正确：
O：打造行业标杆产品
KR1：用户满意度达到4.5分（当前3.8）
KR2：市场份额提升5个百分点
```

**问题2：目标过于激进**
```
❌ 错误：
O：实现100倍增长
KR1：用户从100万到1亿
KR2：收入从1000万到10亿

✓ 正确：
O：实现快速增长
KR1：用户增长100%（100万到200万）
KR2：收入增长80%（1000万到1800万）
```

### 6.2 KR设定问题

**问题1：KR不可衡量**
```
❌ 错误：
KR：提升用户体验
KR：优化产品功能
KR：加强团队协作

✓ 正确：
KR：用户满意度从3.8提升至4.5
KR：核心功能使用率提升30%
KR：跨部门协作满意度达到4.0
```

**问题2：KR过多**
```
❌ 错误：每个O有8-10个KR

✓ 正确：每个O有3-5个KR
- 保持聚焦
- 突出重点
- 便于追踪
```

```sql
CREATE TABLE okr_tracking (
    okr_id VARCHAR(50) PRIMARY KEY,
    quarter VARCHAR(10),
    department VARCHAR(50),
    objective TEXT,
    key_result TEXT,
    metric_name VARCHAR(100),
    baseline_value DECIMAL(15,2),
    target_value DECIMAL(15,2),
    current_value DECIMAL(15,2),
    owner VARCHAR(50),
    status VARCHAR(20),
    last_updated TIMESTAMP
);

-- 更新OKR进度
UPDATE okr_tracking
SET 
    current_value = (
        SELECT COUNT(DISTINCT user_id)
        FROM users
        WHERE register_date >= '2024-01-01'
    ),
    status = CASE 
        WHEN current_value / target_value >= 0.7 THEN 'on_track'
        WHEN current_value / target_value >= 0.5 THEN 'at_risk'
        ELSE 'behind'
    END,
    last_updated = NOW()
WHERE metric_name = '新增用户数' AND quarter = '2024-Q1';
```

## 10. 总结

OKR是强大的目标管理工具：

**核心价值**
- 聚焦：明确优先级
- 对齐：上下目标一致
- 透明：全员可见
- 挑战：激发潜能

**实施要点**
- 目标要有激励性
- 结果要可衡量
- 定期追踪复盘
- 与绩效适度分离

**成功关键**
- 高层重视
- 文化支撑
- 持续执行
- 工具赋能

记住：**OKR不是考核工具，而是聚焦和对齐的管理方法。**
