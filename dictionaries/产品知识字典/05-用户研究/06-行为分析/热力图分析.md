# 热力图分析

## 概述
热力图(Heatmap)是通过颜色编码直观展示用户行为数据分布的可视化方法,能够快速识别用户关注的区域、点击模式和浏览行为。

## 核心价值

### 相比数字报表
```
热力图:
- 直观可视化
- 快速定位问题
- 无需技术背景
- 一眼看出模式

数字报表:
- 需要解读
- 难以发现位置问题
- 需要分析能力
```

### 应用场景
```
设计优化:
- 识别热点区域
- 发现盲区
- 评估视觉层次

转化优化:
- CTA位置优化
- 内容布局调整
- 表单优化

可用性评估:
- 导航效率
- 信息查找
- 误点击分析
```

## 热力图类型

### 1. 点击热力图(Click Map)
```
定义: 显示用户点击行为的分布

数据:
- 点击次数
- 点击位置
- 点击密度

颜色编码:
- 红色: 高点击区域
- 黄色: 中等点击
- 绿色: 低点击区域
- 无色: 无点击

应用:
- 按钮效果评估
- 链接吸引力
- 可点击元素识别
- 误点击发现
```

### 2. 注意力热力图(Attention Map)
```
定义: 显示用户停留和关注的区域

数据:
- 停留时长
- 鼠标悬停时间
- 浏览时长

颜色编码:
- 红色: 高关注
- 黄色: 中等关注
- 绿色: 低关注
- 无色: 未关注

应用:
- 内容吸引力
- 视觉层次评估
- 重要信息位置
- 文本阅读模式
```

### 3. 滚动热力图(Scroll Map)
```
定义: 显示页面滚动深度分布

数据:
- 滚动到达率
- 平均滚动深度
- 停止滚动位置

颜色编码:
- 红色: 100%用户看到
- 黄色: 50%用户看到
- 绿色: 20%用户看到
- 无色: 很少用户看到

应用:
- 首屏内容评估
- 内容位置优化
- 折叠线影响
- 页面长度决策
```

### 4. 移动热力图(Move Map)
```
定义: 显示鼠标移动轨迹

数据:
- 鼠标路径
- 悬停时间
- 移动速度

应用:
- 用户浏览模式
- 犹豫区域识别
- 注意力流向
```

### 5. 触摸热力图(Touch Map) - 移动端
```
定义: 显示移动端触摸行为

数据:
- 触摸位置
- 触摸次数
- 手势类型

应用:
- 移动端优化
- 按钮大小
- 触摸区域
```

## 使用场景

### 1. 首页优化

#### 案例场景
```
问题: 首页转化率低

分析步骤:
1. 查看点击热力图
   → 发现用户点击了非链接图片
   
2. 查看滚动热力图
   → 发现80%用户未看到底部CTA
   
3. 查看注意力热力图
   → 发现轮播图被忽视

优化措施:
1. 为热门图片添加链接
2. 将CTA上移到80%可见区域
3. 简化首屏,突出核心内容
```

### 2. 表单优化

#### 案例场景
```
问题: 表单填写率低

热力图发现:
1. 点击热力图: 某字段点击率低
2. 注意力热力图: 字段标签被忽视
3. 停留热力图: 某字段停留时间长

可能原因:
- 字段不清晰
- 字段太多
- 验证问题

优化措施:
- 简化字段
- 优化标签
- 添加提示说明
```

### 3. 落地页优化

#### 案例场景
```
问题: 广告落地页转化率低

分析:
1. 点击热力图: CTA点击率低
2. 滚动热力图: 50%用户未看到CTA
3. 注意力热力图: 文本被忽视,图片受关注

优化:
1. CTA位置上移
2. 增加视觉引导到CTA
3. 减少文本,增强视觉元素
```

### 4. 内容页面优化

#### 案例场景
```
问题: 内容消费深度不够

分析:
1. 滚动热力图: 用户在30%处停止
2. 注意力热力图: 首屏停留时间短
3. 点击热力图: 推荐内容点击少

优化:
1. 优化首屏吸引力
2. 调整内容结构
3. 推荐内容位置调整
```

## 分析方法

### 1. 整体浏览
```
步骤:
1. 观察整体模式
2. 识别热点区域
3. 发现冷点区域
4. 寻找意外模式

问题:
- 热点区域符合预期吗?
- 哪些区域被忽视?
- 有意外的高/低关注区域吗?
```

### 2. 分群对比
```
维度:
- 新用户 vs 老用户
- 转化用户 vs 未转化用户
- 高价值 vs 低价值用户
- 不同流量来源

发现:
- 不同群体的行为差异
- 针对性优化方向
```

### 3. A/B对比
```
方法:
- 对比A/B版本的差异
- 验证设计假设
- 选择最优方案

示例:
版本A: CTA在右侧
版本B: CTA在中央
对比点击热力图,看哪个CTR更高
```

### 4. 时间序列
```
方法:
- 对比不同时段
- 发现趋势变化
- 评估优化效果

示例:
- 优化前的热力图
- 优化后的热力图
- 对比变化
```

### 5. 设备对比
```
维度:
- 桌面端 vs 移动端
- iOS vs Android
- 不同屏幕尺寸

发现:
- 设备差异
- 针对性优化
```

## 指标解读

### 1. 点击率(CTR)
```
定义: 点击次数 / 页面浏览量

基准(参考):
- 主要CTA: 2-5%
- 导航链接: 5-15%
- 内容链接: 1-3%

分析:
- 高于基准 → 效果好
- 低于基准 → 需优化
```

### 2. 滚动深度
```
定义: 用户滚动到的页面位置

基准(参考):
- 100%可见: 首屏
- 75%可见: 前两屏
- 50%可见: 页面中部
- 25%可见: 页面底部

分析:
- 重要内容应在高可见区域
- 折叠线下方内容需谨慎
```

### 3. 停留时间
```
定义: 用户在某区域停留时长

分析:
- 停留长: 感兴趣 or 困惑
- 停留短: 不感兴趣 or 一目了然

结合:
- 内容类型判断
- 其他数据验证
```

### 4. 活跃度
```
定义: 页面交互的活跃程度

指标:
- 点击密度
- 互动次数
- 活跃区域比例

分析:
- 高活跃: 页面有效
- 低活跃: 页面问题
```

## 最佳实践

### 1. 数据收集

#### 样本量
```
最小样本:
- 点击热力图: 1000+ PV
- 滚动热力图: 500+ PV
- 注意力热力图: 2000+ PV

原则:
- 样本太小 → 模式不稳定
- 样本足够 → 可靠结论
```

#### 时间范围
```
建议:
- 至少7天数据
- 覆盖完整周期
- 排除异常时期

考虑:
- 工作日vs周末
- 节假日影响
- 营销活动期间
```

#### 重要维度
```
用户类型:
- 新用户

流量来源:
- 自然搜索

设备:
- 桌面端
- 移动端
```

## 常见误区

### ❌ 误区1: 红色=好
```
错误认知: 红色区域就是好

正确理解:
- 红色只是表示高频
- 需要判断是否符合预期
- 可能是误点击

示例:
错误位置的大量点击(红色)是问题而非好现象
```

### ❌ 误区2: 单独使用
```
错误做法: 只看热力图做决策

正确做法:
- 结合其他数据
- 验证假设
- 了解原因
```

### ❌ 误区3: 忽视样本量
```
错误做法: 少量样本就下结论

正确做法:
- 确保样本充足
- 验证模式稳定
- 分时段验证
```

### ❌ 误区4: 过度解读
```
错误做法: 对每个细节过度解读

正确做法:
- 关注主要模式
- 聚焦可行动的洞察
- 结合业务目标
```

## 案例分析

### 案例: 电商详情页优化

**背景**
某电商详情页转化率3%,低于行业平均5%

**热力图分析**

#### 1. 点击热力图发现
```
发现:
- 主图点击率高(45%)
- "加入购物车"点击率低(8%)
- 大量点击在无链接的空白区

洞察:
- 用户期望主图可放大
- CTA不够醒目
- 布局存在误导
```

#### 2. 滚动热力图发现
```
发现:
- 100%看到: 商品图+价格
- 50%看到: 商品详情
- 20%看到: 用户评价

洞察:
- 评价可见性太低
- 重要内容应上移
```

#### 3. 注意力热力图发现
```
发现:
- 高关注: 商品图、价格
- 中关注: 标题
- 低关注: 促销标签、评价

洞察:
- 促销信息被忽视
- 评价入口不明显
```

#### 优化措施
```
1. 主图增加放大功能
2. CTA按钮颜色更醒目
3. 评价入口上移到50%可见区域
4. 促销标签位置调整
```

#### 优化效果
```
转化率: 3% → 4.2%
提升: 40%
```
