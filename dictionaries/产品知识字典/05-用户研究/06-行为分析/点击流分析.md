# 点击流分析

## 概述
点击流分析(Clickstream Analysis)是通过收集和分析用户在网站或应用中的点击行为序列,理解用户浏览路径、行为模式和转化过程的方法。

## 核心价值

### 数据特点
```
点击流数据:
- 序列性: 有时间顺序
- 关联性: 行为前后关联
- 细粒度: 精确到每次交互
- 大规模: 数据量大
- 真实性: 客观行为记录
```

### 应用价值
```
用户洞察:
- 理解浏览模式
- 发现行为规律
- 识别用户意图

产品优化:
- 导航优化
- 内容布局
- 转化路径优化

营销分析:
- 流量来源效果
- 营销活动评估
- 用户归因

个性化:
- 推荐系统
- 个性化内容
- 行为预测
```

### 2. 数据字段

#### 标准字段
```
标识字段:

时间字段:
- timestamp: 精确时间戳

- time: 时间

事件字段:
- event_type: 事件类型
- event_name: 事件名称
- event_category: 事件类别

页面字段:
- page_url: 页面URL
- page_path: 页面路径
- page_title: 页面标题
- referrer: 来源页面

设备字段:
- platform: 平台
- device_type: 设备类型

- os: 操作系统
- screen_resolution: 屏幕分辨率

地理字段:
```

## 数据处理

### 1. 数据清洗

#### 数据质量问题
```
缺失值:
- 字段缺失
- 值为空

异常值:
- 非法字符
- 超出范围
- 格式错误

重复值:
- 重复发送
- 重试导致

无效数据:
- 机器人流量
- 测试数据
- 异常用户
```

#### 清洗步骤
```
1. 去重
   - 基于event_id去重
   - 基于user_id+timestamp去重

2. 过滤
   - 过滤机器人(User Agent识别)
   - 过滤内部IP
   - 过滤测试账号

3. 填充
   - 缺失值填充
   - 默认值设置

4. 转换
   - 时间格式统一
   - 字段名称统一
   - 编码转换
```

### 2. 会话识别

#### 会话定义
```
会话(Session):
用户在一段时间内连续的交互序列

会话切分规则:
- 时间间隔: 30分钟无活动(常用)
- 午夜切分: 跨天新会话
- 来源变化: 来源改变新会话
```

#### 会话识别方法
```
基于Cookie:
- 会话ID存储在Cookie
- 每次请求携带

基于时间间隔:
- 相邻请求间隔>阈值→新会话

示例:
时间     页面      间隔   会话
10:00    首页      -     1
10:02    列表      2min  1
10:05    详情      3min  1
11:00    首页      55min 2  ← 新会话
```

### 3. 用户识别

#### 跨设备识别挑战
```
问题:
- 同一用户多设备
- Cookie清除
- 无痕模式
- App+Web

影响:
- 用户数虚高
- 路径断裂
- 归因不准
```

#### 识别方法
```
确定性识别:
- 登录账号
- 统一账号体系

概率性识别:
- 设备指纹
- 行为特征
- 机器学习

混合方法:
- 有登录用登录
- 无登录用概率
```

## 分析方法

### 1. 路径分析

#### 基本路径分析
```
目的: 了解用户浏览路径

方法:
- 统计路径频次
- 构建路径图
- 识别常见模式

示例:
路径                频次    占比
首页→列表→详情      5000    25%
首页→搜索→详情      3000    15%
直接访问详情         2000    10%
...
```

#### 桑基图(Sankey Diagram)
```
可视化流向:

       首页(100%)
      /    |    \
   列表(40%) 搜索(30%) 直接(20%)
     |        |         |
   详情(90%) 详情(85%)  详情(70%)
     |________|_________|
           |
        购买(40%)

优点:
- 直观展示流向
- 发现主要路径
- 识别流失点
```

#### 路径挖掘
```
分析维度:
- 起始页面分析
- 退出页面分析
- 路径长度分析
- 路径深度分析

示例:
- 最常见起始页: 首页(60%), 搜索(20%)
- 最常见退出页: 详情页(30%), 购物车(15%)
- 平均路径长度: 5.2页
- 平均访问深度: 3.5层
```

### 2. 漏斗分析

#### 漏斗构建
```
步骤:
1. 定义转化目标
2. 设定漏斗步骤
3. 统计各步骤人数
4. 计算转化率

示例: 电商购买漏斗
步骤         人数    转化率
浏览商品     10000   100%
加入购物车    3000    30%
去结算       2000    67%
提交订单      1500    75%
支付成功      1200    80%

整体转化率: 12%
```

#### 漏斗细分
```
细分维度:
- 流量来源
- 用户类型
- 设备类型
- 时间段
- 地理位置

发现:
- 不同群体差异
- 优化方向
```

#### 漏斗分析技巧
```
转化时间:
- 步骤间耗时
- 整体转化时间
- 识别瓶颈

漏斗形状:
- 线性漏斗
- 分支漏斗
- 循环漏斗

流失分析:
- 流失点识别
- 流失原因分析
- 挽回策略
```

### 3. 归因分析

#### 归因模型
```
单触点归因:
- 最后点击归因: 最后一个触点100%功劳
- 首次点击归因: 第一个触点100%功劳

多触点归因:
- 线性归因: 所有触点平分
- 时间衰减: 越接近转化权重越高
- 位置归因: 首尾权重高

数据驱动归因:
- 基于机器学习
- 动态计算权重
```

#### 归因应用
```
营销评估:
- 渠道效果评估
- ROI计算
- 预算分配

示例:
转化路径: 搜索→社交→直接→转化
线性归因: 各33.3%功劳
最后点击: 直接100%功劳
时间衰减: 搜索20% 社交30% 直接50%
```

### 4. 序列模式挖掘

#### 频繁序列挖掘
```
目的: 发现频繁出现的行为序列

算法:
- Apriori算法
- PrefixSpan算法
- GSP算法

示例:
频繁序列           支持度
(浏览,加购,购买)    15%
(搜索,浏览,加购)    12%
(首页,分类,详情)    10%
```

#### 序列规则挖掘
```
目的: 发现行为之间的关联规则

指标:
- 支持度(Support): A→B同时出现概率
- 置信度(Confidence): A出现时B出现概率
- 提升度(Lift): 规则有效性

示例:
规则: 浏览价格页 → 购买
支持度: 10%
置信度: 30%
提升度: 2.5

解读: 浏览价格页后购买概率是平均的2.5倍
```

### 5. 用户分群

#### 基于行为的分群
```
分群维度:
- 访问频率
- 浏览深度
- 功能使用
- 转化行为

方法:
- K-means聚类
- 层次聚类
- RFM模型

示例分群:
- 高价值用户: 高频+高转化
- 潜力用户: 高频+低转化
- 流失风险: 低频+低转化
```

## 案例分析

### 案例1: 电商转化路径优化

**背景**
某电商平台想提升转化率

**数据收集**
```
时间范围: 30天
数据量: 1000万事件
用户数: 50万
```

**路径分析发现**
```
主要转化路径:
1. 首页(100%) → 列表(60%) → 详情(40%) → 加购(15%) → 购买(8%)
2. 搜索(100%) → 结果(85%) → 详情(50%) → 加购(18%) → 购买(10%)
3. 直接详情(100%) → 加购(25%) → 购买(12%)

发现:
- 搜索路径转化率更高
- 直接访问详情转化最高
- 首页→列表有流失
```

**漏斗分析**
```
浏览 → 加购 → 下单 → 支付
100%   18%    60%    80%

发现:
- 浏览到加购流失严重(82%)
- 加购到下单有40%流失
```

**优化措施**
```
1. 首页增加搜索入口显著性
2. 列表页优化,提升点击率
3. 加购后弹窗引导下单
4. 下单页简化流程
```

**优化效果**
```
整体转化率: 8% → 10.5%
提升: 31%
```

### 案例2: 内容消费路径分析

**背景**
某内容平台想提升内容消费深度

**路径分析**
```
用户旅程:
首页 → 推荐列表 → 内容页 → 相关推荐 → 内容页...

发现:
- 70%用户只看1篇文章
- 30%用户会看相关推荐
- 平均会话: 2.3页
```

**序列模式挖掘**
```
频繁模式:
- 首页→娱乐→娱乐: 20%
- 首页→科技→科技: 15%
- 首页→娱乐→科技: 5%

发现:
- 用户倾向于同类内容
- 跨品类浏览较少
```

**优化措施**
```
1. 强化相关推荐
2. 文末增加更多同类内容
3. 个性化推荐优化
```

**效果**
```
平均会话页数: 2.3 → 3.5
人均阅读时长: 5分钟 → 8分钟
```
