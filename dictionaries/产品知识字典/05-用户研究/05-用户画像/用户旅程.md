# 用户旅程

## 概述
用户旅程(Customer Journey/UX Journey)是用户为实现特定目标而与产品/服务交互的完整过程,包括所有接触点、情感体验和关键事件。

## 核心概念

### 关键要素
```
用户(Who): 谁在经历旅程
目标(Why): 要达成什么目标
阶段(When): 经历哪些阶段
接触点(Where): 在哪里交互
行为(What): 做了什么
情感(How): 什么感受
痛点(Pain): 遇到什么问题
机会(Opportunity): 如何改进
```

### 与相关概念的区别

#### 用户旅程 vs 用户流程
```
用户旅程:
- 全渠道、多触点
- 包含情感和体验
- 时间跨度大
- 关注整体体验

用户流程:
- 单一产品/功能
- 操作步骤
- 范围较小
- 关注任务完成
```

#### 用户旅程 vs 服务蓝图
```
用户旅程:
- 用户视角
- 前台体验
- What & How

服务蓝图:
- 组织视角
- 前后台支持
- Who & How
```

## 旅程类型

### 1. 按时间跨度

#### 当前状态旅程(Current State)
```
定义: 描述用户现在的体验

目的:
- 识别问题
- 发现机会
- 优化改进

时间: 过去-现在
```

#### 未来状态旅程(Future State)
```
定义: 描述理想的用户体验

目的:
- 愿景对齐
- 创新探索
- 目标设定

时间: 未来
```

#### 全生命周期旅程(Lifecycle)
```
定义: 用户从认知到离开的完整历程

阶段:
认知 → 考虑 → 购买 → 使用 → 忠诚 → 流失

时间: 完整生命周期
```

### 2. 按场景范围

#### 核心旅程(Core Journey)
```
定义: 完成核心任务的旅程

示例:
- 购物旅程: 浏览→选择→下单→支付→收货→评价
- 注册旅程: 访问→注册→验证→完善→使用
```

#### 特定场景旅程(Scenario-based)
```
定义: 特定场景下的旅程

示例:
- 新用户首次使用
- 用户遇到问题求助
- 用户升级会员
```

### 3. 按用户类型

#### 单一画像旅程
```
定义: 特定用户画像的旅程

特点:
- 深度具体
- 个性化强

示例: "忙碌宝妈小美的购物旅程"
```

#### 多画像旅程
```
定义: 多类用户的对比旅程

特点:
- 发现差异
- 识别共性
```

### 1. 用户与场景
```
用户画像:
- 基于用户画像
- 明确角色特征

场景与目标:
- 具体场景描述
- 清晰的目标
- 旅程范围
```

### 2. 阶段划分

#### 典型阶段模型

**AISAS模型(营销)**
```
Attention(注意) → Interest(兴趣) → Search(搜索) → Action(行动) → Share(分享)
```

**5E模型(体验)**
```
Entice(吸引) → Enter(进入) → Engage(参与) → Exit(离开) → Extend(延伸)
```

**双钻模型(设计)**
```
Discover → Define → Develop → Deliver
```

**电商购物旅程**
```
认知 → 搜索 → 浏览 → 比较 → 决策 → 购买 → 使用 → 评价 → 复购
```

**SaaS产品旅程**
```
认知 → 试用 → 转化 → 使用 → 扩展 → 续费
```

### 3. 用户行为
```
记录内容:
- 具体行动
- 操作步骤
- 决策过程
- 信息需求

层次:
- 宏观行为: 访问网站
- 中观行为: 浏览商品
- 微观行为: 点击按钮
```

### 4. 接触点(Touchpoints)
```
类型:
- 线上: 网站、App、社交媒体、邮件
- 线下: 门店、电话、活动
- 人际: 客服、销售、朋友推荐

记录:
- 渠道类型
- 具体平台
- 交互方式
```

### 5. 情感体验
```
情感维度:
- 满意-不满
- 兴奋-沮丧
- 自信-困惑
- 轻松-紧张

可视化:
- 情感曲线图
- 表情符号
- 颜色编码

关键点:
- 情感高峰(Peak)
- 情感低谷(Trough)
- 结束时刻(End)
```

### 6. 痛点与机会

#### 痛点类型
```
功能痛点:
- 功能缺失
- 性能问题
- 不可用

体验痛点:
- 操作复杂
- 信息不清
- 等待时间长

情感痛点:
- 沮丧
- 困惑
- 失望
```

#### 机会识别
```
来源:
- 解决痛点
- 提升体验
- 创造惊喜

方向:
- 新功能
- 流程优化
- 服务提升
- 情感设计
```

## 创建流程

### 标准流程
```
1. 确定目标
   ↓
2. 用户研究
   ↓
3. 定义阶段
   ↓
4. 收集数据
   ↓
5. 绘制旅程
   ↓
6. 识别洞察
   ↓
7. 机会建议
   ↓
8. 分享应用
```

### 1. 确定目标

#### 明确目的
```
问题:
- 为什么要做旅程地图?
- 要解决什么问题?
- 谁会使用?

目的示例:
- 优化购物转化率
- 减少用户流失
- 发现服务断点
```

#### 确定范围
```
范围选择:
- 核心旅程还是全生命周期
- 单渠道还是全渠道
- 单画像还是多画像

边界定义:
- 起点和终点
- 包含/排除内容
```

### 2. 用户研究

#### 数据收集
```
定性方法:
- 深度访谈
- 焦点小组
- 日记研究
- 实地观察

定量方法:
- 问卷调查
- 行为数据分析
- NPS调研

数据类型:
- 行为数据
- 情感反馈
- 痛点抱怨
- 期望建议
```

#### 样本量
```
定性: 5-15人/画像
定量: 50-100人/画像
```

### 3. 定义阶段

#### 阶段划分方法
```
自上而下:
- 使用行业标准模型
- 根据业务流程划分

自下而上:
- 从用户行为归纳
- 识别自然分界点

建议: 结合两者
```

#### 阶段数量
```
建议: 4-8个阶段

原则:
- 足够具体,有区分度
- 不过细,保持清晰
```

### 5. 绘制旅程

#### 可视化工具
```
传统工具:
- 白板+便利贴
- 纸笔手绘

数字工具:
- Miro
- MURAL
- Figma
- UXPressia
- Smaply
```

#### 协作工作坊
```
参与者:
- 产品经理
- 设计师
- 研究员
- 运营人员
- 技术人员

流程:
1. 介绍背景和目标
2. 分享研究发现
3. 分组绘制初稿
4. 汇报讨论
5. 整合完善
6. 确认共识
```

### 6. 识别洞察

#### 分析维度
```
模式识别:
- 共性痛点
- 高峰低谷
- 掉线断点

根因分析:
- 为什么出现这个问题?
- 根本原因是什么?

情感分析:
- 哪些阶段体验好?
- 哪些阶段体验差?
- 为什么?
```

### 7. 提出建议

#### 建议框架
```
每个机会点:
1. 问题描述: 当前是什么情况
2. 影响: 对用户和业务的影响
3. 建议: 如何改进
4. 优先级: 高/中/低
5. 负责人: 谁来负责
```

### 8. 分享应用

#### 分享方式
```
形式:
- 工作坊展示
- 海报张贴
- 在线文档
- 演示汇报

频率:
- 定期回顾
- 新人培训
- 项目启动时
```

## 案例分析

### 案例: 在线教育用户旅程

**用户画像**: 在职学习者小王

**场景与目标**
```
场景: 小王想提升职场技能,寻找在线课程学习
目标: 找到合适的课程,完成学习,获得证书
```

**旅程地图**

| 阶段 | 认知 | 搜索 | 评估 | 购买 | 学习 | 完成 |
|------|------|------|------|------|------|------|
| **行为** | 看到广告 | 搜索引擎 | 浏览课程 | 比价 | 观看视频 | 考试 |
| | 朋友推荐 | 应用商店 | 看评论 | 咨询客服 | 做作业 | 获证书 |
| | | 访问网站 | 试看 | 支付 | 笔记 | 评价 |
| **接触点** | 社交媒体 | 搜索引擎 | 课程页 | 支付页 | App | 考试页 |
| | 朋友圈 | App商店 | 评论区 | 客服 | 网页 | 证书页 |
| **思考** | "想提升" | "有哪些平台" | "这个好吗" | "贵不贵" | "能坚持吗" | "学到了吗" |
| **情感** | 😟 焦虑 | 😕 困惑 | 🤔 犹豫 | 😬 纠结 | 😊 充实 | 😄 满足 |
| **痛点** | 不知学什么 | 平台太多 | 信息不全面 | 价格敏感 | 时间不够 | 证书认可度 |
| **机会** | 职业测评 | 智能推荐 | 免费试学 | 分期付款 | 碎片学习 | 求职对接 |

**关键洞察**
```
1. 认知阶段: 需求模糊,需要引导
2. 搜索阶段: 信息过载,选择困难
3. 评估阶段: 信任建立是关键
4. 购买阶段: 价格敏感,需降低门槛
5. 学习阶段: 坚持是最大挑战
6. 完成阶段: 价值感需要强化
```

**改进建议**
```
高优先级:
1. 职业规划测评工具(认知阶段)
2. 课程推荐算法优化(搜索阶段)
3. 学习进度提醒和激励(学习阶段)

中优先级:
4. 分期付款选项(购买阶段)
5. 免费试学延长(评估阶段)

低优先级:
6. 证书展示优化(完成阶段)
```

## 高级应用

### 1. 服务蓝图
```
定义: 描绘前后台服务支持系统的可视化工具

结构:
- 用户行为
- 前台接触
- 后台支持
- 支持过程

关系:
用户旅程 → 服务蓝图
(What) → (How & Who)
```

### 2. 旅程编排(Journey Orchestration)
```
定义: 实时动态优化用户旅程

能力:
- 实时感知
- 智能决策
- 个性化响应
- 全渠道协同

应用:
- 营销自动化
- 个性化推荐
- 智能客服
```

### 3. 旅程分析(Journey Analytics)
```
定义: 量化分析用户旅程

方法:
- 路径分析
- 转化漏斗
- 流失分析
- 归因分析

工具:
- Google Analytics
- Mixpanel
- Amplitude
```

## 常见问题

### 1. 旅程地图有多详细?
```
原则:
- 足够详细以发现洞察
- 不过于复杂难以使用

检查:
- 能识别问题吗?
- 能指导行动吗?
```

### 2. 单一还是多画像?
```
单一画像:
- 深度理解
- 个性化洞察

多画像:
- 对比差异
- 识别共性

建议: 主要画像做深度,必要时对比次要画像
```

### 3. 如何保持更新?
```
时机:
- 产品重大变更
- 用户反馈变化
- 定期回顾(半年/年)

方法:
- 持续收集反馈
- 定期验证更新
```

## 最佳实践

### ✅ 应该做
- 基于真实用户研究
- 跨职能协作创建
- 关注情感体验
- 识别具体可行动的机会
- 持续使用和更新

### ❌ 避免做
- 凭空想象
- 一次性文档
- 只关注功能流程
- 过于复杂
- 创建后不用
