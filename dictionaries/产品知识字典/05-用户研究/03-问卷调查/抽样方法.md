# 抽样方法

## 概述
抽样是从总体中选取部分样本进行研究的方法,使研究结果能够推断总体。合理的抽样是保证研究代表性和可靠性的关键。

## 核心概念

### 基本术语
- **总体(Population)**: 研究对象的全体
- **样本(Sample)**: 从总体中抽取的部分
- **抽样框(Sampling Frame)**: 总体的实际清单
- **抽样单元(Sampling Unit)**: 被抽取的基本单位
- **参数(Parameter)**: 总体的特征值
- **统计量(Statistic)**: 样本的特征值

### 抽样误差与非抽样误差

```
抽样误差:
- 由抽样引起的误差
- 可计算、可控制
- 随样本量增大而减小

非抽样误差:
- 覆盖误差: 抽样框不完整
- 无应答误差: 被选中者未回应
- 测量误差: 问题设计不当
- 处理误差: 数据录入错误
```

## 概率抽样

### 1. 简单随机抽样(Simple Random Sampling)

**方法**
- 每个个体被抽中的概率相等
- 使用随机数表或随机函数

**步骤**
```
1. 建立完整抽样框
2. 确定样本量n
3. 使用随机数生成器抽取
4. 记录被抽中个体
```

**优点**
- 统计理论完善
- 计算简单
- 无偏估计

**缺点**
- 需要完整名单
- 可能分布不均
- 实施成本可能较高

**适用场景**
- 总体较小
- 有完整名单
- 个体差异不大

**示例**
```
总体: 某公司1000名员工
样本量: 100人
方法: 给每员工编号001-1000,随机抽取100个号码
```

### 2. 分层抽样(Stratified Sampling)

**方法**
- 先将总体分层
- 层内随机抽样
- 保证各层代表性

**步骤**
```
1. 确定分层标准(如年龄、地区)
2. 将总体分为若干层
3. 确定各层样本量
4. 各层内随机抽样
```

**样本量分配**
```
比例分配:
n_h = (N_h / N) × n

其中:
n_h = 第h层样本量
N_h = 第h层总体量
N = 总体总量
n = 总样本量

最优分配(考虑层内方差):
n_h = n × (N_h × S_h) / Σ(N_h × S_h)
```

**优点**
- 保证各层代表性
- 提高估计精度
- 可比较各层结果

**缺点**
- 需要分层信息
- 分层标准选择关键
- 操作相对复杂

**适用场景**
- 层间差异大
- 层内差异小
- 需要分层分析

**示例**
```
总体: 全国用户10000人
分层: 
- 一线城市 3000人
- 二线城市 4000人
- 三四线城市 3000人

样本量: 500人
比例分配:
- 一线: (3000/10000) × 500 = 150人
- 二线: (4000/10000) × 500 = 200人
- 三四线: (3000/10000) × 500 = 150人
```

### 3. 整群抽样(Cluster Sampling)

**方法**
- 将总体分成若干群
- 随机抽取部分群
- 对选中群全部调查

**步骤**
```
1. 划分群(如学校、社区)
2. 随机抽取部分群
3. 对选中群全面调查
```

**优点**
- 不需要全部名单
- 实施方便
- 成本较低

**缺点**
- 精度相对较低
- 群间差异影响大

**适用场景**
- 地理分散
- 缺乏完整名单
- 实地调查

**示例**
```
总体: 某市100所小学的所有学生
抽样: 随机抽取10所小学
调查: 被抽中学校的所有学生
```

### 4. 系统抽样(Systematic Sampling)

**方法**
- 按固定间隔抽取
- 随机起点

**步骤**
```
1. 确定样本量n和总体N
2. 计算间隔 k = N/n
3. 随机选择起点r (1 ≤ r ≤ k)
4. 抽取 r, r+k, r+2k, ...
```

**优点**
- 简单易行
- 样本分布均匀

**缺点**
- 周期性可能导致偏差
- 不完全随机

**示例**
```
总体: 1000人
样本: 100人
间隔: k = 1000/100 = 10
起点: 随机选择3
样本: 3, 13, 23, 33, ..., 993
```

## 非概率抽样

### 1. 便利抽样(Convenience Sampling)

**方法**
- 选取容易接触的对象
- 如街头拦截、网络问卷

**优点**
- 简单快速
- 成本低

**缺点**
- 代表性差
- 有偏估计

**适用场景**
- 探索性研究
- 预测试
- 资源有限时

### 2. 判断抽样(Judgmental Sampling)

**方法**
- 研究者根据判断选择
- 选择典型或极端案例

**优点**
- 针对性强
- 专家判断

**缺点**
- 主观性强
- 难以推广

**适用场景**
- 专家调研
- 特殊群体
- 案例研究

### 3. 配额抽样(Quota Sampling)

**方法**
- 按特征分配配额
- 在配额内便利抽样

**优点**
- 保证关键特征分布
- 相对简便

**缺点**
- 非随机
- 仍可能有偏

**示例**
```
配额:
- 男性50人,女性50人
- 18-25岁40人,26-35岁40人,36岁以上20人
在配额内,方便招募
```

### 4. 滚雪球抽样(Snowball Sampling)

**方法**
- 先找少数对象
- 请他们推荐更多对象
- 像滚雪球一样扩大

**优点**
- 适合特殊群体
- 信任度高

**缺点**
- 可能有同质性
- 代表性有限

**适用场景**
- 隐蔽群体
- 稀有群体
- 社会网络研究

## 样本量计算

### 基本公式

#### 估计比例
```
n = (Z² × p × (1-p)) / e²

其中:
n = 样本量
Z = 置信水平Z值(95%:1.96, 99%:2.58)
p = 预期比例(未知时取0.5)
e = 允许误差

示例:
置信水平: 95% (Z=1.96)
预期比例: 50% (p=0.5)
允许误差: 5% (e=0.05)

n = (1.96² × 0.5 × 0.5) / 0.05²
  = 384.16
  ≈ 385人
```

#### 估计均值
```
n = (Z² × σ²) / e²

其中:
σ = 总体标准差(需估计或预调查)
```

### 常用样本量速查

| 置信水平 | 允许误差 | 样本量(无限总体) |
|---------|---------|-----------------|
| 95% | 1% | 9604 |
| 95% | 3% | 1067 |
| 95% | 5% | 385 |
| 95% | 7% | 196 |
| 95% | 10% | 96 |
| 99% | 5% | 664 |

### 有限总体校正
```
当总体N不是远大于样本n时:

n_adj = n / (1 + (n-1)/N)

示例:
总体 N = 1000
计算样本 n = 385
校正后: n_adj = 385 / (1 + 384/1000) = 278
```

### 分层抽样样本量
```
总样本量 n 计算同上
各层样本量按比例分配:
n_h = (N_h / N) × n
```

### 实际考虑
```
理论样本量
  × 响应率倒数
  × 设计效应
  = 实际需要的样本量

示例:
理论: 385人
响应率: 50% → ×2 = 770人
设计效应: 1.5 → ×1.5 = 1155人
```

## 抽样实施流程

```
1. 定义总体
   - 目标总体
   - 调查总体
   
2. 确定抽样框
   - 列出总体清单
   - 评估完整性
   
3. 选择抽样方法
   - 根据研究目的
   - 考虑资源限制
   
4. 计算样本量
   - 统计要求
   - 实际可行
   
5. 实施抽样
   - 严格按方法
   - 记录过程
   
6. 评估样本
   - 响应率
   - 代表性
   - 误差评估
```

## 抽样质量评估

### 响应率
```
响应率 = 有效问卷数 / 接触总数 × 100%

提高响应率:
- 合理激励
- 多次提醒
- 简化问卷
- 个性化邀请
```

### 代表性检验
```
对比样本与总体:
- 人口统计特征
- 已知关键指标
- 抽样框覆盖率
```

### 加权调整
```
样本分布与总体不符时:

权重 w_h = N_h% / n_h%

示例:
总体男性50%,样本男性40%
男性权重 = 50% / 40% = 1.25

加权后:
男性平均 = 样本男性平均 × 1.25
```

## 案例分析

### 案例1: 全国用户满意度调研

**研究背景**
某互联网公司想了解全国用户的满意度

**总体定义**
- 目标总体: 全国所有注册用户
- 调查总体: 过去6个月活跃用户(200万)

**抽样设计**
```
方法: 分层抽样 + 系统抽样

分层标准:
- 城市等级: 一线/二线/三四线
- 用户等级: 普通会员/VIP/SVIP

样本量:
- 置信水平: 95%
- 允许误差: 3%
- 理论样本: 1067
- 考虑响应率40%: 2668
- 最终: 3000人
```

**实施结果**
- 发送邀请: 3000人
- 有效回收: 1250人(41.7%)
- 加权调整后代表性良好

### 案例2: 新功能测试用户招募

**研究背景**
需要招募用户进行新功能可用性测试

**抽样设计**
```
方法: 配额抽样

配额标准:
- 用户类型: 新用户20人,老用户10人
- 设备类型: iOS 15人,Android 15人
- 年龄段: 18-30岁20人,31-45岁10人

招募方式:
- 产品内弹窗邀请
- 用户池筛选
- 用户推荐
```

## 最佳实践

### ✅ 应该做
- 明确定义总体
- 选择合适方法
- 合理计算样本量
- 提高响应率
- 评估代表性

### ❌ 避免做
- 忽视抽样框缺陷
- 样本量不足
- 响应率过低
- 过度推广结论
- 忽视非抽样误差
