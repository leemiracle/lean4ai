# 深度访谈

## 概述
深度访谈(In-depth Interview)是一对一的定性研究方法,通过开放式提问和深度追问,挖掘用户潜在需求、态度和行为动机。

## 核心特点

### 与普通访谈的区别
| 维度 | 深度访谈 | 普通访谈 |
|------|---------|---------|
| 时长 | 60-90分钟 | 15-30分钟 |
| 提问 | 开放式为主 | 封闭式为主 |
| 追问 | 深度挖掘 | 表面了解 |
| 关系 | 平等对话 | 问答关系 |
| 目的 | 探索发现 | 信息收集 |

### 适用场景
- 探索用户深层需求
- 理解复杂决策过程
- 挖掘情感和动机
- 了解个人经历故事
- 敏感话题研究

## 访谈类型

### 1. 结构化访谈
- 预设固定问题
- 按顺序提问
- 便于比较
- 灵活性低

### 2. 半结构化访谈 ⭐推荐
- 准备访谈提纲
- 灵活调整顺序
- 深度追问
- 平衡控制与灵活

### 3. 非结构化访谈
- 仅有主题
- 自然对话
- 高度灵活
- 难以比较

## 准备工作

### 1. 研究设计
```
明确研究目的
  ↓
定义目标用户
  ↓
确定访谈主题
  ↓
设计访谈提纲
  ↓
准备访谈材料
```

### 2. 访谈提纲设计
**基本原则**
- 从宽泛到具体
- 从易到难
- 逻辑流畅
- 灵活可调

**典型结构**
```
1. 开场寒暄(5分钟)
   - 自我介绍
   - 研究说明
   - 建立信任

2. 背景了解(10分钟)
   - 基本信息
   - 使用习惯
   - 总体态度

3. 核心问题(40分钟)
   - 按主题展开
   - 深度追问
   - 具体案例

4. 总结收尾(10分钟)
   - 补充问题
   - 道感谢
   - 说明后续
```

### 3. 参与者招募
**筛选标准**
- 人口统计学特征
- 使用经验程度
- 行为特征
- 态度倾向

**招募数量**
- 探索性: 5-10人
- 验证性: 10-20人
- 饱和原则: 无新发现

**招募渠道**
- 现有用户库
- 社交媒体
- 推荐滚雪球
- 第三方招募

## 访谈技巧

### 1. 提问技巧

#### 开放式提问
```
❌ 你喜欢这个功能吗?
✅ 你觉得这个功能怎么样?

❌ 你每周用几次?
✅ 你通常怎么使用它?

❌ 这个设计好不好?
✅ 能说说你对这个设计的看法吗?
```

#### 追问技巧
```
基本追问:
- 能具体说说吗?
- 还有呢?
- 能举个例子吗?

深度追问:
- 为什么这么说?
- 当时怎么想的?
- 现在怎么看?

情感追问:
- 当时什么感受?
- 这对你意味着什么?
- 最在意的是什么?
```

#### 对比提问
```
- 这和之前用的有什么不同?
- 理想状态是什么样的?
- 如果能改变,你希望是什么样?
```

### 2. 倾听技巧

#### 主动倾听
- 保持眼神接触
- 点头微笑回应
- 适当时复述确认
- 不要打断

#### 同理倾听
- 站在对方角度
- 理解情感层面
- 不带评判
- 表示理解

### 3. 控制技巧

#### 时间控制
- 柔性提醒
- "我们还有15分钟..."
- 重要问题优先
- 合理取舍

#### 话题控制
- 承接过渡
- "说到这个,我想起..."
- 巧妙拉回
- "我们回到刚才的话题"

### 4. 建立关系

#### 破冰技巧
- 轻松话题开始
- 找共同点
- 真诚关心
- 幽默风趣

#### 维持关系
- 保持好奇
- 表示尊重
- 适当自我披露
- 感谢欣赏

## 常见问题处理

### 1. 受访者话太少
**原因分析**
- 性格内向
- 不理解问题
- 有顾虑
- 话题不感兴趣

**应对策略**
- 耐心等待
- 换种方式提问
- 分享类似经历
- 激发兴趣点

### 2. 受访者话太多
**原因分析**
- 表达欲强
- 偏离主题
- 需要倾诉

**应对策略**
- 礼貌打断
- 适时总结
- 引导回主题
- 限定范围

### 3. 回答表面化
**原因分析**
- 习惯性思维
- 未深入思考
- 安全回答

**应对策略**
- 多次追问
- 要求具体案例
- 情境还原
- 对比提问

### 4. 出现矛盾
**原因分析**
- 记忆偏差
- 社会期许
- 态度改变

**应对策略**
- 委婉指出
- 询问原因
- 不评判
- 记录两种说法

## 数据记录

### 1. 录音录像
**录音要点**
- 提前获得许可
- 备用录音设备
- 测试音质
- 完整录制

**录像考虑**
- 丰富非语言信息
- 隐私顾虑更大
- 设备要求高
- 分析更复杂

### 2. 现场笔记
**记录内容**
- 关键语句(原话)
- 非语言行为
- 情绪变化
- 研究者思考
- 追问点

**笔记格式**
```
时间 | 主题 | 内容 | 观察 | 反思
----|------|------|------|------
10:05|使用习惯|"我每天都用"|表情自然|习惯性使用?
```

## 数据分析

### 1. 转录
- 逐字转录
- 标注语气
- 记录停顿
- 保留原话

### 2. 编码
**开放式编码**
- 通读转录稿
- 标记有意义片段
- 初步命名
- 形成码号

**轴心编码**
- 聚类相关码号
- 识别范畴
- 建立联系

**选择性编码**
- 提取核心范畴
- 构建故事线
- 理论饱和

### 3. 主题提取
```
原始数据 → 编码 → 范畴 → 主题 → 理论
```

## 案例分析

### 案例: 在线教育用户需求研究

**研究背景**
某在线教育平台想了解成人学习者需求

**访谈设计**
- 样本: 15名在职学习者
- 时长: 60分钟
- 形式: 视频访谈
- 报酬: 200元/人

**访谈提纲(节选)**
```
1. 学习背景
   - 为什么想学?
   - 之前的学习经历?

2. 平台使用
   - 怎么知道我们的?
   - 使用频率如何?
   - 在什么场景学习?

3. 课程体验
   - 选课时考虑什么?
   - 学习过程中遇到什么困难?
   - 最满意/不满意的是什么?

4. 期望建议
   - 理想的学习体验是什么样?
   - 有什么建议?
```

**关键发现**
1. 时间碎片化是最大痛点
2. 互动性需求被低估
3. 进度可视化激励学习
4. 证书价值认可度分化

**实践建议**
- 开发微课模块
- 增加直播答疑
- 优化进度展示
- 提供多类型认证

## 质量控制

### 信度提升
- 多研究者交叉验证
- 详细记录分析过程
- 参与者反馈确认

### 效度提升
- 长期深入接触
- 三角验证
- 同行评审
- 丰富的描述

## 伦理考量

### 知情同意
- 说明研究目的
- 告知录音录像
- 明确数据用途
- 强调自愿性

### 隐私保护
- 匿名处理
- 数据安全存储
- 限制访问权限
- 及时删除

### 避免伤害
- 敏感话题谨慎
- 尊重受访者
- 允许中止
- 提供支持

## 最佳实践

### ✅ 应该做
- 充分准备
- 建立信任
- 保持中立
- 深度追问
- 及时记录

### ❌ 避免做
- 带着预设
- 引导回答
- 频繁打断
- 表达评判
- 只听不问

## 常见问题FAQ

**Q: 访谈多少人合适?**
A: 达到饱和为准,通常10-20人,复杂问题可能需要更多。

**Q: 线上还是线下访谈?**
A: 各有优劣,根据研究目的和资源选择。线下更深入,线上更便利。

**Q: 如何处理"我不知道"的回答?**
A: 换个角度提问,提供情境,或跳过该问题。

**Q: 访谈可以中途调整提纲吗?**
A: 可以且应该,根据新发现动态调整。
