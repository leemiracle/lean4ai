# 访谈提纲

## 概述
访谈提纲(Interview Guide)是访谈中使用的结构化问题清单,帮助研究者系统地探索研究主题,同时保持足够的灵活性以深入挖掘。

## 核心作用

### 对研究者
- 保证覆盖关键话题
- 维持访谈结构
- 减少遗漏
- 提升专业度

### 对研究
- 便于跨访谈比较
- 提高数据质量
- 确保研究信度

## 提纲类型

### 1. 高度结构化
```
特点:
- 固定问题顺序
- 标准化表述
- 限制追问

优点:
- 可比性强
- 易于分析
- 对研究者要求低

缺点:
- 灵活性差
- 可能错过重要发现
- 互动不自然

适用:
- 多研究者协作
- 需要严格比较
- 初级研究者
```

### 2. 半结构化 ⭐推荐
```
特点:
- 核心问题固定
- 顺序可调整
- 允许深度追问

优点:
- 平衡结构与灵活
- 兼顾深度与广度
- 适应性较强

缺点:
- 需要一定经验
- 分析相对复杂

适用:
- 大多数用户研究
- 探索性研究
```

### 3. 低结构化
```
特点:
- 仅有主题清单
- 对话式展开
- 高度灵活

优点:
- 深度探索
- 发现意外
- 自然互动

缺点:
- 可比性弱
- 依赖研究者能力
- 分析困难

适用:
- 深度探索性研究
- 经验丰富的研究者
```

## 提纲设计流程

```
步骤1: 明确研究目的
  ↓
步骤2: 梳理核心问题
  ↓
步骤3: 设计问题框架
  ↓
步骤4: 编写具体问题
  ↓
步骤5: 内部评审
  ↓
步骤6: 试访谈
  ↓
步骤7: 优化调整
```

## 基本结构

### 1. 开场部分(5-10分钟)

**目的**
- 建立信任
- 说明规则
- 放松氛围

**内容**
```
- 自我介绍
- 研究目的说明
- 隐私承诺
- 录音许可
- 时长说明
- 问题解答

开场话术示例:
"您好,我是XX公司的研究员[名字]。感谢您今天抽出时间参与访谈。这次访谈主要是想了解您对XX产品的使用体验,您的反馈对我们很重要。访谈大约需要60分钟,过程中您可以随时提问或中止。我会做录音以便后续整理,所有信息仅用于研究,会严格保密。您有什么问题吗?"
```

### 2. 热身部分(5-10分钟)

**目的**
- 打破冰
- 了解背景
- 建立关系

**内容**
```
- 个人基本情况
- 与主题相关的背景
- 轻松话题

热身问题示例:
- 能简单介绍一下您自己吗?
- 您平时使用XX产品/服务的频率如何?
- 是什么吸引您开始使用的?
```

### 3. 核心部分(40-60分钟)

**目的**
- 深入探索主题
- 获取关键信息

**组织方式**
- 按主题模块划分
- 从宽泛到具体
- 从简单到复杂

**内容框架**
```
模块1: [主题A]
- 核心问题1.1
- 追问角度
- 备选问题

模块2: [主题B]
- 核心问题2.1
- 追问角度
- 备选问题

...
```

### 4. 收尾部分(10分钟)

**目的**
- 补充遗漏
- 总结确认
- 结束访谈

**内容**
```
- 开放补充问题
- 总结确认关键点
- 感谢与后续说明
- 联系方式交换(可选)

收尾话术示例:
"我们的访谈快结束了。还有什么想补充的吗?或者有什么问题想问我?[等待]非常感谢您的分享,这些信息对我们非常有价值。如果后续有需要补充的,可以联系您吗?谢谢您的时间!"
```

## 问题设计原则

### 1. 开放性
```
❌ 您喜欢这个功能吗?
✅ 您觉得这个功能怎么样?

❌ 您每天使用多长时间?
✅ 能描述一下您典型的使用场景吗?

❌ 这个设计好不好?
✅ 看到这个设计,您的第一反应是什么?
```

### 2. 中立性
```
❌ 您觉得这个优秀的设计怎么样?
✅ 您如何评价这个设计?

❌ 使用这个功能时您是否遇到困难?
✅ 使用这个功能时有什么感受?

❌ 大多数用户都喜欢这个功能,您呢?
✅ 您对这个功能有什么看法?
```

### 3. 具体性
```
❌ 您使用体验如何?
✅ 能举个例子说说最近一次使用的经历吗?

❌ 您对客服满意吗?
✅ 上次联系客服是什么情况?怎么解决的?

❌ 这个功能重要吗?
✅ 如果没有这个功能,会对您有什么影响?
```

### 4. 清晰性
```
❌ 您对产品的信息架构和交互设计有什么看法?
✅ 我们分开聊,先说说您对页面布局的感觉?

❌ 您认为UX/UI设计需要改进吗?
✅ (避免专业术语,使用通俗表达)

❌ 您使用过程中遇到的挑战和障碍是什么?
✅ 使用过程中有什么不顺手的地方吗?
```

## 问题类型及用法

### 1. 描述性问题
**作用**: 了解事实和经历
```
示例:
- 您平时怎么使用这个产品?
- 能描述一下您的工作流程吗?
- 上次遇到问题是什么时候?
```

### 2. 感受性问题
**作用**: 了解情绪和态度
```
示例:
- 当时您什么感觉?
- 这对您意味着什么?
- 这让您最在意的是什么?
```

### 3. 行为性问题
**作用**: 了解具体行为
```
示例:
- 您通常怎么做?
- 遇到这种情况,您会如何处理?
- 上次您具体是怎么操作的?
```

### 4. 意见性问题
**作用**: 了解评价和看法
```
示例:
- 您怎么评价这个功能?
- 您认为理想的情况应该是什么样?
- 对比A和B,您更倾向哪个?为什么?
```

### 5. 知识性问题
**作用**: 了解认知和理解
```
示例:
- 您理解这个功能是做什么的吗?
- 这个图标您觉得代表什么意思?
```

### 6. 投射性问题
**作用**: 了解潜意识和深层态度
```
示例:
- 如果这个产品是一个人,会是什么样的人?
- 如果给这个App起个名字,您会叫它什么?
- 这个品牌让您联想到什么?
```

### 7. 假设性问题
**作用**: 探索可能性和期望
```
示例:
- 如果能改变一个功能,您希望是什么?
- 如果没有时间限制,您会怎么做?
- 如果这个功能被移除,对您有什么影响?
```

## 追问技巧

### 为什么要追问
- 获取更深层信息
- 避免表面化回答
- 理解真实含义

### 追问类型

#### 基础追问
```
- 能具体说说吗?
- 还有呢?
- 比如?
- 能举个例子吗?
```

#### 深度追问
```
- 为什么这么说?
- 当时怎么想的?
- 这对您意味着什么?
- 是什么让您有这样的感受?
```

#### 澄清追问
```
- 您的意思是...?
- 可以再解释一下吗?
- 我理解的对吗: [复述]?
```

#### 情感追问
```
- 当时心情怎么样?
- 这让您感觉如何?
- 这个对您来说重要吗?为什么?
```

#### 对比追问
```
- 和之前用的比,有什么不同?
- 理想状态是什么样的?
- 这和您期望的有什么差距?
```

#### 极端追问
```
- 最让您满意的是什么?
- 最不满意的是什么?
- 如果打分,您打几分?为什么?
```

- 灵活调整顺序
- 控制时间节奏

## 试访谈

### 目的
- 测试问题是否清晰
- 评估时间是否合理
- 发现潜在问题
- 优化提纲

### 方法
1. 找同事扮演受访者
2. 或招募1-2名真实用户
3. 访谈后评估
4. 记录需要改进的地方

### 评估要点
- 哪些问题让人困惑?
- 哪些问题引不出有用信息?
- 时间分配是否合理?
- 逻辑是否流畅?
- 是否有遗漏?

## 常见问题与解决

### 1. 问题太多
**症状**: 时间不够用

**解决**
- 区分核心/次要问题
- 精简合并相似问题
- 控制每部分时长
- 准备必问清单

### 2. 问题太抽象
**症状**: 回答空泛

**解决**
- 增加具体场景
- 要求举例子
- 结合实际经历

### 3. 问题有倾向性
**症状**: 引导性回答

**解决**
- 中立化表述
- 避免评价性词语
- 同事评审

### 4. 逻辑不流畅
**症状**: 跳跃性强

**解决**
- 重新排序
- 增加过渡问题
- 画流程图检查

## 最佳实践

### ✅ 应该做
- 以研究目的为导向
- 试用测试
- 灵活使用
- 持续优化
- 记录经验

### ❌ 避免做
- 问题堆砌
- 一成不变
- 严格照读
- 忽视受访者
- 事后不反思
