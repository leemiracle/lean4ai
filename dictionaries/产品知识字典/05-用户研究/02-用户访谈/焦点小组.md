# 焦点小组

## 概述
焦点小组(Focus Group)是通过有组织的群体讨论,收集参与者对特定主题的观点、态度和经验的定性研究方法。小组互动产生的群体动力能激发更多样化的观点。

## 核心特点

### 与深度访谈对比
| 维度 | 焦点小组 | 深度访谈 |
|------|---------|---------|
| 参与人数 | 6-8人 | 1人 |
| 时长 | 90-120分钟 | 60-90分钟 |
| 互动性 | 群体互动 | 一对一 |
| 信息类型 | 公开观点 | 私密深度 |
| 成本效率 | 较高 | 较低 |
| 适用话题 | 一般话题 | 敏感话题 |

### 优势
- 群体互动激发观点
- 观察共识与分歧
- 收集多样视角
- 效率较高
- 生动有趣

### 局限
- 社会赞许效应
- 主导者影响
- 隐私话题受限
- 组织难度大
- 数据分析复杂

## 适用场景

### ✅ 适合
- 探索新产品概念
- 了解群体态度
- 发现共同需求
- 观察群体决策
- 品牌形象研究
- 创意激发讨论

### ❌ 不适合
- 敏感隐私话题
- 专业性很强的话题
- 需要深度个体挖掘
- 参与者地位差异大
- 已有强烈共识话题

## 小组构成

### 理想规模
- 标准规模: 6-8人
- 小型小组: 4-5人
- 大型小组: 9-12人(较少用)

**原则**: 既能充分讨论,又不至于失控

### 同质性与异质性

#### 同质性分组
- 相似背景特征
- 便于建立信任
- 深入特定群体

#### 异质性分组
- 多样化背景
- 激发不同观点
- 发现差异需求

**实践建议**: 单场小组保持适度同质,多场小组间保持差异

### 常见分组维度
- 人口统计学(年龄、性别、收入)
- 使用经验(新手、专家、流失用户)
- 行为特征(高频、低频、不使用)
- 态度倾向(喜欢、中立、不喜欢)

## 组织流程

### 1. 前期准备(1-2周)

#### 研究设计
```
明确研究目的
  ↓
定义目标群体
  ↓
确定小组数量(通常3-6组)
  ↓
设计讨论提纲
  ↓
选择场地设备
```

#### 参与者招募
**筛选标准**
- 目标用户特征
- 无市场研究经验
- 与主持人/客户无关联
- 表达能力良好
- 愿意参与讨论

**招募人数**
- 实际需要6-8人
- 招募8-10人
- 考虑临时缺席

**确认提醒**
- 初次邀请
- 前一天电话确认
- 当天短信提醒

### 2. 现场组织(2小时)

#### 场地布置
```
观察室(单向镜):
- 客户观摩区
- 录音录像设备

讨论室:
- 圆形/椭圆形桌子
- 每人桌牌
- 白板/投影
- 茶歇点心
```

#### 座位安排
```
    观察室(单向镜)
         ↓
    主持人
   /    \
  👤    👤
 👤  👤  👤
  👤    👤
```

#### 时间流程
```
0:00-0:10  签到、茶歇
0:10-0:20  开场介绍
0:20-0:40  热身破冰
0:40-1:40  核心讨论
1:40-1:50  茶歇休息
1:50-2:10  深入讨论
2:10-2:20  总结收尾
2:20-2:30  填问卷、发报酬
```

### 3. 后期分析(1周)

- 转录录音
- 内容编码
- 主题提取
- 跨组比较
- 报告撰写

## 主持人技巧

### 主持人角色
- 引导者: 把控讨论方向
- 促进者: 鼓励所有人参与
- 中立者: 不表达个人观点
- 观察者: 关注非语言信息

### 核心技能

#### 1. 开场破冰
**目的**: 放松氛围,建立信任

**常用方法**
```
- 自我介绍: 名字+一句话描述自己
- 轻松话题: 周末做了什么
- 关联话题: 使用XX产品的经历
- 游戏互动: 词语联想、快速问答
```

**示例话术**
> "欢迎大家今天来参加我们的讨论。在正式开始前,我们先做个小游戏。请大家用一个词形容你现在的心情..."

#### 2. 提问技巧

**漏斗式提问**
```
宽泛问题 → 一般讨论 → 具体话题 → 深入追问
```

**提问类型**
- 开放式: "你们觉得怎么样?"
- 投射式: "如果这个产品是一个人,会是什么样的人?"
- 比较式: "A和B有什么区别?"
- 假设式: "如果能改变一个功能,你希望是什么?"

#### 3. 促进讨论

**鼓励发言**
- 眼神接触
- 点头微笑
- "很有意思,能多说说吗?"
- 环顾所有参与者

**平衡发言**
- "其他人怎么看?"
- "我们听听XX的意见"
- "有没有不同观点?"

**深度挖掘**
- "为什么这么说?"
- "能举个例子吗?"
- "最看重的是什么?"

#### 4. 控制技巧

**时间控制**
- 明确每个环节时长
- 使用计时器
- 柔性提醒
- 灵活调整

**话题控制**
- 过渡承接
- "说到这个,我想起..."
- 礼貌打断
- 记录待讨论话题

**冲突管理**
- 承认差异
- "看来大家有不同看法,这很好"
- 不评判对错
- 引导理解不同观点

## 常见问题处理

### 1. 霸道者(Dominator)
**表现**: 滔滔不绝,占据过多时间

**应对**
- 礼貌打断:"你的观点很有意思,我们听听其他人的想法"
- 眼光转向其他人
- 会后私下沟通

### 2. 沉默者(Silent participant)
**表现**: 很少发言,仅简单回应

**应对**
- 直接温和提问
- 从简单问题开始
- 给予鼓励肯定
- 会后了解原因

### 3. 偏题者(Rambler)
**表现**: 经常跑题,讲无关内容

**应对**
- 礼貌拉回主题
- "这个很有意思,不过我们还是先聊聊XX"
- 承诺会后讨论

### 4. 争吵者(Arguer)
**表现**: 与他人激烈争论

**应对**
- 承认观点差异
- "你们的看法不同,这正是我们需要的"
- 不评判对错
- 转向其他人

### 5. 专家型(Know-it-all)
**表现**: 以专家自居,否定他人

**应对**
- 肯定专业性
- 强调个人体验
- "我们更关心大家的实际使用感受"

## 讨论提纲设计

### 基本结构
```
1. 开场(5-10分钟)
   - 欢迎与规则说明
   - 自我介绍
   - 热身话题

2. 一般探索(20-30分钟)
   - 使用习惯
   - 总体态度
   - 需求痛点

3. 深度讨论(60-70分钟)
   - 概念反应
   - 方案评价
   - 优先级排序
   - 创意激发

4. 总结收尾(10-15分钟)
   - 补充问题
   - 总结确认
   - 道谢说明
```

### 提纲示例: 移动App焦点小组
```
1. 开场(10分钟)
   - 欢迎与规则(5分钟)
   - 自我介绍: 名字+最常用的3个App(5分钟)

2. 使用习惯(20分钟)
   - 你们平时怎么用手机?
   - 选择App时最看重什么?
   - 哪些App让你印象深刻?为什么?

3. 体验评价(40分钟)
   - [展示产品] 第一印象如何?
   - 会用哪些功能?为什么?
   - 哪里让你困惑/不满意?
   - 如果要打分,你打几分?为什么?

4. 需求探索(30分钟)
   - 理想的产品是什么样的?
   - 如果能改变一个功能...
   - [展示新概念] 这个想法怎么样?

5. 总结(10分钟)
   - 还有什么想补充的?
   - 最重要的建议是什么?
```

## 活动技巧

### 视觉辅助
- 产品/原型展示
- 图片/视频
- 概念板
- 情绪板

### 互动活动

- 卡片分类: 信息架构探索

## 数据分析

### 分析维度

#### 内容分析
- 观点统计
- 主题提取
- 共识与分歧
- 典型引用

#### 过程分析
- 互动模式
- 决策过程
- 影响因素
- 情绪变化

#### 跨组比较
- 不同群体差异
- 共同主题
- 特殊观点
- 模式识别

### 报告结构
```
1. 研究概述
2. 主要发现
   - 发现1: 标题
     - 说明
     - 典型引用
     - 含义解读

4. 建议启示
5. 附录
   - 讨论提纲
   - 参与者信息
   - 详细记录
```

## 案例分析

### 案例: 新功能概念测试

**背景**
某社交App计划推出"语音房间"功能,需验证用户接受度

**研究设计**
- 4组焦点小组
- 每组6-8人
- 分组: 活跃用户、一般用户、年轻用户、中年用户
- 时长: 90分钟

**讨论提纲(节选)**
```
1. 社交习惯探索(20分钟)
2. 现有语音功能评价(15分钟)
3. [概念展示] 语音房间介绍(5分钟)
4. 概念反应与讨论(35分钟)
5. 使用场景想象(15分钟)
```

**关键发现**
1. 年轻用户更开放,中年用户顾虑隐私
2. 兴趣导向比熟人社交更有吸引力
3. 担心内容质量和骚扰
4. 需要明确的社交规则

**实践建议**
- 优先在年轻用户中试点
- 设计主题房间机制
- 完善内容审核
- 建立社区规范

## 线上焦点小组

### 优势
- 地理灵活
- 成本较低
- 便于记录
- 匿名性强

### 工具
- Zoom/腾讯会议
- 专门平台: FocusVision, Discuss.io
- 协作白板: Miro, MURAL

### 特别技巧
- 更严格的时间控制
- 利用聊天功能
- 使用投票工具
- 破冰活动更重要

## 质量控制

### 信度
- 多名分析者
- 编码一致性
- 记录详细过程

### 效度
- 三角验证
- 参与者反馈
- 同行评审

## 伦理考量

### 知情同意
- 说明讨论主题
- 告知录音录像
- 强调自愿参与
- 承诺保密

### 隐私保护
- 匿名报告
- 数据安全
- 限制访问

## 最佳实践

### ✅ 应该做
- 充分准备提纲
- 专业主持人
- 合理激励
- 详细记录
- 多组验证

### ❌ 避免做
- 代替深度访谈
- 参与者差异过大
- 强求共识
- 忽视少数观点
- 过度推广结论
