# 眼动追踪

## 概述
眼动追踪(Eye Tracking)是一种通过记录和分析用户眼球运动来研究视觉注意力和认知过程的技术,能够揭示用户"看什么"和"怎么看"。

## 基本原理

### 眼球运动类型

#### 注视(Fixation)
```
定义: 眼睛在某个位置停留(通常100-500ms)

意义:
- 信息获取和处理
- 认知加工
- 兴趣所在

测量指标:
- 注视点数量
- 注视时长
- 注视顺序
```

#### 眼跳(Saccade)
```
定义: 眼睛在两个注视点之间的快速移动

特点:
- 速度极快(30-500°/s)
- 持续时间短(10-100ms)
- 期间基本不获取信息

测量指标:
- 眼跳次数
- 眼跳幅度
- 眼跳方向
```

#### 追随运动(Pursuit)
```
定义: 眼睛跟随移动物体的平滑运动

应用场景:
- 视频观看
- 动态界面
- 游戏研究
```

## 核心指标

### 1. 注视指标

#### 注视点(Fixation Point)
```
定义: 眼睛停留的位置

可视化:
- 点图(Dot Plot)
- 注视点序列
```

#### 注视次数(Fixation Count)
```
含义: 某区域注视点数量

解读:
- 数量多 → 关注度高或理解困难
- 结合注视时长判断
```

#### 注视时长(Fixation Duration)
```
含义: 单次注视持续时间

解读:
- 时长长 → 深度加工或困惑
- 时长短 → 快速扫描或熟悉
```

#### 总注视时长(Total Fixation Duration)
```
含义: 某区域所有注视时长总和

解读:
- 反映总体关注度
- 识别兴趣区域
```

#### 首次注视时长(First Fixation Duration)
```
含义: 第一次注视该区域的时长

应用:
- 评估初始吸引力
- 视觉层次效果
```

### 2. 区域指标

#### 注视率(Gaze Rate)
```
注视率 = 看过某区域的人数 / 总人数 × 100%

应用:
- 关键元素是否被看到
- 布局有效性
```

#### 时间到首次注视(Time to First Fixation)
```
含义: 从刺激呈现到首次注视某区域的时间

解读:
- 时间短 → 显眼性高
- 时间长 → 不易发现
```

#### 注视顺序(Fixation Sequence)
```
含义: 注视各区域的顺序

应用:
- 视觉路径分析
- 阅读模式研究
```

### 3. 扫视指标

#### 扫视路径(Scan Path)
```
定义: 注视点的空间连接序列

可视化:
- 连线图
- 序列图

分析:
- 路径模式
- 搜索策略
- 个体差异
```

#### 扫视幅度(Saccade Amplitude)
```
含义: 两次注视之间的距离

解读:
- 幅度大 → 宏观扫描
- 幅度小 → 细节检查
```

### 4. 瞳孔指标

#### 瞳孔直径(Pupil Dilation)
```
影响因素:
- 光照变化
- 认知负荷
- 情绪唤醒
- 兴趣程度

应用:
- 难度评估
- 情绪反应
- 隐式偏好
```

## 可视化方法

### 1. 热力图(Heatmap)
```
类型:
- 注视热力图: 显示注视密度
- 注意力热力图: 显示停留时长

颜色编码:
- 红色: 高关注
- 黄色: 中关注
- 绿色: 低关注
- 无色: 未注视

用途:
- 快速识别热点区域
- 发现视觉盲区
- 对比不同设计
```

### 2. 注视点图(Gaze Plot)
```
元素:
- 圆点: 注视位置
- 圆大小: 注视时长
- 序号: 注视顺序
- 连线: 扫视路径

用途:
- 分析注视顺序
- 理解视觉流程
```

### 3. 区域图(AOI)
```
AOI (Area of Interest) 感兴趣区域

定义:
- 人为划分关键区域
- 统计各区域指标

分析内容:
- 注视率
- 注视时长
- 首次注视时间
- 注视顺序
```

### 4. 视频回放
```
内容:
- 实时眼动轨迹
- 用户操作同步
- 时序信息

用途:
- 深度个案分析
- 演示汇报
```

### 5. 蜂群图(Bee Swarm)
```
特点:
- 显示多个用户注视点
- 聚合与分布可视化

用途:
- 群体模式识别
- 个体差异观察
```

## 设备与技术

### 1. 设备类型

#### 桌面式眼动仪
```
特点:
- 固定在显示器下方
- 精度高
- 适合实验室

代表产品:
- Tobii Pro Spectrum
- SMI REDn
- EyeLink 1000 Plus

精度: 0.01°-0.5°
采样率: 60-2000Hz
价格: 数万-数十万
```

#### 头戴式眼动仪
```
特点:
- 移动自由
- 真实环境测试
- 头部运动补偿

代表产品:
- Tobii Pro Glasses 3
- SMI Eye Tracking Glasses
- Pupil Labs

应用场景:
- 实体店购物
- 驾驶研究
- VR/AR研究
```

#### 屏幕集成式
```
特点:
- 眼动仪与屏幕一体
- 易于使用
- 整洁美观

代表产品:
- Tobii Pro Nano
- Gazepoint GP3
```

#### 移动设备集成
```
特点:
- 使用设备前置摄像头
- 无需额外硬件
- 精度相对较低

方案:
- iMotions
- Sticky by EyeQuant
- RealEye

精度: 1°-3°
适用: 大规模测试
```

### 2. 技术原理

#### 瞳孔-角膜反射向量法(PCCR)
```
原理:
1. 红外光照射眼睛
2. 反射形成普尔金耶图像
3. 追踪瞳孔中心和角膜反射
4. 计算视线方向

最常用方法
```

#### 视网膜成像法
```
原理:
- 追踪视网膜血管图案
- 精度极高

应用:
- 科研领域
```

### 3. 校准

#### 校准目的
```
- 建立眼动特征与屏幕坐标映射
- 提高测量精度
```

#### 校准方法
```
1点校准:
- 快速简便
- 精度较低

5点校准: ⭐常用
- 四角+中心
- 平衡精度与效率

9点校准:
- 精度更高
- 耗时较长
```

#### 校准质量评估
```
指标:
- 校准误差 < 1° (优秀)
- 校准误差 < 2° (可接受)
- 校准误差 > 2° (需重新校准)
```

## 研究流程

### 标准流程
```
1. 研究设计
   ↓
2. 刺激材料准备
   ↓
3. 实验设置与校准
   ↓
4. 数据收集
   ↓
5. 数据处理
   ↓
6. 统计分析
   ↓
7. 可视化与报告
```

### 1. 研究设计

#### 定义研究问题
```
示例:
- 导航菜单是否容易被发现?
- 用户是否注意到促销信息?
- 图片和文字哪个更吸引注意?
```

#### 设计任务
```
原则:
- 真实场景
- 明确目标
- 适度难度

示例:
"请在页面上找到退换货政策"
```

### 2. 刺激材料准备

#### 图片/网页
```
要求:
- 分辨率一致
- 尺寸适配屏幕
- 格式标准化
```

#### 视频/动态
```
注意:
- 时长控制
- 标记关键帧
- 同步分析
```

### 3. 实验设置

#### 环境控制
```
光照:
- 恒定柔和照明
- 避免直射光
- 避免反光

距离:
- 屏幕距离60-70cm
- 头部固定或自然

干扰:
- 安静环境
- 减少干扰
```

#### 校准流程
```
1. 说明校准目的
2. 调整座椅位置
3. 进行校准
4. 检查校准质量
5. 不合格则重新校准
```

### 4. 数据收集

#### 任务执行
```
流程:
1. 说明任务
2. 开始记录
3. 用户执行任务
4. 标记任务完成
5. 短暂休息
```

#### 样本量
```
定性分析: 5-10人
定量分析: 30-50人
统计检验: 根据效应量计算
```

### 5. 数据处理

#### 数据清洗
```
内容:
- 剔除校准不良的数据
- 处理信号丢失
- 插值补全(谨慎)
```

#### AOI划分
```
方法:
- 手动绘制区域
- 基于元素自动生成
- 统计区域指标
```

#### 事件检测
```
算法:
- I-DT(速度阈值)
- I-VT(速度阈值)
- I-HMM(隐马尔可夫)

常用: I-DT算法
```

### 6. 统计分析

#### 描述统计
```
- 各指标均值、标准差
- 分布可视化
- 异常值识别
```

#### 推断统计
```
比较:
- t检验: 两组对比
- ANOVA: 多组对比
- 非参数检验: 不满足假设时

相关:
- Pearson相关
- 回归分析
```

### 7. 可视化报告

#### 报告内容
```
1. 研究概述
   - 目的、方法、样本

2. 总体热力图
   - 群体注意模式

3. AOI分析
   - 关键区域指标对比

4. 注视序列
   - 视觉流程

5. 分群分析
   - 不同群体差异

6. 结论建议
   - 设计优化方向
```

## 局限与挑战

### 1. 技术局限
```
精度问题:
- 头部移动影响
- 校准漂移
- 采样率限制

环境要求:
- 光照控制
- 设备校准
- 空间限制
```

### 2. 解释挑战
```
注视 ≠ 注意
- 可能看着但没在注意
- 外周视觉信息

注视 ≠ 喜欢
- 注视可能是困惑
- 需结合其他数据
```

### 3. 样本代表性
```
- 通常样本量小
- 实验室环境
- 不完全自然行为
```

### 4. 成本
```
- 设备昂贵
- 专业软件
- 时间成本高
```

## 最佳实践

### ✅ 应该做
- 明确研究问题
- 控制实验环境
- 严格校准
- 结合其他方法
- 谨慎解读结果

### ❌ 避免做
- 单独使用眼动数据
- 过度解读注视含义
- 忽视个体差异
- 环境控制不严

## 案例分析

### 案例: 电商详情页优化

**背景**
某电商想优化商品详情页布局

**研究问题**
- 用户是否注意到关键信息?
- 视觉流程是否合理?
- 哪些区域被忽视?

**方法**
```
设备: Tobii Pro Nano
样本: 30人
任务: 浏览商品详情页,决定是否购买
刺激: A/B两个版本对比
```

**关键发现**
```
1. 商品图片: 注视率95%,平均注视3.2秒
2. 价格信息: 注视率78%,但首次注视时间较晚
3. 促销标签: 注视率仅42%
4. 用户评价: 高兴趣区域,注视时长长

视觉流程:
图片 → 标题 → 价格 → 评价 → 详情
```

**优化建议**
1. 提高价格区域视觉权重
2. 促销标签位置上移
3. 评价区域保持当前设计
4. 简化商品详情,减少信息过载
