# 用户分层 (User Segmentation)

## 定义
用户分层是根据用户特征、行为、价值等维度，将用户划分为不同群体的方法，用于实现精细化运营和个性化服务。

## 分层价值

### 为什么需要用户分层？

```
用户数 × 平均运营效果 = 总效果

精细化运营后：
Σ(各层用户数 × 各层最优策略效果) >> 总效果
```

**核心价值：**
- 资源精准投放
- 策略针对性
- 提升运营效率
- 改善用户体验

## 分层维度

### 1. 人口属性分层

| 维度 | 示例 | 应用场景 |
|------|------|----------|
| 年龄 | 18-24/25-34/35-44... | 内容推荐、产品设计 |
| 性别 | 男/女 | 商品推荐、广告投放 |
| 地域 | 一线/二线/三四线/农村 | 区域活动、本地化运营 |
| 职业 | 学生/白领/蓝领/自由职业 | 功能设计、内容定位 |
| 收入 | 低/中/高 | 定价策略、商品推荐 |
| 教育 | 高中/本科/硕士/博士 | 内容深度、沟通方式 |

### 2. 行为特征分层

| 维度 | 指标 | 分层方法 |
|------|------|----------|
| 活跃度 | DAU/MAU/访问频次 | 高/中/低/沉默 |
| 使用时长 | 日均时长 | 重度/中度/轻度 |
| 功能使用 | 核心功能使用率 | 深度/浅层 |
| 时间分布 | 使用时段分布 | 早晨型/晚间型/碎片型 |
| 设备偏好 | iOS/Android/Web | 设备分层 |

### 3. 价值分层

#### RFM模型

**定义：** 基于最近消费(Recency)、消费频次(Frequency)、消费金额(Monetary)三个维度。

| 维度 | 说明 | 计算方式 |
|------|------|----------|
| R | 最近一次消费距今时间 | 天数 |
| F | 消费频次 | 次数/周期 |
| M | 消费金额 | 金额/周期 |

**RFM分层矩阵：**

| R | F | M | 用户类型 | 运营策略 |
|---|---|---|----------|----------|
| 高 | 高 | 高 | 重要价值用户 | VIP服务、保持满意 |
| 高 | 低 | 高 | 重要发展用户 | 提升频次、深度运营 |
| 低 | 高 | 高 | 重要保持用户 | 召回激活、防止流失 |
| 低 | 低 | 高 | 重要挽留用户 | 重点挽回、个性化沟通 |
| 高 | 高 | 低 | 一般价值用户 | 提升客单价 |
| 高 | 低 | 低 | 一般发展用户 | 培养习惯 |
| 低 | 高 | 低 | 一般保持用户 | 提升价值 |
| 低 | 低 | 低 | 低价值用户 | 低成本维护 |

**RFM评分方法：**
```
R评分：按时间分段打分
- 30天内：5分
- 30-60天：4分
- 60-90天：3分
- 90-180天：2分
- 180天以上：1分

F评分：按频次分段打分
- 10次以上：5分
- 6-10次：4分
- 3-5次：3分
- 2次：2分
- 1次：1分

M评分：按金额分段打分
- 按消费金额分位数划分

RFM总分 = R分 × 权重R + F分 × 权重F + M分 × 权重M
```

### 4. 生命周期分层

```
新用户 → 活跃用户 → 忠诚用户 → 流失用户
   ↓         ↓          ↓          ↓
 注册激活   持续使用    高频高价值   停止使用
```

| 阶段 | 定义特征 | 运营重点 |
|------|----------|----------|
| 新用户 | 注册<7天 | 激活、引导 |
| 活跃用户 | 近7天有行为 | 提升使用深度 |
| 忠诚用户 | 高频、高留存 | 维护、增值 |
| 流失用户 | 超过N天未使用 | 召回 |

### 5. 需求分层

| 需求类型 | 特征 | 内容/功能 |
|----------|------|-----------|
| 功能型 | 解决具体问题 | 工具功能 |
| 内容型 | 获取信息/娱乐 | 内容推荐 |
| 社交型 | 人际交往需求 | 社交功能 |
| 自我实现 | 成长、展示 | 成长体系 |
| 娱乐型 | 消遣、放松 | 游戏化 |

## 分层方法

### 1. 规则分层

**方法：** 基于业务经验设定规则

```
示例：电商用户分层规则

if 消费金额 > 10000 and 购买频次 > 12:
    用户层级 = "钻石会员"
elif 消费金额 > 5000 and 购买频次 > 6:
    用户层级 = "黄金会员"
elif 消费金额 > 1000 and 购买频次 > 3:
    用户层级 = "白银会员"
else:
    用户层级 = "普通用户"
```

**优点：** 简单直观、可解释性强
**缺点：** 主观性强、边界模糊

### 2. 聚类分层

**方法：** 使用算法自动发现用户群体

**K-Means聚类：**
```
步骤：
1. 选择聚类变量（如：活跃度、付费金额、使用时长）
2. 数据标准化
3. 确定K值（肘部法则）
4. 执行聚类
5. 分析各群特征
```

**层次聚类：**
```
适合场景：需要体现层级关系
结果展示：树状图
```

**DBSCAN密度聚类：**
```
适合场景：发现任意形状的群体
优点：自动确定群数、识别异常点
```

### 3. 决策树分层

**方法：** 通过决策树发现分层规则

```
示例决策树：
                    首次消费金额
                   /           \
               >100           <=100
                /               \
           月活天数          月活天数
          /      \          /      \
        >20     <=20      >15      <=15
         ↓        ↓        ↓         ↓
      高价值    中价值    潜力       低价值
```

### 4. 分位数分层

**方法：** 按指标分位数划分

```
示例：按消费金额分层

第1层(20%)：消费金额 >= P80
第2层(20%)：P60 <= 消费金额 < P80
第3层(20%)：P40 <= 消费金额 < P60
第4层(20%)：P20 <= 消费金额 < P40
第5层(20%)：消费金额 < P20
```

## 分层实施流程

### 步骤1：明确分层目的

| 目的 | 适合的分层方式 |
|------|----------------|
| 精准营销 | RFM、行为分层 |
| 产品优化 | 需求分层、功能使用分层 |
| 资源分配 | 价值分层 |
| 用户增长 | 生命周期分层 |

### 步骤2：选择分层维度

**维度选择原则：**
- 与业务目标相关
- 数据可获取
- 可差异化运营

### 步骤3：确定分层方法

| 场景 | 推荐方法 |
|------|----------|
| 探索性分析 | 聚类分析 |
| 运营执行 | 规则分层 |
| 发现规则 | 决策树 |

### 步骤4：验证分层效果

**评估指标：**
```
1. 层内相似度：同一层用户的相似程度
2. 层间差异度：不同层用户的差异程度
3. 运营可操作性：是否能针对不同层制定策略
4. 业务效果：分层运营后的效果提升
```

### 步骤5：建立分层标签

```
标签体系示例：

一级标签：价值层级
- 高价值
- 中价值
- 低价值

二级标签：行为特征
- 高频活跃
- 中频活跃
- 低频活跃

三级标签：需求偏好
- 内容偏好
- 社交偏好
- 工具偏好
```

## 经典分层模型

### 1. 金字塔模型

```
          /\
         /  \
        / 核心\      1-5%  最高价值
       /------\
      / 价值用户\    5-15%  高价值
     /----------\
    /  活跃用户  \  15-35% 中等价值
   /--------------\
  /    普通用户    \ 35-45% 低价值
 /------------------\
/      新用户        \   5-10% 待培育
----------------------
```

### 2. 用户价值分层

| 层级 | 占比 | 特征 | 贡献 | 策略 |
|------|------|------|------|------|
| 核心 | 1-5% | 高频、高付费、高忠诚 | 40-50% | 专属服务 |
| 重要 | 5-15% | 较高频、较高付费 | 20-30% | 提升留存 |
| 主力 | 15-35% | 中等活跃 | 15-25% | 提升价值 |
| 普通 | 45-65% | 低频、低付费 | 5-15% | 低成本维护 |
| 流失 | 5-15% | 不活跃 | 0% | 召回 |

### 3. 用户成长分层

```
L1 新手 → L2 进阶 → L3 高手 → L4 专家 → L5 大神

升级条件：综合活跃度、贡献度、时长等
```

## 分层运营策略

### 高价值用户

**特征识别：**
- 消费金额TOP 20%
- 高频使用
- 高留存率

**运营策略：**
- 专属客服
- VIP特权
- 优先体验
- 生日关怀

### 中等价值用户

**特征识别：**
- 消费金额中等
- 使用频次不稳定
- 有提升空间

**运营策略：**
- 满减活动
- 成长激励
- 内容推荐
- 社群运营

### 低价值用户

**特征识别：**
- 消费金额低
- 使用频次低
- 流失风险高

**运营策略：**
- 免费福利
- 入门引导
- 低价体验
- 低成本触达

## 分层效果评估

### 评估指标

| 指标 | 计算公式 | 说明 |
|------|----------|------|
| 层间差异度 | F值、卡方值 | 统计显著性 |
| 层内一致性 | 轮廓系数 | 聚类效果 |
| 运营效果 | 各层目标达成率 | 业务价值 |
| 策略响应率 | 响应用户/触达用户 | 策略有效性 |

### A/B测试验证

```
实验设计：
- 对照组：统一运营策略
- 实验组：分层运营策略

对比指标：
- 整体GMV
- 用户活跃度
- 留存率
- 运营ROI
```

## 分层系统搭建

### 技术架构

```
数据层：用户行为数据、交易数据、属性数据
    ↓
计算层：离线计算（T+1）、实时计算（秒级）
    ↓
标签层：分层标签存储、标签更新
    ↓
应用层：运营平台、推荐系统、CRM系统
```

### 数据更新频率

| 分层类型 | 更新频率 | 原因 |
|----------|----------|------|
| 人口属性 | 月度 | 变化慢 |
| 行为特征 | 日度 | 需及时 |
| 价值分层 | 周度 | 平衡效率 |
| 生命周期 | 实时 | 即时响应 |

## 最佳实践

### 1. 分层不宜过细
- 3-8层最佳
- 层级过多运营成本高
- 层级过少精细化不足

### 2. 动态调整
- 定期评估分层效果
- 根据业务变化调整
- 优化分层规则

### 3. 策略可执行
- 每层有明确策略
- 资源分配明确
- 效果可衡量

### 4. 数据驱动
- 基于数据分析分层
- 验证分层假设
- 持续优化迭代

### 5. 用户视角
- 分层对用户透明（部分）
- 提供升级路径
- 公平公正
