# 活动数据 (Event Data Analysis)

## 定义
活动数据分析是对活动各环节数据进行收集、监控、分析和评估的过程，通过数据洞察优化活动效果，指导活动决策。

## 核心指标体系

### 01 流量指标

| 指标 | 计算公式 | 说明 |
|------|----------|------|
| PV | 页面访问次数 | 页面热度 |
| UV | 访问用户数 | 覆盖人数 |
| 人均访问次数 | PV/UV | 访问深度 |
| 平均停留时长 | 总时长/UV | 用户兴趣 |
| 跳出率 | 跳出UV/总UV | 页面吸引力 |

### 02 参与指标

| 指标 | 计算公式 | 说明 |
|------|----------|------|
| 参与人数 | 参与UV | 实际参与 |
| 参与率 | 参与UV/访问UV | 转化效果 |
| 完成人数 | 完成UV | 完成情况 |
| 完成率 | 完成UV/参与UV | 漏斗转化 |
| 人均参与次数 | 参与次数/参与UV | 参与深度 |

### 03 传播指标

| 指标 | 计算公式 | 说明 |
|------|----------|------|
| 分享人数 | 分享UV | 传播用户 |
| 分享率 | 分享UV/参与UV | 传播意愿 |
| 分享次数 | 分享总次数 | 传播量 |
| K因子 | 新增用户/分享用户 | 传播效果 |
| 裂变层级 | 平均传播层数 | 传播深度 |

### 04 转化指标

| 指标 | 计算公式 | 说明 |
|------|----------|------|
| 转化人数 | 转化UV | 实际转化 |
| 转化率 | 转化UV/参与UV | 转化效果 |
| GMV | 销售总额 | 销售规模 |
| 客单价 | GMV/订单数 | 消费水平 |
| ROI | (收益-成本)/成本 | 投入产出比 |

### 05 成本指标

| 指标 | 计算公式 | 说明 |
|------|----------|------|
| 总成本 | 各项成本之和 | 总投入 |
| 人均成本 | 总成本/参与UV | 单用户成本 |
| 获客成本 | 总成本/新增用户 | CAC |
| 转化成本 | 总成本/转化UV | CPA |

### 预警机制

```
预警规则设置：

1. 流量预警
   - PV低于预期50% → 预警
   - UV低于预期50% → 预警

2. 转化预警
   - 参与率低于5% → 预警
   - 完成率低于20% → 预警

3. 成本预警
   - 人均成本超过预算50% → 预警
   - 总成本超过预算80% → 预警

4. 技术预警
   - 页面加载时间 > 3秒 → 预警
   - 错误率 > 1% → 预警
```

## 漏斗分析

### 活动漏斗模型

```
活动参与漏斗：

曝光 → 点击 → 参与 → 完成 → 转化

各环节转化率：
- 点击率：点击/曝光
- 参与率：参与/点击
- 完成率：完成/参与
- 转化率：转化/完成

整体转化率 = 点击率 × 参与率 × 完成率 × 转化率
```

### 漏斗分析示例

```
某活动漏斗数据：

曝光：100,000 (100%)
  ↓ 点击率 10%
点击：10,000 (10%)
  ↓ 参与率 40%
参与：4,000 (4%)
  ↓ 完成率 60%
完成：2,400 (2.4%)
  ↓ 转化率 30%
转化：720 (0.72%)

分析：
- 点击率偏低：标题/封面需优化
- 参与率正常
- 完成率正常
- 转化率良好

优化方向：提升点击率
```

## 分层分析

### 渠道分析

```
各渠道数据对比：

| 渠道 | UV | 参与率 | 转化率 | 成本 | ROI |
|------|-----|--------|--------|------|-----|
| Push | 5000 | 15% | 3% | 低 | 高 |
| Banner | 3000 | 10% | 2% | 低 | 中 |
| 社群 | 2000 | 30% | 8% | 低 | 高 |
| 信息流 | 10000 | 5% | 1% | 高 | 低 |

结论：
- 社群渠道效果最好
- Push性价比较高
- 信息流需优化
```

### 用户分层分析

```
新老用户对比：

| 用户类型 | 占比 | 参与率 | 转化率 | 人均价值 |
|----------|------|--------|--------|----------|
| 新用户 | 30% | 20% | 5% | 50元 |
| 老用户 | 70% | 35% | 12% | 120元 |

分析：
- 老用户参与率和转化率更高
- 新用户人均价值较低
- 需要针对不同用户设计策略
```

### 时间分析

```
按时段分析：

| 时段 | PV | UV | 参与率 | 转化率 |
|------|-----|-----|--------|--------|
| 0-6时 | 5000 | 3000 | 8% | 1% |
| 6-12时 | 20000 | 15000 | 12% | 3% |
| 12-18时 | 25000 | 18000 | 15% | 4% |
| 18-24时 | 40000 | 30000 | 20% | 5% |

结论：
- 18-24时流量最大、效果最好
- 应在高峰时段重点推广
```

```
━━─ 目标达成情况 ─━

| 指标 | 目标值 | 实际值 | 达成率 | 评价 |
|------|--------|--------|--------|------|
| 参与人数 | 10000 | 12000 | 120% | 超额达成 |
| 转化人数 | 500 | 450 | 90% | 基本达成 |
| GMV | 10万 | 8万 | 80% | 未达成 |
| ROI | 200% | 150% | 75% | 未达成 |

总体评价：基本达成目标，GMV和ROI需优化
```

### 活动效果评分

```
活动效果评分模型：

总分 = 参与度 × 0.3 + 转化效果 × 0.3 + 成本控制 × 0.2 + 用户反馈 × 0.2

评分标准：
- 90-100分：优秀
- 80-89分：良好
- 70-79分：合格
- <70分：待改进
```

## 问题诊断

### 常见问题诊断

| 问题现象 | 可能原因 | 诊断方法 | 优化方向 |
|----------|----------|----------|----------|
| PV高UV低 | 重复访问多 | 检查人均访问次数 | 扩大触达范围 |
| 点击率低 | 标题/封面不吸引 | A/B测试 | 优化创意 |
| 参与率低 | 规则复杂/奖励不吸引 | 用户调研 | 简化流程、提升激励 |
| 完成率低 | 任务难度高 | 漏斗分析 | 降低门槛 |
| 转化率低 | 转化路径问题 | 路径分析 | 优化转化流程 |
| 分享率低 | 缺乏分享动力 | 用户调研 | 增加分享激励 |

### 数据异常诊断

```
异常诊断流程：

1. 确认异常
   - 数据是否确实异常
   - 与历史数据对比
   - 与预期对比

2. 定位原因
   - 技术原因：埋点、系统故障
   - 运营原因：活动规则、推广
   - 外部原因：节假日、热点

3. 制定方案
   - 技术问题：修复
   - 运营问题：调整策略
   - 外部原因：应对调整

4. 验证效果
   - 监控调整后数据
   - 确认问题解决
```

## 数据驱动优化

### A/B测试

```
A/B测试流程：

1. 假设：{优化假设}
2. 变量：{测试变量}
3. 分组：A组（对照组） vs B组（实验组）
4. 指标：{评估指标}
5. 周期：{测试周期}
6. 结论：{测试结论}

示例：
假设：红色按钮比蓝色按钮点击率高
变量：按钮颜色
分组：各50%流量
指标：点击率
周期：7天
结论：红色按钮点击率提升15%，采用红色
```

### 数据驱动决策

```
决策流程：

数据收集 → 数据分析 → 洞察发现 → 假设提出 → 实验验证 → 决策执行

关键点：
1. 数据要全面准确
2. 分析要深入透彻
3. 洞察要有价值
4. 假设要可验证
5. 实验要科学
6. 决策要果断
```

## 最佳实践

### 1. 埋点先行
- 活动前完成埋点
- 测试埋点正确性
- 数据口径统一

### 2. 实时监控
- 建立实时看板
- 设置预警机制
- 及时发现问题

### 3. 深度分析
- 不仅看表面数据
- 多维度交叉分析
- 找到根本原因

### 4. 数据驱动
- 用数据说话
- 避免主观判断
- 持续优化迭代

### 5. 及时复盘
- 活动后及时复盘
- 沉淀经验教训
- 形成知识库
