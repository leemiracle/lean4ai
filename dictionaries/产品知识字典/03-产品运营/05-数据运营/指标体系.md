# 指标体系 (Metrics System)

## 定义
指标体系是一套完整的、层次化的数据指标集合，用于衡量产品运营的各个方面。北极星指标 (North Star Metric) 是指标体系中最重要的单一指标，代表产品核心价值。

## 指标体系框架

### OSM模型

```
O (Objective) 目标：业务目标
   ↓
S (Strategy) 策略：达成目标的策略
   ↓
M (Metric) 指标：衡量策略的指标
```

### 指标分层

```
┌─────────────────────────────────────────────┐
│              指标金字塔                       │
├─────────────────────────────────────────────┤
│                 北极星指标                    │  1个
│              (公司级核心指标)                 │
├─────────────────────────────────────────────┤
│               一级指标                        │  3-5个
│           (业务线核心指标)                    │
├─────────────────────────────────────────────┤
│               二级指标                        │  10-20个
│           (部门级关键指标)                    │
├─────────────────────────────────────────────┤
│               三级指标                        │  数十个
│           (执行层过程指标)                    │
└─────────────────────────────────────────────┘
```

## 北极星指标

### 定义与特点

```
北极星指标是最能代表产品核心价值传递给用户的指标

特点：
1. 唯一性：全公司只有一个北极星
2. 核心价值：代表产品核心价值
3. 可衡量：可以被量化
4. 可行动：团队可以影响
5. 领先性：预示长期成功
```

### 各行业北极星指标

| 公司/产品 | 北极星指标 | 理由 |
|-----------|------------|------|
| Airbnb | 每晚预订数 | 核心业务价值 |
| Spotify | 收听时长 | 用户参与度 |
| Facebook | 月活跃用户 | 社交网络价值 |
| Slack | 用户发送消息数 | 核心功能使用 |
| Netflix | 月均观看时长 | 内容消费 |
| Zoom | 每周主持会议数 | 核心功能使用 |
| 电商平台 | GMV | 交易价值 |
| 内容平台 | 用户停留时长 | 内容消费 |
| SaaS | MRR/活跃用户数 | 订阅价值 |

### 北极星指标选择方法

```
选择流程：

1. 明确产品核心价值
   - 产品为用户解决什么问题？
   - 用户使用产品的核心场景是什么？

2. 识别价值传递的关键行为
   - 用户什么行为代表获得了价值？
   - 这个行为可以量化吗？

3. 验证指标
   - 与长期成功相关吗？
   - 团队可以影响吗？
   - 会不会导致不良行为？

示例（内容平台）：
核心价值：优质内容消费
关键行为：用户观看/阅读内容
北极星：用户日均停留时长
```

### 北极星指标拆解

```
北极星指标拆解公式：

北极星指标 = Σ(用户分层 × 各层指标)

示例（电商平台GMV）：
GMV = 流量 × 转化率 × 客单价

进一步拆解：
GMV = (自然流量 + 付费流量) × (新客转化率 + 老客转化率) × 客单价
```

## 一级指标

### 定义
一级指标是各部门/业务线的核心指标，直接支撑北极星指标。

### 常见一级指标

#### 用户增长
```
- DAU/MAU：日/月活跃用户
- 新增用户数：新注册/新下载用户
- 用户增长率：用户增长速度
```

#### 用户留存
```
- 次日留存：D1留存率
- 7日留存：D7留存率
- 30日留存：D30留存率
- 流失率：用户流失比例
```

#### 用户活跃
```
- 人均使用时长：日人均停留时长
- 人均使用次数：日人均打开次数
- 功能渗透率：核心功能使用率
```

#### 商业变现
```
- GMV：商品交易总额
- 收入：营业收入
- ARPU：人均收入
- 付费率：付费用户比例
```

## 二级指标

### 定义
二级指标是一级指标的拆解，用于定位问题和指导执行。

### 指标拆解方法

```
拆解公式：

一级指标 = 二级指标1 × 二级指标2 × ...

示例（DAU拆解）：
DAU = 新增用户 + 留存用户 + 召回用户

DAU = MAU × 日活率
日活率 = 日活跃用户 / 月活跃用户
```

### 常见二级指标

| 一级指标 | 二级指标 |
|----------|----------|
| DAU | 新增DAU、留存DAU、召回DAU |
| GMV | 订单量、客单价 |
| 转化率 | 各环节转化率 |
| 留存率 | 不同用户群留存率 |
| ARPU | 付费ARPU、免费ARPU |

## 三级指标

### 定义
三级指标是执行层的过程指标，指导日常运营动作。

### 特点
```
- 可直接操作
- 变化快
- 数量多
- 过程导向
```

### 示例

| 二级指标 | 三级指标 |
|----------|----------|
| 新增用户 | 各渠道新增用户、注册转化率 |
| 留存用户 | 签到率、任务完成率 |
| 订单量 | 各品类订单量、各时段订单量 |
| 客单价 | 品类客单价、用户分层客单价 |

## 指标体系设计

### 设计原则

```
1. MECE原则：相互独立、完全穷尽
2. 可操作性：指标可以被影响
3. 可衡量性：指标可以被量化
4. 相关性：指标与业务目标相关
5. 简洁性：指标数量适当
```

### 设计流程

```
1. 明确业务目标
   - 公司战略目标
   - 业务发展阶段
   - 核心价值定位

2. 确定北极星指标
   - 选择最能代表核心价值的指标
   - 验证指标的合理性

3. 拆解一级指标
   - 按业务线拆解
   - 按用户旅程拆解

4. 细化二级/三级指标
   - 可执行的粒度
   - 责任到人

5. 建立指标字典
   - 指标定义
   - 计算公式
   - 数据来源
   - 负责人

6. 持续迭代
   - 定期回顾
   - 根据业务调整
```

### 监控看板

```
┌─────────────────────────────────────────────────────┐
│              核心指标监控看板                         │
├───────────────┬───────────────┬─────────────────────┤
│   北极星指标   │   一级指标1    │    一级指标2         │

├───────────────┼───────────────┼─────────────────────┤
│   一级指标3   │   一级指标4    │    一级指标5         │
│    {数据}     │    {数据}     │    {数据}          │
│   {环比/同比} │   {环比/同比} │   {环比/同比}      │
└───────────────┴───────────────┴─────────────────────┘
```

### 预警机制

```
预警规则：

1. 阈值预警
   - 指标 < 下限阈值 → 预警
   - 指标 > 上限阈值 → 预警

2. 波动预警
   - 日环比下降 > 20% → 预警
   - 周环比下降 > 15% → 预警

3. 异常预警
   - 与历史同期对比异常 → 预警
   - 与预测值偏差大 → 预警
```

## 指标分析框架

### 对比分析

```
1. 时间对比
   - 日环比：今天 vs 昨天
   - 周环比：本周 vs 上周
   - 月同比：本月 vs 去年同月

2. 维度对比
   - 渠道对比：不同渠道指标对比
   - 用户对比：新老用户对比
   - 地区对比：不同地区对比

3. 目标对比
   - 实际 vs 目标
   - 达成率分析
```

### 趋势分析

```
1. 长期趋势
   - 年度趋势
   - 季度趋势

2. 中期趋势
   - 月度趋势
   - 周度趋势

3. 短期趋势
   - 日趋势
   - 小时趋势

4. 周期性分析
   - 工作日 vs 周末
   - 节假日效应
```

### 归因分析

```
指标变化归因：

1. 内部因素
   - 产品迭代
   - 运营活动
   - 技术故障

2. 外部因素
   - 市场环境
   - 竞品动态
   - 政策变化

3. 用户因素
   - 用户结构变化
   - 用户行为变化
```

## 指标体系最佳实践

### 1. 聚焦核心
- 明确北极星指标
- 不追求指标数量
- 关注核心指标

### 2. 可操作
- 指标可被影响
- 责任到人
- 有明确的行动路径

### 3. 统一口径
- 建立指标字典
- 统一计算方式
- 避免歧义

### 4. 动态调整
- 根据业务发展调整
- 定期回顾有效性
- 持续优化迭代

### 5. 避免虚荣指标
- 关注实际价值
- 避免只看表面数据
- 关注可行动的指标
