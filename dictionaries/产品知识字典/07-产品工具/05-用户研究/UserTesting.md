# UserTesting教程

## 概述
UserTesting是一个用户测试平台,可以快速招募真实用户进行可用性测试,获取用户对产品的真实反馈,帮助产品团队优化用户体验。

## 核心功能

### 测试创建
- 测试类型选择
- 任务场景设计
- 问题设计
- 受众筛选

### 受众招募
- 用户属性筛选
- 行为特征筛选
- 自定义筛选
- 样板库选择

### 测试执行
- 实时测试
- 录制回放
- 文字记录
- 时间戳标记

### 数据分析
- 视频分析
- 关键发现
- 剪辑分享
- 报告导出

## 使用指南

### 创建测试
```
步骤:
1. 点击"Create Test"
2. 选择测试类型
   - 网站/原型测试
   - App测试
   - 视频测试
   - 对比测试

3. 输入测试URL或上传材料

4. 设计测试任务
   - 任务描述
   - 完成标准
   - 预期时间

5. 添加测试问题
   - 前置问题
   - 任务问题
   - 后置问题

6. 设置受众条件

7. 提交测试
```

### 任务设计
```
任务结构:
1. 场景描述
   - 设定背景
   - 用户角色
   - 初始状态

2. 任务指令
   - 明确目标
   - 具体操作
   - 成功标准

示例:
场景: 您正在计划一次家庭旅行,
需要找一个适合带孩子的目的地。

任务: 请在网站上搜索并找到一个
您认为适合的目的地,并将它添加
到收藏列表中。
```

### 问题设计
```
问题类型:
1. 前置问题
   - 了解用户背景
   - 筛选条件验证
   - "您多久旅行一次?"

2. 任务问题
   - 评估任务难度
   - "这个任务对您来说有多容易?"
   - "您对这个功能的印象如何?"

3. 后置问题
   - 整体评价
   - "您会如何评价这次体验?"
   - "有什么改进建议?"

问题形式:
- 单选题
- 多选题
- 量表题(1-5分)
- 开放题
```

### 受众筛选
```
筛选维度:
- 地理位置
- 年龄性别
- 收入水平
- 教育程度
- 行业职业
- 技术熟练度
- 产品使用频率
- 特定行为特征

筛选器示例:
- 年龄: 25-45岁
- 性别: 全部
- 地区: 一线城市
- 使用过竞品
- 每月网购3次以上
```

## 实战案例

### 案例1:新功能可用性测试
```
测试目标: 评估新购物车功能的易用性

测试设计:
1. 前置问题
   Q1. 您平时多久网购一次?
   Q2. 您最常使用的购物App是?

2. 测试任务
   任务1: 浏览商品并添加到购物车
   - 场景: 您需要购买一件生日礼物
   - 任务: 找到合适的商品并加入购物车
   - 时间: 3分钟
   
   任务2: 修改购物车内容
   - 场景: 您决定更换商品
   - 任务: 修改商品数量或删除商品
   - 时间: 2分钟
   
   任务3: 完成结算流程
   - 场景: 您准备付款
   - 任务: 填写信息并到达支付页面
   - 时间: 5分钟

3. 后置问题
   Q1. 整体购物流程是否顺畅? (1-5分)
   Q2. 最喜欢的功能是?
   Q3. 遇到的困难有哪些?
   Q4. 有什么改进建议?

受众筛选:
- 年龄: 25-45岁
- 网购频率: 每月3次以上
- 设备: 移动端用户
- 样本量: 10人
```

### 案例2:竞品对比测试
```
测试目标: 对比我们的产品与竞品的体验

测试设计:
1. 测试流程
   - 使用产品A完成任务
   - 使用产品B完成同样任务
   - 对比评价

2. 测试任务(两组相同)
   任务1: 搜索商品
   任务2: 筛选结果
   任务3: 查看详情
   任务4: 加入购物车

3. 对比问题
   Q1. 哪个产品更容易使用?
   Q2. 哪个产品界面更美观?
   Q3. 您更愿意使用哪个产品?
   Q4. 为什么?

受众筛选:
- 使用过同类产品
- 年龄: 20-40岁
- 样本量: 15人
```

### 案例3:原型测试
```
测试目标: 验证新设计方案的可行性

测试材料:
- Figma/InVision原型
- 关键流程原型
- 高保真或低保真

测试任务:
1. 探索首页
   - 不给具体任务
   - 观察自然浏览行为
   - 记录第一印象

2. 完成核心任务
   - 搜索功能
   - 内容浏览
   - 分享功能

3. 评价设计
   - 视觉设计
   - 信息架构
   - 交互体验

分析要点:
- 用户能否理解界面
- 操作是否直观
- 哪些地方困惑
- 喜欢和不喜欢的点
```

## 最佳实践

### 1. 测试规划
```
规划要点:
- 明确测试目标
- 确定测试范围
- 设计合理任务
- 预留足够时间

样本建议:
- 发现问题: 5人可发现80%问题
- 验证优化: 建议10-15人
- 对比测试: 每组至少8人
```

### 2. 任务设计
```
设计原则:
- 真实场景
- 明确目标
- 不引导操作
- 控制难度
- 时间合理

避免:
- "请点击右上角按钮" (引导)
- "找到X功能" (不自然)
- 任务过多过难
```

### 3. 问题设计
```
提问技巧:
- 开放式问题优先
- 避免引导性提问
- 问题简洁明确
- 从易到难

示例:
✓ "您会如何完成这个任务?"
✗ "这个按钮是否容易找到?"

✓ "您对这个页面的第一印象是什么?"
✗ "这个页面是否美观?"
```

### 4. 结果分析
```
分析方法:
1. 观看所有视频
2. 记录关键发现
3. 分类整理问题
4. 评估问题严重程度
5. 提出优化建议

报告结构:
- 测试概述
- 关键发现
- 问题描述
- 优化建议
- 视频证据
```

## 进阶技巧

### 标记与剪辑
```
标记类型:
- 严重问题
- 次要问题
- 正面反馈
- 有趣发现

剪辑功能:
- 创建精彩片段
- 合并相关片段
- 添加文字说明
- 分享给团队
```

### 测试模板
```
常用模板:
1. 5秒测试
   - 用户浏览5秒
   - 询问第一印象
   - 回忆页面内容

2. 导航测试
   - 能否找到功能
   - 路径是否合理
   - 耗时多久

3. 转化漏斗测试
   - 测试关键步骤
   - 识别流失点
   - 优化转化率
```

### 迭代测试
```
迭代流程:
1. 第一轮测试发现问题
2. 快速优化设计
3. 第二轮测试验证
4. 持续迭代改进

测试频率:
- 设计阶段: 概念测试
- 开发阶段: 原型测试
- 上线后: 可用性测试
```

## 常见问题

### Q: 需要多少测试用户?
A: 发现问题5-8人足够,验证优化建议10-15人

### Q: 测试需要多长时间?
A: 单个用户15-30分钟,包含多个任务

### Q: 如何控制测试成本?
A: 明确筛选条件减少无效样本,合理设计测试任务
