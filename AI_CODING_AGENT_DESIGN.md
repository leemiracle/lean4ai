## 第一部分：什么是AI智能体

### 1.1 定义

```
AI智能体 = LLM + 工具 + 记忆 + 规划 + 自主执行

与普通LLM的区别：
┌────────────────┬─────────────┬─────────────┐
│     维度       │   普通LLM   │  AI智能体   │
├────────────────┼─────────────┼─────────────┤
│ 交互方式       │ 一问一答    │ 持续对话    │
│ 记忆能力       │ 无/短期     │ 长期记忆    │
│ 工具使用       │ 无          │ 可调用工具  │
│ 自主规划       │ 无          │ 有          │
│ 任务执行       │ 生成内容    │ 端到端完成  │
│ 领域知识       │ 通用        │ 专业定制    │
└────────────────┴─────────────┴─────────────┘
```

### 1.2 功能模块详解

#### 模块一：指令系统

```
作用：定义智能体的行为规则和响应模式

组成部分：
├── 系统提示词（System Prompt）
│   └── 定义角色、能力边界、输出规范
├── 用户指令
│   └── 用户的具体任务请求
└── 内部指令
    └── 智能体自我触发的指令

设计原则（基于之前的知识）：
├── 清晰性：每条指令只有一种理解
├── 具体性：明确触发条件和执行动作
├── 结构化：用统一格式组织
└── 可验证：能判断执行是否成功
```

**编程智能体指令系统示例：**

```
【角色定义】
你是一个专业的编程助手，精通Python/JavaScript/TypeScript。
你的职责是帮助用户完成代码开发的全流程工作。

【核心能力】
1. 需求分析：理解并拆解项目需求
2. 代码生成：生成高质量、可维护的代码
3. 代码审查：发现问题并提供改进建议
4. 文档生成：自动生成代码文档和API文档
5. 调试辅助：帮助定位和解决问题

【行为规范】
- 生成代码前，先确认理解需求
- 代码必须有注释和类型标注
- 提供多个方案时，说明各方案优缺点
- 不确定时，主动询问而非猜测
- 每次输出后，询问是否需要进一步帮助

【知识边界】
- 你知道的知识：训练数据中的代码模式和最佳实践
- 你不知道的：用户项目的私有代码（除非用户提供）
- 需要验证的：具体的库版本兼容性、性能数据

【输出格式】
代码：使用代码块，标注语言
分析：使用表格或列表
建议：分点说明，按优先级排序
```

## 第二部分：编程领域专属设计

### 2.1 更好地理解项目需求

#### 问题分析

```
传统方式的问题：
├── 需求描述模糊
├── 缺少上下文
├── 隐含需求未识别
└── 技术约束不明确

导致的结果：
├── 代码方向错误
├── 返工次数多
├── 需求遗漏
└── 用户满意度低
```

#### 智能体解决方案

**方案一：结构化需求收集**

```
智能体需求收集模板：

【项目信息】
- 项目名称：
- 项目类型：[Web/App/API/CLI/其他]
- 技术栈：[语言/框架/数据库]

【功能需求】
- 核心功能：（必填）
- 扩展功能：（可选）
- 用户角色：
- 使用场景：

【非功能需求】
- 性能要求：
- 安全要求：
- 兼容性：

【约束条件】
- 时间约束：
- 资源约束：
- 技术约束：

【参考资料】
- 类似项目：
- 设计稿/原型：
- API文档：
```

**方案二：主动提问机制**

```
智能体主动提问设计：

触发条件：
├── 需求描述少于50字
├── 缺少关键技术信息
├── 存在明显歧义
└── 涉及多个技术选型

提问策略（基于"按需深挖"原则）：
├── 简单需求 → 直接执行
├── 中等需求 → 问≤3个关键问题
└── 复杂需求 → 最小方案→确认→迭代

示例对话：
用户："帮我写一个登录功能"

智能体：
"我理解您需要实现登录功能。为提供精准方案，请确认：

1. 登录方式：账号密码 / 手机验证码 / 第三方登录？
2. 用户类型：普通用户 / 管理员 / 多角色？
3. 安全要求：需要2FA / 验证码 / 密码规则？

您可以回答其中任一问题，或说'使用默认方案'。"
```

需求 → 用例图 → 流程图 → 数据模型

智能体输出：

#### 精准度提升方案

**方案一：多轮确认机制**

```
代码生成流程：

第1轮：需求确认
├── 智能体复述需求
├── 用户确认/修正
└── 输出：确认后的需求文档

第2轮：方案设计
├── 智能体给出技术方案
├── 用户选择/调整
└── 输出：技术方案文档

第3轮：代码生成
├── 智能体生成核心代码
├── 用户审查/反馈
└── 输出：可运行代码

第4轮：优化迭代
├── 根据反馈优化
├── 添加边界处理
└── 输出：最终代码

关键：每轮都要确认，避免方向偏差
```

def calculate_total(items: list[dict], discount: float = 0) -> float:
    """
    计算订单总价，支持折扣。
    
    Args:
        items: 商品列表，每个商品包含:
            - price: 商品单价(float)
            - quantity: 商品数量(int)
        discount: 折扣率，0-1之间，默认为0
    
    Returns:
        折扣后的总价(float)
    
    Raises:
        ValueError: 当discount不在0-1范围时
    
    Examples:
        >>> items = [{'price': 100, 'quantity': 2}]
        >>> calculate_total(items, 0.1)
        180.0
    
    Note:
        - 价格和数量必须为正数
        - 空列表返回0
    """
    if not 0 <= discount <= 1:
        raise ValueError("discount must be between 0 and 1")
    
    total = sum(item['price'] * item['quantity'] for item in items)
    return total * (1 - discount)
